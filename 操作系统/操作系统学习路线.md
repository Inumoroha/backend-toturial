# 操作系统系统学习路线

> 目标：从“会用操作系统”走到“理解操作系统如何设计与实现”，最终能读懂小型内核代码，完成若干系统编程实验，并能解释进程、内存、并发、文件系统、I/O、虚拟化等核心机制。

## 0. 学习方式

建议周期：12 到 16 周。

每周节奏：

- 2 到 3 次理论学习：读书、看课、整理概念。
- 2 到 3 次动手实验：写 C、跑 Linux 命令、做 xv6/OSTEP 小实验。
- 1 次复盘：用自己的话写一页笔记，回答“这个机制解决了什么问题、代价是什么、现实系统怎么做”。

学习主线：

1. 先理解用户态程序如何使用操作系统。
2. 再理解操作系统如何抽象 CPU、内存、磁盘和设备。
3. 最后进入内核实现，读 xv6，做实验，形成系统级直觉。

## 1. 前置基础

### 必备知识

- C 语言：指针、结构体、内存布局、函数指针、编译链接。
- Linux 基础：shell、文件权限、管道、重定向、进程查看、包管理。
- 计算机组成：CPU、寄存器、内存、缓存、中断、总线、磁盘。
- 数据结构：链表、队列、哈希表、树、位图。
- 基础汇编：能看懂函数调用、栈帧、寄存器传参即可。

### 推荐补课任务

- 写一个 C 程序，分别打印全局变量、局部变量、堆变量、函数地址。
- 使用 `gcc -S` 查看汇编，理解函数调用和栈。
- 使用 `strace` 跟踪 `ls`、`cat`、`echo` 的系统调用。
- 使用 `ps`、`top`、`lsof`、`vmstat`、`iostat` 观察系统状态。

## 2. 阶段一：操作系统总览

建议时间：第 1 周。

核心问题：

- 操作系统为什么存在？
- 操作系统提供了哪些抽象？
- 用户态和内核态有什么区别？
- 系统调用、中断、异常分别是什么？

学习内容：

- OS 的三大主题：虚拟化、并发、持久化。
- 内核、shell、库函数、系统调用之间的关系。
- 程序从源码到运行进程的大致路径。

动手任务：

- 用 C 调用 `fork`、`exec`、`wait`，写一个极简 shell。
- 用 `strace` 观察程序启动、读写文件、创建进程时发生的系统调用。
- 画出“用户程序 -> libc -> syscall -> kernel -> hardware”的调用路径。

验收标准：

- 能解释为什么 `printf` 不是系统调用，而 `write` 是系统调用。
- 能解释一次命令行执行 `ls` 的大致过程。

## 3. 阶段二：进程、线程与 CPU 虚拟化

建议时间：第 2 到 3 周。

核心问题：

- 进程是什么？
- 操作系统如何让多个程序“同时运行”？
- 上下文切换保存了什么？
- 调度算法如何影响响应时间和吞吐量？

学习内容：

- 进程状态：创建、就绪、运行、阻塞、退出。
- PCB/进程控制块的作用。
- 上下文切换。
- 调度算法：FIFO、SJF、STCF、RR、MLFQ。
- 线程与进程的区别。

动手任务：

- 写程序观察 `fork` 后父子进程的变量变化。
- 写一个简单的 Round-Robin 调度模拟器。
- 使用 `time`、`perf` 或系统监控工具观察 CPU 密集型任务。
- 用 pthread 写多线程计数器，先故意写出数据竞争。

验收标准：

- 能解释进程切换为什么有成本。
- 能比较进程、线程、协程的区别。
- 能用例子说明不同调度策略的优缺点。

## 4. 阶段三：并发、锁与同步

建议时间：第 4 到 5 周。

核心问题：

- 为什么多线程程序会出错？
- 锁到底保护的是什么？
- 条件变量、信号量、管程分别解决什么问题？
- 死锁如何产生，如何避免？

学习内容：

- 临界区、竞态条件、原子性、可见性、有序性。
- 互斥锁、自旋锁、读写锁。
- 条件变量、信号量、生产者消费者问题。
- 死锁四个必要条件。
- 常见并发 bug：忘记加锁、锁粒度过大、锁顺序错误、条件变量误用。

动手任务：

- 实现线程安全队列。
- 实现生产者消费者模型。
- 写一个会死锁的程序，再修复它。
- 使用 ThreadSanitizer 或类似工具检查数据竞争。

验收标准：

- 能解释 `while` 检查条件变量为什么通常比 `if` 更稳妥。
- 能画出死锁等待图。
- 能说明锁粒度对性能和正确性的影响。

## 5. 阶段四：内存管理与虚拟内存

建议时间：第 6 到 8 周。

核心问题：

- 每个进程为什么都以为自己拥有完整地址空间？
- 虚拟地址如何转换为物理地址？
- 分页、分段、TLB 分别做什么？
- 页面置换算法为什么重要？

学习内容：

- 地址空间：代码段、数据段、堆、栈、mmap 区域。
- 动态内存分配：空闲链表、伙伴系统、slab 思想。
- 分页、页表、多级页表。
- TLB 与地址转换缓存。
- 缺页异常、按需分页、写时复制。
- 页面置换：FIFO、LRU、Clock。

动手任务：

- 写一个简单 malloc/free 模拟器。
- 用 `/proc/<pid>/maps` 观察进程地址空间。
- 写程序观察栈增长、堆增长、mmap 分配。
- 实现一个页面置换算法模拟器。

验收标准：

- 能手算一个简单虚拟地址到物理地址的转换过程。
- 能解释 `fork` 为什么可以借助写时复制降低开销。
- 能说明 TLB miss 和 page fault 的区别。

## 6. 阶段五：文件系统与持久化

建议时间：第 9 到 10 周。

核心问题：

- 文件和目录在磁盘上如何表示？
- inode 是什么？
- 文件系统如何处理崩溃一致性？
- 缓存为什么既提升性能又带来风险？

学习内容：

- 文件抽象、文件描述符、open/read/write/close。
- inode、目录项、数据块、空闲空间管理。
- 文件系统缓存与写回。
- 崩溃一致性：fsck、日志、写时复制文件系统。
- 磁盘、SSD、I/O 调度的基本差异。

动手任务：

- 用 C 实现 `cp`、`cat`、`ls` 的简化版。
- 使用 `stat` 观察 inode、权限、链接数。
- 写程序比较顺序读写和随机读写的性能。
- 设计一个极简文件系统的数据结构草图。

验收标准：

- 能解释硬链接和软链接的区别。
- 能说明为什么 `write` 返回成功不一定代表数据已经落盘。
- 能解释日志文件系统如何降低崩溃恢复成本。

## 7. 阶段六：I/O、设备与中断

建议时间：第 11 周。

核心问题：

- CPU 如何和设备交互？
- 中断为什么重要？
- DMA 解决了什么问题？
- 阻塞 I/O、非阻塞 I/O、多路复用有什么区别？

学习内容：

- 设备控制器、设备驱动。
- 轮询、中断、DMA。
- 阻塞与非阻塞。
- `select`、`poll`、`epoll` 的基本思想。
- 缓冲区与零拷贝的基本概念。

动手任务：

- 写一个基于 `select` 或 `epoll` 的 echo server。
- 用 `strace` 观察网络程序的 I/O 系统调用。
- 比较阻塞 socket 和非阻塞 socket 的行为。

验收标准：

- 能解释中断处理为什么要尽量短。
- 能说明 `epoll` 适合处理大量连接的原因。

## 8. 阶段七：安全、隔离与虚拟化

建议时间：第 12 周。

核心问题：

- 操作系统如何隔离不同用户和不同进程？
- 权限、能力、命名空间、容器分别是什么层次的机制？
- 虚拟机和容器有什么区别？

学习内容：

- 用户、组、权限位、ACL。
- 进程隔离、地址空间隔离、文件权限。
- chroot、namespace、cgroup 的基本思想。
- 虚拟机、容器、沙箱。
- 常见安全边界：系统调用过滤、最小权限原则。

动手任务：

- 使用 `chmod`、`chown`、`umask` 做权限实验。
- 用 Linux namespace 工具观察容器隔离的基本效果。
- 了解 Docker 容器与宿主机共享内核这一点。

验收标准：

- 能解释容器不是虚拟机。
- 能说明为什么系统调用是重要的安全边界。

## 9. 阶段八：xv6 内核实践

建议时间：第 13 到 16 周。

目标：通过 xv6 把前面的概念和真实内核代码连接起来。

推荐顺序：

1. 搭好 xv6 环境，成功启动内核。
2. 阅读启动流程：boot、kernel main、进程初始化。
3. 阅读系统调用路径：用户态陷入内核，再返回用户态。
4. 阅读进程调度：进程表、上下文切换、scheduler。
5. 阅读内存管理：页表、用户地址空间、sbrk。
6. 阅读文件系统：inode、buffer cache、log。

动手任务：

- 给 xv6 添加一个简单系统调用。
- 实现或修改一个调度策略。
- 给 xv6 添加用户级小工具。
- 跟踪一次 `fork`、`exec`、`open` 的完整路径。

验收标准：

- 能说清楚 xv6 中一次系统调用经过哪些关键函数。
- 能解释 xv6 进程切换的大致流程。
- 能在内核中做一个小功能修改并通过测试。

## 10. 推荐资源

### 主教材

- [Operating Systems: Three Easy Pieces](https://pages.cs.wisc.edu/~remzi/OSTEP/)：非常适合作为主线。它围绕虚拟化、并发、持久化三部分展开，概念清楚，配套练习多。

### 实验课程

- [MIT 6.S081 / 6.1810: Operating System Engineering](https://pdos.csail.mit.edu/6.1810/2025/)：以 xv6 为核心，适合在掌握基础后做内核实验。
- [MIT 6.S081 2021 Schedule](https://pdos.csail.mit.edu/6.S081/2021/schedule.html)：资料相对完整，常被自学者使用。
- [UC Berkeley CS 162](https://cs162.org/)：项目强度高，覆盖系统编程、并发、内存、文件系统等主题。

### 中文辅助

- 先用中文资料建立直觉，再回到英文原始教材和课程。
- 遇到概念不清时，优先查教材、课程讲义和手册页，而不是只看零散博客。

### 手册与工具

- `man 2 fork`
- `man 2 execve`
- `man 2 wait`
- `man 2 open`
- `man 2 mmap`
- `man 7 pthreads`
- `strace`
- `gdb`
- `perf`
- `valgrind`
- `qemu`

## 11. 项目清单

从易到难选择：

1. 极简 shell：支持执行命令、参数、重定向、管道、后台运行。
2. 线程池：支持任务队列、工作线程、优雅退出。
3. malloc 模拟器：实现简单的分配、释放、合并空闲块。
4. 调度模拟器：实现 FIFO、SJF、RR、MLFQ 并比较指标。
5. 页面置换模拟器：实现 FIFO、LRU、Clock。
6. 文件系统模拟器：设计 inode、目录、数据块、空闲位图。
7. epoll echo server：理解事件驱动 I/O。
8. xv6 系统调用：添加一个新 syscall。
9. xv6 调度实验：实现优先级或彩票调度。
10. xv6 文件系统实验：修改文件大小限制或增加简单统计信息。

## 13. 检查点

第 4 周：

- 能写 fork/exec/wait 程序。
- 能解释进程、线程、上下文切换、调度。
- 能写出简单并发 bug 并修复。

第 8 周：

- 能解释虚拟内存、页表、TLB、缺页异常。
- 能读懂 `/proc/<pid>/maps`。
- 能实现简单 malloc 或页面置换模拟器。

第 12 周：

- 能解释 inode、目录、文件描述符、文件系统日志。
- 能写一个基于系统调用的文件复制工具。
- 能写一个简单网络 I/O 程序。

第 16 周：

- 能启动并修改 xv6。
- 能添加一个系统调用。
- 能追踪 fork、exec、open 中至少一个完整路径。
- 能完成一篇 1500 字以上的总结：操作系统如何把硬件资源抽象给应用程序。

## 14. 遇到困难时的处理顺序

1. 先画图：画状态、队列、地址空间、调用路径。
2. 再写最小程序：不要只看概念。
3. 查手册页：确认系统调用参数、返回值和错误码。
4. 看教材对应章节：把零散现象放回主线。
5. 最后再看博客或视频：用来补直觉，不作为唯一依据。

## 15. 最终目标

学完后你应该能够：

- 从应用程序角度解释 OS 提供了什么。
- 从内核角度解释 OS 如何管理 CPU、内存、磁盘和设备。
- 写出常见系统调用程序。
- 分析并发程序中的竞态和死锁。
- 理解虚拟内存和文件系统的核心数据结构。
- 阅读 xv6 这类教学内核，并做小型功能修改。
