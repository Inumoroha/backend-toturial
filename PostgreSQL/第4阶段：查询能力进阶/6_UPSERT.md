# 6. UPSERT：INSERT ... ON CONFLICT

本节目标：理解 UPSERT 的作用，掌握 PostgreSQL 中 `INSERT ... ON CONFLICT ... DO NOTHING` 和 `INSERT ... ON CONFLICT ... DO UPDATE` 的常见写法。

UPSERT 可以理解为：

```text
如果不存在，就插入。
如果已经存在，就忽略或更新。
```

它在后端项目中很常见，例如：

- 标签已存在就复用。
- 用户邮箱已存在就更新资料。
- 统计记录已存在就累加计数。

---

## 一、准备练习表

```sql
CREATE SCHEMA IF NOT EXISTS demo;

DROP TABLE IF EXISTS demo.upsert_tags;

CREATE TABLE demo.upsert_tags (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name text NOT NULL UNIQUE,
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now()
);
```

`name` 上有唯一约束：

```sql
name text NOT NULL UNIQUE
```

`ON CONFLICT` 必须依赖唯一约束、主键或唯一索引来判断冲突。

---

## 二、DO NOTHING：冲突时什么都不做

第一次插入：

```sql
INSERT INTO demo.upsert_tags (name)
VALUES ('PostgreSQL')
RETURNING id, name;
```

再次插入同名标签会冲突。可以使用：

```sql
INSERT INTO demo.upsert_tags (name)
VALUES ('PostgreSQL')
ON CONFLICT (name) DO NOTHING;
```

这表示：如果 `name` 冲突，就忽略这次插入。

---

## 三、DO UPDATE：冲突时更新

```sql
INSERT INTO demo.upsert_tags (name)
VALUES ('PostgreSQL')
ON CONFLICT (name)
DO UPDATE SET updated_at = now()
RETURNING id, name, updated_at;
```

这里的含义是：

- 如果 `name` 不存在，就插入。
- 如果 `name` 已存在，就更新 `updated_at`。

---

## 四、EXCLUDED 是什么

`EXCLUDED` 表示“本来想插入但因为冲突没插进去的那一行数据”。

准备用户表：

```sql
DROP TABLE IF EXISTS demo.upsert_users;

CREATE TABLE demo.upsert_users (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    email text NOT NULL UNIQUE,
    username text NOT NULL,
    updated_at timestamptz NOT NULL DEFAULT now()
);
```

插入或更新用户：

```sql
INSERT INTO demo.upsert_users (email, username)
VALUES ('zhangsan@example.com', 'zhangsan')
ON CONFLICT (email)
DO UPDATE SET
    username = EXCLUDED.username,
    updated_at = now()
RETURNING id, email, username, updated_at;
```

如果邮箱已存在，就把用户名更新成这次新传入的 `EXCLUDED.username`。

## 五、计数累加场景

准备文章统计表：

```sql
DROP TABLE IF EXISTS demo.daily_post_views;

CREATE TABLE demo.daily_post_views (
    post_id bigint NOT NULL,
    view_date date NOT NULL,
    view_count bigint NOT NULL DEFAULT 0,
    PRIMARY KEY (post_id, view_date)
);
```

插入或累加访问量：

```sql
INSERT INTO demo.daily_post_views (post_id, view_date, view_count)
VALUES (1, CURRENT_DATE, 1)
ON CONFLICT (post_id, view_date)
DO UPDATE SET view_count = demo.daily_post_views.view_count + EXCLUDED.view_count
RETURNING post_id, view_date, view_count;
```

这类写法常用于统计：

- 每日访问量。
- 点赞数。
- 下载次数。
- 登录次数。

## 六、常见误区

### 1. 没有唯一约束可以 ON CONFLICT 吗？

通常不行。

`ON CONFLICT` 需要知道什么算冲突，通常依赖 `PRIMARY KEY`、`UNIQUE` 或唯一索引。

### 2. DO NOTHING 会返回已有数据吗？

不会自动返回已有行。

如果冲突后 `DO NOTHING`，通常不会返回那条已存在的数据。需要已有数据时，可以考虑 `DO UPDATE ... RETURNING` 或额外查询。

### 3. EXCLUDED 是旧数据吗？

不是。

`EXCLUDED` 是这次尝试插入的新数据。

## 七、本节达标标准

学完本节后，你应该能够做到：

- 解释 UPSERT 是什么。
- 使用 `ON CONFLICT ... DO NOTHING` 忽略重复插入。
- 使用 `ON CONFLICT ... DO UPDATE` 在冲突时更新数据。
- 理解 `EXCLUDED` 的含义。
- 知道 `ON CONFLICT` 依赖唯一约束、主键或唯一索引。
- 能写出标签复用、用户资料更新、计数累加这类常见 UPSERT。
