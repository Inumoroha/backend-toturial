# 1. 聚合函数：COUNT、SUM、AVG、MAX、MIN

本节目标：理解聚合函数的作用，掌握 PostgreSQL 中常用的 `COUNT`、`SUM`、`AVG`、`MAX`、`MIN`，并能完成用户数、文章数、评论数、平均值、最大值、最小值等基础统计查询。

聚合函数用于“把多行数据汇总成一个结果”。它是统计类 SQL 的基础，也是后端管理后台、数据看板、列表统计中非常常见的能力。

---

## 一、聚合函数是什么

普通查询通常是一行数据返回一行结果。

例如：

```sql
SELECT id, username
FROM demo.users;
```

如果表中有 4 个用户，就返回 4 行。

聚合函数会把多行数据统计成一个结果。

例如：

```sql
SELECT COUNT(*) AS user_count
FROM demo.users;
```

无论表中有多少用户，这条 SQL 都只返回一行，表示用户总数。

常见聚合函数：

| 函数 | 作用 | 示例 |
| --- | --- | --- |
| `COUNT` | 统计数量 | 用户数、文章数、评论数 |
| `SUM` | 求和 | 访问量总和、订单总金额 |
| `AVG` | 求平均值 | 平均年龄、平均价格 |
| `MAX` | 求最大值 | 最大价格、最新时间 |
| `MIN` | 求最小值 | 最小价格、最早时间 |

## 三、COUNT：统计数量

`COUNT` 用来统计行数。

### 1. 统计所有用户数量

```sql
SELECT COUNT(*) AS user_count
FROM demo.users;
```

`COUNT(*)` 表示统计结果集中有多少行。

### 2. 统计文章数量

```sql
SELECT COUNT(*) AS post_count
FROM demo.posts;
```

### 3. 统计已发布文章数量

```sql
SELECT COUNT(*) AS published_post_count
FROM demo.posts
WHERE is_published = true;
```

也可以简写：

```sql
SELECT COUNT(*) AS published_post_count
FROM demo.posts
WHERE is_published;
```

### 4. COUNT(*) 和 COUNT(column) 的区别

`COUNT(*)` 统计行数。

`COUNT(column)` 统计某一列非 NULL 的数量。

例如：

```sql
SELECT
    COUNT(*) AS total_rows,
    COUNT(birthday) AS birthday_count
FROM demo.user_profiles;
```

含义：

- `COUNT(*)`：统计资料表中有多少行。
- `COUNT(birthday)`：统计有生日信息的资料有多少行。

如果某行的 `birthday` 是 `NULL`，不会被 `COUNT(birthday)` 统计。

### 5. COUNT(DISTINCT column)：去重统计

统计有多少个作者发布过文章：

```sql
SELECT COUNT(DISTINCT user_id) AS author_count
FROM demo.posts;
```

如果一个用户发布了多篇文章，也只算一个作者。

## 四、SUM：求和

`SUM` 用来计算数值列的总和。

为了更容易练习求和，先创建一个文章访问量表。

```sql
DROP TABLE IF EXISTS demo.post_stats;

CREATE TABLE demo.post_stats (
    post_id bigint PRIMARY KEY,
    view_count bigint NOT NULL DEFAULT 0,
    like_count bigint NOT NULL DEFAULT 0,
    CONSTRAINT post_stats_post_id_fk
        FOREIGN KEY (post_id)
        REFERENCES demo.posts (id)
        ON DELETE CASCADE
);
```

插入统计数据：

```sql
INSERT INTO demo.post_stats (post_id, view_count, like_count)
VALUES
    (1, 120, 10),
    (2, 80, 6),
    (3, 30, 2),
    (4, 200, 30),
    (5, 150, 15);
```

统计总浏览量：

```sql
SELECT SUM(view_count) AS total_views
FROM demo.post_stats;
```

统计总点赞数：

```sql
SELECT SUM(like_count) AS total_likes
FROM demo.post_stats;
```

同时统计多个指标：

```sql
SELECT
    SUM(view_count) AS total_views,
    SUM(like_count) AS total_likes
FROM demo.post_stats;
```

### SUM 和 NULL

`SUM` 会忽略 `NULL`。

如果所有值都是 `NULL`，结果会是 `NULL`，不是 0。

真实项目中，有时可以用 `COALESCE` 把 NULL 转成 0：

```sql
SELECT COALESCE(SUM(view_count), 0) AS total_views
FROM demo.post_stats;
```

## 五、AVG：求平均值

`AVG` 用来计算平均值。

统计平均浏览量：

```sql
SELECT AVG(view_count) AS avg_views
FROM demo.post_stats;
```

统计平均点赞数：

```sql
SELECT AVG(like_count) AS avg_likes
FROM demo.post_stats;
```

可以配合 `ROUND` 控制小数位：

```sql
SELECT ROUND(AVG(view_count), 2) AS avg_views
FROM demo.post_stats;
```

注意：`AVG` 也会忽略 `NULL`。

## 六、MAX 和 MIN：最大值与最小值

`MAX` 用来求最大值，`MIN` 用来求最小值。

### 1. 查询最高浏览量和最低浏览量

```sql
SELECT
    MAX(view_count) AS max_views,
    MIN(view_count) AS min_views
FROM demo.post_stats;
```

### 2. 查询最新文章时间和最早文章时间

```sql
SELECT
    MAX(created_at) AS latest_post_time,
    MIN(created_at) AS earliest_post_time
FROM demo.posts;
```

`MAX` 和 `MIN` 不只适用于数字，也可以用于时间、文本等可比较类型。

## 七、聚合函数配合 WHERE

聚合前可以先用 `WHERE` 过滤数据。

统计已发布文章数量：

```sql
SELECT COUNT(*) AS published_count
FROM demo.posts
WHERE is_published = true;
```

统计已发布文章总浏览量：

```sql
SELECT SUM(s.view_count) AS published_total_views
FROM demo.posts AS p
JOIN demo.post_stats AS s ON s.post_id = p.id
WHERE p.is_published = true;
```

执行顺序可以简单理解为：

```text
先 FROM / JOIN 找到数据来源。
再 WHERE 过滤行。
最后聚合函数统计过滤后的结果。
```

## 八、聚合函数配合 JOIN

聚合函数经常和 JOIN 一起使用。

### 1. 统计每篇文章的评论数

```sql
SELECT
    p.id,
    p.title,
    COUNT(c.id) AS comment_count
FROM demo.posts AS p
LEFT JOIN demo.comments AS c ON c.post_id = p.id
GROUP BY p.id, p.title
ORDER BY p.id;
```

这里先使用 `LEFT JOIN` 把文章和评论连起来，然后按文章分组，再统计每篇文章的评论数量。

注意：这条 SQL 用到了 `GROUP BY`。下一篇会专门系统讲分组，这里先感受聚合函数的使用场景。

### 2. 统计每个用户的文章数

```sql
SELECT
    u.id,
    u.username,
    COUNT(p.id) AS post_count
FROM demo.users AS u
LEFT JOIN demo.posts AS p ON p.user_id = u.id
GROUP BY u.id, u.username
ORDER BY u.id;
```

为什么用 `COUNT(p.id)`，而不是 `COUNT(*)`？

因为 `LEFT JOIN` 时，即使用户没有文章，也会保留一行用户数据。如果用 `COUNT(*)`，没有文章的用户也可能被统计成 1。使用 `COUNT(p.id)` 可以只统计真实存在的文章。

## 九、聚合函数和别名

聚合结果通常建议起别名。

不推荐：

```sql
SELECT COUNT(*)
FROM demo.users;
```

推荐：

```sql
SELECT COUNT(*) AS user_count
FROM demo.users;
```

好处：

- 查询结果更清晰。
- 后端读取字段名更方便。
- SQL 更容易理解。

## 十一、常见误区

### 1. COUNT(*) 会忽略 NULL 吗？

不会。

`COUNT(*)` 统计行数，不关心某一列是不是 NULL。

### 2. COUNT(column) 会忽略 NULL 吗？

会。

`COUNT(column)` 只统计该列非 NULL 的行。

### 3. SUM 没有数据时一定返回 0 吗？

不一定。

如果没有可统计的行，或者所有值都是 NULL，`SUM` 可能返回 NULL。需要 0 时可以使用：

```sql
COALESCE(SUM(column), 0)
```

### 4. AVG 会统计 NULL 吗？

不会。

`AVG` 会忽略 NULL，只对非 NULL 值求平均。

### 5. LEFT JOIN 后统计数量为什么容易错？

因为 `LEFT JOIN` 会保留左表行。统计右表数量时，通常应该使用：

```sql
COUNT(right_table.id)
```

而不是：

```sql
COUNT(*)
```

### 6. 聚合函数能和普通列一起查吗？

可以，但通常需要 `GROUP BY`。

例如下面这种写法会报错：

```sql
SELECT username, COUNT(*)
FROM demo.users;
```

因为数据库不知道每个 `username` 应该如何对应聚合结果。下一篇会专门讲 `GROUP BY`。
