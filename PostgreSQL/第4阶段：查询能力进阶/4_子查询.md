# 4. 子查询：把一个查询嵌入另一个查询

本节目标：理解子查询的作用，掌握 PostgreSQL 中常见的 `WHERE` 子查询、`FROM` 子查询、`SELECT` 子查询和相关子查询，并能判断什么时候使用子查询，什么时候改用 `JOIN` 更清晰。

前面已经学习了聚合函数、分组查询和多表查询。子查询可以把一个查询的结果作为另一个查询的条件、临时表或计算字段，是后端业务 SQL 中非常常见的能力。

---

## 一、子查询是什么

子查询就是嵌套在另一个 SQL 里面的查询。

最简单的形式：

```sql
SELECT *
FROM demo.posts
WHERE user_id = (
    SELECT id
    FROM demo.users
    WHERE username = 'zhangsan'
);
```

括号里的这段就是子查询：

```sql
SELECT id
FROM demo.users
WHERE username = 'zhangsan'
```

外层查询用它的结果来查询张三发布的文章。

可以简单理解为：

```text
先用内层查询得到一个中间结果。
再把这个中间结果交给外层查询使用。
```

## 三、WHERE 子查询

`WHERE` 子查询最常见，用来把另一个查询的结果作为过滤条件。

### 1. 使用等号匹配单个结果

查询 `zhangsan` 发布的文章：

```sql
SELECT id, title, user_id
FROM demo.posts
WHERE user_id = (
    SELECT id
    FROM demo.users
    WHERE username = 'zhangsan'
);
```

注意：使用 `=` 时，子查询必须返回一行一列。

如果子查询返回多行，会报错。

### 2. 使用 IN 匹配多个结果

查询 `zhangsan` 和 `lisi` 发布的文章：

```sql
SELECT id, title, user_id
FROM demo.posts
WHERE user_id IN (
    SELECT id
    FROM demo.users
    WHERE username IN ('zhangsan', 'lisi')
)
ORDER BY id;
```

`IN` 适合子查询返回多个值的场景。

### 3. 查询有评论的文章

```sql
SELECT id, title
FROM demo.posts
WHERE id IN (
    SELECT post_id
    FROM demo.comments
)
ORDER BY id;
```

这条 SQL 的含义是：

```text
先查出 comments 表中出现过的 post_id。
再查询 id 在这些 post_id 中的文章。
```

### 4. 查询没有评论的文章

```sql
SELECT id, title
FROM demo.posts
WHERE id NOT IN (
    SELECT post_id
    FROM demo.comments
)
ORDER BY id;
```

这个写法能理解，但真实项目中要小心 `NOT IN` 和 `NULL` 的关系。后续会讲 `NOT EXISTS`，通常更稳。

## 四、EXISTS 子查询

`EXISTS` 用来判断子查询是否至少返回一行。

### 1. 查询有评论的文章

```sql
SELECT p.id, p.title
FROM demo.posts AS p
WHERE EXISTS (
    SELECT 1
    FROM demo.comments AS c
    WHERE c.post_id = p.id
)
ORDER BY p.id;
```

这里的子查询引用了外层查询的 `p.id`，所以它是相关子查询。

含义是：

```text
对每一篇文章，检查 comments 表里是否存在对应评论。
存在就返回这篇文章。
```

### 2. 查询没有评论的文章

```sql
SELECT p.id, p.title
FROM demo.posts AS p
WHERE NOT EXISTS (
    SELECT 1
    FROM demo.comments AS c
    WHERE c.post_id = p.id
)
ORDER BY p.id;
```

`NOT EXISTS` 常用于查询“没有关联数据”的记录。

例如：

- 没有评论的文章。
- 没有订单的用户。
- 没有绑定标签的文章。

### 3. EXISTS 里面为什么常写 SELECT 1

```sql
SELECT 1
```

只是一个习惯写法。`EXISTS` 只关心子查询有没有返回行，不关心返回的具体列。

写 `SELECT *` 通常也能运行，但 `SELECT 1` 更明确。

## 五、FROM 子查询：把查询结果当临时表

子查询也可以放在 `FROM` 中，把查询结果当作一张临时表。

### 1. 先统计每个用户文章数，再筛选

```sql
SELECT
    t.user_id,
    t.post_count
FROM (
    SELECT
        user_id,
        COUNT(*) AS post_count
    FROM demo.posts
    GROUP BY user_id
) AS t
WHERE t.post_count > 1
ORDER BY t.user_id;
```

括号里的查询先得到一张临时结果表：

```text
user_id | post_count
```

外层再对这张临时表继续查询。

注意：`FROM` 子查询必须起别名，例如这里的 `AS t`。

### 2. 统计文章评论数后再排序

```sql
SELECT
    t.post_id,
    t.title,
    t.comment_count
FROM (
    SELECT
        p.id AS post_id,
        p.title,
        COUNT(c.id) AS comment_count
    FROM demo.posts AS p
    LEFT JOIN demo.comments AS c ON c.post_id = p.id
    GROUP BY p.id, p.title
) AS t
ORDER BY t.comment_count DESC, t.post_id;
```

这种写法适合把复杂查询拆成两层，提高可读性。

## 六、SELECT 子查询：把子查询结果作为字段

子查询也可以写在 `SELECT` 列表中，作为一个计算字段。

### 1. 查询文章列表，并附带评论数

```sql
SELECT
    p.id,
    p.title,
    (
        SELECT COUNT(*)
        FROM demo.comments AS c
        WHERE c.post_id = p.id
    ) AS comment_count
FROM demo.posts AS p
ORDER BY p.id;
```

这条 SQL 的含义是：

```text
每查询一篇文章，就用子查询统计它的评论数。
```

### 2. 查询用户列表，并附带文章数

```sql
SELECT
    u.id,
    u.username,
    (
        SELECT COUNT(*)
        FROM demo.posts AS p
        WHERE p.user_id = u.id
    ) AS post_count
FROM demo.users AS u
ORDER BY u.id;
```

这种写法可读性不错，但当数据量很大时，要关注执行计划。很多场景也可以改写成 `LEFT JOIN + GROUP BY`。

## 七、相关子查询

相关子查询指的是：子查询引用了外层查询中的列。

例如：

```sql
SELECT
    u.id,
    u.username,
    (
        SELECT COUNT(*)
        FROM demo.posts AS p
        WHERE p.user_id = u.id
    ) AS post_count
FROM demo.users AS u;
```

子查询中的：

```sql
WHERE p.user_id = u.id
```

引用了外层的 `u.id`。

相关子查询的特点：

- 表达某些业务逻辑很自然。
- 可读性有时很好。
- 数据量大时可能需要关注性能。
- 有些场景可以改写成 JOIN。

## 八、子查询和 JOIN 的互相改写

很多子查询可以用 JOIN 改写。

### 1. 查询有评论的文章：IN 写法

```sql
SELECT id, title
FROM demo.posts
WHERE id IN (
    SELECT post_id
    FROM demo.comments
)
ORDER BY id;
```

### 2. 查询有评论的文章：JOIN 写法

```sql
SELECT DISTINCT
    p.id,
    p.title
FROM demo.posts AS p
JOIN demo.comments AS c ON c.post_id = p.id
ORDER BY p.id;
```

这两个查询目标类似。

### 3. 查询每个用户文章数：子查询写法

```sql
SELECT
    u.id,
    u.username,
    (
        SELECT COUNT(*)
        FROM demo.posts AS p
        WHERE p.user_id = u.id
    ) AS post_count
FROM demo.users AS u
ORDER BY u.id;
```

### 4. 查询每个用户文章数：JOIN + GROUP BY 写法

```sql
SELECT
    u.id,
    u.username,
    COUNT(p.id) AS post_count
FROM demo.users AS u
LEFT JOIN demo.posts AS p ON p.user_id = u.id
GROUP BY u.id, u.username
ORDER BY u.id;
```

两种写法都值得掌握。

简单建议：

```text
如果是补充一个统计字段，SELECT 子查询有时很直观。
如果要关联很多数据，JOIN 通常更清晰。
如果要判断是否存在，EXISTS 通常很合适。
```

## 九、NOT IN 和 NULL 的坑

`NOT IN` 遇到 `NULL` 时容易出现意外结果。

示例：

```sql
SELECT id, title
FROM demo.posts
WHERE id NOT IN (
    SELECT post_id
    FROM demo.comments
);
```

如果子查询结果里出现 `NULL`，`NOT IN` 的判断可能变得不符合直觉。

因此查询“没有关联数据”时，更推荐使用 `NOT EXISTS`：

```sql
SELECT p.id, p.title
FROM demo.posts AS p
WHERE NOT EXISTS (
    SELECT 1
    FROM demo.comments AS c
    WHERE c.post_id = p.id
)
ORDER BY p.id;
```

或者使用 `LEFT JOIN + IS NULL`：

```sql
SELECT p.id, p.title
FROM demo.posts AS p
LEFT JOIN demo.comments AS c ON c.post_id = p.id
WHERE c.id IS NULL
ORDER BY p.id;
```

## 十、子查询配合聚合结果

查询浏览量高于平均浏览量的文章：

```sql
SELECT
    p.id,
    p.title,
    s.view_count
FROM demo.posts AS p
JOIN demo.post_stats AS s ON s.post_id = p.id
WHERE s.view_count > (
    SELECT AVG(view_count)
    FROM demo.post_stats
)
ORDER BY s.view_count DESC;
```

这类查询很常见：

```text
找出高于平均值的数据。
找出低于平均值的数据。
找出超过某个统计阈值的数据。
```

查询评论数高于平均评论数的文章：

```sql
SELECT
    t.post_id,
    t.title,
    t.comment_count
FROM (
    SELECT
        p.id AS post_id,
        p.title,
        COUNT(c.id) AS comment_count
    FROM demo.posts AS p
    LEFT JOIN demo.comments AS c ON c.post_id = p.id
    GROUP BY p.id, p.title
) AS t
WHERE t.comment_count > (
    SELECT AVG(comment_count)
    FROM (
        SELECT COUNT(c.id) AS comment_count
        FROM demo.posts AS p
        LEFT JOIN demo.comments AS c ON c.post_id = p.id
        GROUP BY p.id
    ) AS comment_stats
)
ORDER BY t.comment_count DESC;
```

这个例子稍复杂，但能展示子查询可以层层组合。真实项目中，如果查询变得很长，可以考虑使用 CTE，也就是 `WITH` 语句。后续会单独学习。

## 十二、常见误区

### 1. 子查询一定比 JOIN 慢吗？

不一定。

PostgreSQL 查询优化器会对很多 SQL 做优化。是否慢要看具体 SQL、数据量、索引和执行计划。不要只凭写法判断性能。

### 2. 子查询一定比 JOIN 更好吗？

也不一定。

子查询有时更直观，JOIN 有时更清晰。选择哪种写法，要看可读性、业务表达和执行计划。

### 3. 使用 = 时，子查询可以返回多行吗？

不可以。

使用 `=` 时，子查询应该返回单个值。如果可能返回多行，应使用 `IN`、`EXISTS` 或改写查询。

### 4. NOT IN 查询没有关联数据安全吗？

要小心。

如果子查询结果里可能有 `NULL`，`NOT IN` 容易出现不符合直觉的结果。查询“不存在关联数据”时，更推荐 `NOT EXISTS`。

### 5. FROM 子查询为什么必须起别名？

因为 `FROM` 子查询会被当成一张临时表，SQL 需要一个名字来引用它。

例如：

```sql
FROM (...) AS t
```

这个 `t` 就是子查询结果表的别名。
