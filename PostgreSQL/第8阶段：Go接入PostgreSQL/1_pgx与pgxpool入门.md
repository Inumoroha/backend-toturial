# 1. pgx 与 pgxpool 入门

本节目标：理解 `pgx` 和 `pgxpool` 的作用，完成一个最小 Go 程序连接 PostgreSQL 并执行查询。

`pgx` 是 PostgreSQL 的 Go 驱动和工具包。

常用包：

```go
github.com/jackc/pgx/v5
github.com/jackc/pgx/v5/pgxpool
```

对 Web 服务来说，最常用的是 `pgxpool`。

---

## 一、pgx 和 pgxpool 是什么

### pgx

`pgx` 提供 PostgreSQL 原生连接和查询能力。

单连接示例会用到：

```go
pgx.Connect
```

它适合：

- 命令行小工具。
- 简单脚本。
- 需要手动控制单个连接的场景。

### pgxpool

`pgxpool` 是连接池。

Web 服务通常有很多请求同时访问数据库。每个请求都新建连接会非常浪费，也可能压垮数据库。

连接池的作用是：

```text
启动时创建一组可复用连接。
请求来了，从池里借连接。
SQL 执行完，连接归还池中。
```

在 Web 服务中，优先使用：

```go
*pgxpool.Pool
```

mkdir go-pg-demo

```

go get github.com/jackc/pgx/v5/pgxpool
```

## 三、准备数据库连接字符串

PowerShell 中设置环境变量：

```powershell
$env:DATABASE_URL="postgres://postgres:postgres@localhost:5432/learn_pg?sslmode=disable"
```

连接字符串格式：

```text
postgres://用户名:密码@主机:端口/数据库名?sslmode=disable
```

本地开发常用 `sslmode=disable`。生产环境是否使用 SSL 要看部署方式。

## 四、最小连接示例

创建 `main.go`：

```go
package main

import (
	"context"
	"fmt"
	"log"
	"os"

	"github.com/jackc/pgx/v5/pgxpool"
)

func main() {
	ctx := context.Background()

	databaseURL := os.Getenv("DATABASE_URL")
	if databaseURL == "" {
		log.Fatal("DATABASE_URL is empty")
	}

	pool, err := pgxpool.New(ctx, databaseURL)
	if err != nil {
		log.Fatalf("create pool: %v", err)
	}
	defer pool.Close()

	if err := pool.Ping(ctx); err != nil {
		log.Fatalf("ping database: %v", err)
	}

	var now string
	err = pool.QueryRow(ctx, "SELECT now()::text").Scan(&now)
	if err != nil {
		log.Fatalf("query now: %v", err)
	}

	fmt.Println("database time:", now)
}
```

运行：

```powershell
go run .
```

如果连接成功，会输出数据库当前时间。

---

## 五、这段代码做了什么

### 1. 创建 context

```go
ctx := context.Background()
```

数据库操作都应该带 `context.Context`。

它可以控制：

- 超时。
- 取消。
- 请求生命周期。

### 2. 读取连接字符串

```go
databaseURL := os.Getenv("DATABASE_URL")
```

连接信息不要写死在代码里。

### 3. 创建连接池

```go
pool, err := pgxpool.New(ctx, databaseURL)
```

`pool` 是整个程序复用的连接池。

### 4. 关闭连接池

```go
defer pool.Close()
```

程序退出时关闭连接池。

### 5. 检查连接

```go
pool.Ping(ctx)
```

`Ping` 可以验证数据库是否可连接。

### 6. 查询单行

```go
pool.QueryRow(ctx, "SELECT now()::text").Scan(&now)
```

`QueryRow` 查询一行，`Scan` 把结果写入 Go 变量。

## 七、pgxpool 的使用原则

### 1. 程序启动时创建

不要在每个函数里创建连接池。

错误思路：

```go
func handler() {
	pool, _ := pgxpool.New(ctx, databaseURL)
	defer pool.Close()
}
```

这会让每个请求都创建连接池。

正确思路：

```text
main 中创建 pool。
把 pool 传给 repository、service、handler。
```

### 2. 整个服务复用

`*pgxpool.Pool` 可以被多个 goroutine 并发使用。

Web 服务里通常只需要一个全局连接池实例。

### 3. 程序退出时关闭

```go
defer pool.Close()
```

关闭后不能再使用。

## 八、常见错误

### 1. DATABASE_URL 为空

错误：

```text
DATABASE_URL is empty
```

解决：确认环境变量已设置。

### 2. 连接被拒绝

可能原因：

- PostgreSQL 没启动。
- 端口不对。
- Docker 容器没起来。
- 防火墙或网络问题。

### 3. 用户名或密码错误

检查连接字符串：

```text
postgres://postgres:postgres@localhost:5432/learn_pg?sslmode=disable
```

### 4. 数据库不存在

确认数据库名是否正确，例如 `learn_pg`。

## 九、常见误区

### 1. 每次查询都要创建一个连接池吗？

不是。

连接池应该长期复用。

### 2. pgxpool 是不是只适合大型项目？

不是。

只要是 Web 服务或并发程序，都应该使用连接池。

### 3. Ping 成功就代表所有 SQL 都正确吗？

不是。

`Ping` 只能说明数据库连接可用，不能说明表存在、权限正确、SQL 正确。

### 4. 本地开发可以把数据库密码写死吗？

练习可以临时写，但不要养成习惯。项目里优先使用环境变量或配置文件。

## 十、本节达标标准

学完本节后，你应该能够做到：

- 安装 `github.com/jackc/pgx/v5/pgxpool`。
- 使用 `DATABASE_URL` 连接 PostgreSQL。
- 使用 `pgxpool.New` 创建连接池。
- 使用 `Ping` 检查连接。
- 使用 `QueryRow` 执行简单查询。
- 知道连接池应该在程序启动时创建，并在退出时关闭。
