# 4. 查询单行和多行数据

本节目标：掌握 pgx 中 `QueryRow` 和 `Query` 的使用方式，能查询单个用户和用户列表，并正确处理没有数据、扫描错误和多行结果关闭。

Go 操作数据库最常见的两类查询：

```text
查询单行：根据 ID 查用户、根据邮箱查用户。
查询多行：用户列表、搜索结果、分页列表。
```

在 pgxpool 中：

- `QueryRow`：查询单行。
- `Query`：查询多行。

---

## 一、准备 User 结构体

```go
package user

import "time"

type User struct {
	ID          int64
	Email       string
	Username    string
	DisplayName *string
	Bio         *string
	CreatedAt   time.Time
	UpdatedAt   time.Time
}
```

这里 `DisplayName` 和 `Bio` 用 `*string`，因为数据库字段允许 `NULL`：

```sql
display_name text,
bio text
```

如果字段不允许 `NULL`，可以直接用 `string`。

---

## 二、查询单行：QueryRow

根据 ID 查询用户：

```go
func (r *Repository) GetByID(ctx context.Context, id int64) (*User, error) {
	var u User

	err := r.pool.QueryRow(ctx, `
		SELECT id, email, username, display_name, bio, created_at, updated_at
		FROM app.users
		WHERE id = $1
		  AND deleted_at IS NULL
	`, id).Scan(
		&u.ID,
		&u.Email,
		&u.Username,
		&u.DisplayName,
		&u.Bio,
		&u.CreatedAt,
		&u.UpdatedAt,
	)
	if err != nil {
		return nil, err
	}

	return &u, nil
}
```

`QueryRow` 的特点：

```text
它不会立刻返回错误。
错误会在 Scan 时返回。
```

所以必须调用 `Scan`。

---

## 三、处理没有数据：pgx.ErrNoRows

如果没有查到用户，`Scan` 会返回：

```go
pgx.ErrNoRows
```

可以转换成业务错误：

```go
var ErrUserNotFound = errors.New("user not found")
```

完整写法：

```go
func (r *Repository) GetByID(ctx context.Context, id int64) (*User, error) {
	var u User

	err := r.pool.QueryRow(ctx, `
		SELECT id, email, username, display_name, bio, created_at, updated_at
		FROM app.users
		WHERE id = $1
		  AND deleted_at IS NULL
	`, id).Scan(
		&u.ID,
		&u.Email,
		&u.Username,
		&u.DisplayName,
		&u.Bio,
		&u.CreatedAt,
		&u.UpdatedAt,
	)
	if err != nil {
		if errors.Is(err, pgx.ErrNoRows) {
			return nil, ErrUserNotFound
		}
		return nil, err
	}

	return &u, nil
}
```

## 四、根据邮箱查询用户

```go
func (r *Repository) GetByEmail(ctx context.Context, email string) (*User, error) {
	var u User

	err := r.pool.QueryRow(ctx, `
		SELECT id, email, username, display_name, bio, created_at, updated_at
		FROM app.users
		WHERE email = $1
		  AND deleted_at IS NULL
	`, email).Scan(
		&u.ID,
		&u.Email,
		&u.Username,
		&u.DisplayName,
		&u.Bio,
		&u.CreatedAt,
		&u.UpdatedAt,
	)
	if err != nil {
		if errors.Is(err, pgx.ErrNoRows) {
			return nil, ErrUserNotFound
		}
		return nil, err
	}

	return &u, nil
}
```

因为 `email` 有唯一约束：

```sql
CONSTRAINT users_email_unique UNIQUE (email)
```

所以最多返回一行，适合使用 `QueryRow`。

## 五、查询多行：Query

查询用户列表：

```go
func (r *Repository) List(ctx context.Context, limit int32, offset int32) ([]User, error) {
	rows, err := r.pool.Query(ctx, `
		SELECT id, email, username, display_name, bio, created_at, updated_at
		FROM app.users
		WHERE deleted_at IS NULL
		ORDER BY id
		LIMIT $1 OFFSET $2
	`, limit, offset)
	if err != nil {
		return nil, err
	}
	defer rows.Close()

	users := make([]User, 0)
	for rows.Next() {
		var u User
		if err := rows.Scan(
			&u.ID,
			&u.Email,
			&u.Username,
			&u.DisplayName,
			&u.Bio,
			&u.CreatedAt,
			&u.UpdatedAt,
		); err != nil {
			return nil, err
		}
		users = append(users, u)
	}

	if err := rows.Err(); err != nil {
		return nil, err
	}

	return users, nil
}
```

关键点：

- `defer rows.Close()` 必须写。
- `for rows.Next()` 逐行读取。
- 每行都要 `Scan`。
- 循环结束后检查 `rows.Err()`。

---

## 六、为什么 rows.Err 很重要

`rows.Next()` 过程中可能出现错误，例如：

- 网络中断。
- 数据库连接异常。
- 扫描中途失败。
- context 被取消。

如果不检查 `rows.Err()`，可能漏掉迭代过程中的错误。

标准结构应该始终是：

```go
rows, err := pool.Query(ctx, sql, args...)
if err != nil {
	return err
}
defer rows.Close()

for rows.Next() {
	// scan
}

if err := rows.Err(); err != nil {
	return err
}
```

## 七、使用 pgx.CollectRows

pgx v5 提供了一些辅助函数，可以减少样板代码。

例如：

```go
rows, err := r.pool.Query(ctx, `
	SELECT id, email, username, display_name, bio, created_at, updated_at
	FROM app.users
	WHERE deleted_at IS NULL
	ORDER BY id
	LIMIT $1 OFFSET $2
`, limit, offset)
if err != nil {
	return nil, err
}

users, err := pgx.CollectRows(rows, func(row pgx.CollectableRow) (User, error) {
	var u User
	err := row.Scan(
		&u.ID,
		&u.Email,
		&u.Username,
		&u.DisplayName,
		&u.Bio,
		&u.CreatedAt,
		&u.UpdatedAt,
	)
	return u, err
})
if err != nil {
	return nil, err
}

return users, nil
```

学习阶段建议先掌握手写 `rows.Next()` 的方式，因为它能帮助你理解连接释放和错误处理。

熟悉后再使用辅助函数简化代码。

## 八、NULL 字段如何处理

如果数据库字段可能为 `NULL`，Go 端可以用：

- 指针类型，例如 `*string`。
- `pgtype.Text`。
- `sql.NullString`。

本教程为了简单，用 `*string`：

```go
DisplayName *string
Bio         *string
```

如果数据库返回 `NULL`，指针就是 `nil`。

如果返回文本，指针指向对应字符串。

---

## 九、不要 SELECT *

不推荐：

```sql
SELECT *
FROM app.users
WHERE id = $1
```

原因：

- 表结构变更可能影响 `Scan` 顺序。
- 查询了不需要的字段。
- 代码不清楚依赖哪些列。

推荐明确列名：

```sql
SELECT id, email, username, display_name, bio, created_at, updated_at
FROM app.users
WHERE id = $1
```

## 十、常见误区

### 1. QueryRow 不需要 Scan 吗？

需要。

`QueryRow` 的错误会在 `Scan` 时返回。

### 2. 没查到数据是不是返回 nil, nil？

不是。

pgx 会返回 `pgx.ErrNoRows`，你应该把它转换成业务错误。

### 3. Query 多行不关闭 rows 可以吗？

不可以。

必须关闭，否则连接可能无法及时归还连接池。

### 4. SELECT * 更省事吗？

短期省事，长期容易出问题。

后端代码里建议明确列名。

## 十一、本节达标标准

学完本节后，你应该能够做到：

- 使用 `QueryRow` 查询单行。
- 使用 `Scan` 把结果写入结构体。
- 使用 `pgx.ErrNoRows` 处理没有数据。
- 使用 `Query` 查询多行。
- 正确关闭 `rows`。
- 检查 `rows.Err()`。
- 处理 nullable 字段。
- 避免在业务代码中使用 `SELECT *`。
