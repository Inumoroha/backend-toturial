# 5. 插入、更新、删除

本节目标：掌握 Go 中使用 pgx 执行 `INSERT`、`UPDATE`、`DELETE`，理解 `RETURNING` 和 `RowsAffected` 的用法。

在 pgx 中，写操作常用：

- `QueryRow` + `RETURNING`：插入后返回新数据。
- `Exec`：执行不需要返回行的更新或删除。
- `RowsAffected`：判断实际影响了多少行。

---

## 一、创建用户：INSERT RETURNING

创建输入结构体：

```go
type CreateUserParams struct {
	Email       string
	Username    string
	DisplayName *string
	Bio         *string
}
```

创建用户方法：

```go
func (r *Repository) Create(ctx context.Context, p CreateUserParams) (*User, error) {
	var u User

	err := r.pool.QueryRow(ctx, `
		INSERT INTO app.users (email, username, display_name, bio)
		VALUES ($1, $2, $3, $4)
		RETURNING id, email, username, display_name, bio, created_at, updated_at
	`, p.Email, p.Username, p.DisplayName, p.Bio).Scan(
		&u.ID,
		&u.Email,
		&u.Username,
		&u.DisplayName,
		&u.Bio,
		&u.CreatedAt,
		&u.UpdatedAt,
	)
	if err != nil {
		return nil, err
	}

	return &u, nil
}
```

`RETURNING` 是 PostgreSQL 非常实用的能力。

它可以在插入后直接返回：

- 自增 ID。
- 默认值。
- 数据库生成的时间。
- 最终写入的字段。

---

## 二、为什么 INSERT 常用 RETURNING

如果不用 `RETURNING`，你可能要：

```text
先 INSERT。
再 SELECT 查刚刚插入的数据。
```

这样多一次数据库往返。

使用 `RETURNING`：

```sql
INSERT ...
RETURNING ...
```

一次 SQL 就能完成。

## 三、更新用户：UPDATE RETURNING

更新输入：

```go
type UpdateUserParams struct {
	ID          int64
	Username    string
	DisplayName *string
	Bio         *string
}
```

更新并返回最新用户：

```go
func (r *Repository) Update(ctx context.Context, p UpdateUserParams) (*User, error) {
	var u User

	err := r.pool.QueryRow(ctx, `
		UPDATE app.users
		SET
			username = $2,
			display_name = $3,
			bio = $4,
			updated_at = now()
		WHERE id = $1
		  AND deleted_at IS NULL
		RETURNING id, email, username, display_name, bio, created_at, updated_at
	`, p.ID, p.Username, p.DisplayName, p.Bio).Scan(
		&u.ID,
		&u.Email,
		&u.Username,
		&u.DisplayName,
		&u.Bio,
		&u.CreatedAt,
		&u.UpdatedAt,
	)
	if err != nil {
		if errors.Is(err, pgx.ErrNoRows) {
			return nil, ErrUserNotFound
		}
		return nil, err
	}

	return &u, nil
}
```

如果没有匹配的用户，`UPDATE ... RETURNING` 不返回行，`Scan` 会得到 `pgx.ErrNoRows`。

## 四、局部更新怎么做

有些接口允许只更新传入字段。

简单做法是在 Go 里先查旧数据，再合并后整体更新。

也可以用 SQL 的 `COALESCE`：

```sql
UPDATE app.users
SET
    display_name = COALESCE($2, display_name),
    bio = COALESCE($3, bio),
    updated_at = now()
WHERE id = $1
RETURNING ...
```

但要注意：这种写法无法区分：

```text
字段没有传。
字段传了 null，想清空。
```

真实项目中局部更新要认真设计请求结构，避免语义混乱。

## 五、软删除用户

很多业务不直接物理删除用户，而是设置 `deleted_at`。

```go
func (r *Repository) Delete(ctx context.Context, id int64) error {
	tag, err := r.pool.Exec(ctx, `
		UPDATE app.users
		SET
			deleted_at = now(),
			updated_at = now()
		WHERE id = $1
		  AND deleted_at IS NULL
	`, id)
	if err != nil {
		return err
	}

	if tag.RowsAffected() == 0 {
		return ErrUserNotFound
	}

	return nil
}
```

这里使用 `Exec`，因为不需要返回行。

---

## 六、RowsAffected

`Exec` 返回 `pgconn.CommandTag`。

可以通过：

```go
tag.RowsAffected()
```

判断影响行数。

常见用途：

```go
if tag.RowsAffected() == 0 {
	return ErrUserNotFound
}
```

适合：

- 更新某个 ID。
- 删除某个 ID。
- 状态流转。
- 乐观锁更新。

---

## 七、物理删除

物理删除：

```go
func (r *Repository) HardDelete(ctx context.Context, id int64) error {
	tag, err := r.pool.Exec(ctx, `
		DELETE FROM app.users
		WHERE id = $1
	`, id)
	if err != nil {
		return err
	}

	if tag.RowsAffected() == 0 {
		return ErrUserNotFound
	}

	return nil
}
```

生产项目中是否物理删除，要看业务规则。

例如：

- 测试数据可以物理删除。
- 用户账号可能软删除。
- 订单通常不应该随便物理删除。

---

## 八、批量写入

少量批量插入可以多次执行 `Exec` 或 `QueryRow`。

大量批量写入时，pgx 提供 `CopyFrom`，性能更好。

学习阶段先掌握单条 CRUD。等需要导入大量数据时，再学习 `CopyFrom`。

## 十、常见误区

### 1. 插入后一定要再 SELECT 一次吗？

不需要。

PostgreSQL 可以使用 `RETURNING` 直接返回插入后的数据。

### 2. UPDATE 没更新到行会返回错误吗？

不一定。

如果使用 `Exec`，没更新到行通常不是错误，要看 `RowsAffected`。

如果使用 `UPDATE ... RETURNING` + `QueryRow`，没返回行会在 `Scan` 时得到 `pgx.ErrNoRows`。

### 3. 删除一定要 DELETE 吗？

不一定。

很多业务使用软删除，即更新 `deleted_at`。

### 4. `RowsAffected` 大于 1 一定正常吗？

要看业务。

如果你按 ID 更新，理论上最多 1 行。如果影响多行，说明 SQL 条件可能写错了。

---

## 十一、本节达标标准

学完本节后，你应该能够做到：

- 使用 `INSERT ... RETURNING` 创建用户并返回结果。
- 使用 `UPDATE ... RETURNING` 更新用户并返回结果。
- 使用 `Exec` 执行软删除。
- 使用 `RowsAffected` 判断是否真的更新或删除了数据。
- 区分软删除和物理删除。
- 理解 `RETURNING` 可以减少一次查询。
