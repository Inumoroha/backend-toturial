# 2. 唯一索引

本节目标：理解唯一索引的作用，掌握 `CREATE UNIQUE INDEX`，知道唯一索引和唯一约束的关系，并能在业务中用它保证数据不重复。

唯一索引有两个作用：

```text
1. 加速查询。
2. 保证某个字段或字段组合不重复。
```

例如用户邮箱：

```text
每个邮箱只能注册一个用户。
```

这类规则应该交给数据库用唯一约束或唯一索引保证，而不是只靠应用代码判断。

---

## 一、为什么需要唯一索引

错误思路：

```text
1. SELECT 查询邮箱是否存在。
2. 如果不存在，就 INSERT 新用户。
```

在并发场景下，两个请求可能同时查询到邮箱不存在，然后都插入成功。

正确思路是：

```text
数据库层面加唯一约束或唯一索引。
应用代码处理重复错误。
```

数据库是最终守门员。

---

## 二、创建唯一索引

先确保练习表中没有重复邮箱。

创建唯一索引：

```sql
CREATE UNIQUE INDEX perf_users_email_uidx
ON demo.perf_users (email);
```

命名上可以用 `_uidx` 表示 unique index。

之后执行：

```sql
INSERT INTO demo.perf_users (username, email)
VALUES ('duplicate_user', 'user_1@example.com');
```

会失败，因为 `user_1@example.com` 已经存在。

---

## 三、唯一索引也能用于查询

唯一索引不仅保证唯一，也能加速查询：

```sql
EXPLAIN
SELECT id, username, email
FROM demo.perf_users
WHERE email = 'user_50000@example.com';
```

可能看到：

```text
Index Scan using perf_users_email_uidx on perf_users
  Index Cond: (email = 'user_50000@example.com'::text)
```

因为唯一索引告诉 PostgreSQL：

```text
这个 email 最多只有一行。
```

这对优化器估算结果行数也有帮助。

---

## 四、唯一约束和唯一索引的关系

你可以在建表时写：

```sql
CREATE TABLE demo.unique_demo_users (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    email text NOT NULL UNIQUE
);
```

这里的 `UNIQUE` 是唯一约束。

PostgreSQL 会自动创建一个唯一索引用来实现这个约束。

也可以显式写：

```sql
CREATE UNIQUE INDEX unique_demo_users_email_uidx
ON demo.unique_demo_users (email);
```

二者关系可以先这样理解：

```text
唯一约束表达业务规则：这个字段不能重复。
唯一索引是数据库实现这个规则的底层手段，同时也能加速查询。
```

学习和建模时，如果它是明确业务规则，优先写成表约束通常更清晰：

```sql
email text NOT NULL UNIQUE
```

如果需要更灵活的索引能力，例如表达式索引、部分唯一索引，可以使用 `CREATE UNIQUE INDEX`。

---

## 五、组合唯一索引

有些唯一规则不是单个字段，而是多个字段组合。

例如订单明细中：

```text
同一个订单里，同一个商品只能出现一次。
```

可以设计：

```sql
CREATE TABLE demo.order_items (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    order_id bigint NOT NULL,
    product_id bigint NOT NULL,
    quantity integer NOT NULL,
    CONSTRAINT order_items_quantity_check CHECK (quantity > 0)
);

CREATE UNIQUE INDEX order_items_order_id_product_id_uidx
ON demo.order_items (order_id, product_id);
```

它表示：

```text
(order_id, product_id) 这个组合不能重复。
```

允许：

```text
order_id = 1, product_id = 10
order_id = 1, product_id = 11
order_id = 2, product_id = 10
```

不允许：

```text
order_id = 1, product_id = 10
order_id = 1, product_id = 10
```

## 六、大小写不敏感的唯一性

业务中经常希望邮箱大小写不敏感：

```text
Alice@example.com
alice@example.com
```

应该视为同一个邮箱。

一种做法是在写入前统一转小写，并用普通唯一索引：

```text
应用代码保证 email 存入数据库前统一 lower。
数据库对 email 建唯一索引。
```

另一种做法是表达式唯一索引：

```sql
CREATE UNIQUE INDEX perf_users_lower_email_uidx
ON demo.perf_users (lower(email));
```

这样下面两个值会冲突：

```sql
INSERT INTO demo.perf_users (username, email)
VALUES ('case_user_1', 'case_user@example.com');

INSERT INTO demo.perf_users (username, email)
VALUES ('case_user_2', 'CASE_USER@example.com');
```

注意：如果表里已经存在大小写不同但 lower 后相同的数据，创建这个索引会失败，需要先清理数据。

## 七、NULL 和唯一索引

在 PostgreSQL 中，普通唯一索引允许多个 `NULL`。

例如：

```sql
CREATE TABLE demo.unique_null_demo (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    phone text
);

CREATE UNIQUE INDEX unique_null_demo_phone_uidx
ON demo.unique_null_demo (phone);
```

下面通常可以插入多行：

```sql
INSERT INTO demo.unique_null_demo (phone)
VALUES (NULL), (NULL), (NULL);
```

因为 SQL 中 `NULL` 表示未知，多个未知不被认为相等。

如果业务要求字段必须唯一，通常还应该加 `NOT NULL`：

```sql
phone text NOT NULL UNIQUE
```

---

## 八、唯一索引和 ON CONFLICT

`INSERT ... ON CONFLICT` 依赖唯一约束、主键或唯一索引判断冲突。

例如：

```sql
INSERT INTO demo.perf_users (username, email)
VALUES ('new_user', 'new_user@example.com')
ON CONFLICT (email)
DO UPDATE SET username = EXCLUDED.username
RETURNING id, username, email;
```

这里要求 `email` 上存在唯一约束或唯一索引。

否则 PostgreSQL 不知道什么叫“email 冲突”。

---

## 九、删除唯一索引的风险

删除唯一索引：

```sql
DROP INDEX demo.perf_users_email_uidx;
```

如果这个唯一索引承载着业务唯一性规则，删除后数据库就不再阻止重复数据。

所以删除唯一索引前要确认：

- 它是不是业务规则的一部分。
- 有没有唯一约束依赖它。
- 应用代码是否依赖它做 `ON CONFLICT`。

如果是唯一约束自动创建的索引，不能简单通过 `DROP INDEX` 删除，通常需要删除约束。

## 十、常见误区

### 1. 唯一索引只是为了查询更快吗？

不是。

它更重要的作用是保证数据不重复。

### 2. 应用层先查再插入就能保证唯一吗？

不能。

并发请求下会有竞态条件。数据库唯一约束或唯一索引才是最终保障。

### 3. UNIQUE 字段允许多个 NULL 是不是 bug？

不是。

这是 SQL 中 `NULL` 语义导致的行为。业务需要必填唯一时，加 `NOT NULL`。

### 4. 所有唯一规则都只能单字段实现吗？

不是。

可以使用组合唯一索引表达多字段组合唯一。

- 理解唯一约束和唯一索引的关系。
- 创建组合唯一索引。
- 知道普通唯一索引允许多个 `NULL`。
- 使用唯一索引支持 `ON CONFLICT`。
