# 4. Array 类型

本节目标：理解 PostgreSQL 的 Array 类型，掌握数组字段的创建、插入、查询和更新，并知道什么时候应该使用数组，什么时候应该使用关联表。

PostgreSQL 支持数组类型，例如：

```sql
text[]
integer[]
uuid[]
```

数组可以存储多个值：

```text
ARRAY['PostgreSQL', 'JSONB', 'Backend']
```

它很方便，但也很容易被滥用。

---

## 一、创建数组字段

文章标签示例：

```sql
CREATE TABLE demo.array_articles (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title text NOT NULL,
    tags text[] NOT NULL DEFAULT ARRAY[]::text[],
    created_at timestamptz NOT NULL DEFAULT now()
);
```

这里：

```sql
tags text[] NOT NULL DEFAULT ARRAY[]::text[]
```

表示 `tags` 是文本数组，默认是空数组。

---

## 二、插入数组数据

```sql
INSERT INTO demo.array_articles (title, tags)
VALUES
    ('PostgreSQL Array 入门', ARRAY['PostgreSQL', 'Array', 'Backend']),
    ('JSONB 和 GIN 索引', ARRAY['PostgreSQL', 'JSONB', 'GIN']),
    ('Go 后端开发', ARRAY['Go', 'Backend']);
```

也可以使用数组字面量：

```sql
INSERT INTO demo.array_articles (title, tags)
VALUES ('全文搜索基础', '{PostgreSQL,Search}'::text[]);
```

学习阶段更推荐 `ARRAY[...]`，可读性更好。

---

## 三、查询数组

### 1. 查询包含某个标签

```sql
SELECT id, title, tags
FROM demo.array_articles
WHERE 'PostgreSQL' = ANY(tags);
```

`ANY(tags)` 表示和数组中的任意元素比较。

### 2. 使用包含操作符 `@>`

```sql
SELECT id, title, tags
FROM demo.array_articles
WHERE tags @> ARRAY['PostgreSQL'];
```

`@>` 表示左边数组是否包含右边数组。

### 3. 查询同时包含多个标签

```sql
SELECT id, title, tags
FROM demo.array_articles
WHERE tags @> ARRAY['PostgreSQL', 'GIN'];
```

表示 `tags` 必须同时包含 `PostgreSQL` 和 `GIN`。

### 4. 查询是否有任意标签重叠

```sql
SELECT id, title, tags
FROM demo.array_articles
WHERE tags && ARRAY['JSONB', 'Search'];
```

`&&` 表示两个数组是否有交集。

---

## 四、数组展开

使用 `unnest` 把数组拆成多行：

```sql
SELECT
    id,
    title,
    unnest(tags) AS tag
FROM demo.array_articles;
```

如果想统计每个标签出现次数：

```sql
SELECT
    tag,
    COUNT(*) AS article_count
FROM demo.array_articles,
LATERAL unnest(tags) AS tag
GROUP BY tag
ORDER BY article_count DESC, tag;
```

能做，但复杂度已经开始接近关联表。

如果标签统计是高频需求，关联表通常更清晰。

---

## 五、更新数组

### 1. 追加元素

```sql
UPDATE demo.array_articles
SET tags = array_append(tags, 'Database')
WHERE id = 1;
```

### 2. 删除元素

```sql
UPDATE demo.array_articles
SET tags = array_remove(tags, 'Backend')
WHERE id = 1;
```

### 3. 整体替换

```sql
UPDATE demo.array_articles
SET tags = ARRAY['PostgreSQL', 'Array']
WHERE id = 1;
```

---

## 六、防止数组重复元素

PostgreSQL 不会自动阻止数组内部重复：

```sql
ARRAY['PostgreSQL', 'PostgreSQL']
```

如果业务要求不重复，需要应用层处理，或写更复杂的数据库约束。

这也是数组不适合复杂关系的重要原因之一。

关联表可以天然用主键避免重复：

```sql
CREATE TABLE demo.article_tags_relational (
    article_id bigint NOT NULL,
    tag text NOT NULL,
    PRIMARY KEY (article_id, tag)
);
```

---

## 七、给数组建立 GIN 索引

如果经常查询：

```sql
WHERE tags @> ARRAY['PostgreSQL']
```

可以创建 GIN 索引：

```sql
CREATE INDEX array_articles_tags_gin_idx
ON demo.array_articles
USING gin (tags);
```

然后查看执行计划：

```sql
EXPLAIN
SELECT id, title
FROM demo.array_articles
WHERE tags @> ARRAY['PostgreSQL'];
```

GIN 索引很适合数组包含和重叠查询。

---

## 八、数组适合的场景

适合：

- 小规模标签。
- 简单选项集合。
- 不需要单独管理元素属性。
- 不需要外键约束。
- 不需要频繁统计和复杂查询。

例如：

```text
文章临时标签。
用户快捷入口配置。
简单权限标记。
少量搜索关键词缓存。
```

---

## 九、什么时候应该用关联表

如果数组里的元素逐渐变成真正的业务对象，就应该使用关联表。

例如标签系统需要：

- 标签有 ID。
- 标签有名称、颜色、描述。
- 标签需要审核。
- 标签需要统计文章数。
- 标签需要重命名。
- 标签需要权限控制。
- 标签需要排序。
- 标签需要被其他表引用。

这时应该设计：

```text
articles
tags
article_tags
```

而不是：

```text
articles.tags text[]
```

---

## 十、Array 和 JSONB 的区别

Array 是同类型值的列表：

```sql
text[]
integer[]
uuid[]
```

JSONB 可以是对象、数组、字符串、数字、布尔：

```json
{"theme": "dark", "shortcuts": ["search", "new_article"]}
```

如果只是“同类型的一组值”，Array 更直接。

如果是“有层次结构的配置对象”，JSONB 更自然。

---

## 十一、常见误区

### 1. Array 可以替代多对多关联表吗？

不建议。

只有当这个关系非常简单、不需要强约束和复杂维护时，才考虑数组。

### 2. 数组内部会自动去重吗？

不会。

需要应用层或额外约束处理。

### 3. 数组查询一定很慢吗？

不一定。

配合 GIN 索引，包含和重叠查询可以很好用。

### 4. Array 和 JSONB 随便选一个就行吗？

不是。

同类型值列表用 Array 更直接；结构化配置对象用 JSONB 更自然。

---

## 十二、本节达标标准

学完本节后，你应该能够做到：

- 创建数组字段，例如 `text[]`。
- 插入数组数据。
- 使用 `ANY`、`@>`、`&&` 查询数组。
- 使用 `unnest` 展开数组。
- 使用 `array_append`、`array_remove` 更新数组。
- 为数组字段创建 GIN 索引。
- 判断什么时候用 Array，什么时候用关联表。
