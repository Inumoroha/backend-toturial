# 6. 枚举类型

本节目标：理解 PostgreSQL 枚举类型的作用，掌握 `CREATE TYPE ... AS ENUM`，并知道枚举类型、`CHECK` 约束、字典表分别适合什么场景。

枚举类型适合表达：

```text
取值有限、含义稳定、不经常变化的状态集合。
```

例如文章状态：

```text
draft
published
archived
```

订单状态：

```text
pending
paid
cancelled
refunded
```

---

## 一、创建枚举类型

创建文章状态枚举：

```sql
CREATE TYPE demo.article_status_enum AS ENUM (
    'draft',
    'published',
    'archived'
);
```

使用枚举类型建表：

```sql
CREATE TABLE demo.enum_articles (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title text NOT NULL,
    content text NOT NULL,
    status demo.article_status_enum NOT NULL DEFAULT 'draft',
    created_at timestamptz NOT NULL DEFAULT now(),
    published_at timestamptz
);
```

插入数据：

```sql
INSERT INTO demo.enum_articles (title, content, status, published_at)
VALUES
    ('枚举类型入门', '学习 PostgreSQL enum。', 'published', now()),
    ('草稿文章', '还没有发布。', 'draft', NULL);
```

---

## 二、枚举类型会限制非法值

下面这条会失败：

```sql
INSERT INTO demo.enum_articles (title, content, status)
VALUES ('错误状态', '测试非法枚举值。', 'deleted');
```

因为 `deleted` 不在枚举定义中。

这就是枚举的价值：

```text
数据库层面保证字段只能取某些固定值。
```

---

## 三、查询枚举字段

```sql
SELECT id, title, status
FROM demo.enum_articles
WHERE status = 'published';
```

排序：

```sql
SELECT id, title, status
FROM demo.enum_articles
ORDER BY status;
```

注意：枚举的排序顺序由创建枚举时的值顺序决定。

例如：

```sql
CREATE TYPE demo.article_status_enum AS ENUM (
    'draft',
    'published',
    'archived'
);
```

排序时顺序就是：

```text
draft < published < archived
```

不要把枚举排序当成字母排序。

---

## 四、给枚举字段建索引

如果经常按状态筛选：

```sql
SELECT id, title
FROM demo.enum_articles
WHERE status = 'published';
```

可以建普通 B-Tree 索引：

```sql
CREATE INDEX enum_articles_status_idx
ON demo.enum_articles (status);
```

如果还经常按发布时间排序：

```sql
SELECT id, title, published_at
FROM demo.enum_articles
WHERE status = 'published'
ORDER BY published_at DESC
LIMIT 20;
```

可以建组合索引：

```sql
CREATE INDEX enum_articles_status_published_at_idx
ON demo.enum_articles (status, published_at DESC);
```

## 五、添加新的枚举值

如果以后需要增加状态：

```text
reviewing
```

可以：

```sql
ALTER TYPE demo.article_status_enum
ADD VALUE 'reviewing';
```

也可以指定位置：

```sql
ALTER TYPE demo.article_status_enum
ADD VALUE 'reviewing' BEFORE 'published';
```

注意：枚举类型适合变化少的集合。如果状态经常新增、删除、重命名，枚举可能不适合。

---

## 六、枚举值重命名

可以重命名枚举值：

```sql
ALTER TYPE demo.article_status_enum
RENAME VALUE 'archived' TO 'hidden';
```

但生产环境中要谨慎，因为应用代码、接口文档、历史数据、报表逻辑可能都依赖旧值。

状态值不只是数据库字段，它通常也是业务协议的一部分。

## 七、枚举类型、CHECK 约束、字典表对比

### 1. 枚举类型

```sql
CREATE TYPE demo.article_status_enum AS ENUM ('draft', 'published', 'archived');
```

适合：

- 状态集合很稳定。
- 多张表可能复用同一状态类型。
- 希望数据库类型层面限制取值。

缺点：

- 修改不如普通表灵活。
- 删除枚举值不方便。
- 业务状态经常变化时维护成本高。

### 2. CHECK 约束

```sql
status text NOT NULL CHECK (status IN ('draft', 'published', 'archived'))
```

适合：

- 单表使用。
- 取值简单。
- 想保持字段是普通 `text`。

优点：

- 写法直观。
- 修改约束相对容易。

缺点：

- 多表复用时容易重复定义。

### 3. 字典表

```sql
CREATE TABLE demo.article_statuses (
    code text PRIMARY KEY,
    name text NOT NULL,
    sort_order integer NOT NULL
);
```

业务表引用：

```sql
status_code text NOT NULL REFERENCES demo.article_statuses (code)
```

适合：

- 状态需要后台管理。
- 状态有名称、排序、颜色、描述等属性。
- 状态可能经常新增或调整。
- 需要多语言显示。

缺点：

- 多一张表和外键。
- 查询时有时需要 JOIN。

---

## 八、什么时候适合枚举

适合：

- 性别这类非常稳定的有限值，前提是业务定义明确。
- 文章状态：`draft`、`published`、`archived`。
- 任务状态：`pending`、`running`、`success`、`failed`。
- 通知渠道：`email`、`sms`、`push`，如果集合稳定。

不适合：

- 后台可配置的分类。
- 经常新增的业务类型。
- 需要显示名称、颜色、排序的状态。
- 用户自定义标签。
- 多语言字典。

## 九、枚举和应用代码

后端代码里通常也会定义状态常量。

例如 Go 代码中可能有：

```go
const (
    ArticleStatusDraft     = "draft"
    ArticleStatusPublished = "published"
    ArticleStatusArchived  = "archived"
)
```

数据库枚举和应用常量要保持一致。

如果修改枚举值，需要同步：

- 数据库迁移。
- 后端常量。
- 前端显示。
- API 文档。
- 测试用例。

## 十、常见误区

### 1. 所有状态字段都应该用 enum？

不是。

状态非常稳定时适合 enum。变化频繁时，`CHECK` 或字典表可能更合适。

### 2. enum 比 text 一定更好？

不一定。

enum 类型更严格，但也更不灵活。

### 3. enum 可以随便删除值吗？

不建议这样理解。

枚举值一旦被历史数据和应用代码使用，删除或大改都需要谨慎迁移。

### 4. enum 能替代业务状态机吗？

不能。

enum 只能限制字段取值，不能自动保证状态流转合法。例如它不能自动阻止文章从 `archived` 变回 `draft`，这类规则仍然需要业务代码或触发器等机制处理。

---

## 十一、本节达标标准

学完本节后，你应该能够做到：

- 使用 `CREATE TYPE ... AS ENUM` 创建枚举类型。
- 在表字段中使用枚举类型。
- 插入和查询枚举值。
- 使用 `ALTER TYPE ... ADD VALUE` 增加枚举值。
- 知道枚举排序由定义顺序决定。
- 区分 enum、`CHECK` 约束和字典表的适用场景。
- 知道 enum 不能替代业务状态机。
