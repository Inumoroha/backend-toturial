# 7. 范围类型

本节目标：理解 PostgreSQL 范围类型的作用，掌握时间范围和数字范围的基本使用，并能用范围类型判断时间段是否重叠。

范围类型用来表达：

```text
从某个起点到某个终点之间的一段区间。
```

例如：

```text
会议室预约时间：2026-07-10 10:00 到 2026-07-10 12:00
商品价格区间：100 到 200
活动有效期：2026-07-01 到 2026-07-31
```

PostgreSQL 不只是能存两个字段：

```text
start_time
end_time
```

它还提供了专门的范围类型和操作符，方便做包含、重叠、相邻等判断。

---

## 一、常见范围类型

PostgreSQL 常见范围类型：

| 类型 | 含义 |
| --- | --- |
| `int4range` | integer 范围 |
| `int8range` | bigint 范围 |
| `numrange` | numeric 范围 |
| `daterange` | date 范围 |
| `tsrange` | timestamp 范围，不带时区 |
| `tstzrange` | timestamptz 范围，带时区 |

后端业务中最常用的是：

```text
tstzrange
daterange
numrange
```

---

## 二、创建范围值

时间范围：

```sql
SELECT tstzrange(
    '2026-07-10 10:00:00+08',
    '2026-07-10 12:00:00+08',
    '[)'
);
```

最后一个参数 `'[)'` 表示：

```text
包含左边界，不包含右边界。
```

也就是：

```text
[10:00, 12:00)
```

这种写法非常适合时间段，因为两个连续时间段不会重叠：

```text
[10:00, 12:00)
[12:00, 14:00)
```

12:00 这一刻只属于第二个时间段。

---

## 三、边界符号

范围边界常见写法：

| 写法 | 含义 |
| --- | --- |
| `[)` | 包含下界，不包含上界 |
| `[]` | 包含下界，包含上界 |
| `()` | 不包含下界，不包含上界 |
| `(]` | 不包含下界，包含上界 |

时间段通常推荐：

```text
[)
```

也就是包含开始时间，不包含结束时间。

---

## 四、创建活动表

```sql
CREATE TABLE demo.range_events (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name text NOT NULL,
    room_name text NOT NULL,
    active_period tstzrange NOT NULL,
    created_at timestamptz NOT NULL DEFAULT now(),
    CONSTRAINT range_events_period_not_empty_check
        CHECK (NOT isempty(active_period))
);
```

插入数据：

```sql
INSERT INTO demo.range_events (name, room_name, active_period)
VALUES
    (
        'PostgreSQL 分享会',
        'A101',
        tstzrange('2026-07-10 10:00:00+08', '2026-07-10 12:00:00+08', '[)')
    ),
    (
        'Go 后端讨论',
        'A101',
        tstzrange('2026-07-10 14:00:00+08', '2026-07-10 16:00:00+08', '[)')
    );
```

---

## 五、判断是否包含某个时间点

查询某个时间点正在进行的活动：

```sql
SELECT id, name, room_name, active_period
FROM demo.range_events
WHERE active_period @> '2026-07-10 10:30:00+08'::timestamptz;
```

`@>` 表示：

```text
左边范围是否包含右边元素。
```

---

## 六、判断两个时间段是否重叠

查询和某个新预约时间冲突的活动：

```sql
SELECT id, name, room_name, active_period
FROM demo.range_events
WHERE room_name = 'A101'
  AND active_period && tstzrange(
        '2026-07-10 11:00:00+08',
        '2026-07-10 13:00:00+08',
        '[)'
    );
```

`&&` 表示两个范围是否重叠。

上面的新时间段：

```text
[11:00, 13:00)
```

会和已有的：

```text
[10:00, 12:00)
```

重叠。

---

## 七、连续时间段不重叠

查询：

```sql
SELECT tstzrange(
    '2026-07-10 10:00:00+08',
    '2026-07-10 12:00:00+08',
    '[)'
) && tstzrange(
    '2026-07-10 12:00:00+08',
    '2026-07-10 14:00:00+08',
    '[)'
) AS is_overlap;
```

结果是：

```text
false
```

因为第一个不包含 12:00，第二个从 12:00 开始。

这就是 `[)` 在时间段建模中很实用的原因。

---

## 八、取范围上下界

```sql
SELECT
    name,
    lower(active_period) AS starts_at,
    upper(active_period) AS ends_at
FROM demo.range_events;
```

`lower` 取下界。

`upper` 取上界。

判断边界是否包含：

```sql
SELECT
    lower_inc(active_period) AS include_start,
    upper_inc(active_period) AS include_end
FROM demo.range_events;
```

## 九、数字范围示例

价格区间：

```sql
CREATE TABLE demo.range_price_rules (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name text NOT NULL,
    price_range numrange NOT NULL
);
```

插入：

```sql
INSERT INTO demo.range_price_rules (name, price_range)
VALUES
    ('low', numrange(0, 100, '[)')),
    ('middle', numrange(100, 500, '[)')),
    ('high', numrange(500, NULL, '[)'));
```

查询某个价格属于哪个区间：

```sql
SELECT name, price_range
FROM demo.range_price_rules
WHERE price_range @> 199.00;
```

## 十、范围类型和两个字段对比

传统设计：

```sql
start_at timestamptz NOT NULL,
end_at timestamptz NOT NULL
```

范围类型设计：

```sql
active_period tstzrange NOT NULL
```

范围类型优点：

- 一个字段表达一个区间概念。
- 操作符丰富，例如 `@>`、`&&`。
- 边界包含关系清晰。
- 更适合做重叠判断。

两个字段优点：

- 更直观。
- 很多开发者更熟悉。
- 前端表单通常也是开始时间和结束时间。

真实项目中可以这样折中：

```text
表里存 start_at 和 end_at。
查询时用 tstzrange(start_at, end_at, '[)') 做范围判断。
```

或者直接存范围类型，看团队熟悉程度和查询需求。

## 十一、范围重叠约束

会议室预约常见需求：

```text
同一个会议室，同一时间不能有两个预约重叠。
```

PostgreSQL 可以用排他约束表达这类规则，但需要更多索引知识。

学习阶段先理解查询写法：

```sql
SELECT 1
FROM demo.range_events
WHERE room_name = 'A101'
  AND active_period && tstzrange($1, $2, '[)');
```

如果查到记录，说明时间冲突。

后续进阶时可以继续学习：

```text
EXCLUDE USING gist
```

它能把“不允许范围重叠”变成数据库约束。

## 十二、常见误区

### 1. 范围类型只是两个字段的替代品吗？

不只是。

范围类型还提供了包含、重叠、上下界等专门操作。

### 2. 时间范围应该用 `[]` 吗？

通常不推荐。

时间段更常用 `[)`，可以自然表达连续区间不重叠。

### 3. 范围类型适合所有时间字段吗？

不是。

单个时间点仍然用 `timestamptz`。只有表达时间段时才考虑范围类型。

### 4. 有了范围类型，就自动防止预约冲突吗？

不会。

范围类型提供表达和查询能力。防止冲突还需要查询校验、事务锁，或进一步使用排他约束。

学完本节后，你应该能够做到：

- 说出常见范围类型，例如 `tstzrange`、`daterange`、`numrange`。
- 创建范围字段。
- 使用 `[)` 表达时间段。
- 使用 `@>` 判断范围是否包含元素。
- 使用 `&&` 判断范围是否重叠。
- 使用 `lower`、`upper` 获取范围边界。
- 用范围类型实现会议室预约冲突查询。
- 知道范围类型不等于自动防止冲突。
