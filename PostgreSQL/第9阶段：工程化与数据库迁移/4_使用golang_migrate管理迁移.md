# 4. 使用 golang-migrate 管理迁移

本节目标：掌握 golang-migrate 的基本用法，能创建、执行、回滚 PostgreSQL 迁移，并理解它和 goose 的文件组织差异。

golang-migrate 通常使用成对迁移文件：

```text
000001_create_users.up.sql
000001_create_users.down.sql
```

其中：

- `.up.sql`：正向迁移。
- `.down.sql`：回滚迁移。

---

## 一、安装 migrate CLI

常见安装方式：

```powershell
go install github.com/golang-migrate/migrate/v4/cmd/migrate@latest
```

确认：

```powershell
migrate -version
```

如果找不到 `migrate`，检查 Go bin 目录是否在 PATH 中。

---

## 二、准备目录

```text
go-pg-demo/
  migrations/
```

创建：

```powershell
mkdir migrations
```

---

## 三、准备连接字符串

```powershell
$env:DATABASE_URL="postgres://postgres:postgres@localhost:5432/learn_pg?sslmode=disable"
```

golang-migrate 常见命令格式：

```powershell
migrate -path migrations -database $env:DATABASE_URL up
```

也可以使用 `file://` source：

```powershell
migrate -source file://migrations -database $env:DATABASE_URL up
```

学习阶段使用 `-path migrations` 更短。

---

## 四、创建迁移文件

```powershell
migrate create -ext sql -dir migrations -seq create_users
```

可能生成：

```text
migrations/000001_create_users.up.sql
migrations/000001_create_users.down.sql
```

参数含义：

- `-ext sql`：生成 SQL 文件。
- `-dir migrations`：迁移目录。
- `-seq`：使用顺序编号。
- `create_users`：迁移名称。

---

## 五、编写 create_users.up.sql

```sql
CREATE SCHEMA IF NOT EXISTS app;

CREATE TABLE app.users (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    email text NOT NULL,
    username text NOT NULL,
    display_name text,
    bio text,
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now(),
    deleted_at timestamptz,
    CONSTRAINT users_email_unique UNIQUE (email),
    CONSTRAINT users_email_not_empty_check CHECK (length(trim(email)) > 0),
    CONSTRAINT users_username_not_empty_check CHECK (length(trim(username)) > 0)
);

CREATE INDEX users_deleted_at_idx
ON app.users (deleted_at);
```

---

## 六、编写 create_users.down.sql

```sql
DROP TABLE app.users;
```

同样，通常不在这里删除 `app` schema，因为其他表可能使用它。

---

## 七、执行迁移

执行所有未执行迁移：

```powershell
migrate -path migrations -database $env:DATABASE_URL up
```

只执行一步：

```powershell
migrate -path migrations -database $env:DATABASE_URL up 1
```

查看版本：

```powershell
migrate -path migrations -database $env:DATABASE_URL version
```

---

## 八、回滚迁移

回滚一步：

```powershell
migrate -path migrations -database $env:DATABASE_URL down 1
```

注意：golang-migrate 的 `down` 通常需要指定步数，或进入交互确认。学习阶段建议明确写：

```powershell
down 1
```

避免误回滚太多。

---

## 九、新增 avatar_url 字段

创建迁移：

```powershell
migrate create -ext sql -dir migrations -seq add_users_avatar_url
```

`000002_add_users_avatar_url.up.sql`：

```sql
ALTER TABLE app.users
ADD COLUMN avatar_url text;
```

`000002_add_users_avatar_url.down.sql`：

```sql
ALTER TABLE app.users
DROP COLUMN avatar_url;
```

执行：

```powershell
migrate -path migrations -database $env:DATABASE_URL up
```

回滚一步：

```powershell
migrate -path migrations -database $env:DATABASE_URL down 1
```

重新执行：

```powershell
migrate -path migrations -database $env:DATABASE_URL up
```

## 十、dirty 状态是什么

如果迁移执行到一半失败，golang-migrate 可能把数据库标记为 dirty。

这表示：

```text
迁移状态不确定，需要人工检查和修复。
```

这时继续执行迁移通常会失败。

处理思路：

1. 查看失败的迁移文件。
2. 检查数据库实际结构。
3. 手动修复数据库到一致状态。
4. 使用 `force VERSION` 修正迁移版本。

例如：

```powershell
migrate -path migrations -database $env:DATABASE_URL force 1
```

`force` 很危险，不是用来跳过错误的日常命令。只有确认数据库状态后才使用。

## 十一、golang-migrate 常用命令

| 命令 | 作用 |
| --- | --- |
| `create` | 创建成对迁移文件 |
| `up` | 执行迁移 |
| `up 1` | 执行 1 步 |
| `down 1` | 回滚 1 步 |
| `version` | 查看当前版本 |
| `force VERSION` | 强制设置版本，常用于修复 dirty 状态 |

学习阶段重点掌握：

```text
create
up
down 1
version
```

## 十二、goose 和 golang-migrate 怎么选

两者都可以用于 Go + PostgreSQL 项目。

goose 的特点：

- 一个 SQL 文件里包含 up/down。
- CLI 用法直观。
- 支持 Go 迁移函数。
- 支持嵌入迁移等 Go 项目友好能力。

golang-migrate 的特点：

- up/down 分文件。
- 生态广，很多项目使用。
- CLI 和库都常见。
- dirty 状态机制明确。

学习阶段可以选一个深入使用。

如果团队已经有工具，就跟团队。

## 十三、常见误区

### 1. golang-migrate 和 goose 要同时用吗？

不建议。

一个项目选一个迁移工具，避免版本表和迁移状态混乱。

### 2. dirty 状态可以直接 force 跳过吗？

不应该。

必须先检查数据库真实状态。

### 3. down 不写也没关系吗？

学习阶段建议写。生产中不可逆迁移也要明确说明。

### 4. 可以手动改 schema_migrations 吗？

不建议。

除非你正在处理迁移事故，并且非常清楚版本状态。

## 十四、本节达标标准

学完本节后，你应该能够做到：

- 安装 migrate CLI。
- 使用 `migrate create` 创建 up/down 文件。
- 编写 `create_users.up.sql` 和 `create_users.down.sql`。
- 使用 `migrate up` 执行迁移。
- 使用 `migrate down 1` 回滚一步。
- 使用 `migrate version` 查看版本。
- 理解 dirty 状态和 `force` 的风险。
- 知道 goose 和 golang-migrate 不应该在同一项目混用。
