# 3. 使用 goose 管理迁移

本节目标：掌握 goose 的基本用法，能创建、执行、查看、回滚 PostgreSQL 迁移。

goose 是一个数据库迁移工具，既可以作为 CLI 使用，也可以作为 Go 库使用。

本节重点学习 CLI 用法。

---

## 一、安装 goose

常见安装方式：

```powershell
go install github.com/pressly/goose/v3/cmd/goose@latest
```

安装后确认：

```powershell
goose -version
```

如果提示找不到 `goose`，通常是 Go 的 bin 目录没有加入 PATH。

可以查看：

```powershell
go env GOPATH
```

一般可执行文件会安装到：

```text
%GOPATH%\bin
```

## 二、准备目录

项目结构：

```text
go-pg-demo/
  migrations/
```

创建目录：

```powershell
mkdir migrations
```

---

## 三、准备连接字符串

PowerShell：

```powershell
$env:DATABASE_URL="postgres://postgres:postgres@localhost:5432/learn_pg?sslmode=disable"
```

goose 可以直接使用：

```powershell
goose -dir migrations postgres $env:DATABASE_URL status
```

也可以使用环境变量：

```powershell
$env:GOOSE_DRIVER="postgres"
$env:GOOSE_DBSTRING=$env:DATABASE_URL
$env:GOOSE_MIGRATION_DIR="migrations"

goose status
```

## 四、创建迁移文件

创建 `users` 表迁移：

```powershell
goose -dir migrations create create_users sql
```

如果希望使用顺序编号，可以：

```powershell
goose -dir migrations -s create create_users sql
```

可能生成：

```text
migrations/00001_create_users.sql
```

---

## 五、编写 create_users 迁移

```sql
-- +goose Up
-- +goose StatementBegin
CREATE SCHEMA IF NOT EXISTS app;

CREATE TABLE app.users (
    id bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    email text NOT NULL,
    username text NOT NULL,
    display_name text,
    bio text,
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now(),
    deleted_at timestamptz,
    CONSTRAINT users_email_unique UNIQUE (email),
    CONSTRAINT users_email_not_empty_check CHECK (length(trim(email)) > 0),
    CONSTRAINT users_username_not_empty_check CHECK (length(trim(username)) > 0)
);

CREATE INDEX users_deleted_at_idx
ON app.users (deleted_at);
-- +goose StatementEnd

-- +goose Down
-- +goose StatementBegin
DROP TABLE app.users;
-- +goose StatementEnd
```

注意：

```sql
CREATE SCHEMA IF NOT EXISTS app;
```

放在 up 中，确保 schema 存在。

down 中这里只删除表，没有删除 schema，因为其他表可能也使用 `app` schema。

## 六、执行迁移

```powershell
goose -dir migrations postgres $env:DATABASE_URL up
```

执行后，goose 会：

- 创建版本记录表。
- 执行未执行的迁移。
- 记录当前版本。

查看状态：

```powershell
goose -dir migrations postgres $env:DATABASE_URL status
```

查看当前版本：

```powershell
goose -dir migrations postgres $env:DATABASE_URL version
```

---

## 七、回滚迁移

回滚最近一条迁移：

```powershell
goose -dir migrations postgres $env:DATABASE_URL down
```

再查看状态：

```powershell
goose -dir migrations postgres $env:DATABASE_URL status
```

重新执行：

```powershell
goose -dir migrations postgres $env:DATABASE_URL up
```

这就是推荐实践中的：

```text
执行迁移。
回滚一次。
重新执行。
确认 up/down 都能工作。
```

## 八、新增 avatar_url 字段

创建迁移：

```powershell
goose -dir migrations -s create add_users_avatar_url sql
```

文件内容：

```sql
-- +goose Up
-- +goose StatementBegin
ALTER TABLE app.users
ADD COLUMN avatar_url text;
-- +goose StatementEnd

-- +goose Down
-- +goose StatementBegin
ALTER TABLE app.users
DROP COLUMN avatar_url;
-- +goose StatementEnd
```

执行：

```powershell
goose -dir migrations postgres $env:DATABASE_URL up
```

回滚：

```powershell
goose -dir migrations postgres $env:DATABASE_URL down
```

重新执行：

```powershell
goose -dir migrations postgres $env:DATABASE_URL up
```

## 九、goose 常用命令

| 命令 | 作用 |
| --- | --- |
| `create NAME sql` | 创建 SQL 迁移文件 |
| `up` | 执行所有未执行迁移 |
| `up-by-one` | 只执行下一条迁移 |
| `up-to VERSION` | 执行到指定版本 |
| `down` | 回滚最近一条迁移 |
| `down-to VERSION` | 回滚到指定版本 |
| `status` | 查看迁移状态 |
| `version` | 查看当前数据库版本 |
| `redo` | 回滚并重新执行最近一条迁移 |

学习阶段最常用：

```text
create
up
down
status
version
```

## 十、常见问题

### 1. goose 找不到迁移文件

检查：

```powershell
goose -dir migrations ...
```

`-dir` 是否指向正确目录。

### 2. goose 连接不上数据库

检查：

- PostgreSQL 是否启动。
- `DATABASE_URL` 是否正确。
- 数据库是否存在。
- 用户名密码是否正确。

### 3. migration already applied

说明版本表里已经记录该迁移执行过。

不要手动重复执行文件内容。

### 4. 修改了已执行迁移怎么办

本地学习时可以重建数据库。

团队项目中，已合并或已上线的迁移不要修改。应该新增迁移修正。

---

## 十一、常见误区

### 1. goose 只是帮我执行 SQL 吗？

不只是。

它还会记录迁移版本，知道哪些文件已执行。

### 2. down 可以随便在生产跑吗？

不能。

生产回滚要结合应用版本、数据风险和发布流程。

### 3. 迁移文件名无所谓吗？

不建议随便命名。

文件名应该表达业务意图，例如 `add_users_avatar_url`。

### 4. 一个迁移里可以放很多无关变更吗？

不推荐。

迁移越小越容易审查、回滚和排查。

## 十二、本节达标标准

学完本节后，你应该能够做到：

- 安装 goose。
- 创建 goose SQL 迁移。
- 编写 `-- +goose Up` 和 `-- +goose Down`。
- 使用 `goose up` 执行迁移。
- 使用 `goose down` 回滚迁移。
- 使用 `goose status` 查看状态。
- 创建并回滚 `avatar_url` 字段迁移。
