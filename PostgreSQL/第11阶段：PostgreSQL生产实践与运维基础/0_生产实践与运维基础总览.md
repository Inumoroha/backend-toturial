# 0. 生产实践与运维基础总览

本阶段目标：理解 PostgreSQL 在生产环境中如何稳定运行，掌握后端开发者必须具备的数据库运维基础。

前面十个阶段已经完成了从 SQL、表设计、事务、索引、Go 接入、迁移，到短链接项目落地的学习闭环。

但真实项目上线后，还有另一类问题：

```text
数据库坏了怎么办？
数据误删了怎么办？
查询突然变慢怎么办？
连接池把数据库打满了怎么办？
磁盘空间越来越大怎么办？
生产迁移卡住业务怎么办？
应用账号权限是不是太大？
```

这些问题不一定每天发生，但一旦发生，影响通常很大。

第十一阶段要学习的不是“怎么成为专业 DBA”，而是让后端开发者具备基本生产意识。

---

## 一、为什么后端开发者也要懂运维基础

很多团队里，数据库不是一个完全独立的黑盒。

后端开发者经常要负责：

- 设计表结构。
- 写迁移文件。
- 写查询。
- 处理事务。
- 配置连接池。
- 排查慢接口。
- 分析数据库错误。
- 配合上线和回滚。

如果不懂生产实践，就容易写出“本地没问题，上线出事故”的代码。

典型例子：

```text
本地 ALTER TABLE 一秒完成，生产大表执行后锁住业务。
本地 SELECT 很快，生产数据多了变成 Seq Scan。
本地连接池开 100 没感觉，生产 20 个实例把数据库连接打满。
开发账号有超级权限，用到生产后误删表。
以为有备份，但从来没恢复演练过。
```

所以生产运维基础不是额外知识，而是后端工程能力的一部分。

---

## 二、本阶段学习内容

本阶段建议拆成以下主题：

1. 生产环境数据库直觉。
2. 备份与恢复：`pg_dump`、`pg_restore`、恢复演练。
3. 角色、用户与权限控制。
4. 连接数、连接池与应用配置。
5. 慢查询日志与 `pg_stat_activity`。
6. VACUUM、ANALYZE 与表膨胀。
7. 常用生产配置与参数直觉。
8. 生产迁移发布流程。
9. 监控指标与告警。
10. 常见故障排查手册。
11. 综合实践：给短链接服务补生产检查清单。

---

## 三、本阶段不追求什么

本阶段暂时不深入：

- PostgreSQL 源码。
- 复杂高可用架构。
- 分布式数据库。
- 内核级调优。
- 云厂商专有功能。
- 大型 DBA 自动化平台。

这些都很重要，但不适合作为后端学习路线的第一轮生产实践。

本阶段更关注：

```text
知道风险在哪里。
知道怎么观察。
知道上线前要检查什么。
知道出问题时先看哪里。
```

---

## 四、生产环境和本地环境有什么不同

本地环境通常是：

```text
数据少。
请求少。
只有一个开发者。
可以随便重建数据库。
错了可以重来。
```

生产环境是：

```text
数据不能丢。
请求持续进来。
多人同时使用。
迁移不能随便锁表。
错误操作可能不可逆。
性能问题会影响用户。
```

所以生产环境里的每个动作都要多想一步：

```text
会不会锁表？
会不会扫全表？
会不会打满连接？
能不能回滚？
有没有备份？
有没有监控？
```

---

## 五、生产数据库最重要的四个底线

### 1. 数据不能轻易丢

备份和恢复是底线。

没有恢复演练的备份，不能算真正可靠。

### 2. 权限不能过大

应用账号不应该是超级用户。

它只应该拥有完成业务所需的最小权限。

### 3. 变更必须可控

表结构变更必须通过迁移文件和发布流程，而不是生产手动乱改。

### 4. 问题必须可观察

慢查询、连接数、锁等待、磁盘空间、错误日志，都要能看见。

看不见的问题，通常只能靠猜。

---

## 六、后端开发者的生产检查清单

每个 PostgreSQL 项目上线前，至少检查：

- 是否有迁移文件。
- 是否有备份方案。
- 是否做过恢复演练。
- 应用账号是否不是超级用户。
- 连接池最大连接数是否合理。
- 关键查询是否有索引。
- 是否开启慢查询日志。
- 是否知道如何查看当前连接。
- 是否知道如何查看锁等待。
- 是否有基本监控。
- 生产迁移是否评估过锁和数据量。

这份清单不复杂，但能避免很多低级事故。

---

## 七、本阶段达标标准

学完第十一阶段后，你应该能够做到：

- 解释生产数据库和本地数据库的差异。
- 使用 `pg_dump` 备份数据库。
- 使用 `pg_restore` 或 `psql` 恢复备份。
- 设计基本的 PostgreSQL 角色和权限。
- 配置 Go 应用连接池，不打爆数据库连接。
- 使用 `pg_stat_activity` 查看当前连接和慢查询。
- 理解 VACUUM、ANALYZE 和表膨胀的基本原因。
- 知道生产迁移为什么要小心锁表。
- 能列出 PostgreSQL 项目上线前的检查清单。
- 出现慢查询、锁等待、连接耗尽时知道先查什么。
