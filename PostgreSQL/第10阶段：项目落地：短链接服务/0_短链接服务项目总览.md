# 0. 短链接服务项目总览

本阶段目标：把前面学过的 PostgreSQL、迁移、索引、事务和 Go `pgxpool` 串起来，完成一个可以落地的小项目：短链接服务。

前面阶段学到的内容大多是单点能力：

```text
会建表。
会写查询。
会使用事务。
会分析索引。
会用 Go 连接 PostgreSQL。
会用迁移工具管理表结构。
```

第十阶段要做的是把这些能力放到一个真实项目里。

短链接服务虽然功能不大，但非常适合练习后端基础，因为它同时涉及：

- 表结构设计。
- 唯一约束。
- 索引。
- 插入、查询、更新。
- 并发下的访问计数。
- 数据库错误处理。
- API 设计。
- Go 项目分层。
- 数据库迁移。
- 性能分析。

---

## 一、项目要做什么

短链接服务的核心功能很简单：

```text
用户提交一个长链接。
系统生成一个短码。
用户访问短码。
系统跳转到原始长链接。
系统记录访问次数。
```

例如：

```text
原始链接：
https://example.com/posts/postgresql-transaction-index-pgxpool

短链接：
https://s.example.com/aB39xQ
```

访问 `https://s.example.com/aB39xQ` 时，服务会找到 `aB39xQ` 对应的原始链接，然后返回 HTTP 302 跳转。

---

## 二、本阶段最终能力

完成本阶段后，你应该能独立写出一个最小可用版本：

- `POST /links`：创建短链接。
- `GET /:code`：根据短码跳转到原始链接。
- `GET /links/:code`：查询短链接详情。
- `GET /links`：分页查询短链接列表。
- 使用 PostgreSQL 保存数据。
- 使用迁移文件创建表。
- 使用 `pgxpool` 管理连接池。
- 使用参数化 SQL，避免 SQL 注入。
- 使用唯一约束防止短码重复。
- 正确处理唯一约束冲突。
- 使用索引优化核心查询。
- 使用 `EXPLAIN ANALYZE` 检查查询计划。

## 六、项目的最小闭环

最小闭环不是一次做完所有功能，而是先跑通：

```text

-> 创建短链接
```

只要这个闭环跑通，就说明：

后面再逐步补：

- 访问日志。

## 七、为什么短链接项目适合练习 PostgreSQL

短链接项目看起来简单，但它背后有不少数据库细节。

### 1. 短码必须唯一

短码重复会导致一个短链接指向错误资源，所以必须用数据库唯一约束兜底。

不能只依赖 Go 代码判断：

```text
先查 code 是否存在。
如果不存在，再插入。
```

并发情况下，两个请求可能同时认为同一个 code 不存在，最后一起插入。

正确做法是：

```text
数据库 UNIQUE 约束负责最终一致性。
Go 代码捕获唯一冲突并重试。
```

### 2. 跳转查询必须快

短链接服务最频繁的操作是：

```sql
SELECT original_url
FROM app.short_links
WHERE code = $1;
```

所以 `code` 必须有唯一索引。

### 3. 访问次数会频繁更新

每次跳转都要增加访问次数：

```sql
UPDATE app.short_links
SET visit_count = visit_count + 1
WHERE code = $1;
```

这里要理解并发更新、行锁和原子自增。

### 4. 后续统计可以继续扩展

如果要统计每天访问次数，就可以增加访问日志表。

这会练习：

- 一对多关系。
- 插入日志。
- 聚合查询。
- 时间范围查询。
- 索引设计。

## 八、先定边界，不要一开始做太大

第一个版本不要急着做这些功能：

- 用户登录。
- 自定义域名。
- 管理后台。
- Redis 缓存。
- 地域统计。
- 浏览器、设备识别。
- 防刷限流。

这些都可以以后加。

本阶段重点是把数据库和 Go 后端基本功练扎实。

第一个版本只需要做到：

```text
创建短链接。
跳转。
计数。
查询。
分页。
```

## 九、本阶段达标标准

学完本阶段后，你应该能够做到：

- 说清楚短链接服务的核心数据模型。
- 写出 `short_links` 表迁移。
- 使用 `pgxpool` 初始化数据库连接池。
- 用参数化 SQL 创建短链接。
- 用唯一约束处理短码重复。
- 写出跳转接口，并更新访问次数。
- 解释为什么 `visit_count = visit_count + 1` 是原子更新。
- 使用 `EXPLAIN ANALYZE` 查看按 `code` 查询是否走索引。
- 知道分页查询为什么不应该长期依赖大偏移量 `OFFSET`。
- 能把前面阶段的知识串成一个可运行的小项目。
