# 8. 查询详情与分页列表

本节目标：实现短链接详情查询和分页列表查询，让项目不只可以创建和跳转，也能被管理和观察。

第一版需要两个查询接口：

```text
GET /links/{code}
GET /links?limit=20&offset=0
```

它们都属于读取接口，但用途不同：

```text
详情接口：根据一个短码查看具体信息。
列表接口：按创建时间倒序查看最近创建的短链接。
```

---

## 一、详情接口设计

请求：

```http
GET /links/aB39xQ
```

响应：

```json
{
  "id": "8b4f5a38-b7b7-4c20-936c-73fbf81c0a17",
  "code": "aB39xQ",
  "original_url": "https://example.com/articles/postgresql",
  "visit_count": 12,
  "created_at": "2026-07-05T10:00:00Z",
  "updated_at": "2026-07-05T10:30:00Z"
}
```

详情接口不会增加访问次数。

只有真正跳转接口：

```text
GET /{code}
```

才增加 `visit_count`。

## 二、repository 查询详情

在 `repository.go` 增加：

```go
func (r *Repository) GetByCode(ctx context.Context, code string) (Link, error) {
    const query = `
        SELECT id, code, original_url, visit_count, created_at, updated_at
        FROM app.short_links
        WHERE code = $1
    `

    var item Link
    err := r.pool.QueryRow(ctx, query, code).Scan(
        &item.ID,
        &item.Code,
        &item.OriginalURL,
        &item.VisitCount,
        &item.CreatedAt,
        &item.UpdatedAt,
    )
    if err != nil {
        if errors.Is(err, pgx.ErrNoRows) {
            return Link{}, ErrNotFound
        }
        return Link{}, err
    }

    return item, nil
}
```

这个查询会使用 `code` 唯一索引。

因为表结构里有：

```sql
CONSTRAINT short_links_code_unique UNIQUE (code)
```

PostgreSQL 会自动为它建立唯一 B-Tree 索引。

## 三、service 查询详情

在 `service.go` 增加：

```go
func (s *Service) GetByCode(ctx context.Context, code string) (Link, error) {
    code = strings.TrimSpace(code)
    if !isValidCode(code) {
        return Link{}, ErrNotFound
    }

    item, err := s.repo.GetByCode(ctx, code)
    if err != nil {
        return Link{}, err
    }

    item.ShortURL = s.baseURL + "/" + item.Code
    return item, nil
}
```

这里也拼接 `ShortURL`，方便前端或调用方直接展示。

## 四、handler 查询详情

注册路由：

```go
func (h *Handler) RegisterRoutes(mux *http.ServeMux) {
    mux.HandleFunc("POST /links", h.createLink)
    mux.HandleFunc("GET /links/{code}", h.getLink)
    mux.HandleFunc("GET /links", h.listLinks)
    mux.HandleFunc("GET /{code}", h.redirect)
}
```

注意顺序和具体程度：

```text
GET /links/{code}
GET /links
GET /{code}
```

不要让 `/links/aB39xQ` 被错误当成跳转短码。

实现：

```go
func (h *Handler) getLink(w http.ResponseWriter, r *http.Request) {
    code := r.PathValue("code")

    ctx, cancel := context.WithTimeout(r.Context(), 3*time.Second)
    defer cancel()

    item, err := h.service.GetByCode(ctx, code)
    if err != nil {
        switch {
        case errors.Is(err, ErrNotFound):
            writeJSON(w, http.StatusNotFound, map[string]string{
                "error": "link not found",
            })
        default:
            writeJSON(w, http.StatusInternalServerError, map[string]string{
                "error": "internal server error",
            })
        }
        return
    }

    writeJSON(w, http.StatusOK, item)
}
```

## 六、分页参数规则

建议第一版规则：

```text
默认 limit = 20。
最大 limit = 100。
默认 offset = 0。
offset 不能小于 0。
```

原因：

- 不允许用户一次查太多数据。
- 避免数据库和网络响应过大。
- 参数错误时可以自动修正，也可以返回 400。

学习项目可以选择自动修正：

```go
func parsePositiveInt(raw string, fallback int) int {
    if raw == "" {
        return fallback
    }

    value, err := strconv.Atoi(raw)
    if err != nil || value < 0 {
        return fallback
    }

    return value
}
```

## 七、repository 分页查询

在 `repository.go` 增加：

```go
func (r *Repository) List(ctx context.Context, limit int, offset int) ([]Link, error) {
    const query = `
        SELECT id, code, original_url, visit_count, created_at, updated_at
        FROM app.short_links
        ORDER BY created_at DESC
        LIMIT $1 OFFSET $2
    `

    rows, err := r.pool.Query(ctx, query, limit, offset)
    if err != nil {
        return nil, err
    }
    defer rows.Close()

    items := make([]Link, 0, limit)
    for rows.Next() {
        var item Link
        if err := rows.Scan(
            &item.ID,
            &item.Code,
            &item.OriginalURL,
            &item.VisitCount,
            &item.CreatedAt,
            &item.UpdatedAt,
        ); err != nil {
            return nil, err
        }
        items = append(items, item)
    }

    if err := rows.Err(); err != nil {
        return nil, err
    }

    return items, nil
}
```

注意：

```go
defer rows.Close()
```

必须写。

不关闭 `rows` 可能导致连接不能及时归还连接池。

## 八、service 分页查询

在 `service.go` 增加：

```go
func (s *Service) List(ctx context.Context, limit int, offset int) ([]Link, error) {
    if limit <= 0 {
        limit = 20
    }
    if limit > 100 {
        limit = 100
    }
    if offset < 0 {
        offset = 0
    }

    items, err := s.repo.List(ctx, limit, offset)
    if err != nil {
        return nil, err
    }

    for i := range items {
        items[i].ShortURL = s.baseURL + "/" + items[i].Code
    }

    return items, nil
}
```

这里把分页规则放在 service 层。

handler 负责解析，service 负责业务范围限制。

## 九、handler 分页查询

在 `handler.go` 增加：

```go
func (h *Handler) listLinks(w http.ResponseWriter, r *http.Request) {
    query := r.URL.Query()
    limit := parsePositiveInt(query.Get("limit"), 20)
    offset := parsePositiveInt(query.Get("offset"), 0)

    ctx, cancel := context.WithTimeout(r.Context(), 3*time.Second)
    defer cancel()

    items, err := h.service.List(ctx, limit, offset)
    if err != nil {
        writeJSON(w, http.StatusInternalServerError, map[string]string{
            "error": "internal server error",
        })
        return
    }

    if limit > 100 {
        limit = 100
    }

    writeJSON(w, http.StatusOK, ListLinksResponse{
        Items:  items,
        Limit:  limit,
        Offset: offset,
    })
}

func parsePositiveInt(raw string, fallback int) int {
    if raw == "" {
        return fallback
    }

    value, err := strconv.Atoi(raw)
    if err != nil || value < 0 {
        return fallback
    }

    return value
}
```

## 十、为什么第一版用 LIMIT/OFFSET

第一版使用：

```sql
LIMIT $1 OFFSET $2
```

原因是简单直观。

但它不是无限适合大数据量。

例如：

```sql
SELECT ...
FROM app.short_links
ORDER BY created_at DESC
LIMIT 20 OFFSET 100000;
```

数据库仍然需要找到并跳过前面 100000 条记录，然后返回 20 条。

数据量大时，性能会下降。

后续更推荐游标分页。

---

## 十一、游标分页的思路

游标分页可以用：

```text
created_at + id
```

例如第一页：

```sql
SELECT id, code, original_url, visit_count, created_at, updated_at
FROM app.short_links
ORDER BY created_at DESC, id DESC
LIMIT 20;
```

下一页传入上一页最后一条的 `created_at` 和 `id`：

```sql
SELECT id, code, original_url, visit_count, created_at, updated_at
FROM app.short_links
WHERE (created_at, id) < ($1, $2)
ORDER BY created_at DESC, id DESC
LIMIT 20;
```

这种方式适合数据量更大的列表。

但第一版先掌握 `LIMIT/OFFSET` 即可。
