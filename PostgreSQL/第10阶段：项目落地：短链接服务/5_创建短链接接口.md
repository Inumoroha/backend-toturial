# 5. 创建短链接接口

本节目标：实现 `POST /links`，完成从 HTTP 请求到 PostgreSQL 插入记录的完整流程。

创建短链接涉及三层：

```text
handler    读取 JSON，返回 HTTP 响应。
service    校验 URL，生成短码，处理冲突重试。
repository 执行 INSERT SQL。
```

本节先实现主路径，短码冲突重试会在下一节展开。

---

## 一、请求和响应

请求：

```http
POST /links
Content-Type: application/json
```

请求体：

```json
{
  "original_url": "https://example.com/articles/postgresql"
}
```

响应：

```json
{
  "id": "8b4f5a38-b7b7-4c20-936c-73fbf81c0a17",
  "code": "aB39xQ",
  "original_url": "https://example.com/articles/postgresql",
  "short_url": "http://localhost:8080/aB39xQ",
  "visit_count": 0,
  "created_at": "2026-07-05T10:00:00Z",
  "updated_at": "2026-07-05T10:00:00Z"
}
```

## 二、输入结构

在 `internal/link/model.go` 中增加：

```go
type CreateLinkRequest struct {
    OriginalURL string `json:"original_url"`
}

type CreateLinkInput struct {
    Code        string
    OriginalURL string
}
```

为什么分两个结构？

```text
CreateLinkRequest 是 HTTP 请求。
CreateLinkInput 是 repository 插入数据库需要的数据。
```

HTTP 请求里没有 `Code`，因为短码由服务端生成。

## 三、repository 插入数据

在 `repository.go` 增加：

```go
package link

import (
    "context"

    "github.com/jackc/pgx/v5/pgxpool"
)

type Repository struct {
    pool *pgxpool.Pool
}

func NewRepository(pool *pgxpool.Pool) *Repository {
    return &Repository{pool: pool}
}

func (r *Repository) Create(ctx context.Context, input CreateLinkInput) (Link, error) {
    const query = `
        INSERT INTO app.short_links (code, original_url)
        VALUES ($1, $2)
        RETURNING id, code, original_url, visit_count, created_at, updated_at
    `

    var item Link
    err := r.pool.QueryRow(ctx, query, input.Code, input.OriginalURL).Scan(
        &item.ID,
        &item.Code,
        &item.OriginalURL,
        &item.VisitCount,
        &item.CreatedAt,
        &item.UpdatedAt,
    )
    if err != nil {
        return Link{}, err
    }

    return item, nil
}
```

注意这里使用的是参数化 SQL：

```sql
VALUES ($1, $2)
```

不要拼接字符串。

## 四、URL 校验

在 `service.go` 中写校验函数：

```go
package link

import (
    "context"
    "net/url"
    "strings"
)

type Service struct {
    repo    *Repository
    baseURL string
}

func NewService(repo *Repository, baseURL string) *Service {
    return &Service{
        repo:    repo,
        baseURL: strings.TrimRight(baseURL, "/"),
    }
}

func validateOriginalURL(raw string) error {
    raw = strings.TrimSpace(raw)
    if raw == "" {
        return ErrInvalidURL
    }

    parsed, err := url.Parse(raw)
    if err != nil {
        return ErrInvalidURL
    }

    if parsed.Scheme != "http" && parsed.Scheme != "https" {
        return ErrInvalidURL
    }

    if parsed.Host == "" {
        return ErrInvalidURL
    }

    return nil
}
```

这里拒绝：

```text
空字符串。
没有 scheme 的 URL。
不是 http/https 的 URL。
没有 host 的 URL。
```

## 五、service 创建短链接

先写一个简单版本：

```go
func (s *Service) Create(ctx context.Context, originalURL string) (Link, error) {
    originalURL = strings.TrimSpace(originalURL)

    if err := validateOriginalURL(originalURL); err != nil {
        return Link{}, err
    }

    code, err := GenerateCode(6)
    if err != nil {
        return Link{}, err
    }

    item, err := s.repo.Create(ctx, CreateLinkInput{
        Code:        code,
        OriginalURL: originalURL,
    })
    if err != nil {
        return Link{}, err
    }

    item.ShortURL = s.baseURL + "/" + item.Code
    return item, nil
}
```

`GenerateCode` 会在下一节实现。

先把 service 的职责看清楚：

```text
清理输入。
校验 URL。
生成短码。
调用 repository。
拼接 short_url。
```

## 六、handler 读取 JSON

创建：

```text
internal/link/handler.go
```

代码：

```go
package link

import (
    "encoding/json"
    "errors"
    "net/http"
    "time"
)

type Handler struct {
    service *Service
}

func NewHandler(service *Service) *Handler {
    return &Handler{service: service}
}

func (h *Handler) RegisterRoutes(mux *http.ServeMux) {
    mux.HandleFunc("POST /links", h.createLink)
}

func (h *Handler) createLink(w http.ResponseWriter, r *http.Request) {
    ctx := r.Context()

    var req CreateLinkRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        writeJSON(w, http.StatusBadRequest, map[string]string{
            "error": "invalid json body",
        })
        return
    }

    ctx, cancel := contextWithTimeout(ctx, 3*time.Second)
    defer cancel()

    item, err := h.service.Create(ctx, req.OriginalURL)
    if err != nil {
        switch {
        case errors.Is(err, ErrInvalidURL):
            writeJSON(w, http.StatusBadRequest, map[string]string{
                "error": "invalid original_url",
            })
        default:
            writeJSON(w, http.StatusInternalServerError, map[string]string{
                "error": "internal server error",
            })
        }
        return
    }

    writeJSON(w, http.StatusCreated, item)
}
```

这里还缺两个辅助函数。

## 七、HTTP 辅助函数

可以在 `handler.go` 同文件底部写：

```go
func writeJSON(w http.ResponseWriter, status int, value any) {
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(status)
    _ = json.NewEncoder(w).Encode(value)
}

func contextWithTimeout(parent context.Context, timeout time.Duration) (context.Context, context.CancelFunc) {
    return context.WithTimeout(parent, timeout)
}
```

需要补 import：

```go
import (
    "context"
    "encoding/json"
    "errors"
    "net/http"
    "time"
)
```

这里把 timeout 抽成函数不是必须的，只是让示例更清楚。

实际项目也可以直接写：

```go
ctx, cancel := context.WithTimeout(r.Context(), 3*time.Second)
defer cancel()
```

## 八、路由注册注意点

Go 1.22 之后，标准库 `http.ServeMux` 支持：

```go
mux.HandleFunc("POST /links", h.createLink)
```

如果你使用更旧的 Go 版本，可能需要自己判断 method：

```go
mux.HandleFunc("/links", func(w http.ResponseWriter, r *http.Request) {
    if r.Method != http.MethodPost {
        http.Error(w, "method not allowed", http.StatusMethodNotAllowed)
        return
    }
    h.createLink(w, r)
})
```

本教程后续使用新写法。如果你的 Go 版本不支持，可以改成旧写法。

## 十、常见错误

### 1. JSON 字段名写错

请求必须是：

```json
{
  "original_url": "https://example.com"
}
```

不是：

```json
{
  "url": "https://example.com"
}
```

### 2. 忘记执行迁移

如果报错：

```text
relation "app.short_links" does not exist
```

说明表还没创建。

先执行迁移。

### 3. URL 没有 scheme

这个会被拒绝：

```text
example.com
```

应该写：

```text
https://example.com
```

学完本节后，你应该能够做到：
