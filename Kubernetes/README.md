# Kubernetes 零基础学习教程

这是一套面向零基础后端学习者的 Kubernetes 中文教程。你不需要先会运维，也不需要先购买云服务器。整套教程会用一个 Go 短链接服务贯穿镜像、部署、访问、配置、任务、排障和发布。

## 从这里开始

第一次学习，请按下面顺序进行：

1. 阅读[学习路线](./00-学习路线/00-Go后端工程师Kubernetes学习路线.md)，了解全程要完成什么。
2. 打开[零基础术语表与命令速查](./00-学习路线/01-零基础术语表与命令速查.md)。遇到生词时先查这里，不必立刻搜索底层原理。
3. 从[第 1 章](./01-基础环境/01-从后端视角理解Kubernetes.md)开始，按编号顺序学习。
4. 每章至少亲手执行一次示例，并完成章末自测。
5. 学完第 18 章后完成[短链接服务实战](./06-项目实战/19-短链接服务Kubernetes落地项目.md)。

## 你最终会做成什么

你会经历一条完整交付链路：

```text
Go 源代码
  -> Docker 镜像
  -> Deployment 管理 Pod
  -> Service 提供稳定地址
  -> ConfigMap / Secret 注入配置
  -> Ingress 接收外部请求
  -> Job / CronJob 执行后台任务
  -> 日志、指标和事件帮助排障
  -> Helm / Kustomize 管理多环境
  -> CI/CD 发布并在异常时回滚
```

## 建议准备的软件

开始前只需要准备：

- Go：用于运行教程里的后端示例。
- Docker Desktop 或 Docker Engine：用于构建和运行镜像。
- kubectl：用于操作 Kubernetes。
- 一个本地集群：Docker Desktop Kubernetes、Minikube 或 kind 三选一。
- 文本编辑器：Visual Studio Code 或你熟悉的编辑器。

不用一次装完所有附加工具。Helm、Ingress Controller、metrics-server 等会在对应章节再介绍。

检查基础工具：

```powershell
go version
docker version
kubectl version --client
```

看到版本号说明命令已安装。若出现“无法识别命令”，先安装或检查系统 `PATH`，不要继续执行后面的部署命令。

## 每章的学习方式

每一章都按同一套节奏学习：

1. **先理解问题**：这个对象解决什么实际问题。
2. **再运行示例**：逐行输入命令，不要一次粘贴一大段后就跳过输出。
3. **核对结果**：确认资源状态和教程给出的预期一致。
4. **故意改错**：改错标签、镜像名或端口，观察 Kubernetes 如何报告问题。
5. **恢复现场**：改回正确配置，确认服务恢复。
6. **完成自测**：能不用照抄解释关键字段，才进入下一章。

## 命令行约定

- `<pod-name>` 是占位符，要替换成你实际看到的 Pod 名称，不要连尖括号一起输入。
- `$GIT_COMMIT` 表示流水线环境变量，Windows PowerShell 手工测试时通常写成 `$env:GIT_COMMIT`。
- 命令没有输出不一定是失败；以退出状态、后续 `kubectl get` 结果和错误信息为准。
- 教程中的 `short-api:0.1.0` 是本地镜像名。如果集群拉不到它，需要按第 4 章说明把镜像载入 kind 或 Minikube。
- 没有特别说明时，命令使用当前 namespace；从第 10 章开始要特别留意 `-n <namespace>`。

## 安全边界

教程命令优先用于本地练习集群。连接公司或生产集群前，至少先执行：

```bash
kubectl config current-context
kubectl config view --minify --output 'jsonpath={..namespace}'
```

生产环境不要照抄示例密码，不要提交真实 Secret，不要使用 `latest` 镜像标签，也不要在不了解影响时执行删除、回滚或权限变更命令。
