# 3. 把 Go 服务打包成镜像

本节目标：学完后你能把一个最小 Go HTTP 服务打包成 Docker 镜像，为部署到 Kubernetes 做准备。

## 简短引入

Kubernetes 不直接运行你的源代码，它运行容器。对 Go 后端来说，通常是先编译二进制，再把二进制放进镜像。镜像就是后续 Deployment 里的 `image`。

## 一、为什么需要它

真实项目中，同一份服务要部署到开发、测试、生产。镜像解决的是“运行内容一致”的问题：

- 代码版本固定。
- 依赖固定。
- 启动命令固定。
- 发布和回滚可以围绕镜像标签进行。

```text
生产环境不要使用 latest 标签。镜像标签应该能追溯到一次构建，例如 Git commit、日期、CI 构建号。
```

## 二、基本用法

创建一个最小 Go 服务。

Windows PowerShell：

```powershell
New-Item -ItemType Directory -Force -Path short-api
Set-Location short-api
go mod init example.com/short-api
```

Linux/macOS：

```bash
mkdir -p short-api
cd short-api
go mod init example.com/short-api
```

创建 `main.go`：

```go
package main

import (
	"fmt"
	"log"
	"net/http"
	"os"
)

func main() {
	port := os.Getenv("PORT")
	if port == "" {
		port = "8080"
	}

	http.HandleFunc("/healthz", func(w http.ResponseWriter, r *http.Request) {
		_, _ = w.Write([]byte("ok"))
	})

	http.HandleFunc("/links", func(w http.ResponseWriter, r *http.Request) {
		_, _ = w.Write([]byte("short links api"))
	})

	addr := ":" + port
	fmt.Println("listening on", addr)
	log.Fatal(http.ListenAndServe(addr, nil))
}
```

创建 `Dockerfile`：

```dockerfile
FROM golang:1.22-alpine AS builder
WORKDIR /src
COPY go.mod ./
RUN go mod download
COPY . .
RUN CGO_ENABLED=0 GOOS=linux go build -o /out/short-api .

FROM alpine:3.20
WORKDIR /app
COPY --from=builder /out/short-api /app/short-api
EXPOSE 8080
ENTRYPOINT ["/app/short-api"]
```

构建和运行：

```bash
docker build -t short-api:0.1.0 .
docker run --rm -p 8080:8080 short-api:0.1.0
```

另开终端访问：

Windows PowerShell：

```powershell
curl.exe http://127.0.0.1:8080/healthz
curl.exe http://127.0.0.1:8080/links
```

Linux/macOS：

```bash
curl http://127.0.0.1:8080/healthz
curl http://127.0.0.1:8080/links
```

## 三、关键参数/语法/代码结构

- 多阶段构建：第一阶段用 Go 镜像编译，第二阶段只放二进制，镜像更小。
- `CGO_ENABLED=0`：常见于构建静态 Go 二进制，减少运行环境依赖。
- `EXPOSE 8080`：说明容器监听端口，Kubernetes 里仍要配置 `containerPort`。
- `ENTRYPOINT`：容器启动时执行的命令。

真实项目中，Dockerfile 应尽量稳定。不要把数据库地址、密码、生产配置写死在镜像里。

## 四、真实后端场景示例

短链接服务常见接口：

- `POST /links` 创建短链接。
- `GET /{code}` 跳转原始地址。
- `GET /healthz` 健康检查。

镜像只负责打包服务，不负责决定生产数据库是哪台。数据库地址应该通过环境变量注入：

```go
databaseURL := os.Getenv("DATABASE_URL")
if databaseURL == "" {
	log.Fatal("DATABASE_URL is required")
}
```

后续 Kubernetes 里会用 Secret 注入它。

## 五、注意点

- 不要把 `.env`、私钥、数据库密码复制进镜像。
- 不要在容器里依赖本机路径，例如 `C:\Users\...`。
- Go 服务要把日志输出到 stdout/stderr，方便 `kubectl logs` 收集。
- 监听地址通常用 `:8080`，不要只监听 `127.0.0.1`。

## 六、常见错误

- 错误：容器启动了，但访问不到。
  检查服务是否监听 `0.0.0.0` 或 `:8080`，以及端口映射是否正确。
- 错误：镜像构建很慢。
  注意 Dockerfile 先复制 `go.mod` 再下载依赖，可以利用缓存。
- 错误：生产镜像使用 `latest`。
  回滚时你无法明确知道旧版本到底是哪一份代码。
