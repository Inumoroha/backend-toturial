# 13. 第 6 阶段练习与自测

## 1. 本阶段练习目标

完成这些练习后，你应该能：

- 使用 UI 和命令排查 RabbitMQ。
- 设计核心监控指标。
- 设计告警规则。
- 做权限隔离。
- 理解生产部署检查项。
- 处理常见线上故障。

## 2. 练习一：使用诊断命令

执行：

```powershell
docker exec -it rabbitmq-dev rabbitmq-diagnostics status
docker exec -it rabbitmq-dev rabbitmq-diagnostics check_running
docker exec -it rabbitmq-dev rabbitmq-diagnostics check_local_alarms
docker exec -it rabbitmq-dev rabbitmqctl list_queues name messages_ready messages_unacknowledged consumers
```

请记录：

- 当前队列数量。
- 是否有 alarm。
- 是否有 ready 或 unacked 消息。

## 3. 练习二：模拟 consumer 为 0

创建一个队列并发布消息，但不启动消费者。

观察：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_queues name messages_ready messages_unacknowledged consumers
```

回答：

```text
ready、unacked、consumers 分别是多少？
```

## 4. 练习三：模拟 unacked 增长

启动一个消费者：

- manual ack。
- 收到消息后 sleep 30 秒再 ack。
- prefetch 设置为 5。

发布 10 条消息。

观察：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_queues name messages_ready messages_unacknowledged consumers
```

回答：

- 为什么 unacked 会增长？
- prefetch 对 unacked 有什么影响？

## 5. 练习四：设计告警规则

为下面队列设计告警：

```text
order.paid.points.queue
notification.email.worker.queue
order.dlq
```

至少包含：

- ready 阈值。
- unacked 阈值。
- consumers 为 0。
- DLQ 有消息。

请说明每条告警的业务影响。

## 6. 练习五：权限隔离

设计两个用户：

```text
order_publisher
points_worker
```

要求：

- `order_publisher` 只能写 `order.events`。
- `points_worker` 只能读 `order.paid.points.queue`。

写出 `set_permissions` 命令。

参考：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl set_permissions -p / order_publisher "^$" "^order\\.events$" "^$"
docker exec -it rabbitmq-dev rabbitmqctl set_permissions -p / points_worker "^$" "^$" "^order\\.paid\\.points\\.queue$"
```

实际生产中，如果 worker 还要写 retry/DLQ exchange，需要额外授权 write。

## 8. 练习七：Prometheus 指标

启用插件：

```powershell
docker exec -it rabbitmq-dev rabbitmq-plugins enable rabbitmq_prometheus
```

如果容器映射了 15692，访问：

```text
http://localhost:15692/metrics
```

回答：

- 为什么 Management UI 不适合替代 Prometheus？
- 你会在 Grafana 放哪些队列指标？
