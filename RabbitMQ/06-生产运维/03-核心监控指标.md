# 03. 核心监控指标

## 1. 为什么先学指标

没有指标，就没有运维。

RabbitMQ 的业务异常通常会先反映在指标上：

- ready 增长。
- unacked 增长。
- consumer 数量下降。
- publish rate 大于 ack rate。
- memory alarm。
- disk alarm。
- connection 暴涨。

你要知道每个指标背后代表什么业务风险。

## 2. 队列指标

### messages_ready

等待投递给消费者的消息数量。

含义：

```text
消息还在队列里排队。
```

风险：

```text
业务处理延迟增加。
```

常见原因：

- 消费者数量不足。
- 消费者处理慢。
- 没有消费者。
- 下游数据库或外部 API 慢。

### messages_unacknowledged

已经投递给消费者，但还没 ack 的消息数量。

风险：

```text
消费者可能卡住、处理慢、忘记 ack 或 prefetch 太大。
```

### consumers

队列当前消费者数量。

如果关键队列 consumers 变为 0：

```text
应该立即告警。
```

## 3. 速率指标

### publish rate

消息进入 RabbitMQ 的速率。

### deliver / get rate

消息投递给消费者的速率。

### ack rate

消费者 ack 的速率。

关键判断：

```text
publish rate 长期大于 ack rate -> 队列会堆积。
```

如果 publish rate 很高，但 deliver rate 为 0：

```text
可能没有消费者，或 routing/queue 出问题。
```

## 4. 连接和 channel 指标

需要关注：

- connection count。
- channel count。
- connection churn。
- channel churn。

异常：

```text
连接数快速增长
连接频繁创建和关闭
channel 数量持续增长
```

常见原因：

- 应用每条消息创建新连接。
- 连接池配置错误。
- 应用重连风暴。
- channel 泄漏。

## 5. 节点资源指标

### 内存

关注：

- total memory。
- memory high watermark。
- memory alarm。

内存水位过高时，RabbitMQ 可能触发 flow control，影响发布。

### 磁盘

关注：

- disk free。
- disk free limit。
- disk alarm。

磁盘不足是严重风险。

### 文件句柄

关注：

- file descriptors used。
- file descriptors limit。

连接多、队列多时文件句柄可能成为瓶颈。

## 6. 死信和重试指标

业务必须监控：

- DLQ messages_ready。
- retry queue messages_ready。
- retry count 分布。
- 最老 DLQ 消息时间。

DLQ 不应该长期没人看。

原则：

```text
DLQ 有消息，不一定是系统挂了，但一定需要有人处理。
```

## 7. 业务指标

RabbitMQ 指标还不够。

还要结合业务指标：

- 订单创建到库存预占完成耗时。
- 支付成功到积分到账耗时。
- 通知发送成功率。
- Outbox pending 数量。
- Outbox 最老 pending 时间。
- 消费失败次数。
- 重复消息次数。

例如：

```text
RabbitMQ 没有堆积，但积分仍未到账。
```

可能是消费者业务逻辑失败并进入 DLQ。

## 8. 常用命令观察指标

队列：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_queues name messages_ready messages_unacknowledged consumers
```

连接：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_connections name user vhost state channels
```

channel：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_channels connection number prefetch_count messages_unacknowledged
```

节点：

```powershell
docker exec -it rabbitmq-dev rabbitmq-diagnostics status
```

内存：

```powershell
docker exec -it rabbitmq-dev rabbitmq-diagnostics memory_breakdown
```

## 9. 本节小结

RabbitMQ 核心指标分四层：

```text
队列：ready、unacked、consumers
速率：publish、deliver、ack
连接：connections、channels
资源：memory、disk、file descriptors
```

业务系统还要监控：

```text
Outbox、DLQ、业务延迟、失败率、幂等重复次数
```
