# 09. 第 1 阶段练习与自测

## 1. 本阶段练习目标

完成本文件后，你应该能独立完成：

- 启动 RabbitMQ。
- 登录 Management UI。

- 手动发布和获取消息。

- 使用基础命令观察 RabbitMQ 状态。

## 2. 练习一：启动环境

```powershell

```

检查容器：

```powershell
docker ps
```

打开管理后台：

```text
http://localhost:15672
```

登录：

```text
go_learner / go_learner_pwd
```

### 自查

- [ ] 我知道 `5672` 是 AMQP 端口。
- [ ] 我知道 `15672` 是管理后台端口。

## 3. 练习二：创建 direct 路由

### 任务

创建：

```text
Exchange: stage1.direct
Type: direct

Queue: stage1.email.queue

Binding:
stage1.direct -- email.send --> stage1.email.queue
```

发布消息：

```text
exchange: stage1.direct
routing key: email.send
payload:
{
  "message_id": "stage1-001",
  "event_type": "email.send",
  "payload": {
    "to": "user@example.com",
    "subject": "hello"
  }
}
```

### 预期

`stage1.email.queue` 的 Ready 数量变为 1。

然后使用 `Get messages` 取出消息。

### 自查

- [ ] routing key 匹配时，消息进入队列。
- [ ] 取出并 ack 后，Ready 变回 0。

## 4. 练习三：测试 routing key 不匹配

### 任务

继续使用：

```text
exchange: stage1.direct
queue: stage1.email.queue
binding key: email.send
```

发布消息时 routing key 改成：

```text
sms.send
```

### 预期

`stage1.email.queue` 不会增加消息。

### 思考

请回答：

```text
为什么发布操作看起来成功了，但队列没有收到消息？
```

参考答案：

```text
direct exchange 需要 routing key 和 binding key 精确匹配。sms.send 不匹配 email.send，所以消息没有路由到 stage1.email.queue。
```

## 5. 练习四：fanout 广播

### 任务

创建：

```text
Exchange: stage1.fanout
Type: fanout

Queue:
stage1.fanout.a.queue
stage1.fanout.b.queue
stage1.fanout.c.queue
```

将三个 queue 都绑定到 `stage1.fanout`。

发布一条消息：

```text
exchange: stage1.fanout
routing key: 留空或任意
payload: fanout hello
```

### 预期

三个队列各收到一条消息。

### 自查

- [ ] 我能解释 fanout 为什么适合广播。
- [ ] 我知道 fanout 通常不关心 routing key。

## 6. 练习五：topic 通配符

### 任务

创建：

```text
Exchange: stage1.topic
Type: topic
```

创建队列和绑定：

```text
stage1.order.all.queue      binding key: order.#
stage1.order.created.queue  binding key: order.created
stage1.created.all.queue    binding key: *.created
```

分别发布：

```text
routing key: order.created
routing key: order.paid
routing key: user.created
```

### 预期

`order.created` 进入：

```text
stage1.order.all.queue
stage1.order.created.queue
stage1.created.all.queue
```

`order.paid` 进入：

```text
stage1.order.all.queue
```

`user.created` 进入：

```text
stage1.created.all.queue
```

## 7. 练习六：使用命令观察

查看 exchange：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_exchanges name type durable auto_delete
```

查看 queue：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_queues name messages_ready messages_unacknowledged consumers
```

查看 binding：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_bindings
```

查看 vhost：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_vhosts
```
