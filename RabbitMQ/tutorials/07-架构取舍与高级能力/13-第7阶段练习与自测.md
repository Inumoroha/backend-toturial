# 13. 第 7 阶段练习与自测

## 1. 本阶段练习目标

完成这些练习后，你应该能：

- 判断 RabbitMQ 是否适合某个场景。
- 对比 RabbitMQ、Kafka、Redis Streams、数据库任务表。
- 选择 Classic、Quorum、Stream 队列类型。
- 区分 task、event、RPC、同步调用。
- 写出架构选型说明。
- 在面试中讲清 RabbitMQ 项目设计。

## 2. 练习一：场景选型

为下面场景选择方案并说明理由：

| 场景 | 推荐方案 |
| --- | --- |
| 用户注册后发欢迎邮件 | RabbitMQ 或数据库任务表 |
| 支付成功后通知积分、通知、风控 | RabbitMQ |
| 用户行为日志进入实时数仓 | Kafka |
| 已有 Redis，小规模任务流 | Redis Streams |
| 每天 100 个报表任务 | 数据库任务表 |
| 资金流水事件，不能丢 | Outbox + RabbitMQ Quorum Queue |
| 需要历史事件回放 | Kafka / RabbitMQ Stream |

## 3. 练习二：RabbitMQ vs Kafka

请回答：

1. RabbitMQ 的核心模型是什么？
2. Kafka 的核心模型是什么？
3. 为什么 Kafka 更适合日志和回放？
4. 为什么 RabbitMQ 更适合复杂业务路由？
5. 一个系统能否同时使用 RabbitMQ 和 Kafka？

## 4. 练习三：RabbitMQ vs Redis Streams

请回答：

1. Redis Streams 的核心结构是什么？
2. Redis Streams 中 consumer group 的作用是什么？
3. RabbitMQ 的 exchange/binding 相比 Redis Streams 有什么优势？
4. 什么情况下选择 Redis Streams 更合适？

## 5. 练习四：RabbitMQ vs 数据库任务表

请判断：

```text
每天 500 个后台任务，任务必须和数据库事务一起提交，允许秒级延迟。
```

你会选什么？为什么？

参考：

```text
数据库任务表可能更合适，因为任务量小、强事务、路由简单、运维成本低。
```

## 6. 练习五：队列类型取舍

为下面队列选择类型：

| 队列 | 推荐类型 |
| --- | --- |
| 普通邮件发送任务 | Classic Queue |
| 支付成功账务处理 | Quorum Queue |
| 用户行为事件流，需要回放 | Stream 或 Kafka |
| 临时日志订阅 | 临时 Classic Queue |
| 订单关键状态事件 | Quorum Queue |

## 7. 练习六：task、event、RPC 判断

判断下面消息语义：

| 名称 | 类型 |
| --- | --- |
| `notification.email.send` | task |
| `order.created` | event |
| `payment.succeeded` | event |
| `report.generate` | task |
| 查询用户详情 | 同步 HTTP/gRPC |
| 通过消息请求并等待响应 | RPC |

## 8. 练习七：写一份选型说明

场景：

```text

消息不能丢，可以重复但业务不能重复执行。
不需要长期回放。
```
