# 11. Go 客户端常见问题排查

## 1. 连接不上 RabbitMQ

错误可能类似：

```text
dial tcp [::1]:5672: connectex: No connection could be made
```

检查 RabbitMQ 是否运行：

```powershell
docker ps
```

启动：

```powershell
docker start rabbitmq-dev
```

查看日志：

```powershell
docker logs rabbitmq-dev
```

## 2. 登录失败

错误可能类似：

```text
username or password not allowed
```

检查连接地址：

```text
amqp://go_learner:go_learner_pwd@localhost:5672/
```

如果你忘了账号，可以进入管理后台看用户：

```text
http://localhost:15672
```

也可以查看 RabbitMQ 用户：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_users
```

## 3. vhost 权限错误

如果连接地址中使用了自定义 vhost：

```text
amqp://app_order:app_order_pwd@localhost:5672/rabbitmq_learning
```

需要确保用户有权限：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_permissions -p rabbitmq_learning
```

设置权限：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl set_permissions -p rabbitmq_learning app_order ".*" ".*" ".*"
```

## 4. inequivalent arg 错误

错误可能类似：

```text
PRECONDITION_FAILED - inequivalent arg 'durable' for queue
```

原因：

```text
同名 exchange 或 queue 已经存在，但这次声明的参数和之前不一致。
```

解决：

1. 保持代码中的声明参数一致。
2. 或删除旧资源后重建。

删除 queue：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl delete_queue stage3.hello.queue
```

删除 exchange：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl delete_exchange stage3.events.topic
```

## 5. 发布成功但队列没有消息

可能原因：

- exchange 名称写错。
- routing key 不匹配。
- queue 没有绑定到 exchange。
- 使用 direct/topic 时 binding key 不符合规则。
- 消息不可路由但你没有处理 return。

检查 binding：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_bindings
```

查看队列：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_queues name messages_ready messages_unacknowledged consumers
```

注意：

```text
PublishWithContext 返回 nil，不代表某个业务队列一定收到消息。
```

publisher confirm、mandatory publish 会在第 4 阶段学习。

## 6. 消费者启动了但收不到消息

检查：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_queues name messages_ready messages_unacknowledged consumers
```

如果 `consumers=0`：

```text
消费者没有注册成功。
```

如果 `messages_ready=0`：

```text
队列里没有可消费消息。
```

如果 `messages_unacknowledged` 很高：

```text
消息已经投递给消费者，但没有 ack。
```

## 7. 消息一直 Unacked

常见原因：

- 消费者处理逻辑卡住。
- 忘记调用 `d.Ack(false)`。
- 处理太慢。
- prefetch 太大。
- 消费者进程还活着但不工作。

查看消费者：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_consumers
```

查看 channel：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_channels connection number user vhost prefetch_count messages_unacknowledged
```

## 8. 程序退出后临时队列消失

如果你声明队列：

```go
ch.QueueDeclare("", false, true, true, false, nil)
```

这是临时队列：

```text
server-named
non-durable
auto-delete
exclusive
```

消费者退出后队列消失是预期行为。

业务队列不要这样声明。

## 9. import 报错

如果代码提示找不到：

```text
github.com/rabbitmq/amqp091-go
```

执行：

```powershell
go get github.com/rabbitmq/amqp091-go
go mod tidy
```

确认当前目录有 `go.mod`：

```powershell
go env GOMOD
```

## 10. Windows PowerShell 路径问题

运行 Go 子命令时，建议使用：

```powershell
go run .\cmd\hello_producer
```

不要在项目根目录外执行。

先进入项目目录：

```powershell
cd C:\Users\MSI\Desktop\RabbitMQ\demos\stage3-go-client
```

## 11. 快速排查顺序

遇到问题按这个顺序查：

1. RabbitMQ 容器是否运行。
2. 连接地址、用户名、密码、vhost 是否正确。
3. exchange 是否存在。
4. queue 是否存在。
5. binding 是否正确。
6. routing key 是否匹配。
7. 队列 `messages_ready` 是否增加。
8. 消费者 `consumers` 是否大于 0。
9. `messages_unacknowledged` 是否持续增长。
10. Go 程序日志是否有 channel 关闭或声明参数错误。

## 12. 本节小结

Go 客户端排查不要只看代码。

要同时看：

- Go 日志。
- RabbitMQ Management UI。
- `rabbitmqctl list_queues`。
- `rabbitmqctl list_bindings`。
- exchange、queue、routing key 是否一致。
