# 10. 消费者服务实现

## 1. 本节目标

实现四类消费者：

- inventory-worker
- points-worker
- notification-worker
- timeout-worker

统一要求：

- manual ack。
- prefetch。
- 幂等。
- retry。
- DLQ。
- 优雅退出。

## 2. 消费者通用流程

```text
收到消息
  -> 解析 Event
  -> 幂等检查
  -> 执行业务
  -> 记录处理完成
  -> ack
失败
  -> retry 或 DLQ
```

## 3. inventory-worker

消费：

```text
order.created.inventory.queue
```

幂等键：

```text
order:{order_id}:inventory:reserve
```

处理：

```text
检查库存
更新 inventory.reserved
插入 inventory_reservations
写 outbox inventory.reserved
```

库存不足：

```text
写 outbox inventory.reserve_failed
```

## 4. points-worker

消费：

```text
order.paid.points.queue
```

幂等键：

```text
order:{order_id}:points:add
```

处理：

```text
插入 processed_messages
插入 points_records
更新用户积分
ack
```

重复消息：

```text
唯一键冲突 -> 直接 ack
```

## 5. notification-worker

消费：

```text
order.notify.queue
```

处理：

```text
order.created -> 创建订单通知
order.paid -> 支付成功通知
order.cancelled -> 订单取消通知
```

学习项目可以只写 notifications 表，不调用真实短信或邮件服务。

幂等键：

```text
event_type:{order_id}:notification
```

## 6. timeout-worker

消费：

```text
order.timeout.check.queue
```

处理：

```text
查询订单
如果 paid -> ack
如果 cancelled -> ack
如果 pending_payment -> 条件取消
取消成功 -> 写 outbox order.cancelled
ack
```

条件取消：

```sql
UPDATE orders
SET status = 'cancelled',
    cancelled_reason = 'payment_timeout'
WHERE id = ?
  AND status = 'pending_payment';
```

## 7. 统一错误分类

建议定义：

```text
BadMessageError -> DLQ
DuplicateMessage -> ack
RetryableError -> retry
NonRetryableError -> DLQ
```

不要所有错误都重试。

## 8. 消费者配置

建议配置：

```text
QUEUE_NAME
CONSUMER_TAG
PREFETCH_COUNT
MAX_RETRIES
RETRY_EXCHANGE
DLX_EXCHANGE
```

## 9. 本节小结

消费者服务最重要的是业务正确性：

- 库存不能重复预占。
- 积分不能重复增加。
- 通知不能无限重复。
- 超时取消不能取消已支付订单。
