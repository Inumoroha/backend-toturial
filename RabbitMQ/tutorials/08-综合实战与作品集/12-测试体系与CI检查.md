# 12. 测试体系与 CI 检查

## 1. 测试分层

项目至少包含三层测试：

```text
单元测试
集成测试
故障演练
```

单元测试验证业务函数。

集成测试验证数据库和 RabbitMQ 链路。

故障演练验证异常恢复能力。

## 2. 单元测试

适合测试：

- 订单状态机。
- 积分计算。
- 幂等键生成。
- retry 间隔计算。
- 消息 JSON 编解码。

示例：

```powershell
go test .\internal\domain\...
```

## 3. 集成测试

适合测试：

- 创建订单写 outbox。
- Outbox Worker 发布 RabbitMQ。
- 消费者处理消息。
- 重复消息幂等。
- retry 和 DLQ。

建议目录：

```text
tests/integration/
  order_flow_test.go
  outbox_test.go
  idempotency_test.go
  retry_dlq_test.go
```

## 4. 集成测试启动依赖

测试前：

```powershell
docker compose -f .\deployments\docker-compose.yml up -d
```

执行迁移：

```powershell
docker exec -i ecommerce-mysql mysql -uapp -papp_pwd ecommerce_lab < .\migrations\001_init.sql
```

运行：

```powershell
go test .\tests\integration\...
```

## 5. 测试数据清理

每个测试前清理：

```sql
DELETE FROM processed_messages;
DELETE FROM outbox_messages;
DELETE FROM notifications;
DELETE FROM points_records;
DELETE FROM inventory_reservations;
DELETE FROM order_items;
DELETE FROM orders;
```

也要清理 RabbitMQ 队列。

学习阶段可以通过测试专用队列名避免互相污染。

## 6. CI 检查建议

最小 CI：

```text
go test ./...
go vet ./...
go fmt 检查
```

命令：

```powershell
go test ./...
go vet ./...
gofmt -w .
```

如果使用 GitHub Actions，可以在 README 中说明。

## 7. 必测场景

必须覆盖：

- 创建订单成功。
- 支付订单成功。
- Outbox 发布成功。
- RabbitMQ 不可用时 outbox 保留 pending。
- 重复 `order.paid` 不重复加积分。
- 超时未支付订单自动取消。
- 已支付订单不会被超时取消。
- 消费失败进入 retry。
- 超过次数进入 DLQ。

## 8. 本节小结

作品级项目不能只有代码。

你要有：

- 测试。
- 演练。
- 文档。
- 可重复启动环境。
