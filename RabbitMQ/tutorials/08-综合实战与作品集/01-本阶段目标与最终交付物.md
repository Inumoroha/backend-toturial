# 01. 本阶段目标与最终交付物

## 1. 为什么需要第 8 阶段

前 7 个阶段已经覆盖了：

- RabbitMQ 基础模型。
- Go 客户端开发。
- 可靠性机制。
- 业务模式落地。
- 生产运维。
- 架构取舍。

但学习最终要落到一个问题：

```text
你能不能独立做出一个像样的 RabbitMQ 后端项目？
```

第 8 阶段就是为这个目标设计的。

## 2. 项目定位

项目不是完整电商系统，而是一个聚焦 RabbitMQ 的异步订单系统。

它重点展示：

- 订单事件。
- 异步消费者。
- Outbox。
- 幂等。
- retry。
- DLQ。
- 订单超时取消。
- 监控和故障演练。

你不需要实现复杂前端，也不需要实现真实支付。

## 3. 最终功能范围

必须实现：

- 创建订单。
- 支付订单。
- 订单超时取消。
- 发布 `order.created`。
- 发布 `order.paid`。
- 发布 `order.cancelled`。
- 库存服务消费 `order.created`。
- 积分服务消费 `order.paid`。
- 通知服务消费订单事件。
- Outbox Worker 可靠发布消息。
- 消费者幂等。
- 失败重试和 DLQ。

可以选做：

- Prometheus 指标。
- Grafana 看板。
- 管理后台。
- 多级重试队列。
- Quorum Queue。

```text

```

## 5. 完成标准

完成本阶段后，你应该能对别人说：

```text
我做过一个 Go + RabbitMQ 的异步订单系统，里面用 Outbox 保证数据库事务和消息发布一致，用 publisher confirm 保证生产端可靠，用 manual ack、retry queue 和 DLQ 处理消费失败，用业务唯一键保证消费者幂等，并且做了队列堆积、死信和重复消费的故障演练。
```
