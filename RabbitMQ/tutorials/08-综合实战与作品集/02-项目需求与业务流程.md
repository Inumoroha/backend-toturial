# 02. 项目需求与业务流程

## 1. 项目业务背景

系统模拟一个简化电商订单流程。

用户可以：

- 创建订单。
- 支付订单。
- 等待库存预占。
- 订单超时未支付时被自动取消。

后台系统会异步处理：

- 库存预占。
- 积分发放。
- 通知发送。
- 审计记录。

## 2. 核心业务流程

### 创建订单

```text
客户端调用创建订单 API
  -> order-service 写 orders 和 order_items
  -> order-service 写 outbox order.created
  -> order-service 写 outbox order.timeout.check
  -> 返回 order_id 和 pending_inventory
```

### 库存预占

```text
Outbox Worker 发布 order.created
  -> inventory-worker 消费
  -> 预占库存
  -> 发布 inventory.reserved 或 inventory.reserve_failed
```

### 支付订单

```text
客户端调用支付 API
  -> order-service 条件更新订单为 paid
  -> 写 outbox order.paid
  -> 返回支付成功
```

### 积分和通知

```text
Outbox Worker 发布 order.paid
  -> points-worker 加积分
  -> notification-worker 发送通知
  -> audit-worker 记录事件
```

### 超时取消

```text
order.timeout.check 延迟任务到期
  -> timeout-worker 查询订单
  -> 如果仍未支付，条件取消订单
  -> 写 outbox order.cancelled
```

## 3. API 需求

### 创建订单

```text
POST /api/orders
```

请求：

```json
{
  "user_id": 1001,
  "items": [
    {
      "sku_id": 2001,
      "quantity": 2
    }
  ]
}
```

响应：

```json
{
  "order_id": 1,
  "status": "pending_inventory"
}
```

### 支付订单

```text
POST /api/orders/{order_id}/pay
```

响应：

```json
{
  "order_id": 1,
  "status": "paid"
}
```

### 查询订单

```text
GET /api/orders/{order_id}
```

响应：

```json
{
  "order_id": 1,
  "user_id": 1001,
  "amount": 9900,
  "status": "paid"
}
```

## 4. 订单状态

建议状态：

```text
pending_inventory
pending_payment
paid
cancelled
closed
```

状态流转：

```text
pending_inventory -> pending_payment
pending_inventory -> cancelled
pending_payment -> paid
pending_payment -> cancelled
paid -> closed
```

必须使用状态条件更新，避免重复和乱序事件导致错误状态。

## 5. 事件列表

订单事件：

```text
order.created
order.paid
order.cancelled
```

库存事件：

```text
inventory.reserved
inventory.reserve_failed
```

任务消息：

```text
order.timeout.check
notification.email.send
notification.sms.send
```

## 6. 非功能需求

必须满足：

- 消息至少一次投递。
- 消费者幂等。
- 订单创建和 outbox 写入同事务。
- Outbox Worker 使用 publisher confirm。
- 消费者使用 manual ack。
- 失败消息进入 retry queue。
- 超过重试次数进入 DLQ。
- 可以观察 ready、unacked、DLQ。

## 7. 本节小结

这个项目的核心业务闭环是：

```text
订单 API
  -> 数据库事务
  -> Outbox
  -> RabbitMQ
  -> 消费者
  -> 幂等处理
  -> retry / DLQ
```
