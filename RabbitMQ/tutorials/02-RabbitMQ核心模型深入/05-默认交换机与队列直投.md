# 05. 默认交换机与队列直投

## 1. 什么是默认交换机

RabbitMQ 有一个特殊的默认 exchange。

它的名称是空字符串：

```text
""
```

它是 direct 类型的 exchange。

每个队列创建时，都会自动绑定到默认 exchange，binding key 就是队列名。

## 2. 默认交换机的路由规则

如果存在一个队列：

```text
stage2.default.demo.queue
```

那么它会自动拥有这样一条隐式绑定：

```text
default exchange -- stage2.default.demo.queue --> stage2.default.demo.queue
```

所以生产者发布消息时可以这样投递：

```text
exchange: ""
routing key: stage2.default.demo.queue
```

消息会进入同名队列。

## 3. 为什么初学者容易误会

很多入门教程里，代码看起来像是直接给队列发消息。

例如概念上写成：

```text
send to queue: hello
```

但底层通常是：

```text
exchange: ""
routing key: hello
```

也就是通过默认 exchange 路由到名为 `hello` 的队列。

所以更准确的理解是：

```text
RabbitMQ 中消息仍然是发给 exchange，只是默认 exchange 让队列直投变得很方便。
```

## 4. 手动实验：默认交换机

创建队列：

```text
Name: stage2.default.demo.queue
Type: Classic
Durability: Durable
```

进入：

```text
Exchanges
```

找到默认 exchange。不同 UI 版本可能显示为：

```text
(AMQP default)
```

或者名称为空。

进入默认 exchange 的详情页，发布消息：

```text
Routing key: stage2.default.demo.queue
Payload:
{
  "message_id": "default-001",
  "text": "hello default exchange"
}
```

然后查看：

```text
Queues and Streams -> stage2.default.demo.queue
```

预期：

```text
Ready: 1
```

## 5. routing key 写错会怎样

继续向默认 exchange 发布：

```text
Routing key: not.exists.queue
Payload: wrong routing key
```

如果没有名为 `not.exists.queue` 的队列，这条消息不会进入任何队列。

这再次说明：

```text
发布成功不等于业务队列一定收到。
```

后续可靠性阶段会学习 unroutable message 的处理。

## 6. 默认交换机适合什么

适合：

- Hello World 入门。
- 简单任务队列。
- 明确知道目标 queue 名称的场景。

不适合：

- 复杂业务路由。
- 事件广播。
- 多消费者服务订阅同一事件。
- 需要清晰解耦生产者和消费者的系统。

## 7. 为什么真实业务更推荐显式 exchange

如果生产者直接使用 queue name 作为 routing key，生产者就知道了消费者队列名。

这会让生产者和消费者耦合。

更推荐：

```text
Producer -> order.events -> order.created -> consumer queues
```

而不是：

```text
Producer -> specific.consumer.queue
```

业务事件应该表达事实：

```text
order.created
```

而不是表达要送给谁：

```text
inventory-service-queue
```

## 8. 默认交换机和 direct exchange 的关系

默认 exchange 本质上是一个特殊 direct exchange。

区别是：

- 它由 RabbitMQ 自动创建。
- 名称为空字符串。
- 每个队列会自动绑定到它。
- binding key 是队列名称。

## 9. 本节小结

你要记住：

- 默认 exchange 的名字是空字符串。
- 队列会自动绑定到默认 exchange。
- routing key 等于队列名时，可以直接投递到该队列。
- 入门教程中的“发给队列”通常是通过默认 exchange 实现的。
- 真实业务中更推荐显式 exchange，降低生产者和消费者耦合。
