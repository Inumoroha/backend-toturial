## 建议学习时间

建议用 14 到 21 天完成：

- 第 1 到 2 天：明确项目需求、业务流程和系统架构。
- 第 3 到 4 天：搭建仓库结构、本地 Docker Compose 环境和数据库迁移。
- 第 5 到 6 天：设计 RabbitMQ 拓扑和消息契约。
- 第 7 到 9 天：实现订单 API 和 Transactional Outbox。
- 第 10 到 12 天：实现 Outbox Worker 和可靠发布。
- 第 13 到 15 天：实现库存、积分、通知、超时取消消费者。
- 第 16 到 17 天：完成可靠性验收和故障演练。
- 第 18 到 19 天：补齐测试体系和 CI 检查。
- 第 20 到 21 天：整理项目 README、简历描述和面试表达。

## 最终项目名称

建议项目名：

```text
rabbitmq-go-ecommerce-lab
```

也可以使用中文名：

```text
Go 电商异步订单系统
```

## 最终你要交付什么

完成本阶段后，你应该拥有：

- 一个可运行的 Go 项目。
- 一份完整的项目 README。
- 一个 Docker Compose 本地环境。
- MySQL 或 PostgreSQL 数据库表结构。
- RabbitMQ exchange、queue、binding 拓扑。
- 订单 API。
- Outbox Worker。
- 库存、积分、通知、订单超时消费者。
- retry queue 和 DLQ。
- 幂等消费者。
- 集成测试。
- 故障演练脚本或文档。
- 简历项目描述。
- 面试讲解稿。

## 本阶段重要提醒

这个阶段不是“看完就算”。

你要真的写代码、跑服务、制造故障、观察队列、修复问题。

RabbitMQ 是实践型技术。只有把消息从 API、数据库、Outbox、RabbitMQ、Consumer、Retry、DLQ 全链路跑通，你才算真正掌握。
