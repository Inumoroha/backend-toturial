## 2. 初始化命令

在工作区中执行：

```powershell
mkdir demos
mkdir demos\rabbitmq-go-ecommerce-lab
cd demos\rabbitmq-go-ecommerce-lab
go mod init rabbitmq-go-ecommerce-lab
go get github.com/rabbitmq/amqp091-go
```

如果使用 HTTP 框架，可以选择：

```powershell
go get github.com/go-chi/chi/v5
```

如果使用数据库驱动，根据数据库选择：

```powershell
go get github.com/go-sql-driver/mysql
```

或：

```powershell
go get github.com/jackc/pgx/v5
```

## 4. 配置项规划

建议使用环境变量：

```text
APP_ENV=dev
HTTP_ADDR=:8080
DATABASE_DSN=...
RABBITMQ_URL=amqp://go_learner:go_learner_pwd@localhost:5672/
OUTBOX_BATCH_SIZE=100
OUTBOX_POLL_INTERVAL=1s
MQ_PREFETCH=5
```

不要把生产密码写进代码。

## 5. cmd/main.go 职责

每个 `main.go` 只做启动编排：

```text
加载配置
连接数据库
连接 RabbitMQ
初始化 repository / service / worker
启动服务
监听退出信号
优雅关闭
```

不要把业务逻辑堆在 `main.go`。

## 6. internal/mq 职责

```text
connection.go  -> 建立连接
topology.go    -> 声明 exchange/queue/binding
publisher.go   -> 可靠发布
consumer.go    -> 消费封装
message.go     -> 消息结构
retry.go       -> retry/DLQ 辅助
```

## 7. internal/outbox 职责

```text
message.go     -> OutboxMessage
repository.go  -> 数据库访问
worker.go      -> 扫描和发布
```

## 8. 本节小结

好的脚手架能让项目更像真实后端工程。

核心原则：

- `cmd` 只启动。
- `internal/mq` 封装 RabbitMQ。
- `internal/outbox` 管 Outbox。
- `domain` 放业务规则。
- `docs` 放架构和演练文档。
