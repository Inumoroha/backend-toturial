# 01. 本阶段目标与第 1 阶段复习

## 1. 本阶段学习位置

前两个阶段的关系是：

```text
第 1 阶段：会启动、会点击、会手动发消息
第 2 阶段：理解参数、理解模式、能设计拓扑
```

第 1 阶段你已经知道：

```text
Producer -> Exchange -> Queue -> Consumer
```

第 2 阶段要继续追问：

- Exchange 应该选 direct、fanout 还是 topic？
- Queue 应该 durable、exclusive、auto-delete 吗？
- 一个事件要绑定到几个队列？
- routing key 应该怎么命名？
- 任务队列和事件广播有什么区别？
- 为什么同一个消息可能进入多个队列？

## 2. 先复习核心对象

### Producer

生产者负责发布消息。

生产者发布消息时通常指定：

```text
exchange
routing key
message body
message properties
```

### Exchange

交换机负责路由消息。

它决定：

```text
这条消息应该进入哪些队列
```

### Queue

队列负责存储消息。

消费者通常从队列中消费消息：

```text
Queue -> Consumer
```

### Binding

Binding 是 exchange 和 queue 之间的路由规则。

```text
Exchange -- binding key --> Queue
```

### Routing Key

Routing key 是生产者发布消息时携带的路由信息。

exchange 会根据自身类型解释 routing key。

## 3. 你要建立的新理解

第 1 阶段可以把 RabbitMQ 看成简单管道：

```text
发消息 -> 收消息
```

第 2 阶段要把它看成消息拓扑：

```text
一个业务事件
  -> 进入哪个 exchange
  -> 使用什么 routing key
  -> 命中哪些 binding
  -> 进入哪些 queue
  -> 被哪些 consumer 处理
```

真实项目中，设计拓扑比写几行发送代码更重要。

## 4. 本阶段统一实验环境

本阶段默认你已经启动 RabbitMQ：

```powershell
docker start rabbitmq-dev
```

如果容器不存在，重新启动：

```powershell
docker run -d --hostname rabbitmq-dev --name rabbitmq-dev -p 5672:5672 -p 15672:15672 -e RABBITMQ_DEFAULT_USER=go_learner -e RABBITMQ_DEFAULT_PASS=go_learner_pwd rabbitmq:4-management
```

管理后台：

```text
http://localhost:15672
```

账号：

```text
go_learner / go_learner_pwd
```

本阶段默认使用默认 vhost：

```text
/
```

如果你使用上一阶段创建的 `rabbitmq_learning` vhost，也可以继续使用，但 UI 操作时要注意切换 vhost。

## 5. 常用观察命令

查看 exchange：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_exchanges name type durable auto_delete internal
```

查看 queue：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_queues name durable exclusive auto_delete messages_ready messages_unacknowledged consumers
```

查看 binding：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_bindings
```

查看连接：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_connections name user vhost state channels
```

查看 channel：

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_channels connection number user vhost
```

## 6. 本阶段命名约定

为了不和第 1 阶段混淆，本阶段统一使用 `stage2` 前缀：

```text
Exchange:
stage2.direct
stage2.fanout
stage2.topic

Queue:
stage2.email.queue
stage2.sms.queue
stage2.order.created.queue
stage2.order.all.queue

Routing key:
email.send
sms.send
order.created
order.paid
```

学习结束后，如果想清理这些资源，可以在 Management UI 中删除对应 exchange 和 queue。
