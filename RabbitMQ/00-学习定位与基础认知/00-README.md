# 第 0 阶段：学习定位与基础认知

> 本阶段目标：在正式学习 RabbitMQ 的命令、API 和 Go 代码之前，先建立正确的后端工程思维。你要知道消息队列解决什么问题、带来什么代价，以及作为 Go 后端工程师应该重点掌握哪些能力。

请按下面顺序阅读和练习：

2. [02-消息队列到底解决什么问题.md](./02-消息队列到底解决什么问题.md)

## 建议学习时间

建议用 2 到 3 天完成：

- 第 1 天：阅读 01、02，理解为什么需要消息队列。
- 第 2 天：阅读 03、04，理解异步系统的代价和可靠性问题。
- 第 3 天：阅读 05，完成 06 的练习与自测。

## 本阶段你应该掌握什么

完成本阶段后，你应该能用自己的话讲清楚：

- RabbitMQ 在后端系统中扮演什么角色。
- 同步调用和异步消息各自适合什么场景。
- 为什么消息队列可以解耦、削峰、异步处理。
- 为什么引入消息队列后，系统会出现最终一致性、重复消费、消息堆积等新问题。
- Go 后端学习 RabbitMQ 前需要补哪些基础。

## 本阶段不要急着做什么

这一阶段不要急着封装 MQ SDK，也不要急着研究集群、高可用、复杂调优。先把“为什么用”和“用它会带来什么后果”想明白。

后面的学习会逐步进入：

- RabbitMQ 安装和管理后台
- Exchange、Queue、Binding、Routing Key
- Go 客户端 `amqp091-go`
- ack、prefetch、publisher confirm
- 重试、死信队列、延迟任务
- Outbox、幂等和最终一致性
