# 01. 本阶段目标与项目准备

## 1. 本阶段从哪里开始

前两个阶段你主要在 RabbitMQ Management UI 中手动操作：

```text
创建 exchange
创建 queue
创建 binding
发布消息
获取消息
```

第 3 阶段开始，你要用 Go 完成这些事情。

Go 程序做的事情，本质上仍然是：

```text
连接 RabbitMQ
打开 channel
声明拓扑
发布消息
消费消息
关闭资源
```

## 2. 本阶段使用的 Go 客户端

使用 RabbitMQ 官方团队维护的 AMQP 0-9-1 Go 客户端：

```text
github.com/rabbitmq/amqp091-go
```

安装命令：

```powershell
go get github.com/rabbitmq/amqp091-go
```

代码中通常这样导入：

```go
import amqp "github.com/rabbitmq/amqp091-go"
```

## 3. 确认 RabbitMQ 正在运行

先启动 RabbitMQ：

```powershell
docker start rabbitmq-dev
```

如果容器不存在，重新创建：

```powershell
docker run -d --hostname rabbitmq-dev --name rabbitmq-dev -p 5672:5672 -p 15672:15672 -e RABBITMQ_DEFAULT_USER=go_learner -e RABBITMQ_DEFAULT_PASS=go_learner_pwd rabbitmq:4-management
```

检查容器：

```powershell
docker ps
```

打开管理后台：

```text
http://localhost:15672
```

账号：

```text
go_learner / go_learner_pwd
```

## 4. 创建本阶段 Go 项目

建议在当前 RabbitMQ 学习目录下创建 demo 项目：

```powershell
mkdir demos
mkdir demos\stage3-go-client
cd demos\stage3-go-client
go mod init rabbitmq-go-stage3
go get github.com/rabbitmq/amqp091-go
```

检查：

```powershell
go list -m all
```

你应该能看到：

```text
github.com/rabbitmq/amqp091-go
```

## 5. 推荐目录结构

本阶段可以使用下面结构：

```text
demos/stage3-go-client/

      main.go
    fanout_receive/
      main.go

      main.go

      main.go
    topic_emit/
      main.go
    topic_receive/
      main.go

  go.mod

```

刚开始也可以简单一点：

```text
producer.go
consumer.go
```

但为了后续扩展，推荐使用 `cmd/xxx/main.go`。

## 6. 本阶段统一连接地址

如果你使用前面教程的 Docker 命令，连接地址是：

```text
amqp://go_learner:go_learner_pwd@localhost:5672/
```

含义：

| 部分 | 含义 |
| --- | --- |
| `amqp://` | AMQP 协议 |
| `go_learner` | 用户名 |
| `go_learner_pwd` | 密码 |
| `localhost` | RabbitMQ 地址 |
| `5672` | AMQP 端口 |
| `/` | 默认 vhost |

如果你使用自定义 vhost，例如 `rabbitmq_learning`：

```text
amqp://app_order:app_order_pwd@localhost:5672/rabbitmq_learning
```

## 7. 创建基础连接 helper

在后面的代码里，为了先看清楚每个 demo 的完整流程，大部分示例会直接在 `main.go` 中连接 RabbitMQ。

如果你想提前抽一个 helper，可以创建：

```text
internal/mq/connection.go
```

内容：

```go
package mq

import amqp "github.com/rabbitmq/amqp091-go"

const URL = "amqp://go_learner:go_learner_pwd@localhost:5672/"

func Dial() (*amqp.Connection, error) {
	return amqp.Dial(URL)
}
```

注意：生产环境不会把连接地址硬编码在代码里，应该从配置或环境变量读取。本阶段为了学习先保持简单。

## 8. 检查 Go 版本

执行：

```powershell
go version
```

建议使用 Go 1.22 或更新版本。

检查当前模块：

```powershell
go env GOMOD
```

如果输出是 `NUL` 或空，说明你当前不在 Go module 目录里。请进入：

```powershell
cd demos\stage3-go-client
```

## 9. 本阶段代码风格

本阶段代码会使用：

- `log.Fatalf` 处理启动失败。
- `defer conn.Close()` 关闭连接。
- `defer ch.Close()` 关闭 channel。
- `context.WithTimeout` 控制发布超时。
- `d.Ack(false)` 手动确认部分消息。

这些 demo 不是最终生产代码。

生产代码还要补：

- 配置管理。
- 连接重试。
- channel 重建。
- publisher confirm。
- 统一日志。
- 指标监控。
- 优雅退出。
