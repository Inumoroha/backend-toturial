# 12. 第 3 阶段练习与自测

## 2. 练习一：Hello World

### 任务

写两个程序：

```text
cmd/hello_producer/main.go
cmd/hello_consumer/main.go
```

要求：

- 声明 `stage3.hello.queue`。
- producer 通过默认 exchange 发布消息。
- consumer 从队列消费消息。
- consumer 使用 manual ack。

### 检查命令

```powershell
docker exec -it rabbitmq-dev rabbitmqctl list_queues name messages_ready messages_unacknowledged consumers
```

## 3. 练习二：Work Queue

### 任务

实现：

```text
Exchange: stage3.task.direct
Queue: stage3.email.task.queue
Routing key: email.send
```

要求：

- producer 可以从命令行读取任务内容。
- 启动两个 worker。
- 发布多条任务。
- 观察任务被多个 worker 分担。

### 思考

请回答：

```text
Work Queue 中一条任务会被所有 worker 都处理吗？
```

参考答案：

```text
通常不会。一条任务会进入一个队列，多个 worker 共享消费这个队列，每条消息通常只由其中一个 worker 处理。
```

## 4. 练习三：Fanout 发布订阅

### 任务

实现：

```text
Exchange: stage3.logs.fanout
Type: fanout
```

要求：

- 每个 receiver 创建临时队列。
- 每个临时队列绑定到 fanout exchange。
- 启动两个 receiver。
- 发布一条消息，两个 receiver 都收到。

### 思考

请回答：

```text
为什么 fanout receiver 使用临时队列？
```

参考答案：

```text
每个订阅者需要自己的一份消息。临时队列适合跟随客户端生命周期，客户端退出后自动清理。
```

## 5. 练习四：Direct Routing

### 任务

实现：

```text
Exchange: stage3.notify.direct
Type: direct

Binding keys:
email.send
sms.send
```

要求：

- `direct_receive email.send` 只收邮件消息。
- `direct_receive sms.send` 只收短信消息。
- `direct_receive email.send sms.send` 同时收两类消息。

### 思考

请回答：

```text
Direct exchange 是不是只能一对一？
```

参考答案：

```text
不是。多个队列可以绑定同一个 binding key，匹配的消息会进入多个队列。
```

## 6. 练习五：Topic Routing

### 任务

实现：

```text
Exchange: stage3.business.topic
Type: topic
```

启动三个 receiver：

```powershell
go run .\cmd\topic_receive "order.#"
go run .\cmd\topic_receive "*.created"
go run .\cmd\topic_receive "payment.*"
```

发布：

```powershell
go run .\cmd\topic_emit order.created "order created"
go run .\cmd\topic_emit order.payment.succeeded "payment succeeded"
go run .\cmd\topic_emit user.created "user created"
go run .\cmd\topic_emit payment.failed "payment failed"
```

### 预期

- `order.created` 被 `order.#` 和 `*.created` 收到。
- `order.payment.succeeded` 被 `order.#` 收到。
- `user.created` 被 `*.created` 收到。
- `payment.failed` 被 `payment.*` 收到。

## 7. 练习六：JSON 消息结构

### 任务

定义：

```go
type Event struct {
	MessageID     string          `json:"message_id"`
	EventType     string          `json:"event_type"`
	Version       int             `json:"version"`
	OccurredAt    time.Time       `json:"occurred_at"`
	CorrelationID string          `json:"correlation_id"`
	Payload       json.RawMessage `json:"payload"`
}
```

发布一条：

```text
event_type: order.created
routing key: order.created
```

consumer 解析外层 `Event`，再解析 `OrderCreatedPayload`。

### 思考

请回答：

- `message_id` 用来做什么？
- `correlation_id` 用来做什么？
- `version` 为什么重要？
- 为什么不要把敏感信息放进消息？

## 8. 练习七：优雅退出

### 任务

实现一个 consumer：

- 使用 `signal.NotifyContext` 监听 `Ctrl+C`。
- 收到退出信号后调用 `ch.Cancel`。
- 当前消息处理完成后 ack。
- 打印退出日志。

### 思考

请回答：

```text
如果消费者处理成功，但 ack 前进程崩溃，可能会发生什么？
```

参考答案：

```text
RabbitMQ 没收到 ack，可能重新投递这条消息。因此消费者需要幂等。
```
