# 12. 综合项目：Go 电商异步订单系统

## 1. 项目目标

实现一个简化电商异步订单系统，把第 5 阶段所有模式串起来。

你要完成：

- 创建订单。
- Outbox 发布 `order.created`。
- 库存服务消费并预占库存。
- 支付成功后发布 `order.paid`。
- 积分服务消费并加积分。
- 通知服务消费并发送通知。
- 订单超时未支付自动取消。
- 所有消费者幂等。
- 失败消息重试和进入 DLQ。

## 2. 服务划分

```text
cmd/

```

业务包：

```text

  storage/
```

## 3. RabbitMQ 拓扑

订单事件：

```text
Exchange: order.events
Type: topic

Queues:
order.created.inventory.queue   binding: order.created
order.paid.points.queue         binding: order.paid
order.notify.queue              binding: order.created, order.paid, order.cancelled
order.all.audit.queue           binding: order.#
```

通知任务：

```text
Exchange: notification.task.direct
Type: direct

Queues:
notification.email.worker.queue binding: notification.email.send
notification.sms.worker.queue   binding: notification.sms.send
```

订单超时：

```text
Exchange: order.delay.exchange
Queue: order.timeout.15m.delay.queue
DLX -> order.task.exchange

Exchange: order.task.exchange
Queue: order.timeout.check.queue binding: order.timeout.check
```

## 4. 数据库表

核心表：

```text
orders
order_items
inventory_reservations
points_records
notifications
outbox_messages
processed_messages
```

订单表简化：

```sql
CREATE TABLE orders (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    amount BIGINT NOT NULL,
    status VARCHAR(32) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

处理记录表：

```sql
CREATE TABLE processed_messages (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    idempotency_key VARCHAR(128) NOT NULL,
    message_id VARCHAR(128) NOT NULL,
    event_type VARCHAR(64) NOT NULL,
    processed_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE KEY uk_idempotency_key (idempotency_key)
);
```

## 5. 创建订单流程

API：

```text
POST /api/orders
```

流程：

```text
开启事务
  -> 写 orders status=pending_inventory
  -> 写 order_items
  -> 写 outbox order.created
  -> 写 outbox order.timeout.check
提交事务
返回 order_id 和 status
```

注意：

```text
API 返回不代表库存已经预占完成。
```

## 6. 库存 worker

消费：

```text
order.created.inventory.queue
```

流程：

```text
收到 order.created
幂等键 order:{order_id}:inventory:reserve
检查库存
预占库存
写 inventory_reservations
写 outbox inventory.reserved
ack
```

库存不足：

```text
写 outbox inventory.reserve_failed
ack
```

订单服务消费 `inventory.reserve_failed` 后取消订单。

## 7. 支付成功流程

模拟支付 API：

```text
POST /api/orders/{id}/pay
```

流程：

```text
开启事务
  -> 条件更新 orders status paid
  -> 写 outbox order.paid
提交事务
```

条件更新：

```sql
UPDATE orders
SET status = 'paid'
WHERE id = ? AND status IN ('pending_payment', 'pending_inventory');
```

真实业务要更严格，这里是学习项目简化。

## 8. 积分 worker

消费：

```text
order.paid.points.queue
```

流程：

```text
收到 order.paid
幂等键 order:{order_id}:points:add
写 points_records
更新用户积分
ack
```

重复消息：

```text
唯一键冲突 -> 直接 ack
```

## 9. 通知 worker

消费：

```text
order.notify.queue
```

根据事件类型：

```text
order.created -> 发送订单创建通知
order.paid -> 发送支付成功通知
order.cancelled -> 发送取消通知
```

也可以转换成通知任务：

```text
order event -> notification.email.send
```

这样通知发送逻辑更统一。

## 10. 超时 worker

消费：

```text
order.timeout.check.queue
```

流程：

```text
查询订单状态
如果 paid -> ack
如果 cancelled -> ack
如果 pending_payment -> 条件取消订单
取消成功 -> 写 outbox order.cancelled
ack
```

条件取消：

```sql
UPDATE orders
SET status = 'cancelled'
WHERE id = ? AND status = 'pending_payment';
```

## 11. 失败处理

所有 worker 统一：

```text
成功 -> ack
重复 -> ack
可重试失败 -> retry queue
不可重试失败 -> DLQ
超过最大重试 -> DLQ
```

每个 DLQ 都要监控。

学习项目可以先共用一个：

```text
app.dlq
```

生产环境建议按业务拆分。

## 12. 项目完成标准

你完成项目后，应该能演示：

1. 创建订单后，outbox 有 `order.created`。
2. Outbox Worker 发布成功后，outbox 状态变 `sent`。
3. 库存 worker 消费 `order.created`。
4. 支付后发布 `order.paid`。
5. 积分 worker 只加一次积分。
6. 通知 worker 收到订单事件。
7. 未支付订单超时取消。
8. 重复消息不会造成重复业务结果。
9. 失败消息会重试。
10. 超过次数后进入 DLQ。
