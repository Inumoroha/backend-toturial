# 7. 压测、pprof、连接池、背压与高性能直觉

本节目标：建立 Go 后端网络性能分析的基础直觉，掌握压测、pprof、连接数和背压的基本观察方法。

---

## 一、压测工具

```bash
hey -n 10000 -c 100 http://127.0.0.1:8080/
```

或：

```bash
wrk -t4 -c100 -d30s http://127.0.0.1:8080/
```

关注：

- QPS。
- 平均延迟。
- P95/P99。
- 错误率。

不要只看平均值，尾延迟更重要。

---

## 二、pprof

```go
import _ "net/http/pprof"

go func() {
    http.ListenAndServe("127.0.0.1:6060", nil)
}()
```

查看 goroutine：

```bash
curl http://127.0.0.1:6060/debug/pprof/goroutine?debug=1
```

CPU：

```bash
go tool pprof http://127.0.0.1:6060/debug/pprof/profile
```

---

## 三、连接池

连接池太小：

- 请求排队。
- 延迟升高。

连接池太大：

- 下游压力过大。
- fd 和内存占用升高。

连接池要结合下游能力配置。

---

## 四、背压

背压表示下游处理不过来时，上游要减速或拒绝，而不是无限堆积。

常见方式：

- 限流。
- 有界队列。
- 超时。
- 熔断。
- 快速失败。

---

## 补充实验：压测时同时观察服务端

压测不要只看 `hey` 输出。建议同时开三个窗口。

窗口 1 启动服务：

```bash
go run .
```

窗口 2 压测：

```bash
hey -n 10000 -c 100 http://127.0.0.1:8080/
```

窗口 3 观察连接：

```bash
watch -n 1 "ss -ant | grep ':8080' | wc -l"
```

如果开启了 pprof，再观察 goroutine：

```bash
curl http://127.0.0.1:6060/debug/pprof/goroutine?debug=1 | head -40
```

你要把这些指标一起看：

```text
QPS 是否上升。
P95/P99 是否恶化。
错误率是否出现。
连接数是否持续堆积。
goroutine 是否持续增长。
CPU 是否打满。
```

只看 QPS 很容易误判。一个服务 QPS 很高但 P99 很差，对用户来说依然是慢服务。

## 补充理解：有界队列就是一种背压

错误示例：

```go
jobs := make(chan Job) // 无缓冲或无限堆积思路都危险
```

更可控的方式：

```go
jobs := make(chan Job, 1000)

select {
case jobs <- job:
    // accepted
default:
    http.Error(w, "server busy", http.StatusServiceUnavailable)
}
```

当队列满时，服务快速拒绝请求，而不是无限占用内存。

这就是背压思想：

```text
下游处理不过来时，上游必须减速、排队上限或失败返回。
```

对后端系统来说，明确失败通常比无限排队更容易恢复。

## 补充压测复盘：一次压测至少记录这些

压测结束后不要只贴一张 QPS 截图。建议记录：

```text
压测工具和命令。
并发数。
总请求数或持续时间。
QPS。
平均延迟。
P95/P99 延迟。
错误率。
CPU、内存、连接数。
goroutine 数量。
下游依赖是否被打满。
瓶颈判断。
下一步优化动作。
```

示例结论：

```text
并发从 50 到 100 时 QPS 只提升 5%，P99 从 80ms 升到 900ms。
服务 CPU 未打满，但 goroutine 数和到 PostgreSQL 的等待增加。
初步判断瓶颈在数据库连接池或慢查询，不是 Go HTTP 层。
```

这类结论才对工程优化有价值。

## 五、常见问题

### 1. QPS 越高就一定越好吗？

不是。还要看错误率、P95/P99 延迟、资源使用和下游压力。

### 2. 压测本机结果能代表生产吗？

不能完全代表。本机压测适合理解趋势，生产容量还要结合真实网络、机器规格和依赖。

### 3. pprof 能直接告诉我怎么优化吗？

pprof 提供证据，优化仍需要结合业务和代码结构判断。
