# 0. 后端网络工程能力总览

本阶段目标：从单个 Go 服务走向真实后端系统，理解 Nginx、反向代理、负载均衡、API 网关、限流、熔断、重试、WebSocket、gRPC 和高性能网络基础。

---

## 一、本阶段学习顺序

```text
1_Nginx反向代理与七层转发
2_负载均衡策略与健康检查
3_API网关_路由_鉴权_日志_限流
4_超时_重试_熔断_幂等与故障放大
5_WebSocket长连接与心跳
6_gRPC_HTTP2_Protobuf与服务间通信
7_压测_pprof_连接池_背压与高性能直觉
```

---

## 二、本阶段达标标准

学完本阶段后，你应该能够做到：

- 用 Nginx 代理 Go 服务。
- 解释四层和七层负载均衡。
- 设计 API 网关的基础能力。
- 解释重试为什么可能放大故障。
- 说明 WebSocket 和 gRPC 的适用场景。
- 使用 hey/wrk 和 pprof 做基础性能观察。
