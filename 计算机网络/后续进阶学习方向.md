# 计算机网络后续进阶学习方向

本篇目标：在完成 `第1阶段` 到 `第9阶段` 后，继续规划更深入的计算机网络学习路线。前面的课程帮助你建立 Go 后端工程师必备的网络基础；后续进阶则面向生产环境、分布式系统、高性能网关、云原生网络和可观测性。

如果说前 9 个阶段解决的是：

```text
我能不能写出可靠的 Go 网络服务？
我能不能排查常见网络问题？
我能不能理解 HTTP、TCP、DNS、TLS、Nginx、gRPC？
```

那么后续进阶要解决的是：

```text
我能不能理解 Kubernetes 网络？
我能不能分析复杂链路中的性能瓶颈？
我能不能读懂网关、RPC、Service Mesh 的网络设计？
我能不能用 eBPF、pprof、抓包和指标定位疑难问题？
```

---

## 一、进阶学习总路线

建议按下面顺序继续学习：

```text
1. Go runtime netpoller 与高并发网络模型
2. Linux 网络内核基础
3. QUIC、HTTP/3 与现代传输协议
4. Kubernetes 网络
5. Service Mesh 与东西向流量治理
6. eBPF 网络观测与故障定位
7. 高性能网关与代理源码阅读
8. 生产网络安全与零信任基础
9. 大型项目综合实践
```

不要急着全部展开。建议每个方向都配一个可运行实验，否则很容易变成“知道名词但没有手感”。

## 二、方向一：Go runtime netpoller 与高并发网络模型

### 为什么要学

前面你已经会写：

```go
for {
    conn, _ := ln.Accept()
    go handleConn(conn)
}
```

但还没有深入理解：

- 为什么 Go 可以用大量 goroutine 处理连接。
- goroutine 阻塞在网络 I/O 时，线程是否也阻塞。
- Go runtime 如何借助 epoll/kqueue/IOCP 管理网络事件。
- 高并发下 goroutine、fd、连接池、GC 如何互相影响。

### 学什么

- goroutine 调度基础。
- netpoller 基本思想。
- epoll / kqueue / IOCP。
- goroutine-per-connection 模型。
- 事件循环模型。
- Go 网络 I/O 与 runtime 调度关系。

### 怎么练

1. 写一个 TCP Echo Server。
2. 建立 1 万个空闲连接。
3. 观察 goroutine 数、内存、fd 数。
4. 使用 pprof 查看 goroutine 堆栈。
5. 对比 Go TCP Server 和事件循环框架的设计差异。

### 达标标准

你能解释：

- goroutine-per-connection 为什么不是 thread-per-connection。
- Go 网络 I/O 阻塞时 runtime 大致如何处理。
- 高连接数服务要关注哪些资源指标。

## 三、方向二：Linux 网络内核基础

### 为什么要学

生产环境中的网络问题经常发生在应用和内核之间：

```text
连接队列满。
fd 耗尽。
临时端口耗尽。
TCP 重传。
网卡丢包。
conntrack 表满。
iptables 规则异常。
```

只懂 Go 代码不够，需要能看懂 Linux 网络栈的关键指标。

### 学什么

- socket 与文件描述符。
- listen backlog。
- SYN 队列与 accept 队列。
- TCP 状态机。
- conntrack。
- iptables / nftables。
- netfilter。
- 网卡队列。
- `/proc/sys/net` 常见参数。

### 怎么练

常用命令：

```bash
ss -s
ss -lnt
ss -ant state time-wait
sysctl net.core.somaxconn
sysctl net.ipv4.tcp_max_syn_backlog
cat /proc/sys/net/ipv4/ip_local_port_range
sudo iptables -L -n -v
sudo conntrack -L
```

可以做的实验：

- 制造大量短连接，观察 TIME_WAIT。
- 限制 fd 数，观察 `too many open files`。
- 用 iptables 拦截端口，观察 `timeout` 和 `refused` 的区别。
- 用压测工具观察 accept 队列和连接状态变化。

### 达标标准

你能解释：

- SYN 队列和 accept 队列的区别。
- `Connection refused` 和 `timeout` 的常见原因。
- fd、临时端口、conntrack 如何影响网络连接。

## 四、方向三：QUIC、HTTP/3 与现代传输协议

### 为什么要学

HTTP/3 已经在浏览器、CDN、边缘网关中越来越常见。它不是 HTTP/2 的简单升级，而是把底层从 TCP 换成了 QUIC。

### 学什么

- QUIC 基于 UDP。
- QUIC 内置 TLS 1.3。
- 0-RTT / 1-RTT 建连。
- 多路复用。
- 连接迁移。
- TCP 队头阻塞与 QUIC 的改进。
- HTTP/2 与 HTTP/3 对比。

### 怎么练

```bash
curl -v --http3 https://cloudflare.com
```

如果本机 curl 不支持 HTTP/3，可以先学习理论，再使用支持 QUIC 的工具或浏览器开发者工具观察协议版本。

推荐实验：

- 对比 HTTP/1.1、HTTP/2、HTTP/3 请求。
- 观察浏览器 Network 面板中的协议列。
- 阅读 quic-go 的基础示例。

### 达标标准

你能解释：

- HTTP/3 为什么基于 QUIC。
- QUIC 为什么基于 UDP。
- HTTP/2 和 HTTP/3 的多路复用差异。

## 五、方向四：Kubernetes 网络

### 为什么要学

Go 后端服务最终很可能运行在 Kubernetes 中。此时你会遇到：

```text
Pod IP。
Service IP。
NodePort。
ClusterIP。
Ingress。
CoreDNS。
CNI。
NetworkPolicy。
```

这些都和计算机网络直接相关。

### 学什么

- Pod 网络模型。
- CNI 插件。
- Service：ClusterIP、NodePort、LoadBalancer。
- kube-proxy：iptables / IPVS。
- Ingress。
- CoreDNS。
- NetworkPolicy。
- Pod 到 Pod 通信。
- Pod 到 Service 通信。
- 集群内外访问链路。

### 怎么练

建议使用 kind 或 minikube：

```bash
kind create cluster
kubectl get pods -A
kubectl get svc -A
kubectl run curl --image=curlimages/curl -it --rm -- sh
```

实验：

- 创建两个 Deployment。
- 用 Service 暴露其中一个。
- 在 Pod 内访问 Service DNS。
- 查看 CoreDNS 日志。
- 使用 `kubectl exec` 进入 Pod 查看 `ip addr` 和 `ip route`。

### 达标标准

你能解释：

- Pod IP、Service IP、Node IP 的区别。
- Service 如何把请求转发到 Pod。
- 为什么 Kubernetes 中 DNS 很重要。
- Ingress、Service、Pod 之间的请求链路。

## 六、方向五：Service Mesh 与东西向流量治理

### 为什么要学

当服务越来越多，服务间通信会变复杂：

```text
超时。
重试。
熔断。
限流。
灰度发布。
mTLS。
流量镜像。
可观测性。
```

Service Mesh 把这些网络治理能力从业务代码中抽离出来，交给 sidecar 或代理层处理。

### 学什么

- Sidecar 模式。
- Envoy 基础。
- Istio 基本概念。
- mTLS。
- VirtualService。
- DestinationRule。
- 流量灰度。
- 熔断和重试策略。
- 服务间观测指标。

### 怎么练

可以先不急着完整部署 Istio，先理解：

```text
应用容器 -> sidecar proxy -> 目标服务 sidecar -> 目标应用
```

实验建议：

- 部署一个简单 Istio demo。
- 配置请求按比例分流。
- 配置超时和重试。
- 观察 Envoy access log。

### 达标标准

你能解释：

- Service Mesh 解决什么问题。
- Sidecar 如何劫持服务流量。
- mTLS、灰度、重试、熔断在 Mesh 中如何表达。

## 七、方向六：eBPF 网络观测与故障定位

### 为什么要学

传统排障依赖日志、指标、抓包。但有些问题发生在内核层，应用日志看不到。eBPF 可以在低开销下观测内核事件，是现代 Linux 可观测性的重要方向。

### 学什么

- eBPF 基本概念。
- kprobe / tracepoint。
- XDP。
- tc。
- bpftrace。
- Cilium。
- 网络延迟、丢包、重传观测。

### 怎么练

先从工具开始，不急着写 eBPF 程序：

```bash
sudo bpftrace -l
```

可以学习：

- bcc tools。
- bpftrace。
- Cilium Hubble。

实验方向：

- 观察 TCP connect。
- 观察网络重传。
- 观察进程发起的连接。
- 在 Kubernetes 中使用 Hubble 查看服务调用图。

### 达标标准

你能解释：

- eBPF 为什么适合做网络观测。
- 它和 tcpdump、日志、指标的区别。
- Cilium 为什么和 Kubernetes 网络联系紧密。

## 八、方向七：高性能网关与代理源码阅读

### 为什么要学

Nginx、Envoy、Traefik、Caddy、HAProxy、Go 自研网关都属于后端网络工程的重要基础设施。

读源码不是为了背实现，而是学习：

- 连接管理。
- 事件循环。
- 配置加载。
- 负载均衡。
- 健康检查。
- 超时。
- 限流。
- 指标。
- 热更新。

### 推荐阅读对象

先易后难：

```text
Go 标准库 net/http
Caddy 部分模块
Traefik 路由和中间件
Envoy 架构文档
Nginx 事件模型资料
HAProxy 配置与架构文章
```

### 怎么练

可以自己实现一个小网关：

- 静态路由。
- 轮询负载均衡。
- 超时。
- 健康检查。
- 限流。
- 请求日志。
- Prometheus 指标。

### 达标标准

你能解释：

- 一个网关的核心模块有哪些。
- 连接池和超时如何设计。
- 健康检查如何影响负载均衡。
- 指标和日志如何帮助排障。

## 九、方向八：生产网络安全与零信任基础

### 为什么要学

后端网络不是只要能连通，还要安全。

生产中要考虑：

- TLS。
- mTLS。
- 证书轮换。
- API 鉴权。
- WAF。
- 防重放。
- 内网访问控制。
- NetworkPolicy。
- 零信任访问。

### 学什么

- TLS 证书生命周期。
- mTLS 双向认证。
- JWT 与 opaque token。
- API Gateway 鉴权。
- NetworkPolicy。
- 最小权限访问。
- 服务到服务身份。

### 怎么练

实验建议：

- 为 Go HTTP 服务启用 TLS。
- 使用自签 CA 做 mTLS。
- 给网关加入 JWT 校验。
- 在 Kubernetes 中配置 NetworkPolicy。

### 达标标准

你能解释：

- TLS 和 mTLS 的区别。
- 为什么内网服务也需要身份认证。
- NetworkPolicy 如何限制 Pod 间访问。

## 十三、最终达标标准

完成后续进阶学习后，你应该能够做到：

- 解释 Go 高并发网络模型。
- 看懂 Linux 网络关键参数。
- 理解 QUIC 和 HTTP/3。
- 排查 Kubernetes 中的 DNS、Service、Ingress 问题。
- 理解 Service Mesh 的流量治理模型。
- 使用 pprof、tcpdump、ss、eBPF 工具定位复杂网络问题。
- 设计一个具备超时、限流、负载均衡、健康检查和观测能力的 Go 网关。
- 从“会写网络服务”进阶到“能治理、观测、优化生产网络系统”。

网络进阶不是为了堆名词，而是为了在复杂系统里保持判断力。你最终要形成的能力是：看到一个请求失败，能把它拆成 DNS、路由、TCP、TLS、HTTP、网关、服务、下游依赖和内核资源这些层次，并用证据一步步定位。
