# 0. 环境搭建与网络直觉总览

本阶段目标：搭建学习计算机网络需要的本地实验环境，并建立最基本的网络直觉：一台机器如何访问另一台机器，一个 Go 服务如何被请求命中，命令行工具如何帮助我们证明网络现象。

学习计算机网络不能只看概念。你需要反复做三类事情：

```text
看地址：IP、端口、网关、DNS。
看连接：监听端口、TCP 状态、连接来源。
看报文：抓包观察请求到底有没有发出去、有没有回来。
```

如果一开始只背 OSI 七层、TCP 三次握手、HTTP 状态码，很容易出现一种错觉：文字都看懂了，但服务一部署就不知道从哪里排查。本阶段就是为了先把“能观察网络”的能力建立起来。

---

## 一、本阶段你要学什么

本阶段包含 5 个具体小节：

```text
1_安装 Go、Docker 与基础网络工具
2_认识本机网络信息：IP、网关、DNS、端口
3_使用 Docker 理解容器网络和端口映射
4_使用 Wireshark 与 tcpdump 做第一次抓包
5_启动一个 Go HTTP 服务并完成端到端访问
```

每一节都不是单纯安装工具，而是围绕一个问题展开。

例如：

- 为什么服务监听 `127.0.0.1` 时，外部机器访问不到？
- 为什么 Docker 容器里访问 `localhost` 不是访问宿主机？
- 为什么 `ping` 通了，但 `curl` 仍然失败？
- 为什么 Go HTTP 服务要显式配置超时？

这些问题比背定义更接近真实后端开发。

## 二、推荐学习环境

如果你使用 Windows，推荐：

```text

Docker Desktop
WSL2 Ubuntu
Go 1.22+
Wireshark
VS Code 或 GoLand
```

本教程会尽量同时给出：

- Linux / macOS 命令。

但后续线上排障基本都以 Linux 为主，因为 Go 后端服务最终大概率运行在 Linux 服务器、Docker 容器或 Kubernetes 集群中。

## 三、为什么后端工程师要先学工具

计算机网络不是纯理论课。线上遇到问题时，你需要回答的是：

```text
请求有没有到达机器？
机器有没有监听端口？
端口是不是被防火墙挡了？
DNS 有没有解析到正确 IP？
TCP 连接有没有建立？
TLS 握手有没有成功？
HTTP 服务有没有返回状态码？
```

这些问题不能靠猜，必须靠工具证明。

本阶段会用到的工具包括：

| 工具 | 作用 |
| --- | --- |
| `ip` / `ipconfig` | 查看本机 IP、网卡、网关 |
| `curl` | 发起 HTTP 请求并观察细节 |
| `dig` / `nslookup` | 查询 DNS |
| `ss` / `netstat` | 查看端口监听和连接状态 |
| `lsof` | 查看端口被哪个进程占用 |
| `tcpdump` | 命令行抓包 |
| Wireshark | 图形化抓包分析 |
| Docker | 模拟服务部署和网络隔离 |

工具不是附属内容，它们是你理解网络的“眼睛”。

## 四、本阶段完成后的能力

学完本阶段，你应该能够做到：

- 安装并验证 Go、Docker、curl、dig、ss、tcpdump、Wireshark。
- 查看本机 IP、默认网关、DNS。
- 解释 `127.0.0.1`、`0.0.0.0`、局域网 IP 的区别。
- 使用 Docker 启动容器，并理解 `8080:8080` 端口映射的含义。
- 使用 Wireshark 或 tcpdump 抓到一次 HTTP 请求。
- 启动一个 Go HTTP 服务，并从本机、容器或另一个终端访问它。
- 初步判断“服务启动了但访问不到”应该从哪些方向排查。

---

## 五、本阶段学习方式

每节建议按下面方式学习：

1. 先读目标和背景，不急着敲命令。
2. 按步骤执行命令。
3. 对照教程理解输出。
4. 故意制造一个小错误，例如端口写错、服务停止、监听地址改成 `127.0.0.1`。
5. 用命令证明错误原因。

学习网络最重要的不是“一次成功”，而是知道失败时怎么看。

## 六、目录建议

建议在本阶段单独建立实验目录：

```bash
mkdir network-stage-1
cd network-stage-1
```

后续文件可以这样组织：

```text
network-stage-1/
  hello-http/
  docker-demo/
  captures/
  notes/
```

其中：

- `hello-http`：放 Go HTTP 服务。
- `docker-demo`：放 Docker 相关实验。
- `captures`：放抓包文件。
- `notes`：放自己的实验记录。

---

## 七、本阶段达标标准

学完本阶段后，你应该能够独立完成下面的任务：

- 用命令说明本机当前 IP、网关、DNS 是什么。
- 启动一个 Go HTTP 服务并访问成功。
- 查看该服务监听在哪个端口。
- 抓到一次访问该服务的 TCP 或 HTTP 报文。
- 用 Docker 运行一个容器，并解释端口映射。
- 遇到端口访问失败时，能按“服务是否启动 -> 端口是否监听 -> 地址是否正确 -> 防火墙或容器映射是否正确”的顺序排查。

如果这些还做不到，不建议急着进入 TCP 三次握手。先把工具和直觉练熟，后面的协议学习会轻松很多。
