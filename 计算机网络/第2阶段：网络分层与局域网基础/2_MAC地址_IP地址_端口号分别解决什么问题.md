# 2. MAC 地址、IP 地址、端口号分别解决什么问题

本节目标：真正区分 MAC 地址、IP 地址和端口号。它们经常同时出现在网络排障中，但解决的是三个不同层次的问题。

一句话先建立直觉：

```text
MAC 地址：这一跳交给哪块网卡。
IP 地址：最终要找哪台主机。
端口号：主机上的哪个进程接收数据。
```

---

## 一、为什么需要三种地址

假设你要访问一个 Go 服务：

```text
http://192.168.1.20:8080/hello
```

这里至少有三个问题：

```text
目标主机是谁？192.168.1.20
目标进程入口是谁？8080
本地网络中下一跳交给谁？某个 MAC 地址
```

如果没有 IP，网络不知道目标主机在哪里。

如果没有端口，目标主机不知道交给哪个进程。

如果没有 MAC，同一链路内设备不知道帧交给哪块网卡。

---

## 二、MAC 地址

MAC 地址是数据链路层地址，通常和网卡相关。

格式类似：

```text
00:1a:2b:3c:4d:5e
```

查看本机 MAC：

```bash
ip link
```

你会看到类似：

```text
link/ether 00:1a:2b:3c:4d:5e
```

MAC 地址主要在同一局域网或同一链路中使用。跨过路由器后，链路层头部会变化，下一段链路会使用新的源 MAC 和目标 MAC。

---

## 三、IP 地址

IP 地址是网络层地址，用来标识主机或网络接口。

IPv4 示例：

```text
192.168.1.10
```

查看本机 IP：

```bash
ip addr
```

常见私有 IP 范围：

| 范围 | 常见场景 |
| --- | --- |
| `10.0.0.0/8` | 企业内网、云内网 |
| `172.16.0.0/12` | Docker、企业网络 |
| `192.168.0.0/16` | 家庭路由器 |

私有 IP 不能直接在公网路由，需要通过 NAT 或代理访问公网。

---

## 四、端口号

端口号是传输层概念，用来区分同一台主机上的不同进程。

例如一台机器可以同时运行：

```text
22    SSH
80    HTTP
443   HTTPS
5432  PostgreSQL
6379  Redis
8080  Go Web 服务
```

查看监听端口：

```bash
ss -lntp
```

如果 Go 服务监听 `8080`，你会看到：

```text
LISTEN 0 4096 0.0.0.0:8080
```

端口不是机器地址，它是进程入口。

---

## 五、一个请求中的三种地址

你访问：

```bash
curl http://192.168.1.20:8080/hello
```

如果目标在同一局域网：

```text
目标 IP：192.168.1.20
目标端口：8080
目标 MAC：192.168.1.20 对应的网卡 MAC
```

如果目标不在同一局域网：

```text
目标 IP：最终服务器 IP
目标端口：最终服务器端口
目标 MAC：默认网关的 MAC
```

这是很多人容易混的点：

```text
跨网段通信时，链路层目标 MAC 通常是下一跳网关，不是最终服务器。
```

---

## 六、实验：查看 IP、MAC、端口

查看 IP 和 MAC：

```bash
ip addr
ip link
```

查看路由：

```bash
ip route
```

查看 ARP 或邻居缓存：

```bash
ip neigh
```

查看监听端口：

```bash
ss -lntp
```

启动一个简单服务：

```bash
python3 -m http.server 8080
```

另一个终端查看：

```bash
ss -lntp | grep 8080
```

## 七、Go 后端中的典型错误

### 1. 监听地址写错

只监听本机：

```go
http.ListenAndServe("127.0.0.1:8080", nil)
```

外部机器通常访问不到。

监听所有网卡：

```go
http.ListenAndServe(":8080", nil)
```

### 2. 注册了错误 IP

服务注册中心中如果注册了容器内部 IP，其他机器可能无法访问。

例如：

```text
172.17.0.2:8080
```

这个地址可能只在 Docker 宿主机内部有效。

### 3. 端口映射理解反了

Docker：

```bash
docker run -p 8081:8080 app
```

访问时应该访问宿主机：

```text
宿主机IP:8081
```

不是 `8080`。

## 八、常见问题

### 1. IP 和 MAC 都是地址，为什么不能只用一个？

MAC 更适合同一链路内转发，IP 更适合跨网络寻址。互联网规模很大，需要层次化的 IP 路由体系。

### 2. 端口是不是应用层概念？

不是。端口属于传输层 TCP/UDP，用来把数据交给主机上的具体进程。

### 3. 一个端口能被多个进程监听吗？

通常不能。同一个 IP、协议、端口组合只能被一个服务监听。特殊场景可以通过 `SO_REUSEPORT` 等机制实现多进程共享，但初学阶段先按“不能重复监听”理解。

### 4. 为什么服务启动报 address already in use？

说明对应地址和端口已经被占用。用：

```bash
ss -lntp | grep 端口
```

找到占用进程。

## 九、本节达标标准

学完本节后，你应该能够做到：

- 解释 MAC、IP、端口分别解决什么问题。
- 使用 `ip addr` 查看 IP。
- 使用 `ip link` 查看 MAC。
- 使用 `ss -lntp` 查看端口。
- 解释为什么跨网段时目标 MAC 是网关。
- 解释 Go 服务监听 `127.0.0.1` 和 `0.0.0.0` 的区别。

掌握这些之后，就可以进入下一节：局域网通信、交换机、路由器和网关。
