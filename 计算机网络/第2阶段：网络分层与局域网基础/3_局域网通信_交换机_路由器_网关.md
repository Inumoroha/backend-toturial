# 3. 局域网通信：交换机、路由器、网关

本节目标：理解同网段通信和跨网段通信的区别，掌握交换机、路由器、默认网关分别扮演什么角色。

很多网络问题的本质都是：

```text
目标在不在同一个网段？
如果不在，默认网关是谁？
数据包有没有被正确转发到下一跳？
```

---

## 一、先理解网段

看到一个地址：

```text
192.168.1.10/24
```

它表示：

- IP：`192.168.1.10`
- 网络前缀：`/24`
- 大致网段：`192.168.1.0/24`

同一网段中的常见地址可能是：

```text
192.168.1.1
192.168.1.20
192.168.1.100
```

不在同一网段的地址：

```text
192.168.2.10
10.0.0.5
8.8.8.8
```

主机发包前会先判断：

```text
目标 IP 是否和我在同一网段？
```

---

## 二、同网段通信流程

假设：

```text
A: 192.168.1.10
B: 192.168.1.20
网段: 192.168.1.0/24
```

A 访问 B：

```text
curl http://192.168.1.20:8080
```

大致流程：

```text
1. A 判断 B 和自己在同一网段。
2. A 需要知道 B 的 MAC 地址。
3. A 发送 ARP 广播：谁是 192.168.1.20？
4. B 回复自己的 MAC。
5. A 把数据帧发给 B 的 MAC。
6. 交换机根据 MAC 表把帧转发给 B。
```

关键点：

```text
同网段通信不需要经过默认网关。
```

---

## 三、跨网段通信流程

假设：

```text
A: 192.168.1.10/24
目标: 8.8.8.8
默认网关: 192.168.1.1
```

A 访问 `8.8.8.8`：

```text
1. A 判断 8.8.8.8 不在本地网段。
2. A 查询默认网关 192.168.1.1 的 MAC。
3. A 把 IP 包交给网关的 MAC。
4. 网关根据路由表继续转发。
```

注意：

```text
IP 包的目标 IP 是 8.8.8.8。
以太网帧的目标 MAC 是默认网关的 MAC。
```

每经过一个路由器，链路层 MAC 会变化，但目标 IP 通常仍然是最终目标。

---

## 四、交换机做什么

交换机主要工作在二层。

它维护一张 MAC 地址表：

```text
MAC 地址 -> 交换机端口
```

当交换机收到一个以太网帧时，会看目标 MAC，然后决定从哪个端口转发出去。

交换机不负责公网路由。它只关心当前二层网络里的 MAC 转发。

---

## 五、路由器做什么

路由器主要工作在三层。

它维护路由表：

```text
目标网段 -> 下一跳 / 出接口
```

当路由器收到 IP 包时，会看目标 IP，并决定下一跳发给谁。

家庭路由器通常同时做很多事：

- 默认网关。
- NAT。
- DHCP。
- Wi-Fi 接入点。
- 简单防火墙。

但从网络模型上看，路由器最核心的能力是三层转发。

---

## 六、默认网关是什么

默认网关是当目标不在本地网段时，主机默认交给的下一跳。

查看默认网关：

```bash
ip route
```

你可能看到：

```text
default via 192.168.1.1 dev eth0
```

含义：

```text
访问不知道怎么走的目标，就交给 192.168.1.1。
```

如果默认网关错误，可能出现：

- 能访问同网段机器。
- 不能访问外网。
- 不能访问其他网段服务。

---

## 七、实验：判断目标怎么走

查看本机地址：

```bash
ip addr
```

查看路由：

```bash
ip route
```

查询访问某个目标的路由：

```bash
ip route get 8.8.8.8
```

输出可能类似：

```text
8.8.8.8 via 192.168.1.1 dev eth0 src 192.168.1.10
```

说明：

- 目标是 `8.8.8.8`。
- 下一跳是 `192.168.1.1`。
- 从 `eth0` 发出。
- 源 IP 是 `192.168.1.10`。

---

## 八、实验：查看 ARP 缓存

查看邻居表：

```bash
ip neigh
```

或者：

```bash
arp -a
```

先 ping 网关：

```bash
ping 网关IP
```

再查看：

```bash
ip neigh
```

你应该能看到网关 IP 对应的 MAC 地址。

这说明本机已经知道下一跳网关的链路层地址。

## 九、Go 后端中的场景

### 1. 应用访问数据库超时

如果数据库在另一个网段，你需要检查：

```bash
ip route get 数据库IP
ping 数据库IP
nc -vz 数据库IP 5432
```

问题可能不是 Go 代码，而是路由或防火墙。

### 2. 容器可以访问同网络服务，但不能访问外部服务

检查容器路由：

```bash
docker exec -it 容器ID sh
ip route
```

可能是默认网关、NAT 或 DNS 问题。

### 3. 服务注册了内网 IP

如果调用方和服务方不在同一个网络环境，内网 IP 可能不可达。

## 十、常见问题

### 1. 交换机和路由器最简单的区别是什么？

交换机主要看 MAC，路由器主要看 IP。

### 2. 同网段通信一定不经过路由器吗？

通常不经过三层路由器，直接在二层网络中转发。但真实网络设备可能有三层交换机等组合设备。初学阶段先按基本模型理解。

### 3. 默认网关是不是 DNS？

不是。默认网关负责转发非本地网段流量，DNS 负责把域名解析成 IP。

### 4. 为什么跨网段时目标 MAC 是网关？

因为链路层只负责当前这一跳。当前主机无法直接把帧发到远端服务器的网卡，只能先交给下一跳网关。

---

## 十一、本节达标标准

学完本节后，你应该能够做到：

- 判断两个 IP 是否大致在同一网段。
- 解释同网段通信为什么需要 ARP。
- 解释跨网段通信为什么需要默认网关。
- 区分交换机和路由器。
- 使用 `ip route get` 判断访问目标的下一跳。
- 使用 `ip neigh` 或 `arp -a` 查看邻居缓存。

掌握这些之后，就可以进入下一节：MTU、MSS 与网络分片问题。
