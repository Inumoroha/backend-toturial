# 0. 网络分层与局域网基础总览

本阶段目标：理解网络为什么要分层，弄清 MAC 地址、IP 地址、端口号分别解决什么问题，并建立局域网通信的基本直觉。

第一阶段你已经能启动服务、查看端口、抓包。现在开始回答更底层的问题：

```text
为什么访问服务需要 IP 和端口？
为什么同一个局域网内要先找 MAC？
为什么跨网段要经过网关？
为什么 HTTP 请求会被一层层封装？
为什么大包有时比小包更容易出问题？
```

这些问题是后续学习 TCP、HTTP、Docker 网络和线上排障的基础。

---

## 一、本阶段包含哪些内容

本阶段建议按下面顺序学习：

```text
1_OSI七层与TCPIP四层模型
2_MAC地址_IP地址_端口号分别解决什么问题
3_局域网通信_交换机_路由器_网关
4_MTU_MSS与网络分片问题
```

每一节都围绕一个核心直觉：

- 分层模型：复杂网络如何拆成可理解的层次。
- 地址体系：不同层次如何定位设备、主机和进程。
- 局域网通信：同网段和跨网段访问有什么不同。
- MTU/MSS：数据太大时网络如何处理。

---

## 二、为什么后端工程师要学这一阶段

很多后端问题表面看是代码问题，实际是网络基础问题。

例如：

```text
服务监听 127.0.0.1，导致容器外访问不到。
服务注册了容器 IP，其他机器无法访问。
Nginx upstream 写错 IP，返回 502。
VPN 场景下小请求正常，大文件上传失败。
跨网段访问数据库超时。
```

这些问题都离不开：

- IP。
- 端口。
- 路由。
- 网关。
- MTU。
- 分层排查。

---

## 三、本阶段的学习方法

不要只背：

```text
MAC 是二层，IP 是三层，TCP 是四层，HTTP 是七层。
```

更重要的是能解释：

```text
MAC 负责哪一跳？
IP 负责找哪台主机？
端口负责找哪个进程？
网关什么时候出现？
HTTP 请求在发送前被封装了什么头？
```

每节学习时都建议画一张小图。网络知识如果只留在文字里，很容易混。

## 五、本阶段达标标准

学完本阶段后，你应该能够做到：

- 解释 OSI 七层和 TCP/IP 四层的对应关系。
- 说清 MAC、IP、端口的区别。
- 解释同网段通信和跨网段通信的差异。
- 解释交换机、路由器、网关的作用。
- 解释为什么服务监听地址会影响访问范围。
- 使用 `ip addr`、`ip route`、`arp -a` 查看本机网络信息。
- 解释 MTU/MSS 对大请求、大响应、容器网络的影响。

如果你能把“浏览器访问一个 Go HTTP 服务”从应用层一路讲到链路层，说明本阶段已经基本掌握。
