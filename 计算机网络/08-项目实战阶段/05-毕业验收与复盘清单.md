- MAC、IP、端口的区别。
- 同网段通信和跨网段通信流程。
- ARP、ICMP、NAT 的作用。

- TIME_WAIT 和 CLOSE_WAIT。

- UDP 和 TCP 的区别。
- DNS 解析流程。
- HTTP 请求响应格式。
- HTTP Keep-Alive 和 TCP Keepalive 的区别。

- HTTP/1.1、HTTP/2、HTTP/3 的基本差异。

## 五、排障验收

给出下面问题的排查路径：

### 1. 用户反馈接口很慢

你应该想到：

- DNS 是否慢。
- TCP 连接是否慢。
- TLS 是否慢。
- 网关是否等待上游。
- 后端 handler 是否慢。
- 下游 DB/RPC 是否慢。
- 连接池是否耗尽。

### 2. 服务出现大量 CLOSE_WAIT

你应该想到：

- 程序没有关闭连接。
- HTTP Client 没有关闭 body。
- TCP 读写循环没有处理 EOF。

### 3. Nginx 返回 502

你应该想到：

- 上游服务没启动。
- 上游端口错了。
- 上游连接被拒绝。
- Nginx upstream 配置错了。

### 4. Nginx 返回 504

你应该想到：

- 上游服务处理慢。
- 下游依赖慢。
- 网关超时太短。
- 请求排队或连接池耗尽。

## 六、面试表达训练

每个问题按这个结构回答：

```text
先讲结论。
再讲机制。
再讲 Go 中的体现。
最后讲排障或工程实践。
```

示例：为什么要关闭 `resp.Body`？

```text
结论：为了释放响应体资源，并让连接有机会回到连接池。
机制：HTTP Client 通过 Transport 管理连接复用，如果响应体不关闭，连接生命周期无法结束。
Go 体现：client.Do 返回 resp 后，应 defer resp.Body.Close()，必要时读完 body。
工程实践：忘记关闭会导致连接泄漏、CLOSE_WAIT 增多或文件描述符耗尽。
```

## 九、验收标准

完成本篇后，你应该能：

- 对照原理、编码、命令、项目、排障五个维度完成自测。
- 至少完成 TCP 聊天室、HTTP 反向代理、迷你 API 网关、gRPC 用户服务四个项目说明中的基础要求。
- 面对接口慢、502、504、CLOSE_WAIT、端口不通等问题，说出可执行的排查路径。
- 用“结论、机制、Go 体现、工程实践”的结构回答常见网络面试题。
