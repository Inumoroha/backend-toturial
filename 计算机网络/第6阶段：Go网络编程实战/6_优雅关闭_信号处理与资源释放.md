# 6. 优雅关闭：信号处理与资源释放

本节目标：让 Go HTTP 服务在收到退出信号时停止接收新请求，并等待正在处理的请求完成。

---

## 一、为什么需要优雅关闭

如果直接杀进程：

- 正在处理的请求会中断。
- 客户端可能收到连接重置。
- 日志和指标可能丢失。
- 数据库事务可能未按预期结束。

优雅关闭的目标：

```text
不接新请求。
给旧请求一点时间完成。
超时后退出。
```

---

## 二、示例代码

```go
package main

import (
    "context"
    "log"
    "net/http"
    "os"
    "os/signal"
    "syscall"
    "time"
)

func main() {
    mux := http.NewServeMux()
    mux.HandleFunc("/hello", func(w http.ResponseWriter, r *http.Request) {
        w.Write([]byte("hello"))
    })

    srv := &http.Server{
        Addr:              ":8080",
        Handler:           mux,
        ReadHeaderTimeout: 2 * time.Second,
    }

    go func() {
        log.Println("listen on :8080")
        if err := srv.ListenAndServe(); err != nil && err != http.ErrServerClosed {
            log.Fatal(err)
        }
    }()

    ctx, stop := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)
    defer stop()

    <-ctx.Done()
    log.Println("shutdown start")

    shutdownCtx, cancel := context.WithTimeout(context.Background(), 10*time.Second)
    defer cancel()

    if err := srv.Shutdown(shutdownCtx); err != nil {
        log.Println("shutdown error:", err)
    }

    log.Println("shutdown done")
}
```

## 补充实验：验证正在处理的请求能否完成

增加一个慢接口：

```go
mux.HandleFunc("/slow", func(w http.ResponseWriter, r *http.Request) {
    time.Sleep(5 * time.Second)
    fmt.Fprintln(w, "done")
})
```

启动服务后请求：

```bash
curl http://127.0.0.1:8080/slow
```

在请求未结束时按 `Ctrl+C`。

如果使用 `Shutdown(ctx)`，服务会：

```text
停止接收新连接。
等待正在处理的请求完成。
超过 ctx 超时时间后返回错误。
```

如果直接 `Close()`，正在处理的请求更可能被中断。

## 补充清单：优雅关闭还要释放什么

真实项目里不只 HTTP Server 要关，还可能包括：

```text
数据库连接池。
Redis Client。
消息队列消费者。
后台 goroutine。
定时任务。
日志缓冲。
pprof 或 metrics server。
```

建议在 main 中把资源关闭顺序写清楚，并给每一步记录日志。这样发布、重启、容器滚动更新时更容易判断服务是否真的退出干净。

## 补充检查：容器环境下的退出信号

在 Docker 或 Kubernetes 中，服务通常收到的是 `SIGTERM`，不是你手动按下的 `Ctrl+C`。

所以代码里要监听：

```go
signal.Notify(stop, os.Interrupt, syscall.SIGTERM)
```

并确认：

```text
收到信号后不再接收新请求。
已有请求在超时时间内完成。
后台 goroutine 能退出。
日志能看到 shutdown start 和 shutdown done。
```

如果服务关闭太慢，容器编排系统可能会强制杀掉进程。

## 四、常见问题

### 1. Shutdown 和 Close 区别是什么？

`Shutdown` 会优雅关闭，等待已有请求。`Close` 会立即关闭。

### 2. 优雅关闭能等待无限久吗？

不应该。必须设置超时。

### 3. 只处理 Ctrl+C 够吗？

本地开发够用。部署环境还应处理 `SIGTERM`。
