# 01-IP 地址、子网与路由表

## 学习目标

理解 IP 地址、子网掩码、CIDR、默认网关和路由表。后端工程师不需要成为网络工程师，但必须能看懂基本路由信息，否则排查容器、云服务器、Kubernetes 网络问题会很吃力。

## 一、IPv4 地址

IPv4 地址由 32 位组成，通常写成点分十进制：

```text
192.168.1.10
```

常见私有地址范围：

| 范围 | 用途 |
| --- | --- |
| `10.0.0.0/8` | 大型内网 |
| `172.16.0.0/12` | Docker、企业内网常见 |
| `192.168.0.0/16` | 家庭路由器常见 |

私有地址不能直接在公网路由，需要通过 NAT 访问公网。

## 二、CIDR 表示法

`192.168.1.10/24` 表示：

- IP 地址：`192.168.1.10`
- 网络前缀长度：24 位
- 子网掩码：`255.255.255.0`
- 网络号：`192.168.1.0`
- 可用主机大致范围：`192.168.1.1` 到 `192.168.1.254`

`/24` 的含义是前 24 位是网络部分，后 8 位是主机部分。

## 三、如何判断是否同网段

主机会用自己的 IP 和子网掩码计算网络号，再用目标 IP 和子网掩码计算目标网络号。

如果网络号相同，认为目标在同一网段，可以直接发给目标。

如果网络号不同，交给默认网关。

## 四、路由表

查看路由表：

```bash
ip route
```

典型输出：

```text
default via 192.168.1.1 dev eth0
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.10
```

含义：

- `192.168.1.0/24 dev eth0`：访问本地网段走 `eth0`。
- `default via 192.168.1.1`：其他地址交给默认网关 `192.168.1.1`。

## 五、实验：查看访问目标走哪条路由

```bash
ip route get 8.8.8.8
```

你会看到类似信息：

```text
8.8.8.8 via 192.168.1.1 dev eth0 src 192.168.1.10
```

这说明访问 `8.8.8.8` 会经过默认网关。

## 六、实验：traceroute 观察路径

```bash
traceroute example.com
```

Windows：

```powershell
tracert example.com
```

每一跳通常是一个路由器或网关。线上排查跨地域网络慢、运营商链路问题时，traceroute 可以提供线索。

## 七、Go 后端关联

Go 程序通常不直接操作路由表，但它的网络访问受路由表影响。

常见场景：

- 服务访问数据库超时，实际是路由不可达。
- 容器内能访问外网，宿主机不能，或反过来。
- Kubernetes Pod 能访问 Service，但不能访问外部 IP。
- 云服务器有公网 IP，但安全组或路由配置不正确。

## 八、练习题

1. `10.1.2.3/16` 的网络号是什么？
2. 默认网关解决什么问题？
3. `ip route get 目标IP` 能帮助你判断什么？
4. 私有 IP 为什么不能直接在公网通信？

## 九、验收标准

你能看懂 `ip route` 的基本输出，并能判断访问一个目标地址是本地直连还是经过默认网关。
