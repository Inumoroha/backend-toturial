# 1. IP 地址、CIDR、私有地址与公网地址

本节目标：理解 IPv4 地址、CIDR、子网掩码、私有地址和公网地址。学完后你应该能看懂 `192.168.1.10/24`，并知道后端服务部署时为什么会遇到内网 IP、公网 IP、容器 IP。

---

## 一、IPv4 地址是什么

IPv4 地址由 32 位组成，通常写成点分十进制：

```text
192.168.1.10
```

它用于网络层寻址，表示某个网络接口在 IP 网络中的地址。

查看本机 IP：

```bash
ip addr
```

Windows：

```powershell
ipconfig
```

---

## 二、CIDR 是什么

你经常会看到：

```text
192.168.1.10/24
```

这里：

- `192.168.1.10` 是 IP 地址。
- `/24` 表示前 24 位是网络部分。
- 对应子网掩码通常是 `255.255.255.0`。
- 所在网段大致是 `192.168.1.0/24`。

同网段的地址通常包括：

```text
192.168.1.1
192.168.1.20
192.168.1.200
```

不在这个网段的地址：

```text
192.168.2.10
10.0.0.1
8.8.8.8
```

---

## 三、为什么要有子网

如果整个互联网都是一个平面网络，路由器无法高效管理转发规则。

子网的作用是：

- 把地址空间分块。
- 让路由表按网段转发。
- 控制广播范围。
- 方便企业、云厂商、容器网络规划。

后端工程师常见的子网场景：

- 云服务器 VPC 网段。
- Kubernetes Pod 网段。
- Docker bridge 网段。
- 数据库内网地址。

---

## 四、私有地址

常见私有 IPv4 地址范围：

| 范围 | 常见场景 |
| --- | --- |
| `10.0.0.0/8` | 企业内网、云 VPC |
| `172.16.0.0/12` | Docker、企业网络 |
| `192.168.0.0/16` | 家庭和小型办公网络 |

私有地址不能直接在公网路由。

例如你的电脑可能是：

```text
192.168.1.10
```

它访问公网时，通常要经过路由器 NAT。

---

## 五、公网地址

公网地址可以在互联网中被路由。

云服务器通常会有：

- 内网 IP：用于同 VPC 内通信。
- 公网 IP：用于互联网访问。

一个 Go 服务能不能被公网访问，不只取决于它是否启动，还取决于：

- 是否监听公网网卡或 `0.0.0.0`。
- 云安全组是否开放端口。
- 系统防火墙是否允许。
- 路由是否正确。

---

## 六、实验：识别本机地址

执行：

```bash
ip addr
```

记录：

```text
网卡名：
IPv4 地址：
CIDR：
是否私有地址：
```

再查看公网出口 IP：

```bash
curl ifconfig.me
```

如果本机 IP 是 `192.168.x.x`，但公网出口是另一个地址，说明中间发生了 NAT。

## 七、Go 后端中的常见地址问题

### 1. 服务注册了错误地址

服务运行在容器里，注册了：

```text
172.17.0.2:8080
```

其他机器可能访问不到，因为这是 Docker bridge 内部地址。

### 2. 只监听本机地址

```go
http.ListenAndServe("127.0.0.1:8080", nil)
```

外部访问失败。

### 3. 公网 IP 能 ping，但端口不通

可能是：

- 服务没监听。
- 防火墙没放行。
- 安全组没开放。
- 服务监听在 `127.0.0.1`。

---

## 八、常见问题

### 1. 私有 IP 能不能访问公网？

可以，但通常需要 NAT。私有 IP 本身不能直接在公网路由。

### 2. 公网 IP 能不能访问私有 IP？

通常不能直接访问，除非有 VPN、专线、端口映射、NAT 网关或代理。

### 3. `/24` 是什么意思？

表示前 24 位是网络部分，剩下 8 位是主机部分。

### 4. 容器 IP 是不是稳定的？

通常不稳定。容器重建后 IP 可能变化，不应该在配置中长期写死容器 IP。
