## 学习要求

1. 按阶段顺序学习，不跳过总览和实验。
2. 每节至少留下命令输出、抓包、运行结果或故障复现中的一种证据。
3. 每阶段结束后完成复盘，并检查阶段达标标准。
4. 项目阶段按最小链路、协议与错误、超时与清理、并发与可靠性、观测与文档五轮推进。

## 总体目标

- 解释一次 HTTPS 请求从域名解析到服务响应的完整过程。
- 编写 TCP、UDP、HTTP、WebSocket 和 gRPC 的 Go 程序。
- 正确配置超时、连接池、重试、限流、熔断与优雅关闭。
- 使用常见 Linux 网络命令和抓包工具逐层排障。
- 完成四个网络项目，并提供运行、测试、排障和复盘记录。
