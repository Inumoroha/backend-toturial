# 01-网络分层与数据传输全流程

## 学习目标

理解为什么网络要分层，以及一次请求从 Go 客户端发出到服务端返回，中间经历了哪些步骤。

## 一、为什么需要分层

网络通信很复杂，涉及应用数据、可靠传输、寻址、路由、物理介质等问题。如果所有问题都混在一起，协议会非常难设计、难实现、难排障。

分层的好处：

- 每层只解决一类问题。
- 上层不用关心下层细节。
- 排障时可以逐层定位。
- 协议可以独立演进。

## 二、TCP/IP 四层模型

| 层级 | 典型协议 | 解决的问题 |
| --- | --- | --- |
| 应用层 | HTTP、DNS、TLS、gRPC | 应用如何表达请求和响应 |
| 传输层 | TCP、UDP | 进程到进程之间如何通信 |
| 网络层 | IP、ICMP | 主机到主机之间如何寻址和路由 |
| 网络接口层 | Ethernet、Wi-Fi、ARP | 同一链路内如何传输帧 |

作为 Go 后端工程师，最常接触的是应用层和传输层，但线上排障经常要向下看到 IP、路由、ARP、网卡和防火墙。

## 三、一次 HTTP 请求的完整过程

假设 Go 程序请求：

```text
https://api.example.com/users/1
```

大致流程：

1. DNS 解析 `api.example.com`，得到服务器 IP。
2. 客户端选择一个本地临时端口。
3. 客户端与服务器 `443` 端口建立 TCP 连接。
4. 双方完成 TLS 握手。
5. 客户端发送 HTTP 请求。
6. 服务端处理请求并返回 HTTP 响应。
7. 连接可能复用，也可能关闭。

每一层看到的数据不同：

```text
应用层：HTTP 请求
传输层：TCP 段，包含源端口和目标端口
网络层：IP 包，包含源 IP 和目标 IP
链路层：以太网帧，包含源 MAC 和目标 MAC
```

## 四、封装与解封装

发送数据时，每经过一层都会加上该层头部，这叫封装。

```text
HTTP 数据
=> TCP 头 + HTTP 数据
=> IP 头 + TCP 头 + HTTP 数据
=> Ethernet 头 + IP 头 + TCP 头 + HTTP 数据
```

接收数据时则反过来逐层拆开，这叫解封装。

## 五、Go 中如何体现分层

### HTTP 层

```go
resp, err := http.Get("https://example.com")
```

你操作的是 URL、Header、StatusCode、Body。

### TCP 层

```go
conn, err := net.Dial("tcp", "example.com:80")
```

你操作的是连接、读写字节流。

### IP 和端口

```go
ln, err := net.Listen("tcp", "0.0.0.0:8080")
```

`0.0.0.0` 表示监听本机所有网卡地址，`8080` 是端口。

## 六、实验：观察请求路径

### 1. DNS 解析

```bash
dig example.com
```

关注：

- `ANSWER SECTION`
- 返回的 A 记录
- 查询耗时

### 2. 连通性测试

```bash
ping example.com
```

关注：

- 是否有响应
- 延迟
- 是否丢包

### 3. 路由路径

```bash
traceroute example.com
```

Windows：

```powershell
tracert example.com
```

关注请求经过了哪些网关。

### 4. HTTP 细节

```bash
curl -v https://example.com
```

关注：

- DNS 是否成功
- TCP 是否连接成功
- TLS 是否成功
- HTTP 状态码

## 七、常见误区

- HTTP 不是传输层协议，它运行在 TCP 或 QUIC 之上。
- TCP 负责可靠传输，但不理解 HTTP 业务含义。
- IP 只负责尽力把包送到目标主机，不保证可靠。
- 端口不是机器的地址，而是机器上进程通信的入口。

## 八、练习题

1. TCP/IP 四层模型每层解决什么问题？
2. 浏览器访问 HTTPS 网站时，DNS、TCP、TLS、HTTP 的顺序是什么？
3. `net.Dial("tcp", "example.com:80")` 里的 `tcp`、`example.com`、`80` 分别代表什么？
4. 为什么排查网络问题时要分层定位？

## 九、验收标准

你能画出一次 HTTP 请求从应用层到链路层的封装过程，并能用 `dig`、`ping`、`traceroute`、`curl -v` 分别观察 DNS、连通性、路径和 HTTP 细节。
