# 02-MAC、IP、端口与局域网基础

## 学习目标

区分 MAC 地址、IP 地址、端口号的作用，理解局域网内通信和跨网段通信的差异。

## 一、三个地址分别解决什么问题

### MAC 地址

MAC 地址用于同一链路内的设备识别，通常由网卡提供。

示例：

```text
00:1A:2B:3C:4D:5E
```

它更接近“这一跳要交给哪块网卡”。

### IP 地址

IP 地址用于网络层寻址，帮助数据跨网络找到目标主机。

示例：

```text
192.168.1.10
```

它更接近“目标主机在哪里”。

### 端口号

端口号用于定位主机上的具体进程。

示例：

```text
127.0.0.1:8080
```

`127.0.0.1` 是主机地址，`8080` 是进程监听入口。

## 二、局域网内通信

同一网段内，主机可以直接通过二层网络通信。流程通常是：

1. A 判断 B 与自己在同一网段。
2. A 使用 ARP 查询 B 的 MAC 地址。
3. A 把 IP 包封装进以太网帧，目标 MAC 填 B 的 MAC。
4. 交换机根据 MAC 表转发帧。

## 三、跨网段通信

如果目标 IP 不在同一网段，主机会把包交给默认网关。

流程：

1. A 判断目标 IP 不在本地网段。
2. A 查询默认网关的 MAC 地址。
3. A 把帧发给网关。
4. 网关根据路由表继续转发。

注意：跨网段时，以太网帧的目标 MAC 是下一跳网关，不是最终服务器。

## 四、监听地址的区别

Go 服务常见监听地址：

```text
127.0.0.1:8080
0.0.0.0:8080
:8080
```

区别：

- `127.0.0.1`：只允许本机访问。
- `0.0.0.0`：监听所有网卡。
- `:8080`：通常等价于监听所有网卡的 `8080` 端口。

如果你在云服务器或容器里启动服务，只监听 `127.0.0.1`，外部通常无法访问。

## 五、实验：查看本机网络信息

Linux：

```bash
ip addr
ip route
```

Windows：

```powershell
ipconfig
route print
```

观察：

- 本机 IP 地址
- 子网掩码或 CIDR
- 默认网关
- DNS 服务器

## 六、实验：监听地址对访问范围的影响

创建 Go HTTP 服务：

```go
package main

import (
    "fmt"
    "net/http"
)

func main() {
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintln(w, "hello")
    })

    fmt.Println("listen on 127.0.0.1:8080")
    http.ListenAndServe("127.0.0.1:8080", nil)
}
```

本机访问：

```bash
curl http://127.0.0.1:8080
```

然后把监听地址改成：

```go
http.ListenAndServe("0.0.0.0:8080", nil)
```

再尝试用局域网 IP 访问：

```bash
curl http://你的局域网IP:8080
```

## 七、Go 后端关联

部署服务时，常见问题都和地址有关：

- 服务只监听 `127.0.0.1`，导致 Nginx 或外部机器访问不到。
- Docker 容器内服务监听正确，但端口没有映射到宿主机。
- 安全组或防火墙没有开放端口。
- 服务注册的地址是容器内 IP，其他机器无法访问。

## 八、练习题

1. MAC 地址、IP 地址、端口号分别解决什么问题？
2. 为什么访问不同网段的主机要经过网关？
3. `127.0.0.1:8080` 和 `0.0.0.0:8080` 有什么区别？
4. 为什么云服务器服务启动了，但公网访问不到？

## 九、验收标准

你能说清楚“一台机器上的一个 Go HTTP 服务”至少需要 IP 和端口才能被访问，也能解释为什么局域网访问失败时要检查监听地址、防火墙和路由。
