# 0. 项目实战总览

本阶段目标：按统一迭代方法完成 TCP 聊天室、HTTP 反向代理、迷你 API 网关和 gRPC 用户服务，并用测试、日志、连接状态和文档证明交付质量。

项目阶段把协议、Go API、超时、并发、治理和排障串成可运行成果，重点不是代码数量，而是可验证、可排障和可复现。

---

## 一、本阶段学习顺序

1. `1. 项目落地方法：目录结构、迭代节奏与交付标准`
2. `2. TCP 聊天室项目总览与协议设计`
3. `3. TCP 聊天室：连接管理、广播、心跳`
4. `4. HTTP 反向代理项目总览与单上游转发`
5. `5. HTTP 反向代理：负载均衡、超时、日志`
6. `6. 迷你 API 网关：路由、限流、健康检查`
7. `7. gRPC 用户服务：proto、server、client`
8. `8. 项目验收、排障与复盘清单`
9. `9. TCP 聊天室完整落地：从目录到运行`
10. `10. HTTP 反向代理完整落地：从目录到运行`
11. `11. 迷你 API 网关完整落地：配置、路由、限流与健康检查`
12. `12. gRPC 用户服务完整落地：proto 生成、Server 与 Client`

请按顺序完成正文、命令或代码实验，并使用每节末尾的达标标准验收。
