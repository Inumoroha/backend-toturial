# 02-学习方法与实验规范

## 学习目标

建立一套稳定的学习方法。计算机网络不是靠背概念学会的，而是靠“概念 + 报文 + 命令 + 代码 + 故障案例”一起建立直觉。

## 一、每个知识点的学习闭环

建议每个主题都按下面顺序学习：

1. 先理解它解决什么问题。
2. 再看它在协议栈中属于哪一层。
3. 用命令或抓包观察真实现象。
4. 写一个最小 Go 程序验证。
5. 思考它在后端系统里的工程影响。

例如学习 TCP TIME_WAIT：

- 问题：为什么连接关闭后还要保留一段时间？
- 层级：传输层 TCP。
- 实验：制造短连接，用 `ss` 查看 TIME_WAIT。
- Go：写一个 HTTP Client 循环请求本地服务。
- 工程：短连接过多可能导致临时端口耗尽。

## 三、常用命令分类

### 连通性

```bash
ping example.com
traceroute example.com
curl -v https://example.com
```

### DNS

```bash
dig example.com
dig +trace example.com
nslookup example.com
```

### 端口与连接

```bash
ss -lntp
ss -antp
lsof -i :8080
```

### 抓包

```bash
sudo tcpdump -i any port 8080 -nn
sudo tcpdump -i any host 8.8.8.8 -nn
```

### 系统限制

```bash
ulimit -n
cat /proc/sys/net/ipv4/ip_local_port_range
```

## 四、Go 网络实验规范

写 Go 网络实验时，尽量遵守：

- 每个实验放一个独立目录。
- 程序启动后打印监听地址。
- 所有网络请求都设置超时。
- Client 请求结束后关闭响应体。
- 错误不要忽略，必须打印出来。
- 用 `context.Context` 控制取消和超时。

一个最小 HTTP Client 示例：

```go
package main

import (
    "context"
    "fmt"
    "io"
    "net/http"
    "time"
)

func main() {
    ctx, cancel := context.WithTimeout(context.Background(), 3*time.Second)
    defer cancel()

    req, err := http.NewRequestWithContext(ctx, http.MethodGet, "https://example.com", nil)
    if err != nil {
        panic(err)
    }

    client := &http.Client{Timeout: 5 * time.Second}
    resp, err := client.Do(req)
    if err != nil {
        panic(err)
    }
    defer resp.Body.Close()

    body, _ := io.ReadAll(resp.Body)
    fmt.Println(resp.Status)
    fmt.Println(len(body))
}
```

## 五、不要跳过的问题

学习时遇到下面问题，不要只背答案，要做实验：

- 为什么 `localhost` 可以访问，换成局域网 IP 不行？
- 为什么服务监听 `127.0.0.1` 后容器外访问不到？
- 为什么接口偶发超时但服务日志没有错误？
- 为什么 `curl` 很快，Go 程序请求很慢？
- 为什么大量短连接会出现很多 TIME_WAIT？
- 为什么忘记关闭 `resp.Body` 会导致连接泄漏？

## 六、阶段复盘模板

每学完一个阶段，回答这些问题：

1. 这一层协议解决了什么问题？
2. 它依赖下层提供什么能力？
3. 它向上层提供什么能力？
4. Go 标准库里哪里体现了这个知识？
5. 线上出问题时用什么命令证明？

## 七、验收标准

完成本篇后，你应该能：

- 为每次网络实验写出目的、环境、步骤、观察结果和结论。
- 说清楚一个知识点如何从概念落到命令、抓包和 Go 代码。
- 按连通性、DNS、端口、连接状态、抓包、系统限制对常用命令分类。
- 在 Go 网络实验中主动设置超时、关闭连接并记录错误。
