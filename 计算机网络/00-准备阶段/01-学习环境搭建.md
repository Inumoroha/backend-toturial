# 01-学习环境搭建

## 学习目标

完成本课程需要的基础环境。网络学习非常依赖实验环境，你需要能运行 Go 程序、抓包、查看端口、启动容器、模拟服务之间的访问。

## 一、安装 Go

检查 Go 是否已安装：

```bash
go version
```

如果没有安装，请安装 Go 1.22 或更新版本。

验证 Go 编译运行：

```bash
mkdir network-lab
cd network-lab
go mod init network-lab
cat > main.go <<'EOF'
package main

import "fmt"

func main() {
    fmt.Println("hello network")
}
EOF
go run .
```

在 Windows PowerShell 中可以手动创建 `main.go`，不必照抄 `cat` 命令。重点是确认 `go run .` 能正常执行。

## 二、准备 Linux 环境

推荐使用 WSL2 Ubuntu。原因是很多网络命令和线上服务器环境接近 Linux。

常用命令：

```bash
uname -a
ip addr
ip route
ss -lntp
```

如果你使用 Windows，也要熟悉这些替代命令：

```powershell
ipconfig
route print
netstat -ano
Test-NetConnection example.com -Port 443
```

## 三、安装常用网络工具

Ubuntu / Debian：

```bash
sudo apt update
sudo apt install -y curl wget dnsutils iproute2 net-tools tcpdump traceroute lsof netcat-openbsd
```

验证：

```bash
curl --version
dig example.com
ss -lnt
tcpdump --version
```

## 四、安装 Wireshark

Wireshark 用来图形化查看网络报文。学习 TCP 三次握手、HTTP 明文请求、DNS 查询时非常直观。

使用建议：

- 初学阶段先抓 `ping`、DNS、HTTP。
- HTTPS 抓包时先观察 TLS 握手，不急着解密内容。
- 抓包过滤器先从简单条件开始，例如 `tcp.port == 8080`、`dns`、`icmp`。

## 五、安装 Docker

Docker 用来模拟多服务、多容器、多网络环境。

验证：

```bash
docker version
docker run --rm hello-world
```

后续会用 Docker 模拟：

- 两个容器互相通信。
- Nginx 代理 Go 服务。
- 多个 Go 服务实例做负载均衡。
- 容器端口映射问题。

## 六、准备课程目录

建议你在课程根目录下新建一个实验目录：

```bash
mkdir labs
```

以后每个实验都放到独立子目录，例如：

```text
labs/
  tcp-echo/
  udp-echo/
  http-server/
  reverse-proxy/
  grpc-user-service/
```

## 七、验收标准

完成本篇后，你应该能：

- 运行 `go version`。
- 编译并运行一个最简单的 Go 程序。
- 在 Linux 或 WSL2 中执行 `ip addr`、`ip route`、`ss -lnt`。
- 使用 `curl` 访问网站。
- 使用 `dig` 查询 DNS。
- 使用 Docker 运行 `hello-world`。
- 打开 Wireshark 并知道如何选择网卡抓包。
