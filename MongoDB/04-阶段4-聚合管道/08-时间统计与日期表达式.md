# 08. 时间统计与日期表达式

后台报表最常见的需求就是按时间统计：按日订单金额、按月销售额、最近 30 天活跃用户等。本节学习时间范围过滤和日期分组。

## 1. 时间范围过滤

统计 2026-07-01 到 2026-07-05 之间的已支付订单：

```javascript
db.orders.aggregate([
  {
    $match: {
      status: "paid",
      paid_at: {
        $gte: new Date("2026-07-01T00:00:00Z"),
        $lt: new Date("2026-07-05T00:00:00Z")
      }
    }
  }
])
```

推荐使用左闭右开：

```text
[start, end)
```

也就是 `>= start` 且 `< end`，避免边界重复。

## 2. 使用 $dateToString 按天统计

```javascript
db.orders.aggregate([
  {
    $match: {
      status: "paid",
      paid_at: {
        $gte: new Date("2026-07-01T00:00:00Z"),
        $lt: new Date("2026-07-05T00:00:00Z")
      }
    }
  },
  {
    $group: {
      _id: {
        $dateToString: {
          format: "%Y-%m-%d",
          date: "$paid_at",
          timezone: "Asia/Shanghai"
        }
      },
      order_count: { $sum: 1 },
      total_amount: { $sum: "$total_amount" }
    }
  },
  { $sort: { _id: 1 } }
])
```

输出：

```javascript
{ _id: "2026-07-01", order_count: 2, total_amount: 35700 }
```

## 3. 使用 $dateTrunc 按天统计

较新的 MongoDB 版本支持 `$dateTrunc`，可以把日期截断到指定单位。

```javascript
db.orders.aggregate([
  { $match: { status: "paid" } },
  {
    $group: {
      _id: {
        $dateTrunc: {
          date: "$paid_at",
          unit: "day",
          timezone: "Asia/Shanghai"
        }
      },
      order_count: { $sum: 1 },
      total_amount: { $sum: "$total_amount" }
    }
  },
  { $sort: { _id: 1 } }
])
```

`$dateToString` 输出字符串，`$dateTrunc` 输出日期。报表展示常用字符串，后续继续处理常用日期。

## 4. 按月统计

```javascript
db.orders.aggregate([
  { $match: { status: "paid" } },

      total_amount: { $sum: "$total_amount" }
    }
  },

```

## 5. 最近 30 天活跃用户

假设当前分析时间是 2026-07-05，为了练习固定范围：

```javascript
db.events.aggregate([
  {
    $match: {
      created_at: {
        $gte: new Date("2026-06-05T00:00:00Z"),
        $lt: new Date("2026-07-06T00:00:00Z")
      }
    }
  },
  {
    $group: {
      _id: "$user_id",
      event_count: { $sum: 1 },
      active_days: {
        $addToSet: {
          $dateToString: {
            format: "%Y-%m-%d",
            date: "$created_at",
            timezone: "Asia/Shanghai"
          }
        }
      }
    }
  },
  {
    $project: {
      user_id: "$_id",
      _id: 0,
      event_count: 1,
      active_day_count: { $size: "$active_days" }
    }
  },
  { $sort: { active_day_count: -1, event_count: -1 } }
])
```

这里用 `$addToSet` 对活跃日期去重，再用 `$size` 计算活跃天数。

## 6. 按天统计事件数

```javascript
db.events.aggregate([
  {
    $group: {
      _id: {
        day: {
          $dateToString: {
            format: "%Y-%m-%d",
            date: "$created_at",
            timezone: "Asia/Shanghai"
          }
        },
        type: "$type"
      },
      count: { $sum: 1 }
    }
  },
  { $sort: { "_id.day": 1, "_id.type": 1 } }
])
```

输出按日期和事件类型分组。

## 7. 日期字段缺失问题

不是所有订单都有 `paid_at`。

如果直接按 `paid_at` 分组，未支付订单可能出现 null 或报错风险。

建议先过滤：

```javascript
{
  $match: {
    status: "paid",
    paid_at: { $exists: true }
  }
}
```

## 8. 时区问题

数据库中建议存 UTC 时间。

统计展示时指定时区：

```javascript
timezone: "Asia/Shanghai"
```

否则按 UTC 截断日期，可能和中国业务日期不一致。

例如北京时间 2026-07-02 00:30，对应 UTC 还是 2026-07-01 16:30。按 UTC 分组会算到前一天。

## 9. 补零问题

聚合只会返回有数据的日期。

如果某一天没有订单，结果里不会自动出现：

```javascript
{ day: "2026-07-03", total_amount: 0 }
```

常见做法：

- 应用层补零。
- 报表服务生成日期序列后合并。
- 使用更复杂的聚合生成时间序列。

学习阶段推荐应用层补零。

## 11. 本节小结

你需要记住：

- 时间范围推荐左闭右开。
- 按天统计可以用 `$dateToString` 或 `$dateTrunc`。
- 报表通常要指定业务时区。
- 未支付订单没有 `paid_at`，要先过滤。
- 聚合结果不会自动补零，通常由应用层处理。
