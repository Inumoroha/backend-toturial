## 3. 常用 stage 速查

### $match

```javascript
{ $match: { status: "paid" } }
```

### $project

```javascript
{
  $project: {
    _id: 0,
    order_no: 1,
    amount: "$total_amount"
  }
}
```

### $addFields / $set

```javascript
{
  $addFields: {
    total_yuan: { $divide: ["$total_amount", 100] }
  }
}
```

### $group

```javascript
{
  $group: {
    _id: "$user_id",
    count: { $sum: 1 },
    total: { $sum: "$total_amount" }
  }
}
```

### $sort

```javascript
{ $sort: { total: -1 } }
```

### $limit

```javascript
{ $limit: 10 }
```

### $skip

```javascript
{ $skip: 20 }
```

### $unwind

```javascript
{ $unwind: "$items" }
```

完整形式：

```javascript
{
  $unwind: {
    path: "$items",
    preserveNullAndEmptyArrays: true,
    includeArrayIndex: "item_index"
  }
}
```

### $lookup

```javascript
{
  $lookup: {
    from: "users",
    localField: "user_id",
    foreignField: "_id",
    as: "user"
  }
}
```

## 4. 常用表达式速查

### 数学计算

```javascript
{ $add: ["$a", "$b"] }
{ $subtract: ["$a", "$b"] }
{ $multiply: ["$price", "$quantity"] }
{ $divide: ["$total_amount", 100] }
```

### 条件

```javascript
{
  $cond: [
    { $gte: ["$total_amount", 20000] },
    "high",
    "normal"
  ]
}
```

### 空值

```javascript
{ $ifNull: ["$paid_at", "not paid"] }
```

### 数组大小

```javascript
{ $size: "$items" }
```

安全写法：

```javascript
{ $size: { $ifNull: ["$items", []] } }
```

### 日期格式化

```javascript
{
  $dateToString: {
    format: "%Y-%m-%d",
    date: "$paid_at",
    timezone: "Asia/Shanghai"
  }
}
```

## 5. Go Driver 速查

```go
pipeline := mongo.Pipeline{
    bson.D{{"$match", bson.D{{"status", "paid"}}}},
    bson.D{{"$group", bson.D{
        {"_id", "$user_id"},
        {"count", bson.D{{"$sum", 1}}},
    }}},
}

cursor, err := collection.Aggregate(ctx, pipeline)
if err != nil {
    return err
}
defer cursor.Close(ctx)

var results []Result
if err := cursor.All(ctx, &results); err != nil {
    return err
}
```

## 6. 聚合评审清单

写 pipeline 后检查：

- 是否先 `$match` 缩小数据量？
- `$match` 字段是否有索引？
- 是否不必要地 `$lookup` 大集合？
- `$unwind` 后数据量会膨胀多少？
- `$group` 是否会产生巨大内存压力？
- 输出字段是否只包含接口需要的数据？
- 时间统计是否指定 timezone？
- 金额是否使用整数分？
- 排序是否可能利用索引？
- Go 代码是否关闭 Cursor？

## 7. 进入第 5 阶段的标准

满足下面条件，可以进入第 5 阶段：

- 能独立写基础 pipeline。
- 能使用 `$match`、`$project`、`$group`、`$sort`、`$limit`。
- 能用 `$unwind` 做商品销量统计。
- 能用 `$lookup` 做集合关联，并知道它的边界。
- 能按天、按月统计时间数据。
- 能在 Go 中构造 `mongo.Pipeline`。
- 能正确处理 `Aggregate` 返回的 Cursor。
- 能说出一个 pipeline 的潜在性能风险。

第 5 阶段会进入索引与性能优化，重点学习为什么某些聚合慢、如何用 explain 看执行计划、如何设计索引支持查询和排序。
