# 10. 阶段练习：订单与内容统计

本练习把第 4 阶段的内容串起来。你要完成一组真实后端常见的统计需求。

## 1. 练习目标

完成下面统计：

- 订单金额按天统计。
- 用户消费排行榜。
- 商品销量排行榜。
- 文章阅读量和点赞排行。
- 每篇文章可见评论数。
- 用户最近 30 天活跃情况。
- 订单状态分布。

## 2. 订单金额按天统计

```javascript
db.orders.aggregate([
  {
    $match: {
      status: "paid",
      paid_at: {
        $gte: new Date("2026-07-01T00:00:00Z"),
        $lt: new Date("2026-07-05T00:00:00Z")
      }
    }
  },
  {
    $group: {
      _id: {
        $dateToString: {
          format: "%Y-%m-%d",
          date: "$paid_at",
          timezone: "Asia/Shanghai"
        }
      },
      order_count: { $sum: 1 },
      total_amount: { $sum: "$total_amount" }
    }
  },
  {
    $project: {
      _id: 0,
      day: "$_id",
      order_count: 1,
      total_amount: 1,
      total_yuan: { $divide: ["$total_amount", 100] }
    }
  },
  { $sort: { day: 1 } }
])
```

## 3. 用户消费排行榜

```javascript
db.orders.aggregate([
  { $match: { status: "paid" } },
  {
    $group: {
      _id: "$user_id",
      order_count: { $sum: 1 },
      total_amount: { $sum: "$total_amount" }
    }
  },
  { $sort: { total_amount: -1 } },
  { $limit: 10 },
  {
    $lookup: {
      from: "users",
      localField: "_id",
      foreignField: "_id",
      as: "user"
    }
  },
  { $unwind: "$user" },
  {
    $project: {
      _id: 0,
      user_id: "$_id",
      user_name: "$user.name",
      city: "$user.city",
      order_count: 1,
      total_amount: 1
    }
  }
])
```

## 4. 商品销量排行榜

```javascript
db.orders.aggregate([
  { $match: { status: "paid" } },
  { $unwind: "$items" },
  {
    $group: {
      _id: "$items.product_id",
      product_name: { $first: "$items.product_name" },
      sales_quantity: { $sum: "$items.quantity" },
      sales_amount: {
        $sum: { $multiply: ["$items.price", "$items.quantity"] }
      }
    }
  },
  { $sort: { sales_quantity: -1, sales_amount: -1 } }
])
```

## 5. 文章阅读量和点赞排行

```javascript
db.posts.aggregate([
  { $match: { status: "published" } },
  {
    $project: {
      _id: 0,
      title: 1,
      views: "$stats.views",
      likes: "$stats.likes",
      score: {
        $add: [
          "$stats.views",
          { $multiply: ["$stats.likes", 5] }
        ]
      }
    }
  },
  { $sort: { score: -1 } }
])
```

这里的 `score` 是示例公式：浏览量 + 点赞数 * 5。

真实项目中排序公式需要产品和业务共同定义。

## 6. 每篇文章可见评论数

从 comments 集合统计，再关联 posts：

```javascript
db.comments.aggregate([
  { $match: { status: "visible" } },
  {
    $group: {
      _id: "$post_id",
      visible_comment_count: { $sum: 1 }
    }
  },
  {
    $lookup: {
      from: "posts",
      localField: "_id",
      foreignField: "_id",
      as: "post"
    }
  },
  { $unwind: "$post" },
  {
    $project: {
      _id: 0,
      post_id: "$_id",
      title: "$post.title",
      visible_comment_count: 1
    }
  },
  { $sort: { visible_comment_count: -1 } }
])
```

为什么不从 posts `$lookup` 所有 comments 再 `$size`？

因为评论很多时，把所有评论拉进文章文档会很重。从 comments 先过滤和分组更合理。

## 7. 用户最近 30 天活跃情况

```javascript
db.events.aggregate([
  {
    $match: {
      created_at: {
        $gte: new Date("2026-06-05T00:00:00Z"),
        $lt: new Date("2026-07-06T00:00:00Z")
      }
    }
  },
  {
    $group: {
      _id: "$user_id",
      event_count: { $sum: 1 },
      active_days: {
        $addToSet: {
          $dateToString: {
            format: "%Y-%m-%d",
            date: "$created_at",
            timezone: "Asia/Shanghai"
          }
        }
      },
      event_types: { $addToSet: "$type" }
    }
  },
  {
    $lookup: {
      from: "users",
      localField: "_id",
      foreignField: "_id",
      as: "user"
    }
  },
  { $unwind: "$user" },
  {
    $project: {
      _id: 0,
      user_id: "$_id",
      user_name: "$user.name",
      event_count: 1,
      active_day_count: { $size: "$active_days" },
      event_types: 1
    }
  },
  { $sort: { active_day_count: -1, event_count: -1 } }
])
```

## 8. 订单状态分布

```javascript
db.orders.aggregate([
  {
    $group: {
      _id: "$status",
      count: { $sum: 1 },
      amount: { $sum: "$total_amount" }
    }
  },
  {
    $project: {
      _id: 0,
      status: "$_id",
      count: 1,
      amount: 1
    }
  },
  { $sort: { count: -1 } }
])
```

注意：这里把 pending 和 cancelled 的金额也算进 amount，只是状态分布示例。如果是收入报表，应该只统计 paid。
