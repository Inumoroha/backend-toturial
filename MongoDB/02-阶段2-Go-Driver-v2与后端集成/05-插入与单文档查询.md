# 05. 插入与单文档查询

本节实现 `Create`、`FindByID`、`FindByEmail`。这三个方法是 Repository 的基本骨架。

## 1. Create 方法

在 `internal/user/repository.go` 中增加：

```go
package user

import (
    "context"
    "time"

    "go.mongodb.org/mongo-driver/v2/bson"
    "go.mongodb.org/mongo-driver/v2/mongo"
)

type Repository struct {
    collection *mongo.Collection
}

func NewRepository(db *mongo.Database) *Repository {
    return &Repository{
        collection: db.Collection("users"),
    }
}

func (r *Repository) Create(ctx context.Context, u *User) error {
    now := time.Now().UTC()

    if u.ID.IsZero() {
        u.ID = bson.NewObjectID()
    }
    if u.CreatedAt.IsZero() {
        u.CreatedAt = now
    }
    u.UpdatedAt = now
    if u.Status == "" {
        u.Status = "active"
    }

    _, err := r.collection.InsertOne(ctx, u)
    if err != nil {
        return err
    }

    return nil
}
```

要点：

- 插入前补齐 `_id`。
- 插入前补齐 `created_at`、`updated_at`。
- Repository 可以设置数据库层默认值，但复杂业务规则应放 Service。
- 先直接返回错误，后面错误处理章节再转换。

## 2. 为什么 Create 接收指针

```go
func (r *Repository) Create(ctx context.Context, u *User) error
```

因为方法内部会补齐：

```go
u.ID = bson.NewObjectID()
u.CreatedAt = now
u.UpdatedAt = now
```

如果传值，外面的调用者拿不到补齐后的 ID 和时间。

## 3. FindByEmail

```go
func (r *Repository) FindByEmail(ctx context.Context, email string) (*User, error) {
    filter := bson.D{
        {"email", email},
        {"deleted_at", bson.D{{"$exists", false}}},
    }

    var u User
    err := r.collection.FindOne(ctx, filter).Decode(&u)
    if err != nil {
        return nil, err
    }

    return &u, nil
}
```

要点：

- `FindOne` 返回 `*mongo.SingleResult`。
- 真正错误通常在 `Decode` 时返回。
- 如果没找到，`Decode` 返回 `mongo.ErrNoDocuments`。
- 加上 `deleted_at` 条件，避免查到软删除用户。

## 4. FindByID

```go
func (r *Repository) FindByID(ctx context.Context, id bson.ObjectID) (*User, error) {
    filter := bson.D{
        {"_id", id},
        {"deleted_at", bson.D{{"$exists", false}}},
    }

    var u User
    err := r.collection.FindOne(ctx, filter).Decode(&u)
    if err != nil {
        return nil, err
    }

    return &u, nil
}
```

注意：这里接收的是 `bson.ObjectID`，不是字符串。

字符串转换可以在上层做：

```go
id, err := bson.ObjectIDFromHex(idString)
if err != nil {
    return err
}
```

## 5. 在 main 中测试

修改 `cmd/api/main.go`：

```go
package main

import (
    "context"
    "fmt"
    "log"
    "time"

    "mongodb-go-stage2/internal/config"
    "mongodb-go-stage2/internal/database"
    "mongodb-go-stage2/internal/user"
)

func main() {
    cfg := config.Load()

    ctx, cancel := context.WithTimeout(context.Background(), cfg.MongoTimeout)
    defer cancel()

    client, err := database.Connect(ctx, cfg.MongoURI)
    if err != nil {
        log.Fatalf("connect mongodb: %v", err)
    }
    defer client.Disconnect(context.Background())

    db := client.Database(cfg.MongoDatabase)
    repo := user.NewRepository(db)

    u := &user.User{
        Email: "alice@example.com",
        Name:  "Alice",
        Age:   25,
        Roles: []string{"user"},
    }

    if err := repo.Create(ctx, u); err != nil {
        log.Fatalf("create user: %v", err)
    }

    fmt.Println("created user id:", u.ID.Hex())

    found, err := repo.FindByEmail(ctx, "alice@example.com")
    if err != nil {
        log.Fatalf("find user: %v", err)
    }

    fmt.Printf("found user: %+v\n", found)
}
```

运行：

```powershell
go run .\cmd\api
```

## 6. 重复执行的问题

如果你多次运行上面的程序，会插入多个相同邮箱用户。真实业务中邮箱应该唯一。

先在 `mongosh` 中创建唯一索引：

```javascript
use go_mongodb_stage2
db.users.createIndex({ email: 1 }, { unique: true })
```

如果集合里已经有重复邮箱，创建唯一索引会失败。学习阶段可以清空集合：

```javascript
db.users.drop()
db.users.createIndex({ email: 1 }, { unique: true })
```

后面错误处理章节会把重复键错误转换成 `ErrEmailExists`。

## 7. InsertOne 返回值

如果不手动设置 ID，也可以从返回值拿：

```go
result, err := r.collection.InsertOne(ctx, u)
if err != nil {
    return err
}

id, ok := result.InsertedID.(bson.ObjectID)
if ok {
    u.ID = id
}
```

不过学习阶段推荐插入前自己设置 `u.ID = bson.NewObjectID()`，代码更直接，也方便后续业务使用。

## 8. projection 查询

单文档查询也可以设置投影：

```go
opts := options.FindOne().SetProjection(bson.D{
    {"password_hash", 0},
})

err := r.collection.FindOne(ctx, filter, opts).Decode(&u)
```

如果使用了这个代码，需要导入：

```go
"go.mongodb.org/mongo-driver/v2/mongo/options"
```

本阶段用户模型暂时没有密码字段，所以先了解即可。

## 9. 本节常见错误

### 忘记 Decode

错误：

```go
result := r.collection.FindOne(ctx, filter)
return result, nil
```

`FindOne` 返回的是 `SingleResult`，你需要 `Decode` 到结构体。

### Decode 传值

错误：

```go
var u User
err := r.collection.FindOne(ctx, filter).Decode(u)
```

正确：

```go
err := r.collection.FindOne(ctx, filter).Decode(&u)
```

### 查软删除数据

如果业务采用软删除，查询时要加：

```go
{"deleted_at", bson.D{{"$exists", false}}}
```

## 10. 本节验收

你应该能做到：

- 用 `InsertOne` 创建用户。
- 插入前设置 `bson.NewObjectID()`。
- 用 `FindOne().Decode(&u)` 查询单个用户。
- 用 `bson.ObjectIDFromHex` 解析字符串 ID。
- 解释 `mongo.ErrNoDocuments` 在什么时候出现。
- 解释为什么邮箱需要唯一索引。
