# 01. Go 项目初始化与驱动安装

本节创建一个干净的 Go 项目，并安装 MongoDB 官方 Go Driver v2。

## 1. 创建项目目录

建议把练习代码放在教程目录外，避免 Markdown 教程和 Go 项目混在一起。

示例：

```powershell
mkdir C:\Users\HP\Desktop\mongodb-go-stage2
cd C:\Users\HP\Desktop\mongodb-go-stage2
```

初始化 Go module：

```powershell
go mod init mongodb-go-stage2
```

此时会生成：

```text
go.mod
```

## 2. 安装 Go Driver v2

执行：

```powershell
go get go.mongodb.org/mongo-driver/v2/mongo
go mod tidy
```

安装完成后查看：

```powershell
go list -m all | Select-String "mongo-driver"
```

你应该看到类似：

```text
go.mongodb.org/mongo-driver/v2 v2.x.x
```

版本号不需要和教程完全一致，以当前官方稳定版本为准。

## 3. 为什么导入路径带 /v2

Go module 的主版本号规则要求：当模块进入 v2 或更高版本时，导入路径通常要带主版本号。

所以 Go Driver v2 的导入路径是：

```go
go.mongodb.org/mongo-driver/v2/mongo
```

而不是：

```go
go.mongodb.org/mongo-driver/mongo
```

旧教程经常使用 v1 路径。你现在学习的是 v2，请统一使用 `/v2`。

## 4. 创建最小 main.go

创建目录：

```powershell
mkdir cmd
mkdir cmd\api
```

创建 `cmd/api/main.go`：

```go
package main

import "fmt"

func main() {
    fmt.Println("mongodb go stage2")
}
```

运行：

```powershell
go run .\cmd\api
```

如果看到：

```text
mongodb go stage2
```

说明项目初始化正常。

## 5. 建议目录结构

先创建目录：

```powershell
mkdir internal
mkdir internal\config
mkdir internal\database
mkdir internal\user
```

最终结构：

```text
mongodb-go-stage2/
  go.mod
  go.sum
  cmd/
    api/
      main.go
  internal/
    config/
    config.go
    database/
      mongo.go
    user/
      model.go
      repository.go
      errors.go
      repository_test.go
```

学习阶段先不接 HTTP，也不引入复杂配置库。后面项目做大了，再引入 Viper、envconfig、fx/wire 等工具。

## 6. 配置环境变量

在 PowerShell 当前会话设置：

```powershell
$env:MONGO_URI="mongodb://root:example@localhost:27017/?authSource=admin"
$env:MONGO_DATABASE="go_mongodb_stage2"
```

查看：

```powershell
echo $env:MONGO_URI
echo $env:MONGO_DATABASE
```

注意：这种方式只对当前 PowerShell 会话有效。关闭窗口后会消失。

如果想设置用户环境变量：

```powershell
[Environment]::SetEnvironmentVariable("MONGO_URI", "mongodb://root:example@localhost:27017/?authSource=admin", "User")
[Environment]::SetEnvironmentVariable("MONGO_DATABASE", "go_mongodb_stage2", "User")
```

设置后重新打开 PowerShell。

## 7. 读取配置

创建 `internal/config/config.go`：

```go
package config

import (
    "os"
    "time"
)

type Config struct {
    MongoURI      string
    MongoDatabase string
    MongoTimeout  time.Duration
}

func Load() Config {
    return Config{
        MongoURI:      getEnv("MONGO_URI", "mongodb://root:example@localhost:27017/?authSource=admin"),
        MongoDatabase: getEnv("MONGO_DATABASE", "go_mongodb_stage2"),
        MongoTimeout:  5 * time.Second,
    }
}

func getEnv(key, fallback string) string {
    value := os.Getenv(key)
    if value == "" {
        return fallback
    }
    return value
}
```

这段代码的目的：

- 不把 MongoDB URI 散落在业务代码里。
- 本地没有环境变量时使用默认值。
- 后面测试和部署时可以覆盖配置。

## 8. 本节常见问题

### go get 拉不下来

如果网络慢，先检查代理：

```powershell
go env GOPROXY
```

国内网络可以设置：

```powershell
go env -w GOPROXY=https://goproxy.cn,direct
```

### 项目里出现 v1 和 v2 两套依赖

查看：

```powershell
go list -m all | Select-String "mongo-driver"
```

如果同时看到：

```text
go.mongodb.org/mongo-driver
go.mongodb.org/mongo-driver/v2
```

说明你可能混用了旧导入路径。

检查代码：

```powershell
Select-String -Path .\**\*.go -Pattern "go.mongodb.org/mongo-driver/" -Recurse
```

统一改成 v2 路径。

### GOROOT 异常

如果 `go version` 都不能正常运行，先回到第 0 阶段修 Go 环境。MongoDB Driver 问题不要和 Go 环境问题混在一起排。
