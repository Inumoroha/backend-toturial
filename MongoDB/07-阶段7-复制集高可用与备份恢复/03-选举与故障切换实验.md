# 03. 选举与故障切换实验

本节通过停止 primary 观察复制集重新选主。这个实验能让你直观看到 MongoDB 高可用的基本过程。

## 1. 查看当前 primary

连接复制集：

```powershell
mongosh "mongodb://localhost:27017,localhost:27018,localhost:27019/?replicaSet=rs0"
```

查看：

```javascript
rs.status().members.map(m => ({
  name: m.name,
  stateStr: m.stateStr
}))
```

记录当前 PRIMARY 是哪个容器。

也可以在 Docker 中看容器名：

```powershell
docker ps
```

## 2. 停掉 primary

假设 primary 是 `mongo-rs1`：

```powershell
docker stop mongo-rs1
```

等待几秒到几十秒。

重新连接复制集：

```powershell
mongosh "mongodb://localhost:27018,localhost:27019/?replicaSet=rs0"
```

查看：

```javascript
rs.status().members.map(m => ({
  name: m.name,
  stateStr: m.stateStr
}))
```

应该看到新的 PRIMARY。

## 3. 写入测试

在新 primary 上：

```javascript
use rs_stage7
db.failover_test.insertOne({
  event: "after primary stopped",
  created_at: new Date()
})
```

如果写入成功，说明复制集仍然可写。

## 4. 恢复旧 primary

```powershell
docker start mongo-rs1
```

等待它重新加入。

查看状态：

```javascript
rs.status().members.map(m => ({
  name: m.name,
  stateStr: m.stateStr
}))
```

旧 primary 恢复后通常会作为 SECONDARY 加入，不一定重新成为 PRIMARY。

## 5. 多数派理解

三节点复制集：

```text
多数 = 2
```

停掉 1 个节点，还剩 2 个，可以选主。

如果再停掉一个节点，只剩 1 个，通常无法维持 primary。

实验：

```powershell
docker stop mongo-rs2
```

再观察是否还能写入。

做完后恢复：

```powershell
docker start mongo-rs1
docker start mongo-rs2
docker start mongo-rs3
```

## 6. Driver 如何处理故障切换

Go Driver 使用复制集连接串：

```text
mongodb://localhost:27017,localhost:27018,localhost:27019/?replicaSet=rs0
```

Driver 会监控拓扑变化。

primary 切换时，短时间内操作可能失败或超时。应用应该：

- 设置合理 timeout。
- 对可重试操作做重试。
- 对非幂等操作使用幂等键。
- 不要假设 primary 永远不变。

## 7. 故障切换中的写入风险

如果写关注是 `w:1`：

```text
primary 确认写入后返回成功。
```

但如果写入尚未复制到多数节点，primary 就故障，极端情况下可能发生回滚。

重要写入建议考虑：

```text
writeConcern: majority
```

这会在后面章节展开。

## 9. 本节小结

你需要记住：

- primary 故障后，secondary 可以选举为新 primary。
- 三节点复制集可以容忍一个节点故障。
- 多数派是选举和 majority 写入的基础。
- 故障切换期间应用要能处理短暂错误。
- 重要写入应考虑 majority。
