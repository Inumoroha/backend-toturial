# 10. Go 中配置读写关注与监听变更

本节学习 Go Driver v2 中配置 readPreference、writeConcern，以及使用 Change Streams 监听订单变化。

## 1. 复制集连接串

```text
mongodb://localhost:27017,localhost:27018,localhost:27019/?replicaSet=rs0
```

Go 配置：

```go
clientOpts := options.Client().ApplyURI(uri)
client, err := mongo.Connect(clientOpts)
```

## 2. 设置 writeConcern majority

Client 级别：

```go
clientOpts := options.Client().
    ApplyURI(uri).
    SetWriteConcern(writeconcern.Majority())
```

Collection 级别：

```go
orders := db.Collection(
    "orders",
    options.Collection().SetWriteConcern(writeconcern.Majority()),
)
```

## 3. 设置 readPreference

```go
clientOpts := options.Client().
    ApplyURI(uri).
    SetReadPreference(readpref.Primary())
```

secondaryPreferred：

```go
clientOpts := options.Client().
    ApplyURI(uri).
    SetReadPreference(readpref.SecondaryPreferred())
```

强一致业务建议 primary。

## 4. 设置 readConcern

Collection 级别：

```go
orders := db.Collection(
    "orders",
    options.Collection().SetReadConcern(readconcern.Majority()),
)
```

事务级别：

```go
txnOpts := options.Transaction().
    SetReadConcern(readconcern.Snapshot()).
    SetWriteConcern(writeconcern.Majority())
```

## 5. 监听 Change Streams

```go
func WatchOrderChanges(ctx context.Context, orders *mongo.Collection) error {
    pipeline := mongo.Pipeline{
        bson.D{{"$match", bson.D{
            {"operationType", bson.D{{"$in", bson.A{"insert", "update", "replace"}}}},
        }}},
    }

    stream, err := orders.Watch(ctx, pipeline, options.ChangeStream().SetFullDocument(options.UpdateLookup))
    if err != nil {
        return err
    }
    defer stream.Close(ctx)

    for stream.Next(ctx) {
        var event bson.M
        if err := stream.Decode(&event); err != nil {
            return err
        }

        // 学习阶段先打印；生产中应解析事件并幂等处理。
        fmt.Printf("order changed: %+v\n", event)
    }

    if err := stream.Err(); err != nil {
        return err
    }

    return nil
}
```

需要导入：

```go
import (
    "context"
    "fmt"

    "go.mongodb.org/mongo-driver/v2/bson"
    "go.mongodb.org/mongo-driver/v2/mongo"
    "go.mongodb.org/mongo-driver/v2/mongo/options"
)
```

以及：

```go
"go.mongodb.org/mongo-driver/v2/mongo/readconcern"
"go.mongodb.org/mongo-driver/v2/mongo/readpref"
"go.mongodb.org/mongo-driver/v2/mongo/writeconcern"
```

## 6. 保存 resume token

每个事件都有 resume token。

Go 中可以：

```go
token := stream.ResumeToken()
```

生产中要把 token 保存到可靠存储。

服务重启后使用：

```go
options.ChangeStream().SetResumeAfter(token)
```

学习阶段先知道有这个机制。

## 7. 事件处理要幂等

Change Streams 处理可能重复。

例如订单 paid 事件：

```text
发送通知
更新搜索索引
清理缓存
```

都应该能重复处理而不造成错误。

可以记录：

```text
event_id / resume_token / order_id + status
```

防止重复副作用。

## 9. 本节小结

你需要记住：

- Go Driver 可以配置 readPreference、readConcern、writeConcern。
- Change Streams 用 `collection.Watch`。
- Change Stream 返回 Cursor 风格的 stream。
- 要 Close、Next、Decode、Err。
- 生产中要保存 resume token，并保证事件幂等。
