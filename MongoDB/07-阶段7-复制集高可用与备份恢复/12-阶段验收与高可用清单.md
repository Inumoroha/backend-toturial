## 2. 必须会的命令

### 查看复制集状态

```javascript
rs.status()
```

### 查看当前节点信息

```javascript
db.hello()
```

### majority 写入

```javascript
db.orders.insertOne(
  { order_no: "O1", created_at: new Date() },
  { writeConcern: { w: "majority", wtimeout: 5000 } }
)
```

### 备份

```powershell
mongodump --uri="mongodb://localhost:27017,localhost:27018,localhost:27019/?replicaSet=rs0" --db=stage7_practice --out="C:\backup"
```

### 恢复

```powershell
mongorestore --uri="mongodb://localhost:27017,localhost:27018,localhost:27019/?replicaSet=rs0" --db=stage7_practice "C:\backup\stage7_practice"
```

### Change Streams

```javascript
const stream = db.orders.watch([], { fullDocument: "updateLookup" })
while (stream.hasNext()) {
  printjson(stream.next())
}
```

## 3. 高可用评审清单

生产 MongoDB 方案至少要问：

- 是否使用复制集？
- 有几个数据节点？
- 是否跨机器或可用区？
- majority 写入是否用于关键数据？
- 应用连接串是否包含多个节点？
- Driver 是否设置合理 timeout？
- 是否处理故障切换中的短暂错误？
- 是否有备份策略？
- 是否做过恢复演练？
- 是否定义 RPO/RTO？
- 是否监控复制延迟？
- 是否监控磁盘和 oplog 窗口？
- 是否限制 secondary 读的业务场景？
- 是否有 Change Streams resume token 机制？

## 4. 常见错误

### 只连一个节点

错误连接：

```text
mongodb://localhost:27017
```

复制集应用应使用多个节点和 `replicaSet`：

```text
mongodb://host1:27017,host2:27017,host3:27017/?replicaSet=rs0
```

### 重要写入只用 w:1

可能在极端故障下有回滚风险。

### secondary 读用于强一致业务

可能读到旧数据。

### 只备份不恢复演练

没有恢复演练的备份是不完整的。

### Change Streams 不保存 resume token

服务重启后可能丢事件或重复处理。
