本阶段使用 `performance_stage5` 数据库。为了让 `explain` 对比更明显，本节会插入一批模拟数据。

```powershell

```

## 3. 插入订单

```javascript
const userIds = db.users.find({}, { _id: 1 }).toArray().map(u => u._id)
const statuses = ["pending_payment", "paid", "cancelled", "shipped", "completed"]
const orders = []

for (let i = 1; i <= 20000; i++) {
  const userId = userIds[i % userIds.length]
  const status = statuses[i % statuses.length]
  const createdAt = new Date(2026, 6, (i % 28) + 1, i % 24, i % 60, 0)

  orders.push({
    order_no: `O202607${String(i).padStart(6, "0")}`,
    user_id: userId,
    status,
    total_amount: ((i % 50) + 1) * 1000,
    items: [
      {
        product_id: ObjectId("661000000000000000000001"),
        product_name: "MongoDB 入门课",
        price: 9900,
        quantity: (i % 3) + 1
      }
    ],
    created_at: createdAt,
    paid_at: status === "paid" || status === "shipped" || status === "completed" ? createdAt : null
  })

  if (orders.length === 1000) {
    db.orders.insertMany(orders)
    orders.length = 0
  }
}

if (orders.length > 0) {
  db.orders.insertMany(orders)
}
```

## 7. 清理索引

为了练习从无索引到有索引的对比，先只保留默认 `_id` 索引。

查看索引：

```javascript
db.users.getIndexes()
db.orders.getIndexes()
db.verify_tokens.getIndexes()
db.posts.getIndexes()
```

如果你重复练习，需要删除非 `_id` 索引：

```javascript
db.users.dropIndexes()
db.orders.dropIndexes()
db.verify_tokens.dropIndexes()
db.posts.dropIndexes()
```

注意：`dropIndexes()` 不会删除 `_id` 默认索引。

## 8. 本节提醒

这批数据不算大，但足够让你观察：

- 没有索引时 `COLLSCAN`。
- 有索引时 `IXSCAN`。
- `totalDocsExamined` 的变化。
- 深分页 skip 的问题。
- 复合索引字段顺序的影响。

真实生产数据量会大得多，所以学习阶段要先建立分析习惯。
