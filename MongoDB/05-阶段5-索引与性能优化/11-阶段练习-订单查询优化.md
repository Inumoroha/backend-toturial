# 11. 阶段练习：订单查询优化

本练习把第 5 阶段串起来。你要对一组订单相关查询做索引设计、explain 对比和优化记录。

## 1. 练习目标

完成：

- 用户订单列表优化。
- 用户按状态筛选订单优化。
- 后台按状态查询订单优化。
- 订单号唯一查询优化。
- 验证码 TTL 索引。
- 深分页改游标分页。
- 批量订单状态更新。

## 2. 查询 1：订单号查询

业务：

```text
根据 order_no 查询订单详情。
```

查询：

```javascript
db.orders.findOne({ order_no: "O202607000100" })
```

索引：

```javascript
db.orders.createIndex({ order_no: 1 }, { unique: true, name: "uniq_orders_order_no" })
```

验收：

```javascript
db.orders.find({ order_no: "O202607000100" }).explain("executionStats")
```

检查是否使用 `IXSCAN`。

## 3. 查询 2：用户订单列表

查询：

```javascript
db.orders.find({ user_id: user1._id })
  .sort({ created_at: -1 })
  .limit(20)
```

索引：

```javascript
db.orders.createIndex(
  { user_id: 1, created_at: -1 },
  { name: "idx_orders_user_created_at" }
)
```

记录：

优化后 totalDocsExamined:

## 4. 查询 3：用户按状态筛选订单

查询：

```javascript
db.orders.find({
  user_id: user1._id,
  status: "paid"
}).sort({ created_at: -1 }).limit(20)
```

索引：

```javascript
db.orders.createIndex(
  { user_id: 1, status: 1, created_at: -1 },
  { name: "idx_orders_user_status_created_at" }
)
```

ESR：

```text
Equality: user_id, status
Sort: created_at
Range: none
```

## 5. 查询 4：后台 paid 订单列表

查询：

```javascript
db.orders.find({ status: "paid" })
  .sort({ created_at: -1 })
  .limit(50)
```

索引：

```javascript
db.orders.createIndex(
  { status: 1, created_at: -1 },
  { name: "idx_orders_status_created_at" }
)
```

注意：

这个索引不是为了用户订单列表，而是为了后台状态列表。

## 6. 查询 5：时间范围订单

查询：

```javascript
db.orders.find({
  status: "paid",
  created_at: {
    $gte: new Date("2026-07-01T00:00:00Z"),
    $lt: new Date("2026-07-08T00:00:00Z")
  }
}).sort({ created_at: -1 })
```

索引：

```javascript
db.orders.createIndex(
  { status: 1, created_at: -1 },
  { name: "idx_orders_status_created_at" }
)
```

这里 `status` 是等值，`created_at` 同时用于排序和范围，需要用 explain 验证实际效果。

## 7. 查询 6：验证码过期

索引：

```javascript
db.verify_tokens.createIndex(
  { expired_at: 1 },
  { expireAfterSeconds: 0, name: "ttl_verify_tokens_expired_at" }
)
```

验收：

```javascript
db.verify_tokens.getIndexes()
```

TTL 删除不会秒级立即发生，不要用它做精确定时任务。

## 8. 查询 7：文章文本搜索

索引：

```javascript
db.posts.createIndex(
  { title: "text", content: "text" },
  { name: "text_posts_title_content" }
)
```

查询：

```javascript
db.posts.find(
  { $text: { $search: "MongoDB 索引" } },
  { score: { $meta: "textScore" }, title: 1 }
).sort({ score: { $meta: "textScore" } })
```

说明：

文本索引适合基础搜索，复杂搜索应考虑专业搜索方案。

## 9. 深分页改游标分页

旧查询：

```javascript
db.orders.find({ user_id: user1._id })
  .sort({ created_at: -1 })
  .skip(10000)
  .limit(20)
```

新索引：

```javascript
db.orders.createIndex(
  { user_id: 1, created_at: -1, _id: -1 },
  { name: "idx_orders_user_created_id_cursor" }
)
```

第一页：

```javascript
db.orders.find({ user_id: user1._id })
  .sort({ created_at: -1, _id: -1 })
  .limit(20)
```

下一页：

```javascript
db.orders.find({
  user_id: user1._id,
  $or: [
    { created_at: { $lt: lastCreatedAt } },
    { created_at: lastCreatedAt, _id: { $lt: lastId } }
  ]
}).sort({ created_at: -1, _id: -1 }).limit(20)
```
