# 04. 查询文档

查询是 MongoDB 最常用的能力。本节学习 `find`、`findOne`、过滤条件、投影、排序、分页和常见查询操作符。

## 2. find 查询全部

```javascript
db.query_demo.find()
```

格式化：

```javascript
db.query_demo.find().pretty()
```

在新版 `mongosh` 中，很多输出已经比较易读，但 `.pretty()` 仍然适合学习阶段。

## 3. findOne 查询一条

```javascript
db.query_demo.findOne({ email: "alice@example.com" })
```

`findOne` 返回匹配到的第一条文档。

如果没有匹配，返回 `null`：

```javascript
db.query_demo.findOne({ email: "not-exists@example.com" })
```

## 4. 等值查询

简写：

```javascript
db.query_demo.find({ status: "active" })
```

等价于：

```javascript
db.query_demo.find({ status: { $eq: "active" } })
```

常用简写即可。

## 5. 多条件查询

```javascript
db.query_demo.find({
  status: "active",
  age: { $gt: 26 }
})
```

多个字段条件默认是 AND 关系。

等价于：

```javascript
db.query_demo.find({
  $and: [
    { status: "active" },
    { age: { $gt: 26 } }
  ]
})
```

通常优先使用隐式 AND，写起来更清爽。

## 6. 比较操作符

年龄大于 25：

```javascript
db.query_demo.find({ age: { $gt: 25 } })
```

年龄大于等于 25：

```javascript
db.query_demo.find({ age: { $gte: 25 } })
```

年龄小于 30：

```javascript
db.query_demo.find({ age: { $lt: 30 } })
```

年龄小于等于 30：

```javascript
db.query_demo.find({ age: { $lte: 30 } })
```

年龄不等于 30：

```javascript
db.query_demo.find({ age: { $ne: 30 } })
```

注意：`$ne` 会匹配字段不存在的文档。后面做复杂查询时要特别小心。

## 7. 范围查询

年龄在 20 到 30 之间：

```javascript
db.query_demo.find({
  age: { $gte: 20, $lte: 30 }
})
```

分数在 80 到 95 之间：

```javascript
db.query_demo.find({
  score: { $gte: 80, $lte: 95 }
})
```

时间范围查询：

```javascript
db.query_demo.find({
  created_at: {
    $gte: new Date("2026-07-02T00:00:00Z"),
    $lt: new Date("2026-07-05T00:00:00Z")
  }
})
```

时间查询建议使用左闭右开：

```text
[start, end)
```

也就是 `>= start` 且 `< end`，这样按天、按月统计时更不容易重叠。

## 8. $in 和 $nin

查询状态是 `active` 或 `blocked` 的用户：

```javascript
db.query_demo.find({
  status: { $in: ["active", "blocked"] }
})
```

查询状态不是 `inactive` 和 `blocked` 的用户：

```javascript
db.query_demo.find({
  status: { $nin: ["inactive", "blocked"] }
})
```

`$in` 很常用，例如按一批 ID 查询、按多个状态查询。

## 9. $exists

查询存在 `profile.company` 的文档：

```javascript
db.query_demo.find({
  "profile.company": { $exists: true }
})
```

查询不存在 `deleted_at` 的文档：

```javascript
db.query_demo.find({
  deleted_at: { $exists: false }
})
```

软删除场景中，经常会查询：

```javascript
db.query_demo.find({
  deleted_at: { $exists: false }
})
```

## 10. $or

查询状态是 `blocked` 或分数低于 70：

```javascript
db.query_demo.find({
  $or: [
    { status: "blocked" },
    { score: { $lt: 70 } }
  ]
})
```

`$or` 的每个条件最好都能被合适索引支持。后面索引阶段会展开。

## 11. 正则查询

查询邮箱以 `alice` 开头：

```javascript
db.query_demo.find({
  email: { $regex: "^alice" }
})
```

忽略大小写：

```javascript
db.query_demo.find({
  name: { $regex: "alice", $options: "i" }
})
```

正则查询要谨慎。没有前缀约束的模糊匹配通常很难高效使用普通索引。

后面如果需要复杂搜索，应考虑 MongoDB Atlas Search 或 Elasticsearch 等搜索方案。

## 12. 投影 Projection

只返回 `name`、`email`、`age`：

```javascript
db.query_demo.find(
  { status: "active" },
  { name: 1, email: 1, age: 1 }
)
```

默认会返回 `_id`。

不返回 `_id`：

```javascript
db.query_demo.find(
  { status: "active" },
  { name: 1, email: 1, age: 1, _id: 0 }
)
```

排除某个字段：

```javascript
db.query_demo.find(
  {},
  { profile: 0 }
)
```

一般不要在同一个投影里混用包含和排除，`_id` 是例外。

## 13. 排序 sort

按年龄升序：

```javascript
db.query_demo.find().sort({ age: 1 })
```

按年龄降序：

```javascript
db.query_demo.find().sort({ age: -1 })
```

按状态升序、分数降序：

```javascript
db.query_demo.find().sort({ status: 1, score: -1 })
```

排序字段如果没有索引，数据量大时可能很慢。

## 14. limit 和 skip

返回 2 条：

```javascript
db.query_demo.find().limit(2)
```

跳过 2 条，再返回 2 条：

```javascript
db.query_demo.find().skip(2).limit(2)
```

分页示例：

```javascript
const page = 2
const pageSize = 2

db.query_demo.find()
  .sort({ created_at: -1 })
  .skip((page - 1) * pageSize)
  .limit(pageSize)
```

注意：`skip` 深分页在大数据量下有性能问题。第 5 阶段会学习游标分页。

## 15. 查询数量

```javascript
db.query_demo.countDocuments()
```

带条件：

```javascript
db.query_demo.countDocuments({ status: "active" })
```

## 16. 查询时常见错误

### 类型不匹配

```javascript
db.query_demo.find({ age: "25" })
```

如果数据库里 `age` 是数字，这个查询匹配不到数字 25。

### ObjectId 当字符串查

```javascript
db.query_demo.findOne({ _id: "66a1..." })
```

如果 `_id` 是 ObjectId，应该写：

```javascript
db.query_demo.findOne({ _id: ObjectId("66a1...") })
```

### 忘记软删除条件

如果业务使用软删除，每次列表查询都要考虑：

```javascript
{ deleted_at: { $exists: false } }
```

## 18. 本节小结

你现在需要记住：

- `find` 返回多条，`findOne` 返回一条。
- 多字段条件默认是 AND。
- 常见比较操作符：`$gt`、`$gte`、`$lt`、`$lte`、`$ne`。
- `$in` 适合多个候选值。
- 点语法可以查询嵌套字段。
- 投影用于控制返回字段。
- 排序和分页要注意索引和深分页性能。
