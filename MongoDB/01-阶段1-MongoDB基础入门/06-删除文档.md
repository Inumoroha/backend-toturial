# 06. 删除文档

删除是最危险的基础操作之一。本节学习 `deleteOne`、`deleteMany`、`drop`，以及真实后端项目中常用的软删除思路。

## 1. 准备数据

```javascript
use mongodb_stage1
db.delete_demo.drop()

db.delete_demo.insertMany([
  {
    name: "Alice",
    email: "alice@example.com",
    status: "active",
    created_at: new Date()
  },
  {
    name: "Bob",
    email: "bob@example.com",
    status: "inactive",
    created_at: new Date()
  },
  {
    name: "Charlie",
    email: "charlie@example.com",
    status: "inactive",
    created_at: new Date()
  },
  {
    name: "Diana",
    email: "diana@example.com",
    status: "blocked",
    created_at: new Date()
  }
])
```

## 2. deleteOne

删除一条文档：

```javascript
db.delete_demo.find({ email: "diana@example.com" }).pretty()

db.delete_demo.deleteOne({
  email: "diana@example.com"
})
```

返回结果：

```javascript
{
  acknowledged: true,
  deletedCount: 1
}
```

如果没有匹配：

```javascript
db.delete_demo.deleteOne({
  email: "not-exists@example.com"
})
```

`deletedCount` 会是 0。

## 3. deleteMany

删除所有 inactive 用户：

```javascript
db.delete_demo.find({ status: "inactive" }).pretty()

db.delete_demo.deleteMany({
  status: "inactive"
})
```

`deleteMany` 一定要先查询确认范围。

危险写法：

```javascript
db.delete_demo.deleteMany({})
```

这会删除集合中的所有文档。

如果你只是学习清理数据，可以使用；真实项目中必须极其谨慎。

## 4. 删除所有文档但保留集合

```javascript
db.delete_demo.deleteMany({})
```

这会清空集合里的文档，但集合和索引仍可能保留。

学习阶段可以用它重置数据。

## 5. 删除集合 drop

```javascript
db.delete_demo.drop()
```

`drop` 会删除整个集合，包括集合里的文档和索引。

适合学习阶段重建集合，不适合真实环境随手执行。

## 6. 删除数据库 dropDatabase

```javascript
use mongodb_stage1
db.dropDatabase()
```

这会删除当前数据库。

执行前一定确认当前数据库：

```javascript
db
```

真实项目里，删除数据库属于高危操作。

## 7. 软删除

真实后端系统中，很多业务数据不会直接物理删除，而是软删除。

软删除通常是设置字段：

```javascript
deleted_at: new Date()
```

示例：

```javascript
db.delete_demo.updateOne(
  { email: "alice@example.com" },
  {
    $set: {
      deleted_at: new Date(),
      updated_at: new Date()
    }
  }
)
```

查询未删除数据：

```javascript
db.delete_demo.find({
  deleted_at: { $exists: false }
})
```

软删除优点：

- 可以恢复。
- 保留审计记录。
- 避免误删造成不可逆影响。

软删除缺点：

- 所有查询都要记得过滤。
- 数据量会继续增长。
- 唯一索引设计会变复杂。

## 8. 软删除恢复

恢复软删除：

```javascript
db.delete_demo.updateOne(
  { email: "alice@example.com" },
  {
    $unset: { deleted_at: "" },
    $set: { updated_at: new Date() }
  }
)
```

查询：

```javascript
db.delete_demo.find({ email: "alice@example.com" }).pretty()
```

## 9. 删除前备份思维

学习阶段你可以随便删；真实环境不行。

删除前至少考虑：

- 有没有备份？
- 删除条件是否足够精确？
- 是否需要先导出待删除数据？
- 是否需要先统计数量？
- 是否可以先软删除？
- 是否要分批删除？
- 是否会影响关联数据？

统计待删除数量：

```javascript
db.delete_demo.countDocuments({ status: "inactive" })
```

查看样本：

```javascript
db.delete_demo.find({ status: "inactive" }).limit(5).pretty()
```

## 10. 分批删除的思想

如果数据很多，不建议一次性删除海量数据。

学习阶段先了解思路：

```javascript
const ids = db.delete_demo
  .find({ status: "inactive" }, { _id: 1 })
  .limit(100)
  .map(doc => doc._id)

db.delete_demo.deleteMany({ _id: { $in: ids } })
```

真实批量清理会写脚本，控制每批数量、间隔、日志和失败重试。

## 12. 本节小结

你现在需要记住：

- `deleteOne` 删除一条。
- `deleteMany` 删除多条。
- 删除前必须先 `find` 或 `countDocuments` 确认范围。
- `drop` 删除集合。
- `dropDatabase` 删除数据库。
- 真实业务中常使用软删除。
- 软删除会让查询和索引设计更复杂。
