# 07. 数组与嵌套文档查询

数组和嵌套文档是 MongoDB 文档模型的核心能力。本节重点学习点语法、数组匹配、`$all`、`$size`、`$elemMatch`。

## 1. 准备数据

```javascript
use mongodb_stage1
db.nested_demo.drop()

db.nested_demo.insertMany([
  {
    title: "Go 后端入门",
    author: {
      id: "u1",
      name: "Alice",
      city: "Shanghai"
    },
    tags: ["go", "backend", "api"],
    comments: [
      { user_id: "u2", content: "good", likes: 3 },
      { user_id: "u3", content: "useful", likes: 5 }
    ],
    stats: {
      views: 120,
      likes: 12
    },
    status: "published",
    created_at: new Date("2026-07-01T10:00:00Z")
  },
  {
    title: "MongoDB 文档模型",
    author: {
      id: "u2",
      name: "Bob",
      city: "Beijing"
    },
    tags: ["mongodb", "database", "backend"],
    comments: [
      { user_id: "u1", content: "clear", likes: 6 },
      { user_id: "u4", content: "thanks", likes: 1 }
    ],
    stats: {
      views: 300,
      likes: 45
    },
    status: "published",
    created_at: new Date("2026-07-02T10:00:00Z")
  },
  {
    title: "Docker 本地开发",
    author: {
      id: "u3",
      name: "Charlie",
      city: "Hangzhou"
    },
    tags: ["docker", "devops"],
    comments: [],
    stats: {
      views: 80,
      likes: 6
    },
    status: "draft",
    created_at: new Date("2026-07-03T10:00:00Z")
  }
])
```

## 2. 查询嵌套字段

使用点语法：

```javascript
db.nested_demo.find({
  "author.city": "Shanghai"
})
```

查询统计字段：

```javascript
db.nested_demo.find({
  "stats.views": { $gt: 100 }
})
```

只返回标题和作者城市：

```javascript
db.nested_demo.find(
  {},
  { title: 1, "author.city": 1, _id: 0 }
)
```

## 3. 更新嵌套字段

```javascript
db.nested_demo.updateOne(
  { title: "Go 后端入门" },
  {
    $set: {
      "author.city": "Shenzhen",
      updated_at: new Date()
    }
  }
)
```

查询确认：

```javascript
db.nested_demo.find(
  { title: "Go 后端入门" },
  { title: 1, author: 1, _id: 0 }
).pretty()
```

## 4. 数组包含某个元素

查询 tags 包含 `backend` 的文章：

```javascript
db.nested_demo.find({
  tags: "backend"
})
```

等价于：

```javascript
db.nested_demo.find({
  tags: { $eq: "backend" }
})
```

MongoDB 会匹配数组中是否存在这个元素。

## 5. $all

查询同时包含 `mongodb` 和 `backend` 标签的文章：

```javascript
db.nested_demo.find({
  tags: { $all: ["mongodb", "backend"] }
})
```

`$all` 要求数组包含所有指定元素，不要求顺序一致。

## 6. $size

查询标签数量为 2 的文章：

```javascript
db.nested_demo.find({
  tags: { $size: 2 }
})
```

查询评论数量为 0 的文章：

```javascript
db.nested_demo.find({
  comments: { $size: 0 }
})
```

注意：`$size` 不适合范围查询。如果你经常需要按数组长度范围查询，通常应该额外维护一个计数字段。

例如：

```javascript
comment_count: 2
```

## 7. 查询数组中的嵌套文档字段

查询有用户 `u1` 评论过的文章：

```javascript
db.nested_demo.find({
  "comments.user_id": "u1"
})
```

查询有点赞数大于 3 的评论的文章：

```javascript
db.nested_demo.find({
  "comments.likes": { $gt: 3 }
})
```

## 8. 为什么需要 $elemMatch

先看一个容易误解的查询：

```javascript
db.nested_demo.find({
  "comments.user_id": "u4",
  "comments.likes": { $gt: 3 }
})
```

这个查询的意思不是“同一条评论同时满足 user_id 是 u4 且 likes 大于 3”。

它可能匹配：

- 某条评论的 `user_id` 是 `u4`。
- 另一条评论的 `likes` 大于 3。

如果要求同一个数组元素同时满足多个条件，要用 `$elemMatch`。

```javascript
db.nested_demo.find({
  comments: {
    $elemMatch: {
      user_id: "u4",
      likes: { $gt: 3 }
    }
  }
})
```

这才表示同一条评论既是 `u4` 发的，又点赞数大于 3。

## 9. 数组更新：追加和去重

追加标签：

```javascript
db.nested_demo.updateOne(
  { title: "Docker 本地开发" },
  {
    $push: { tags: "backend" },
    $set: { updated_at: new Date() }
  }
)
```

避免重复追加：

```javascript
db.nested_demo.updateOne(
  { title: "Docker 本地开发" },
  {
    $addToSet: { tags: "backend" },
    $set: { updated_at: new Date() }
  }
)
```

移除标签：

```javascript
db.nested_demo.updateOne(
  { title: "Docker 本地开发" },
  {
    $pull: { tags: "devops" },
    $set: { updated_at: new Date() }
  }
)
```

## 10. 向嵌套数组追加评论

```javascript
db.nested_demo.updateOne(
  { title: "Docker 本地开发" },
  {
    $push: {
      comments: {
        user_id: "u5",
        content: "nice",
        likes: 0,
        created_at: new Date()
      }
    },
    $inc: { "stats.likes": 1 },
    $set: { updated_at: new Date() }
  }
)
```

注意：这里故意展示了一个潜在问题。评论如果会无限增长，不适合一直嵌入文章文档。第 3 阶段文档建模会系统讨论。

## 11. 投影数组字段

只返回标题和标签：

```javascript
db.nested_demo.find(
  {},
  { title: 1, tags: 1, _id: 0 }
)
```

只返回匹配到的第一个数组元素，可以使用位置投影：

```javascript
db.nested_demo.find(
  { "comments.user_id": "u1" },
  { title: 1, "comments.$": 1 }
)
```

学习阶段了解即可，后面项目中更常用聚合管道处理复杂数组。

## 12. 数组和嵌套文档的常见坑

### 坑 1：数组无限增长

例如把所有评论、所有登录日志、所有订单都嵌入用户文档，后期会出问题。

### 坑 2：忘记 $elemMatch

当同一数组元素需要同时满足多个条件时，要考虑 `$elemMatch`。

### 坑 3：数组字段重复

标签、角色通常不希望重复，优先用 `$addToSet`。

### 坑 4：嵌套太深

嵌套太深会让查询、更新和索引都变复杂。

## 14. 本节小结

你现在需要记住：

- 点语法可以查询和更新嵌套字段。
- 数组字段可以直接用元素值匹配。
- `$all` 要求数组包含所有指定值。
- `$size` 匹配数组长度。
- 查询数组中嵌套文档字段可以用点语法。
- 同一个数组元素需要满足多个条件时，用 `$elemMatch`。
- 数组很强大，但不要让它无限增长。
