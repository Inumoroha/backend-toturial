# 05. 更新文档

更新是 CRUD 中风险较高的一类操作，因为它会修改已有数据。本节学习 `updateOne`、`updateMany` 和常见更新操作符。

## 1. 准备数据

```javascript
use mongodb_stage1
db.update_demo.drop()

db.update_demo.insertMany([
  {
    name: "Alice",
    email: "alice@example.com",
    age: 25,
    status: "active",
    roles: ["user"],
    profile: { city: "Shanghai", company: "Example Inc" },
    login_count: 3,
    tags: ["go", "mongodb"],
    created_at: new Date(),
    updated_at: new Date()
  },
  {
    name: "Bob",
    email: "bob@example.com",
    age: 30,
    status: "active",
    roles: ["user", "admin"],
    profile: { city: "Beijing", company: "Demo Tech" },
    login_count: 10,
    tags: ["backend"],
    created_at: new Date(),
    updated_at: new Date()
  },
  {
    name: "Charlie",
    email: "charlie@example.com",
    age: 19,
    status: "inactive",
    roles: ["user"],
    profile: { city: "Hangzhou", company: "Student" },
    login_count: 0,
    tags: [],
    created_at: new Date(),
    updated_at: new Date()
  }
])
```

## 2. updateOne

更新 Alice 的年龄：

```javascript
db.update_demo.updateOne(
  { email: "alice@example.com" },
  {
    $set: {
      age: 26,
      updated_at: new Date()
    }
  }
)
```

返回结果类似：

```javascript
{
  acknowledged: true,
  matchedCount: 1,
  modifiedCount: 1
}
```

含义：

- `matchedCount`：匹配到几条。
- `modifiedCount`：实际修改了几条。

如果新值和旧值一样，可能匹配到了，但 `modifiedCount` 是 0。

## 3. updateMany

把所有 inactive 用户标记为 archived：

```javascript
db.update_demo.updateMany(
  { status: "inactive" },
  {
    $set: {
      archived: true,
      updated_at: new Date()
    }
  }
)
```

使用 `updateMany` 前一定要先查：

```javascript
db.update_demo.find({ status: "inactive" }).pretty()
```

不要直接写：

```javascript
db.update_demo.updateMany({}, { $set: { status: "inactive" } })
```

除非你真的要更新整个集合。

## 4. $set

`$set` 用于设置字段值。

```javascript
db.update_demo.updateOne(
  { email: "bob@example.com" },
  {
    $set: {
      "profile.city": "Shanghai",
      updated_at: new Date()
    }
  }
)
```

可以用点语法更新嵌套字段。

如果字段不存在，`$set` 会创建字段：

```javascript
db.update_demo.updateOne(
  { email: "bob@example.com" },
  {
    $set: {
      last_login_at: new Date()
    }
  }
)
```

## 5. $unset

`$unset` 删除字段：

```javascript
db.update_demo.updateOne(
  { email: "bob@example.com" },
  {
    $unset: {
      archived: ""
    }
  }
)
```

`$unset` 的值通常写空字符串，重点是字段名。

注意：删除字段不是删除文档。

## 6. $inc

`$inc` 用于数字递增或递减。

登录次数加 1：

```javascript
db.update_demo.updateOne(
  { email: "alice@example.com" },
  {
    $inc: { login_count: 1 },
    $set: { updated_at: new Date() }
  }
)
```

减少积分：

```javascript
db.update_demo.updateOne(
  { email: "alice@example.com" },
  {
    $inc: { score: -10 },
    $set: { updated_at: new Date() }
  }
)
```

如果 `score` 不存在，`$inc` 会创建它并按 0 计算。

## 7. $push

`$push` 向数组追加元素。

```javascript
db.update_demo.updateOne(
  { email: "alice@example.com" },
  {
    $push: { tags: "database" },
    $set: { updated_at: new Date() }
  }
)
```

重复执行会追加重复值。

追加多个：

```javascript
db.update_demo.updateOne(
  { email: "alice@example.com" },
  {
    $push: {
      tags: {
        $each: ["docker", "api"]
      }
    },
    $set: { updated_at: new Date() }
  }
)
```

## 8. $addToSet

`$addToSet` 向数组添加元素，但避免重复。

```javascript
db.update_demo.updateOne(
  { email: "alice@example.com" },
  {
    $addToSet: { roles: "editor" },
    $set: { updated_at: new Date() }
  }
)
```

重复执行不会重复添加 `editor`。

添加多个：

```javascript
db.update_demo.updateOne(
  { email: "alice@example.com" },
  {
    $addToSet: {
      roles: {
        $each: ["author", "vip"]
      }
    },
    $set: { updated_at: new Date() }
  }
)
```

角色、标签这类不希望重复的数组，通常优先考虑 `$addToSet`。

## 9. $pull

`$pull` 从数组中移除匹配元素。

```javascript
db.update_demo.updateOne(
  { email: "alice@example.com" },
  {
    $pull: { tags: "database" },
    $set: { updated_at: new Date() }
  }
)
```

从角色中移除：

```javascript
db.update_demo.updateOne(
  { email: "alice@example.com" },
  {
    $pull: { roles: "vip" },
    $set: { updated_at: new Date() }
  }
)
```

## 10. $rename

字段重命名：

```javascript
db.update_demo.updateMany(
  {},
  {
    $rename: {
      company: "organization"
    }
  }
)
```

学习阶段了解即可。真实项目中字段重命名属于数据迁移，要谨慎规划。

## 11. upsert

`upsert` 表示：有则更新，没有则插入。

```javascript
db.update_demo.updateOne(
  { email: "new-user@example.com" },
  {
    $set: {
      name: "New User",
      status: "active",
      updated_at: new Date()
    },
    $setOnInsert: {
      created_at: new Date()
    }
  },
  { upsert: true }
)
```

如果没有匹配 `email: "new-user@example.com"` 的文档，会插入新文档。

`$setOnInsert` 只在插入时生效。

适合场景：

- 配置初始化。
- 幂等写入。
- 同步外部数据。

但要注意唯一约束，否则并发 upsert 可能产生意料之外的数据。

## 12. 替换文档 replaceOne

`replaceOne` 会替换整条文档，除了 `_id`。

```javascript
db.update_demo.replaceOne(
  { email: "charlie@example.com" },
  {
    name: "Charlie New",
    email: "charlie@example.com",
    status: "active",
    updated_at: new Date()
  }
)
```

注意：原来的 `age`、`roles`、`profile` 等字段会消失。

学习阶段建议优先使用更新操作符，少用整文档替换。

## 13. 更新安全习惯

每次更新前：

```javascript
db.update_demo.find({ status: "inactive" }).pretty()
```

确认范围后再：

```javascript
db.update_demo.updateMany(
  { status: "inactive" },
  { $set: { archived: true } }
)
```

真实项目中还要考虑：

- 是否有备份。
- 是否可以回滚。
- 是否需要分批更新。
- 是否影响索引。
- 是否会触发业务逻辑。

## 14. 本节练习

完成下面操作：

1. 把 Alice 的年龄改成 27。
2. 给 Bob 增加 `editor` 角色，不能重复。
3. 给 Charlie 增加 `login_count` 1 次。
4. 给所有 inactive 用户设置 `archived: true`。
5. 从 Alice 的 `tags` 中移除 `go`。
6. 给所有用户设置 `updated_at`。
7. 使用 upsert 创建一个新用户。
8. 查询每一步修改后的结果。

## 15. 本节小结

你现在需要记住：

- `updateOne` 修改一条，`updateMany` 修改多条。
- 更新前先 `find` 确认范围。
- `$set` 设置字段。
- `$unset` 删除字段。
- `$inc` 数字增减。
- `$push` 数组追加，可能重复。
- `$addToSet` 数组追加，避免重复。
- `$pull` 从数组移除。
- `upsert` 是有则更新、无则插入。
- `replaceOne` 会替换整条文档，使用要谨慎。
