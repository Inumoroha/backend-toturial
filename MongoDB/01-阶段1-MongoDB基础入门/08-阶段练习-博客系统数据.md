# 08. 阶段练习：博客系统数据

本练习把第 1 阶段的内容串起来。你会创建一个简单博客系统的数据，包括用户、文章、评论，然后完成不少于 30 条 `mongosh` 查询和修改操作。

## 1. 练习目标

完成后你应该能熟练操作：

- 创建数据库和集合。
- 插入用户、文章、评论。
- 查询文章列表。
- 查询用户文章。
- 查询评论。
- 更新文章状态。
- 更新点赞、浏览量。
- 使用数组和嵌套字段查询。
- 使用投影、排序、分页。
- 安全删除测试数据。

## 6. 查询练习：基础查询

## 7. 查询练习：字段和条件

7. 查询城市是 Shanghai 的用户：

```javascript
db.users.find({ "profile.city": "Shanghai" }).pretty()
```

8. 查询角色包含 admin 的用户：

```javascript
db.users.find({ roles: "admin" }).pretty()
```

9. 查询标签包含 mongodb 的文章：

```javascript
db.posts.find({ tags: "mongodb" }).pretty()
```

10. 查询同时包含 mongodb 和 backend 标签的文章：

```javascript
db.posts.find({ tags: { $all: ["mongodb", "backend"] } }).pretty()
```

11. 查询浏览量大于 100 的文章：

```javascript
db.posts.find({ "stats.views": { $gt: 100 } }).pretty()
```

12. 查询点赞数大于等于 40 的文章：

```javascript
db.posts.find({ "stats.likes": { $gte: 40 } }).pretty()
```

13. 查询评论数不为 0 的文章：

```javascript
db.posts.find({ "stats.comments": { $ne: 0 } }).pretty()
```

14. 查询 2026-07-03 之后创建的文章：

```javascript
db.posts.find({
  created_at: { $gte: new Date("2026-07-03T00:00:00Z") }
}).pretty()
```

15. 查询状态为 published 或 draft 的文章：

```javascript
db.posts.find({
  status: { $in: ["published", "draft"] }
}).pretty()
```

## 8. 查询练习：投影、排序、分页

16. 只返回文章标题、slug 和状态：

```javascript
db.posts.find(
  {},
  { title: 1, slug: 1, status: 1, _id: 0 }
).pretty()
```

17. 按发布时间倒序查询已发布文章：

```javascript
db.posts.find({ status: "published" })
  .sort({ published_at: -1 })
  .pretty()
```

18. 查询浏览量最高的前 2 篇文章：

```javascript
db.posts.find()
  .sort({ "stats.views": -1 })
  .limit(2)
  .pretty()
```

19. 查询第 2 页文章，每页 2 条：

```javascript
db.posts.find()
  .sort({ created_at: -1 })
  .skip(2)
  .limit(2)
  .pretty()
```

20. 查询文章数量：

```javascript
db.posts.countDocuments()
```

21. 查询已发布文章数量：

```javascript
db.posts.countDocuments({ status: "published" })
```

## 10. 更新练习

25. 给文章浏览量加 1：

```javascript
db.posts.updateOne(
  { slug: "go-backend-intro" },
  {
    $inc: { "stats.views": 1 },
    $set: { updated_at: new Date() }
  }
)
```

26. 给文章点赞数加 1：

```javascript
db.posts.updateOne(
  { slug: "mongodb-document-model" },
  {
    $inc: { "stats.likes": 1 },
    $set: { updated_at: new Date() }
  }
)
```

27. 给文章增加标签，避免重复：

```javascript
db.posts.updateOne(
  { slug: "go-backend-intro" },
  {
    $addToSet: { tags: "golang" },
    $set: { updated_at: new Date() }
  }
)
```

28. 从文章移除标签：

```javascript
db.posts.updateOne(
  { slug: "docker-local-dev" },
  {
    $pull: { tags: "devops" },
    $set: { updated_at: new Date() }
  }
)
```

29. 发布草稿文章：

```javascript
db.posts.updateOne(
  { slug: "docker-local-dev" },
  {
    $set: {
      status: "published",
      published_at: new Date(),
      updated_at: new Date()
    }
  }
)
```

30. 隐藏一条评论：

```javascript
db.comments.updateOne(
  { content: "广告内容" },
  {
    $set: {
      status: "hidden",
      updated_at: new Date()
    }
  }
)
```

## 11. 删除练习

31. 先查询要删除的评论：

```javascript
db.comments.find({ status: "hidden" }).pretty()
```

32. 删除隐藏评论：

```javascript
db.comments.deleteMany({ status: "hidden" })
```

33. 软删除 inactive 用户：

```javascript
db.users.updateMany(
  { status: "inactive" },
  {
    $set: {
      deleted_at: new Date(),
      updated_at: new Date()
    }
  }
)
```

34. 查询未软删除用户：

```javascript
db.users.find({
  deleted_at: { $exists: false }
}).pretty()
```

## 13. 思考题

1. 为什么 `posts.author_id` 存的是用户 `_id`，而不是用户名？
2. 为什么评论拆成了 `comments` 集合，而不是嵌入 `posts`？
3. `stats.comments` 是冗余字段，它有什么好处和风险？
4. 查询文章列表时，是否应该返回完整 `content`？
5. 如果用户改了 display_name，历史评论是否要跟着变？

这些问题没有唯一答案，但它们会引出第 3 阶段的数据建模。
