# 03. 插入文档

本节学习 MongoDB 的创建操作，也就是 CRUD 中的 Create。主要命令是 `insertOne` 和 `insertMany`。

## 1. 准备数据库

连接 MongoDB：

```powershell
mongosh "mongodb://root:example@localhost:27017/?authSource=admin"
```

切换数据库：

```javascript
use mongodb_stage1
```

为了避免旧数据干扰，可以先清理：

```javascript
db.insert_demo.drop()
```

## 2. insertOne

插入一条文档：

```javascript
db.insert_demo.insertOne({
  name: "Alice",
  email: "alice@example.com",
  age: 25,
  status: "active",
  created_at: new Date()
})
```

返回结果类似：

```javascript
{
  acknowledged: true,
  insertedId: ObjectId("...")
}
```

含义：

- `acknowledged: true` 表示写入被确认。
- `insertedId` 是插入文档的 `_id`。

查询：

```javascript
db.insert_demo.find().pretty()
```

## 3. 插入包含数组和嵌套文档的数据

```javascript
db.insert_demo.insertOne({
  name: "Bob",
  email: "bob@example.com",
  age: 30,
  roles: ["user", "admin"],
  profile: {
    city: "Beijing",
    company: "Demo Tech",
    skills: ["Go", "MongoDB", "Docker"]
  },
  created_at: new Date()
})
```

查询：

```javascript
db.insert_demo.find({ email: "bob@example.com" }).pretty()
```

你会看到 `roles` 是数组，`profile` 是嵌套文档。

## 4. insertMany

插入多条文档：

```javascript
db.insert_demo.insertMany([
  {
    name: "Charlie",
    email: "charlie@example.com",
    age: 19,
    status: "active",
    created_at: new Date()
  },
  {
    name: "Diana",
    email: "diana@example.com",
    age: 28,
    status: "inactive",
    created_at: new Date()
  },
  {
    name: "Eric",
    email: "eric@example.com",
    age: 35,
    status: "active",
    created_at: new Date()
  }
])
```

返回结果类似：

```javascript
{
  acknowledged: true,
  insertedIds: {
    '0': ObjectId("..."),
    '1': ObjectId("..."),
    '2': ObjectId("...")
  }
}
```

## 5. 查看文档数量

统计集合中文档数量：

```javascript
db.insert_demo.countDocuments()
```

带条件统计：

```javascript
db.insert_demo.countDocuments({ status: "active" })
```

注意：学习阶段可以用 `countDocuments` 熟悉数据量。真实项目中，频繁统计大集合也要注意性能。

## 6. 插入时指定 _id

```javascript
db.insert_demo.insertOne({
  _id: "user_custom_001",
  name: "Frank",
  email: "frank@example.com",
  status: "active",
  created_at: new Date()
})
```

查询：

```javascript
db.insert_demo.findOne({ _id: "user_custom_001" })
```

再次插入相同 `_id`：

```javascript
db.insert_demo.insertOne({
  _id: "user_custom_001",
  name: "Frank 2",
  email: "frank2@example.com"
})
```

会报重复键错误：

```text
E11000 duplicate key error
```

## 7. 字段可以不完全一致

MongoDB 同一个集合里的文档不要求字段完全一样。

```javascript
db.insert_demo.insertOne({
  name: "Grace",
  email: "grace@example.com"
})
```

这条文档没有 `age`、`status`、`created_at`。

灵活性是 MongoDB 的优势，但在 Go 后端项目中，不能因此放任字段混乱。你应该通过：

- Go 结构体。
- 请求参数校验。
- 默认值填充。
- Schema Validation。
- 数据迁移脚本。

来维持数据质量。

## 8. 有序插入和无序插入

`insertMany` 默认是有序插入。如果中间某条出错，后面的可能不会继续插入。

先准备一个重复 `_id`：

```javascript
db.insert_order_demo.drop()

db.insert_order_demo.insertOne({
  _id: "u1",
  name: "User 1"
})
```

有序插入：

```javascript
db.insert_order_demo.insertMany([
  { _id: "u2", name: "User 2" },
  { _id: "u1", name: "Duplicate User 1" },
  { _id: "u3", name: "User 3" }
])
```

中间 `_id: "u1"` 重复，后面的 `u3` 可能不会继续插入。

无序插入：

```javascript
db.insert_order_demo.insertMany(
  [
    { _id: "u4", name: "User 4" },
    { _id: "u1", name: "Duplicate User 1" },
    { _id: "u5", name: "User 5" }
  ],
  { ordered: false }
)
```

无序插入遇到错误时，会尽量继续插入其他可写入的文档。

适用场景：

- 批量导入。
- 希望跳过个别失败数据。

但你仍然需要记录失败项，不能静默忽略。

## 9. 插入时间字段

建议每条业务文档都保存创建时间：

```javascript
db.insert_demo.insertOne({
  name: "Helen",
  email: "helen@example.com",
  created_at: new Date(),
  updated_at: new Date()
})
```

常见字段：

```text
created_at
updated_at
deleted_at
```

`deleted_at` 常用于软删除。后面更新和删除章节会讲。

## 10. 插入前的检查习惯

插入用户前，通常要考虑：

- 邮箱是否格式正确。
- 邮箱是否唯一。
- 必填字段是否为空。
- 默认状态是什么。
- 创建时间和更新时间是否设置。

学习阶段先在 `mongosh` 中手动插入。后面 Go 项目中，这些检查会放在 Handler 和 Service 层。

## 11. 本节练习

创建 `products` 集合，插入下面类型的数据：

1. 一本书。
2. 一门课程。
3. 一个电子设备。
4. 一个包含标签数组的商品。
5. 一个包含嵌套规格对象的商品。

示例：

```javascript
db.products.insertOne({
  name: "MongoDB 入门课",
  category: "course",
  price: 9900,
  tags: ["database", "backend"],
  specs: {
    level: "beginner",
    duration_hours: 12
  },
  status: "on_sale",
  created_at: new Date(),
  updated_at: new Date()
})
```

然后完成：

```javascript
db.products.find().pretty()
db.products.countDocuments()
db.products.countDocuments({ status: "on_sale" })
```

## 12. 本节小结

你现在需要记住：

- `insertOne` 插入单条文档。
- `insertMany` 插入多条文档。
- 不提供 `_id` 时 MongoDB 会自动生成 ObjectId。
- `_id` 重复会报 duplicate key error。
- 同一个集合可以存字段不同的文档，但工程上仍然要保持结构稳定。
- 时间字段应在插入时明确设置。
