# 02. 数据库、集合、文档与 _id

本节学习 MongoDB 的基本组织结构：database、collection、document，以及每条文档默认拥有的 `_id` 字段。

## 1. 三层结构

MongoDB 中的数据大致按三层组织：

```text
MongoDB Server
  |
  |-- database
        |
        |-- collection
              |
              |-- document
```

例如：

```text
mongodb_stage1
  |
  |-- users
  |-- posts
  |-- comments
```

`mongodb_stage1` 是数据库，`users`、`posts`、`comments` 是集合，集合里的一条条数据是文档。

## 2. 切换数据库

连接 MongoDB：

```powershell
mongosh "mongodb://root:example@localhost:27017/?authSource=admin"
```

切换数据库：

```javascript
use mongodb_stage1
```

查看当前数据库：

```javascript
db
```

注意：`use mongodb_stage1` 不会立刻创建数据库。只有当你写入数据后，数据库才会真正出现。

查看数据库列表：

```javascript
show dbs
```

如果你刚 `use`，但没有插入数据，`show dbs` 里可能看不到它。

## 3. 集合是什么

集合是一组文档的容器。

例如：

```javascript
db.users.insertOne({
  name: "Alice",
  email: "alice@example.com",
  created_at: new Date()
})
```

这里的 `users` 就是集合。

查看集合：

```javascript
show collections
```

如果集合不存在，插入文档时 MongoDB 会自动创建集合。

## 4. 文档是什么

文档是 MongoDB 中的一条数据记录。

示例：

```javascript
{
  _id: ObjectId("66a1f1c2e138237e2f7d0001"),
  name: "Alice",
  email: "alice@example.com",
  age: 25,
  created_at: ISODate("2026-07-05T10:00:00Z")
}
```

它相当于关系型数据库中的一行，但比“行”更灵活，因为它可以有嵌套结构和数组。

## 5. _id 是什么

每个 MongoDB 文档都必须有 `_id` 字段。

如果你插入文档时不提供 `_id`，MongoDB 会自动生成一个 ObjectId。

执行：

```javascript
db.users.insertOne({
  name: "Bob",
  email: "bob@example.com"
})
```

查询：

```javascript
db.users.find({ email: "bob@example.com" }).pretty()
```

你会看到类似：

```javascript
{
  _id: ObjectId("66a1f1c2e138237e2f7d0002"),
  name: "Bob",
  email: "bob@example.com"
}
```

## 6. ObjectId 的特点

ObjectId 通常长这样：

```javascript
ObjectId("66a1f1c2e138237e2f7d0002")
```

它是一个 12 字节的 BSON 类型，常用作默认主键。

学习阶段你需要知道：

- ObjectId 不是普通字符串。
- 查询 `_id` 时要用 `ObjectId("...")`。
- ObjectId 中包含时间信息。
- `_id` 默认唯一。
- `_id` 默认有索引。

查看 ObjectId 的生成时间：

```javascript
ObjectId("66a1f1c2e138237e2f7d0002").getTimestamp()
```

注意：不要把 ObjectId 的时间信息当作业务时间。业务时间还是应该显式保存 `created_at`。

## 7. 查询 _id

先插入：

```javascript
const result = db.users.insertOne({
  name: "Charlie",
  email: "charlie@example.com",
  created_at: new Date()
})
```

查看插入结果：

```javascript
result
```

查询：

```javascript
db.users.findOne({ _id: result.insertedId })
```

如果你复制了某个 ObjectId 字符串：

```javascript
db.users.findOne({ _id: ObjectId("66a1f1c2e138237e2f7d0002") })
```

错误写法：

```javascript
db.users.findOne({ _id: "66a1f1c2e138237e2f7d0002" })
```

除非你的 `_id` 本来就是字符串，否则这个查询匹配不到 ObjectId 类型的 `_id`。

## 8. 自定义 _id

你可以自己指定 `_id`：

```javascript
db.categories.insertOne({
  _id: "go",
  name: "Go 后端",
  description: "Go backend articles"
})
```

查询：

```javascript
db.categories.findOne({ _id: "go" })
```

自定义 `_id` 的好处：

- 业务含义清晰。
- 某些配置类、分类类数据查询简单。

风险：

- 需要自己保证唯一。
- 业务含义变化时不方便修改。
- 长字符串主键可能比 ObjectId 更占空间。

学习阶段建议：大多数业务文档使用默认 ObjectId，少量固定枚举或配置数据可以考虑字符串 `_id`。

## 9. 重复 _id 会怎样

执行：

```javascript
db.categories.insertOne({
  _id: "go",
  name: "Go 后端"
})
```

如果再执行一次相同 `_id` 的插入：

```javascript
db.categories.insertOne({
  _id: "go",
  name: "Go 新分类"
})
```

会报重复键错误，类似：

```text
E11000 duplicate key error
```

这是因为 `_id` 必须唯一。

## 10. 字段命名建议

建议：

- 使用小写加下划线：`created_at`、`updated_at`。
- 字段名语义清晰。
- 不要频繁变更字段名。
- 不要混用 `createdAt` 和 `created_at`。
- 不要用太长的字段名。

Go 中可以通过 tag 映射：

```go
CreatedAt time.Time `bson:"created_at" json:"created_at"`
```

## 11. 删除学习数据

删除当前数据库：

```javascript
db.dropDatabase()
```

删除集合：

```javascript
db.users.drop()
```

注意：学习阶段可以清理数据，真实环境不要随便执行。

## 13. 本节小结

你现在需要记住：

- MongoDB 使用 database、collection、document 组织数据。
- `use` 不会立刻创建数据库，写入数据后才会出现。
- 每个文档都有 `_id`。
- 默认 `_id` 是 ObjectId，不是字符串。
- 查询 ObjectId 类型的 `_id` 时，要使用 `ObjectId("...")`。
- `_id` 唯一且默认有索引。
