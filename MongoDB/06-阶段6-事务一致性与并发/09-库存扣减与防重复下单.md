# 09. 库存扣减与防重复下单

库存扣减比看起来复杂：既要防止库存扣成负数，又要防止重复下单，还要写库存流水。不同复杂度下，可以选择单文档原子更新、事务或事件补偿。

## 1. 单商品库存扣减

商品：

```javascript
{
  _id: ObjectId("..."),
  name: "MongoDB 入门课",
  stock: 100,
  sold: 0
}
```

扣减：

```javascript
db.products.updateOne(
  {
    _id: productId,
    stock: { $gte: quantity }
  },
  {
    $inc: {
      stock: -quantity,
      sold: quantity
    },
    $set: { updated_at: new Date() }
  }
)
```

这是单文档原子更新，不一定需要事务。

## 2. SKU 数组库存扣减

商品：

```javascript
{
  _id: ObjectId("..."),
  skus: [
    { id: skuId, name: "标准版", stock: 100, sold: 0 }
  ]
}
```

扣减：

```javascript
db.products.updateOne(
  {
    _id: productId,
    "skus.id": skuId,
    "skus.stock": { $gte: quantity }
  },
  {
    $inc: {
      "skus.$.stock": -quantity,
      "skus.$.sold": quantity
    },
    $set: { updated_at: new Date() }
  }
)
```

注意：数组定位和并发更新要充分测试。

## 3. 创建订单 + 扣库存

如果只扣一个商品，且可以先扣库存再创建订单失败后补偿，可能不用事务，但复杂度会上升。

如果要求：

```text
订单创建
库存扣减
库存流水
购物车清理
```

必须一起成功，通常考虑事务。

## 4. 防重复下单

订单集合：

```javascript
db.orders.createIndex({ idempotency_key: 1 }, { unique: true })
```

订单文档：

```javascript
{
  order_no: "O202607050001",
  idempotency_key: "user1-cart-submit-abc123",
  user_id: ObjectId("..."),
  status: "pending_payment",
  items: [...],
  created_at: new Date()
}
```

重复请求使用同一个 `idempotency_key`，唯一索引阻止重复订单。

## 5. 库存流水

```javascript
{
  _id: ObjectId("..."),
  idempotency_key: "stock-deduct-O202607050001-sku1",
  product_id: ObjectId("..."),
  sku_id: ObjectId("..."),
  order_id: ObjectId("..."),
  type: "order_deduct",
  quantity: -1,
  created_at: new Date()
}
```

索引：

```javascript
db.inventory_records.createIndex({ idempotency_key: 1 }, { unique: true })
db.inventory_records.createIndex({ product_id: 1, created_at: -1 })
db.inventory_records.createIndex({ order_id: 1 })
```

流水也要幂等，避免重复记录。

## 6. 多商品下单

多商品下单涉及多个库存扣减：

```text
商品 A 扣减成功
商品 B 扣减失败
```

如果没有事务，需要补偿商品 A。

更稳妥方案：

- 事务中逐个扣减库存。
- 任何一个失败则回滚。
- 写订单。
- 写库存流水。

但事务中不要处理太多商品，订单 items 数量也应有限制。

## 7. 下单事务步骤

```text
1. 根据 idempotency_key 检查或插入订单
2. 校验商品和价格
3. 扣减每个商品库存
4. 创建订单
5. 写库存流水
6. 清理购物车选中项
7. 提交事务
```

如果 `idempotency_key` 重复：

- 查询已有订单。
- 返回已有订单结果。

## 8. 状态条件

取消订单回滚库存：

```javascript
db.orders.updateOne(
  { _id: orderId, status: "pending_payment" },
  {
    $set: {
      status: "cancelled",
      cancelled_at: new Date()
    }
  }
)
```

只有待支付订单允许取消并回滚库存。

## 10. 本节小结

你需要记住：

- 单商品库存扣减可用单文档条件更新。
- 多商品下单更容易需要事务。
- 下单必须考虑幂等键。
- 库存流水也要防重复。
- 订单状态流转必须带条件。
