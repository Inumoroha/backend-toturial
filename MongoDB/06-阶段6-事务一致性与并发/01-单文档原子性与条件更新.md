# 01. 单文档原子性与条件更新

MongoDB 的一个重要特性是：单个文档上的写操作是原子的。很多看起来需要事务的问题，可以通过单文档条件更新解决。

## 1. 什么是单文档原子性

单文档原子性表示：

```text
一次写操作作用在单个文档上时，要么完整成功，要么不发生部分修改。
```

例如：

```javascript
db.products.updateOne(
  { _id: productId },
  {
    $inc: { stock: -1 },
    $set: { updated_at: new Date() }
  }
)
```

`stock` 和 `updated_at` 的修改在同一文档中原子完成。

## 2. 并发更新会重新检查条件

官方文档强调：多个更新命令并发发生时，每个更新命令会检查查询条件是否仍然匹配。

这让条件更新可以解决很多并发问题。

例如库存扣减：

```javascript
db.products.updateOne(
  {
    _id: productId,
    stock: { $gte: 1 }
  },
  {
    $inc: { stock: -1 },
    $set: { updated_at: new Date() }
  }
)
```

如果库存只剩 1，两个请求同时来：

- 第一个匹配 `stock >= 1`，扣减成功，库存变 0。
- 第二个再检查条件时，`stock >= 1` 不成立，更新失败。

这就防止了库存扣成负数。

## 3. 准备数据

```javascript
use transaction_stage6
db.dropDatabase()
use transaction_stage6

db.products.insertOne({
  _id: ObjectId("670000000000000000000001"),
  name: "MongoDB 入门课",
  stock: 5,
  sold: 0,
  version: 1,
  updated_at: new Date()
})
```

保存：

```javascript
const productId = ObjectId("670000000000000000000001")
```

## 4. 安全扣减库存

```javascript
const quantity = 2

const result = db.products.updateOne(
  {
    _id: productId,
    stock: { $gte: quantity }
  },
  {
    $inc: {
      stock: -quantity,
      sold: quantity
    },
    $set: { updated_at: new Date() }
  }
)

result
```

判断结果：

```javascript
if (result.matchedCount === 0) {
  print("库存不足或商品不存在")
} else {
  print("扣减成功")
}
```

关键点：

- 不要先查库存再扣库存。
- 要把库存条件放进更新 filter。
- 通过 `matchedCount` 判断是否成功。

## 5. 错误示例：先查再写

错误流程：

```javascript
const product = db.products.findOne({ _id: productId })

if (product.stock >= quantity) {
  db.products.updateOne(
    { _id: productId },
    { $inc: { stock: -quantity } }
  )
}
```

并发下可能两个请求都看到库存足够，然后都扣减。

真实后端里，这就是超卖的典型来源。

## 6. 原子更新多个字段

扣库存同时增加销量：

```javascript
db.products.updateOne(
  {
    _id: productId,
    stock: { $gte: quantity }
  },
  {
    $inc: {
      stock: -quantity,
      sold: quantity
    },
    $set: { updated_at: new Date() }
  }
)
```

同一文档内这几个字段一起原子更新。

## 7. 条件更新适用场景

适合：

- 单商品库存扣减。
- 单钱包余额扣减。
- 单文章点赞数增加。
- 单用户状态流转。
- 单订单状态从 pending 改 paid。

订单状态更新：

```javascript
db.orders.updateOne(
  { _id: orderId, status: "pending_payment" },
  {
    $set: {
      status: "paid",
      paid_at: new Date(),
      updated_at: new Date()
    }
  }
)
```

状态条件能防止重复支付或非法状态流转。

## 8. 条件更新不适合什么

不适合单独解决：

- A 钱包扣款 + B 钱包加款。
- 创建订单 + 扣库存 + 写库存流水必须一起成功。
- 多个商品库存同时扣减。
- 多个集合需要强一致更新。

这些可能需要事务，或者需要更复杂的幂等和补偿机制。

## 10. 本节小结

你需要记住：

- MongoDB 单文档写操作是原子的。
- 并发更新会重新检查 filter 条件。
- 库存扣减要把 `stock >= quantity` 放进 update filter。
- 通过 `matchedCount` 判断条件更新是否成功。
- 单文档原子性可以避免很多不必要的事务。
