# 02. 并发问题与唯一索引幂等

真实后端里，很多事故不是数据库不会写，而是请求重复、并发提交、网络重试导致重复写。幂等设计就是为了解决这类问题。

## 1. 什么是幂等

幂等表示：同一个请求执行一次和执行多次，最终效果一致。

例如创建订单：

```text
用户点击下单
网络超时
前端重试
后端收到两次请求
```

如果没有幂等，可能创建两笔订单。

## 2. 唯一索引防重复

订单号唯一：

```javascript
db.orders.createIndex({ order_no: 1 }, { unique: true })
```

幂等键唯一：

```javascript
db.orders.createIndex({ idempotency_key: 1 }, { unique: true })
```

插入订单：

```javascript
db.orders.insertOne({
  order_no: "O202607050001",
  idempotency_key: "user1-submit-cart-abc123",
  user_id: ObjectId("..."),
  status: "pending_payment",
  created_at: new Date()
})
```

如果同一个幂等键再次插入，会报重复键错误。

## 3. 幂等键怎么设计

幂等键可以来自：

- 前端生成的请求 ID。
- 后端生成的业务操作 ID。
- 用户 ID + 购物车版本 + 提交时间窗口。
- 第三方回调流水号。
- 支付平台交易号。

好的幂等键应该：

- 能唯一标识一次业务操作。
- 可重复提交时保持不变。
- 不容易冲突。
- 有生命周期或可归档。

## 4. 重复请求的处理方式

如果插入订单遇到重复键：

```text
duplicate key: idempotency_key
```

不要直接返回 500。

应该查询已有订单：

```javascript
db.orders.findOne({ idempotency_key: "user1-submit-cart-abc123" })
```

如果订单存在，返回已有结果。

这就是“重复请求返回同一结果”。

## 5. 第三方回调幂等

支付回调可能重复发送。

支付流水号唯一：

```javascript
db.payments.createIndex(
  { third_party_trade_no: 1 },
  { unique: true }
)
```

或者：

```javascript
db.payment_callbacks.createIndex(
  { provider: 1, callback_id: 1 },
  { unique: true }
)
```

处理回调时，先记录回调事件，重复事件直接忽略或返回成功。

## 6. 幂等和状态条件

只靠幂等键不够，状态更新也要带条件。

支付订单：

```javascript
db.orders.updateOne(
  {
    _id: orderId,
    status: "pending_payment"
  },
  {
    $set: {
      status: "paid",
      paid_at: new Date(),
      updated_at: new Date()
    }
  }
)
```

如果订单已经是 `paid`，重复回调不会再次修改。

## 7. 幂等和事务

事务不能替代幂等。

事务保证“一次执行中的多个写操作要么都成功，要么都失败”。

幂等保证“同一个业务请求重复执行，不会产生重复效果”。

两者经常一起用。

例如创建订单事务中：

- 插入订单，带唯一 `idempotency_key`。
- 扣减库存。
- 写库存流水。

重复请求因为唯一索引被挡住。

## 8. 常见唯一索引

```javascript
db.users.createIndex({ email: 1 }, { unique: true })
db.orders.createIndex({ order_no: 1 }, { unique: true })
db.orders.createIndex({ idempotency_key: 1 }, { unique: true })
db.payments.createIndex({ payment_no: 1 }, { unique: true })
db.wallet_transactions.createIndex({ transaction_no: 1 }, { unique: true })
db.inventory_records.createIndex({ idempotency_key: 1 }, { unique: true })
```

唯一索引是并发安全的重要基础。

## 10. 本节小结

你需要记住：

- 幂等解决重复请求问题。
- 唯一索引是幂等的关键工具。
- 重复键错误不一定是系统异常，可能是重复请求。
- 事务不能替代幂等。
- 状态更新要带条件，防止重复状态流转。
