# 10. 阶段练习：交易与库存系统

本练习要求你设计并实现两个核心场景：钱包转账和库存扣减。重点是写清楚一致性边界，而不是只写几条 update。

## 1. 练习目标

完成：

- 钱包转账模型。
- 钱包转账事务。
- 交易流水唯一索引。
- 库存扣减模型。
- 库存扣减条件更新。
- 下单幂等键。
- 库存流水。
- 并发安全说明。

## 2. 钱包转账集合

```javascript
db.wallets.createIndex({ user_id: 1 }, { unique: true })
db.wallet_transactions.createIndex({ transaction_no: 1 }, { unique: true })
```

准备数据：

```javascript
db.wallets.insertMany([
  { user_id: ObjectId("660000000000000000000001"), balance: 10000, status: "active", updated_at: new Date() },
  { user_id: ObjectId("660000000000000000000002"), balance: 5000, status: "active", updated_at: new Date() }
])
```

要求：

- A 转给 B 1000 分。
- A 余额不足时全部失败。
- B 不存在时全部失败。
- 流水重复时不能重复转账。

要求：

## 9. 项目结构建议

可以按下面结构实现：

```text
cmd/api/main.go
internal/mongo/client.go
internal/wallet/repository.go
internal/wallet/service.go
internal/order/repository.go
internal/order/service.go
internal/inventory/repository.go
internal/testutil/mongo.go
```

不要把所有 MongoDB 操作写在 handler 里。handler 只负责 HTTP 入参和响应，业务一致性放在 service，具体数据库操作放在 repository。


交易流水：

```javascript
db.wallet_transactions.createIndex(

```

订单：

```javascript

  { user_id: 1, request_id: 1 },
  { unique: true }

```

```javascript
db.inventory_logs.createIndex(
  { order_id: 1, product_id: 1 },
  { unique: true }

```

这些索引不是性能装饰，而是并发安全的一部分。

## 11. 钱包转账事务要求

事务内步骤：

```text
1. 插入交易流水，request_id 唯一。
2. 扣减付款方余额，条件是 balance >= amount。
3. 增加收款方余额。
4. 更新交易流水为 success。
```

关键点：

- 重复 `request_id` 不能重复转账。
- 扣款条件必须检查余额。
- 余额不足要返回业务错误。
- 事务回调里不能发短信、HTTP 请求。

错误处理：

```text
DuplicateKey -> 查询已有交易并返回
ModifiedCount = 0 -> 余额不足或账户不存在
TransientTransactionError -> 允许事务机制重试
UnknownTransactionCommitResult -> 查询流水确认最终状态
```

## 12. 库存扣减要求

单商品库存扣减优先用单文档条件更新：

```javascript
db.products.updateOne(
  { _id: productId, stock: { $gte: qty } },
  { $inc: { stock: -qty } }
)
```

多商品订单可以使用事务：

```text
1. 插入订单 pending。
2. 逐个商品条件扣库存。
3. 写库存流水。
4. 更新订单 confirmed。
```

如果任意商品库存不足，事务回滚。

## 13. 并发测试示例

测试目标：100 个 goroutine 同时扣库存，库存不能小于 0。

伪代码：

```go
var wg sync.WaitGroup
successCh := make(chan bool, 100)

for i := 0; i < 100; i++ {
    wg.Add(1)
    go func() {
        defer wg.Done()
        ok, err := repo.Deduct(ctx, productID, 1)
        if err != nil {
            successCh <- false
            return
        }
        successCh <- ok
    }()
}

wg.Wait()
close(successCh)

success := 0
for ok := range successCh {
    if ok {
        success++
    }
}
```

断言：

```text
success == 初始库存
最终 stock == 0
没有重复流水
没有负库存
```
