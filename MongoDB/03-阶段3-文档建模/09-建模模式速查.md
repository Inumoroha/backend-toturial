# 09. 建模模式速查

MongoDB 官方和社区中有很多常见 schema design patterns。本节不是让你背模式名字，而是给你一个速查表：遇到某类问题时，可以想到哪些建模方案。

## 1. Embedded Pattern

把相关数据嵌入同一文档。

适合：

- 一起读取。
- 数量有限。
- 生命周期一致。

例子：

```javascript
{
  name: "Alice",
  profile: {
    city: "Shanghai"
  }
}
```

## 2. Reference Pattern

通过 ID 引用其他集合。

适合：

- 数据量大。
- 独立查询。
- 独立更新。
- 生命周期不同。

例子：

```javascript
{
  post_id: ObjectId("..."),
  user_id: ObjectId("..."),
  content: "comment"
}
```

## 3. Subset Pattern

主文档保存常用的一小部分数据，完整数据放在单独集合。

例如文章保存最近 3 条评论：

```javascript
{
  title: "文章",
  recent_comments: [
    { content: "good" }
  ]
}
```

全部评论在 `comments` 集合。

适合：

- 列表或详情页只展示前几条。
- 完整数据很多。

## 4. Extended Reference Pattern

保存引用 ID，同时冗余少量展示字段。

```javascript
{
  author_id: ObjectId("..."),
  author_snapshot: {
    display_name: "Alice",
    avatar: "https://example.com/a.png"
  }
}
```

适合：

- 高频展示。
- 不想每次额外查询。
- 冗余字段少且变化不频繁。

## 5. Computed Pattern

预计算统计结果。

```javascript
{
  title: "文章",
  stats: {
    views: 120,
    likes: 15,
    comments: 8
  }
}
```

适合：

- count 或聚合很频繁。
- 实时精确性要求可控。

风险：

- 统计可能不一致。
- 需要 `$inc`、校准或异步修复。

## 6. Bucket Pattern

把多条事件或时间序列数据放进一个桶文档。

```javascript
{
  device_id: "d1",
  bucket_start: ISODate("2026-07-05T10:00:00Z"),
  readings: [
    { ts: ISODate("2026-07-05T10:01:00Z"), value: 23.5 }
  ]
}
```

适合：

- 时间序列。
- 日志。
- 高频事件。

## 7. Outlier Pattern

大多数文档用简单模型，少数异常大文档单独处理。

```javascript
{
  user_type: "enterprise",
  has_external_addresses: true
}
```

地址拆集合。

适合：

- 大多数对象数据量小。
- 少数对象数据量极大。
- 不想为了极端情况让所有对象都走复杂模型。

## 8. Archive Pattern

热数据和历史数据分离。

```text
orders
orders_archive
```

适合：

- 历史数据很大。
- 最近数据访问频繁。
- 旧数据很少查。

## 9. Schema Version Pattern

文档保存 schema 版本。

```javascript
{
  schema_version: 2,
  name: "Alice"
}
```

适合：

- 业务快速演进。
- 老数据和新数据结构不同。
- 需要渐进迁移。

## 10. Approximation Pattern

近似计数，减少高频写入。

例如浏览量不是每次都写数据库，而是先缓存或批量累加。

适合：

- 浏览量。
- 曝光数。
- 点赞数等可接受短暂延迟的统计。

第 5、6 阶段会结合性能再展开。

## 11. 模式选择表

| 问题 | 可考虑模式 |
| --- | --- |
| 经常一起读取 | Embedded |
| 子数据无限增长 | Reference |
| 只展示前几条 | Subset |
| 引用对象只需少量字段 | Extended Reference |
| 统计频繁 | Computed |
| 时间序列或日志 | Bucket |
| 少数对象异常大 | Outlier |
| 历史数据很多 | Archive |
| Schema 演进 | Schema Version |
| 高频计数 | Approximation |

## 12. 本节练习

为下面场景选择模式：

1. 文章详情展示最近 3 条评论。
2. 文章列表展示作者昵称头像。
3. 商品详情展示评价总数和平均分。
4. 设备每秒上传温度数据。
5. 企业用户有几万个地址，普通用户只有 3 个。
6. 订单三年后很少查询。
7. 用户资料字段经历多次版本变化。
8. 视频播放量每秒大量增长。


## 13. 本节小结

你需要记住：

- 建模模式是工具箱，不是教条。
- 一个集合可以组合多个模式。
- 模式选择必须回到访问模式和数据规模。
- 模式会带来维护成本，不能只看查询方便。
