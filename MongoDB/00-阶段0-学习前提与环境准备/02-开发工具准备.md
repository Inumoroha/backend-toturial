# 02. 开发工具准备

这一节把后续学习需要的工具准备好。建议你不要只安装，必须做验证。能验证，才算环境搭好。

## 1. 工具清单

你至少需要：

| 工具 | 用途 | 必需程度 |
| --- | --- | --- |
| Go | 编写后端服务和 MongoDB 访问代码 | 必需 |
| Git | 管理代码和学习笔记 | 必需 |
| VS Code 或 GoLand | 编写代码 | 必需 |
| Docker Desktop | 本地启动 MongoDB | 强烈建议 |
| mongosh | MongoDB 命令行客户端 | 必需 |
| MongoDB Compass | 图形化查看数据库 | 强烈建议 |
| Postman 或 Bruno | 调试 HTTP API | 建议 |

你现在使用的是 Windows 环境，下面命令以 PowerShell 为主。

## 2. 检查 PowerShell

打开 PowerShell，执行：

```powershell
$PSVersionTable.PSVersion
```

能看到版本号即可。

如果后续命令出现中文乱码，优先尝试：

```powershell
chcp 65001
```

这会把当前终端切到 UTF-8 编码。

## 3. 安装和验证 Go

下载地址：

<https://go.dev/dl/>

安装完成后，重新打开 PowerShell：

```powershell
go version
```

期望看到类似输出：

```text
go version go1.xx.x windows/amd64
```

继续检查 Go 环境：

```powershell
go env GOPATH
go env GOMODCACHE
go env GOPROXY
```

如果你在国内网络环境下拉包较慢，可以设置 Go 代理：

```powershell
go env -w GOPROXY=https://goproxy.cn,direct
```

验证 Go 能正常编译：

```powershell
mkdir C:\Users\HP\Desktop\go-env-check
cd C:\Users\HP\Desktop\go-env-check
go mod init go-env-check
```

创建 `main.go`：

```go
package main

import "fmt"

func main() {
    fmt.Println("Go environment is ready")
}
```

运行：

```powershell
go run .
```

看到下面输出说明 Go 环境可用：

```text
Go environment is ready
```

## 4. 安装和验证 Git

下载地址：

<https://git-scm.com/downloads>

安装后验证：

```powershell
git --version
```

配置用户名和邮箱：

```powershell
git config --global user.name "your-name"
git config --global user.email "your-email@example.com"
```

查看配置：

```powershell
git config --global --list
```

建议你把整个 MongoDB 学习目录纳入 Git 管理：

```powershell
cd C:\Users\HP\Desktop\MongoDB
git init
git status
```

如果以后要提交学习笔记，可以使用：

```powershell
git add .
git commit -m "docs: add mongodb learning notes"
```

## 5. 安装 VS Code

下载地址：

<https://code.visualstudio.com/>

推荐扩展：

- Go：Go 官方扩展。
- Docker：查看容器和镜像。
- MongoDB for VS Code：可选，用于连接 MongoDB。
- Markdown All in One：写学习笔记更舒服。

在 PowerShell 中打开当前目录：

```powershell
cd C:\Users\HP\Desktop\MongoDB
code .
```

如果提示 `code` 命令不存在，在 VS Code 中打开命令面板，搜索：

```text
Shell Command: Install 'code' command in PATH
```

Windows 版 VS Code 通常安装时也可以选择加入 PATH。

## 6. 安装 Docker Desktop

下载地址：

<https://www.docker.com/products/docker-desktop/>

安装完成后，启动 Docker Desktop，然后验证：

```powershell
docker version
docker info
```

再运行一个最小容器：

```powershell
docker run --rm hello-world
```

看到 hello-world 的提示文本，说明 Docker 可用。

### 常见问题：Docker Desktop 启动失败

可能原因：

- WSL2 没启用。
- 虚拟化没启用。
- Docker Desktop 没完全启动。
- 公司网络或代理影响镜像拉取。

检查 WSL：

```powershell
wsl --status
```

如果 WSL 不可用，请按 Docker Desktop 的提示安装或更新 WSL2。

## 7. 安装 mongosh

mongosh 是 MongoDB 的官方命令行 Shell。

下载地址：

<https://www.mongodb.com/docs/mongodb-shell/install/>

安装后验证：

```powershell
mongosh --version
```

能看到版本号即可。

注意：旧教程里常见的 `mongo` shell 已经不是现在推荐的工具，后续学习统一使用 `mongosh`。

## 8. 安装 MongoDB Compass

MongoDB Compass 是官方图形化客户端。

下载地址：

<https://www.mongodb.com/products/tools/compass>

安装后暂时不用连接，等下一节本地 MongoDB 启动后再使用。

Compass 的作用：

- 查看数据库和集合。
- 查看文档结构。
- 执行简单查询。
- 创建索引。
- 观察 explain 结果。

学习早期不要完全依赖 Compass。你需要同时练习 `mongosh`，因为线上排障和脚本化操作经常靠命令行。

## 9. 安装 Postman 或 Bruno

后面写 Go Web API 时，需要 HTTP 调试工具。

二选一即可：

- Postman：<https://www.postman.com/downloads/>
- Bruno：<https://www.usebruno.com/downloads>

如果你喜欢轻量、本地文件化管理接口，Bruno 很适合；如果你已经熟悉 Postman，继续用 Postman 就行。

## 10. 本节验收清单

逐条执行：

```powershell
go version
git --version
docker version
docker run --rm hello-world
mongosh --version
```

你还应该能做到：

- 用 VS Code 打开 `C:\Users\HP\Desktop\MongoDB`。
- 知道 Go 代码以后放在哪里。
- 知道 MongoDB 会通过 Docker 启动。
- 知道 `mongosh` 是命令行客户端，Compass 是图形化客户端。

如果这里有任何一个工具验证失败，先不要进入下一节。环境问题拖到后面会更难排查。
