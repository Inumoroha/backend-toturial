# 05. Go 后端前置知识

本节不是完整 Go 教程，而是列出学习 MongoDB Go Driver 之前必须掌握的 Go 知识。你不需要成为 Go 高手后才学 MongoDB，但下面这些内容必须熟。

## 1. Go Module

Go 项目通常从 `go mod init` 开始。

示例：

```powershell
mkdir C:\Users\HP\Desktop\mongodb-go-playground
cd C:\Users\HP\Desktop\mongodb-go-playground
go mod init mongodb-go-playground
```

这会生成：

```text
go.mod
```

后面安装 MongoDB Go Driver 时，会使用：

```powershell
go get go.mongodb.org/mongo-driver/v2/mongo
```

再通过：

```powershell
go mod tidy
```

整理依赖。

你要理解：

- `go.mod` 记录模块名和依赖。
- `go.sum` 记录依赖校验信息。
- 不要手动乱改 `go.sum`。
- 每个后端项目通常有自己的 `go.mod`。

## 2. 包和目录

Go 中目录和 package 强相关。

常见结构：

```text
cmd/
  api/
    main.go
internal/
  config/
  database/
  user/
    model.go
    repository.go
    service.go
    handler.go
```

含义：

- `cmd/api/main.go`：程序入口。
- `internal/config`：配置读取。
- `internal/database`：数据库连接。
- `internal/user`：用户模块。

学习阶段可以简单一点：

```text
main.go
```

但进入项目实战后，需要逐步使用分层结构。

## 3. 结构体和 tag

MongoDB 文档会映射到 Go 结构体。

示例：

```go
type User struct {
    ID    string `bson:"_id,omitempty" json:"id"`
    Name  string `bson:"name" json:"name"`
    Email string `bson:"email" json:"email"`
    Age   int    `bson:"age" json:"age"`
}
```

你要理解：

- `bson` tag 控制和 MongoDB 文档字段的映射。
- `json` tag 控制 HTTP API 返回 JSON 时的字段名。
- `omitempty` 表示零值时可以省略。
- 数据库字段名和 API 字段名不一定必须相同，但学习阶段建议保持清晰一致。

后面会使用 MongoDB Driver 中的 `primitive.ObjectID` 或新版驱动对应的 BSON ObjectID 类型来表示 `_id`。

## 4. 指针和值

Go 中经常要判断字段是否传了值。

例如更新用户资料：

```go
type UpdateUserRequest struct {
    Name *string `json:"name"`
    Age  *int    `json:"age"`
}
```

为什么用指针？

因为这样可以区分：

- 用户没传 `age`。
- 用户传了 `age: 0`。

MongoDB 更新时，这个区别很重要。没传的字段不应该被更新成零值。

## 5. 错误处理

Go 不使用异常，通常显式返回 error：

```go
user, err := repo.FindByEmail(ctx, email)
if err != nil {
    return nil, err
}
```

后面学习 MongoDB 时，你会遇到：

- 没查到数据。
- 连接超时。
- 重复键错误。
- BSON 解码失败。
- context 被取消。

不要忽略错误：

```go
user, _ := repo.FindByEmail(ctx, email) // 不推荐
```

应该明确处理：

```go
user, err := repo.FindByEmail(ctx, email)
if err != nil {
    return nil, err
}
```

## 6. context.Context

`context.Context` 是 Go 后端开发非常重要的概念。

它常用于：

- 控制超时。
- 取消请求。
- 在调用链中传递请求范围的信息。

示例：

```go
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()
```

MongoDB 操作一定要带 context：

```go
result, err := collection.InsertOne(ctx, user)
```

为什么重要？

如果不设置超时，数据库卡住时，请求可能一直占用资源。真实服务中，这会导致连接堆积、接口超时、服务雪崩。

## 7. time.Time

后端数据通常要记录时间：

```go
type User struct {
    CreatedAt time.Time `bson:"created_at" json:"created_at"`
    UpdatedAt time.Time `bson:"updated_at" json:"updated_at"`
}
```

建议：

- 后端内部统一使用 UTC 或明确时区策略。
- 数据库中保存标准时间。
- 展示给用户时再转换成本地时区。

学习阶段先养成字段命名习惯：

```text
created_at
updated_at
deleted_at
```

## 8. interface 和依赖反转

Repository 可以定义成接口：

```go
type UserRepository interface {
    Create(ctx context.Context, user *User) error
    FindByEmail(ctx context.Context, email string) (*User, error)
}
```

Service 依赖接口：

```go
type UserService struct {
    repo UserRepository
}
```

好处：

- 方便测试。
- 业务层不关心 MongoDB 细节。
- 以后换存储或做 mock 更容易。

学习早期可以先写具体类型，等你理解 CRUD 后再引入接口。

## 9. 单元测试和集成测试

Go 测试文件命名：

```text
xxx_test.go
```

测试函数：

```go
func TestCreateUser(t *testing.T) {
    // ...
}
```

MongoDB 相关测试通常分两类：

### 单元测试

不连接真实 MongoDB，主要测试业务逻辑。

### 集成测试

连接真实 MongoDB，测试 Repository 是否真的能读写数据。

学习 MongoDB 时，Repository 层更推荐集成测试，因为 mock 数据库很容易掩盖真实 BSON、索引、查询错误。

## 10. 并发基础

Go 的并发基础：

- goroutine。
- channel。
- sync.Mutex。
- sync.WaitGroup。
- context cancel。

学习 MongoDB 时并发会出现在：

- 多请求同时写入。
- 库存扣减。
- 唯一索引防重复。
- 连接池复用。
- 批量写入。

现在不需要深入并发模型，但必须知道：并发安全不能只靠“先查再写”。

例如：

```text
请求 A：查询库存还有 1
请求 B：查询库存还有 1
请求 A：扣减库存
请求 B：扣减库存
```

如果没有原子更新或事务，就可能超卖。

## 12. 本节练习

创建一个 Go 项目，写一个用户结构体并打印 JSON：

```go
package main

import (
    "encoding/json"
    "fmt"
    "time"
)

type User struct {
    ID        string    `json:"id"`
    Name      string    `json:"name"`
    Email     string    `json:"email"`
    CreatedAt time.Time `json:"created_at"`
}

func main() {
    user := User{
        ID:        "u1001",
        Name:      "Alice",
        Email:     "alice@example.com",
        CreatedAt: time.Now(),
    }

    data, err := json.MarshalIndent(user, "", "  ")
    if err != nil {
        panic(err)
    }

    fmt.Println(string(data))
}
```

运行：

```powershell
go run .
```

能看到 JSON 输出，就说明最基础的 Go 编码环境没问题。
