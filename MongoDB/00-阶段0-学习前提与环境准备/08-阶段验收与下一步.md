# 08. 阶段验收与下一步

这是第 0 阶段的收尾文件。请不要只读完就算完成，一定要按清单实际操作。

## 1. 环境验收

在 PowerShell 中执行：

```powershell
go version
git --version
docker version
mongosh --version
```

你应该能看到所有工具的版本信息。

检查 MongoDB 容器：

```powershell
docker ps
```

如果没有看到 `mongodb-learning`，启动它：

```powershell
docker start mongodb-learning
```

如果提示没有这个容器，回到 `03-MongoDB本地环境搭建.md` 重新创建。

## 2. MongoDB 连接验收

执行：

```powershell
mongosh "mongodb://root:example@localhost:27017/?authSource=admin"
```

进入 `mongosh` 后执行：

```javascript
use mongodb_learning
db.stage0_check.insertOne({
  title: "stage 0 check",
  status: "passed",
  created_at: new Date()
})
db.stage0_check.find().pretty()
```

能看到插入的文档，说明命令行连接正常。

## 3. Compass 验收

打开 MongoDB Compass，连接：

```text
mongodb://root:example@localhost:27017/?authSource=admin
```

确认你能看到：

```text
mongodb_learning.stage0_check
```

在 Filter 中输入：

```json
{ "status": "passed" }
```

能查到数据，说明 Compass 连接正常。

## 6. 第 0 阶段练习任务

完成一个小练习：用户环境检查集合。

在 `mongodb_learning` 中创建 `users` 集合，插入至少 5 条数据：

```javascript
db.users.insertMany([
  {
    name: "Alice",
    email: "alice@example.com",
    age: 25,
    roles: ["user"],
    profile: { city: "Shanghai" },
    created_at: new Date()
  },
  {
    name: "Bob",
    email: "bob@example.com",
    age: 30,
    roles: ["user", "admin"],
    profile: { city: "Beijing" },
    created_at: new Date()
  }
])
```

你需要自己补足另外 3 条。

然后完成查询：

```javascript
db.users.find().pretty()
db.users.find({ age: { $gt: 20 } }).pretty()
db.users.find({ "profile.city": "Shanghai" }).pretty()
db.users.find({ roles: "admin" }).pretty()
db.users.find({}, { name: 1, email: 1, _id: 0 }).pretty()
db.users.find().sort({ age: -1 }).limit(3).pretty()
```

更新：

```javascript
db.users.updateOne(
  { email: "alice@example.com" },
  { $set: { age: 26, updated_at: new Date() } }
)
```

删除一条你自己创建的测试用户：

```javascript
db.users.deleteOne({ email: "your-test-user@example.com" })
```

## 8. 常见未通过情况

### Docker 启动不了

检查：

```powershell
docker version
wsl --status
```

确保 Docker Desktop 正在运行。

### MongoDB 端口冲突

检查：

```powershell
docker ps
```

如果 `27017` 被占用，可以改用 `27018`：

```text
mongodb://root:example@localhost:27018/?authSource=admin
```

### 认证失败

检查 URI：

```text
mongodb://root:example@localhost:27017/?authSource=admin
```

重点看：

- 用户名。
- 密码。
- 端口。
- `authSource=admin`。

### Compass 能连但 mongosh 不能连

通常是：

- mongosh 没安装好。
- PATH 没生效。
- URI 复制时多了空格。
- PowerShell 引号问题。

可以重新打开 PowerShell 再试。
