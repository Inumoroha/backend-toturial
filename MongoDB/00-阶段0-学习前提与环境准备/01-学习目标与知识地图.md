# 01. 学习目标与知识地图

第 0 阶段先帮你建立一张地图：MongoDB 不是孤立知识，它会和 Go、Web API、数据建模、索引、部署、监控、测试一起出现在真实后端项目里。

## 1. 你要成为的不是“会 MongoDB 语法的人”

Go 后端工程师学习 MongoDB，最终应该能做到：

- 根据业务场景设计集合和文档结构。
- 用 Go Driver 编写可靠的数据访问代码。
- 正确处理连接池、超时、错误和上下文取消。
- 根据查询条件设计索引。
- 能用 `explain` 判断查询是否走索引。
- 能理解单文档原子性和多文档事务的边界。
- 能在复制集、分片、备份恢复这些线上场景中做正确判断。

简单说，你要从“会写命令”走向“会设计和维护系统”。

## 2. MongoDB 在后端系统中的位置

一个常见 Go 后端系统大致长这样：

```text
Client
  |
HTTP API / gRPC API
  |
Handler / Controller
  |
Service
  |
Repository / DAO
  |
MongoDB / Redis / MySQL / MQ / Object Storage
```

MongoDB 通常位于数据存储层。它不是直接暴露给用户的工具，而是被后端服务封装起来。

在项目里，你通常不会在 Handler 里面直接写 MongoDB 查询，而是这样分层：

```text
handler.go       负责 HTTP 参数解析、响应返回
service.go       负责业务规则、事务、权限和幂等
repository.go    负责 MongoDB 查询、更新和索引相关逻辑
model.go         定义数据库模型
dto.go           定义接口输入输出结构
```

这个分层习惯非常重要。后面进入 Go Driver 学习时，会反复用到。

## 3. MongoDB 和 MySQL 的思维差异

如果你熟悉 MySQL，学习 MongoDB 时最容易犯的错是照搬表设计。

### MySQL 常见思路

```text
users
orders
order_items
products
payments
```

这些表通过外键或业务 ID 建立关系。查询时经常通过 join 组合数据。

### MongoDB 常见思路

MongoDB 不会要求你先拆成很多表。你要先问：

- 业务最常见的查询是什么？
- 这些数据是否总是一起读取？
- 子数据是否会无限增长？
- 数据是否需要独立更新？
- 数据是否有不同生命周期？

例如订单文档可能长这样：

```json
{
  "_id": "order_id",
  "user_id": "user_id",
  "status": "paid",
  "items": [
    {
      "product_id": "p1001",
      "name": "Go Web 实战课",
      "price": 9900,
      "quantity": 1
    }
  ],
  "total_amount": 9900,
  "created_at": "2026-07-05T10:00:00Z"
}
```

订单明细被嵌入到订单里，因为查询订单详情时通常就需要商品快照。

但评论列表不一定适合全部嵌入文章文档，因为热门文章的评论可能无限增长。

## 4. 学习 MongoDB 的核心主线

你可以把整个学习过程理解为 6 条线：

### 第 1 条线：文档与查询

你需要学会：

- 什么是 document。
- 什么是 collection。
- 如何插入、查询、更新、删除。
- 如何查询嵌套字段和数组字段。

### 第 2 条线：Go Driver

你需要学会：

- 如何连接 MongoDB。
- 如何用 `context.Context` 控制超时。
- 如何用结构体映射 BSON。
- 如何封装 Repository。
- 如何处理 cursor 和错误。

### 第 3 条线：数据建模

你需要学会：

- 什么时候嵌入。
- 什么时候引用。
- 如何避免无限增长数组。
- 如何为了查询效率做冗余。
- 如何设计数据生命周期。

### 第 4 条线：索引与性能

你需要学会：

- 什么查询需要索引。
- 如何设计复合索引。
- 如何使用 `explain`。
- 如何识别慢查询。
- 如何避免深分页问题。

### 第 5 条线：可靠性

你需要学会：

- 单文档原子性。
- 多文档事务。
- 读关注和写关注。
- 复制集。
- 备份恢复。

### 第 6 条线：规模化

你需要学会：

- 分片。
- 分片键。
- 热点写入。
- 批量写入。
- 容量估算。
- 归档策略。

## 5. 第 0 阶段你暂时不需要掌握什么

现在不要急着学这些：

- 分片集群部署细节。
- 高级聚合表达式。
- 复杂事务调优。
- Atlas 高级权限。
- Change Streams 生产级消费。
- 复杂监控体系。

这些内容后面会学。第 0 阶段只需要完成“环境可用、概念有轮廓、知道自己要补什么”。

## 6. 本阶段的最终产出

你需要在本机完成下面这条链路：

```text
安装工具
  -> 启动 MongoDB
  -> 用 mongosh 连接
  -> 用 Compass 连接
  -> 插入一条测试数据
  -> 查询这条数据
  -> 用 Go 环境确认后续可以开发
```

完成这条链路，你就可以进入第 1 阶段。
