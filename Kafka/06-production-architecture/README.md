# 第 6 阶段：生产运维与架构设计

这一阶段把视角从“会写代码”扩大到“能设计和维护 Kafka 业务链路”。Go 后端工程师不一定要成为 Kafka 运维专家，但必须能设计合理 topic、评估容量、理解安全和故障处理。

## 本阶段目标

完成后你应该能做到：

- 设计 topic、partition、retention、replication factor。
- 做基础容量估算。
- 理解 TLS、SASL、ACL 的作用。
- 排查常见生产故障。
- 设计事件驱动架构。
- 理解 schema 演进。
