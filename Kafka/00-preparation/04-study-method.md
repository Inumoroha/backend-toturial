# 04 Kafka 的正确学习方法

Kafka 不是只靠看文章就能学会的技术。最好的学习方法是：读一个概念，跑一个实验，制造一个问题，再把问题修好。

## 每个主题都按四步学习

### 第一步：知道它解决什么问题

例如 partition 不是凭空出现的概念，它解决的是：

- 单台机器吞吐有限。
- 单个 consumer 处理能力有限。
- 数据需要并行写入和读取。

### 第二步：看最小实验

比如学习 partition 时：

1. 创建 3 个 partition 的 topic。
2. 启动 1 个 consumer。
3. 启动 2 个 consumer。
4. 启动 4 个 consumer。
5. 观察 partition 分配变化。

### 第三步：故意破坏

学习 Kafka 时要主动制造故障：

- 关掉 consumer。
- 让 consumer 处理变慢。
- 消费成功但不提交 offset。
- 提交 offset 后再让业务失败。
- 给 producer 设置很小的超时时间。

这些实验会帮你真正理解重复消费、消息丢失、lag、rebalance。

### 第四步：写总结

每次实验记录：

- 实验目标。
- 命令或代码。
- 观察到的现象。
- 为什么会这样。
- 真实项目中怎么处理。

## 推荐笔记模板

```markdown
# 实验名称

## 目标

## 环境

## 步骤

## 观察结果

## 原因分析

## Go 后端项目里的处理方式

## 仍然不懂的问题
```

## 不建议的学习方式

- 一上来就背所有参数。
- 一上来就看源码。
- 只用命令行，不写 Go 代码。
- 只写 Go demo，不理解 offset 和 consumer group。
- 不做失败实验。

## 阶段检查

如果你能完成下面的事，就可以进入第 1 阶段：

- 本地 Kafka 能稳定启动。
- 能创建 topic。
- 能发送和消费消息。
- 能查看 consumer group 的 offset 和 lag。
- 能解释为什么 Kafka consumer 处理失败后不能随便提交 offset。
