# Go 后端工程师数据结构实践教程

这套教程面向想做 Go 后端开发的学习者。它不按“算法百科”的方式堆概念，而是围绕真实服务中的用户、订单、权限、库存、日志、缓存、短链接、任务队列等场景，说明数据结构怎么选、怎么写、哪里容易踩坑。

建议学习顺序：

1. 先读 [学习路线](./00-学习路线/01-Go后端数据结构学习路线.md)，知道每一阶段要解决什么问题。
2. 再读 [阶段学习计划与项目产出](./00-学习路线/02-阶段学习计划与项目产出.md)，按周安排练习。
3. 按目录顺序阅读教程，每篇只做一个主题。
4. 每篇代码都可以单独复制到一个临时 Go 项目里运行。
5. 学完后对照 [阶段练习与验收清单](./04-工程综合/16-阶段练习与验收清单.md)，检查自己是否真的能写。
6. 上生产思维前，读 [生产落地检查清单](./04-工程综合/17-生产落地检查清单.md)，确认风险边界。
7. 最后做短链接综合项目，把 map、队列、堆、树、图、缓存和并发安全串起来。

目录结构：

- `00-学习路线`：整体学习路线和阶段目标。
- `01-基础容器`：Go 后端最常用的切片、map、结构体组织方式。
- `02-线性结构`：栈、队列、链表，以及 LRU 这种真实常见组合结构。
- `03-检索与索引`：排序、二分、堆、树、Trie、图，重点放在后端查询和调度。
- `04-工程综合`：Bloom Filter、并发安全结构、综合项目落地和验收清单。

学习方式建议：

- 第一遍先跑通示例，不急着改复杂。
- 第二遍把示例里的用户、订单、短链接换成自己的业务对象。
- 第三遍按验收清单补错误处理、边界判断、并发保护和持久化方案说明。
- 每完成一个阶段，都写一小段复盘：这个结构适合什么、不适合什么、生产中要靠什么兜底。

运行示例前，请确认已安装 Go：

```powershell
go version
```

```bash
go version
```

真实项目中请始终记住：

```text
数据结构不是为了炫技，而是为了让业务代码更清晰、更稳定、更容易回滚。
```
