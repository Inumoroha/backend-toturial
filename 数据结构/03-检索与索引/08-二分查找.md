# 8. 二分查找

本节目标：你能在有序数据中用二分查找定位配置、版本和时间区间。

简短引入：二分查找适合“数据已经排好序”的场景。后端常见例子是按金额找费率档位、按时间找生效配置、按版本号找迁移脚本。

## 一、为什么需要它

顺序扫描像从第一页翻到最后一页，二分像直接翻到中间判断方向。数据越大，差距越明显。但前提是数据必须有序。

```text
二分查找的前提是有序；如果数据没排序，结果没有意义。
```

## 二、基本用法

```powershell

go mod init ds-binary-demo
notepad main.go
go run .
```

```bash
mkdir ds-binary-demo && cd ds-binary-demo

```

`main.go`：

```go
package main

import (
	"fmt"
	"sort"
)

func main() {
	limits := []int{100, 500, 1000, 5000}
	amount := 750

	idx := sort.Search(len(limits), func(i int) bool {
		return limits[i] >= amount
	})
	if idx < len(limits) {
		fmt.Println("matched limit:", limits[idx])
	} else {
		fmt.Println("no limit matched")
	}
}
```

## 三、关键参数/语法/代码结构

- `sort.Search(n, fn)` 会在 `[0, n)` 中找第一个让 `fn(i)` 为 true 的位置。
- 条件函数必须从 false 单调变成 true。
- 返回值可能等于 `len(slice)`，表示没找到。
- 二分查找常用于边界，不只是查精确相等。

## 四、真实后端场景示例

订单金额对应不同手续费档位：

```go
type FeeRule struct {
	MinAmount int
	Rate      float64
}

func MatchFeeRule(rules []FeeRule, amount int) (FeeRule, bool) {
	idx := sort.Search(len(rules), func(i int) bool {
		return rules[i].MinAmount > amount
	})
	if idx == 0 {
		return FeeRule{}, false
	}
	return rules[idx-1], true
}
```

这里找的是“不超过当前金额的最大档位”。真实项目中规则通常来自数据库或配置中心，加载后要先排序并验证是否有重叠。

## 五、注意点

配置变更要可回滚。费率、权限、灰度策略这类规则上线前应有测试数据，发布后保留旧版本，出问题能快速切回。

## 六、常见误区

- 误区一：忘记排序就二分。
- 误区二：没处理 `idx == len(slice)`。
- 误区三：条件函数不单调，比如一会 true 一会 false。
- 误区四：边界理解错，把 `idx` 和 `idx-1` 用反。

## 七、本节达标标准

- 能用 `sort.Search` 写二分查找。
- 能处理没找到和边界条件。
- 能把二分用于金额档位、时间版本、灰度配置。
- 能说明配置类数据上线时为什么要保留可回滚方案。
