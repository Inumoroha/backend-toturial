# 12. 图

本节目标：你能用图表达依赖关系，并检测迁移、权限或任务依赖中是否存在环。

简短引入：树是一种特殊的图，而图能表达更复杂的关系。后端项目里，数据库迁移依赖、服务调用链、权限依赖、任务编排都可以用图来看。

## 一、为什么需要它

如果任务 A 必须先于任务 B，B 又必须先于 C，这就是依赖关系。最怕的是 A 依赖 B，B 又依赖 A，系统就不知道谁先执行。图可以帮助我们发现这种环。

```text
有依赖关系的任务上线前必须能检测环，否则生产执行顺序可能卡死。
```

`main.go`：

```go
package main

import "fmt"

func HasCycle(graph map[string][]string) bool {
	visiting := map[string]bool{}
	visited := map[string]bool{}

	var dfs func(string) bool
	dfs = func(node string) bool {
		if visiting[node] {
			return true
		}
		if visited[node] {
			return false
		}
		visiting[node] = true
		for _, next := range graph[node] {
			if dfs(next) {
				return true
			}
		}
		visiting[node] = false
		visited[node] = true
		return false
	}

	for node := range graph {
		if dfs(node) {
			return true
		}
	}
	return false
}

func main() {
	graph := map[string][]string{
		"create_users": {"create_orders"},
		"create_orders": {"add_order_index"},
	}
	fmt.Println("has cycle:", HasCycle(graph))
}
```

## 三、关键参数/语法/代码结构

- `graph[a] = []string{b}` 表示 a 依赖或指向 b，方向要在项目中统一。
- `visiting` 表示当前递归路径上的节点。
- 再次遇到 `visiting` 中的节点，说明有环。
- `visited` 表示已经确认无环的节点，避免重复计算。

## 四、真实后端场景示例

数据库迁移脚本可能有依赖顺序：先建用户表，再建订单表，再给订单表加索引。上线前可以把脚本依赖建成图，检查是否成环。

真实项目中迁移要使用工具管理版本，不要在生产手写临时 SQL。每个迁移都应有可回滚方案，至少要知道失败后如何停止和修复。

## 五、注意点

图的方向很容易写反。写代码前先在注释里明确：边表示“依赖谁”还是“执行后到谁”。方向不统一，拓扑排序和环检测都会出错。

## 六、常见误区

- 误区一：只看单条链路，不检查全图。
- 误区二：图方向混乱，结果看起来能跑但业务含义错了。
- 误区三：迁移脚本没有版本管理和回滚说明。
- 误区四：服务调用图中出现循环依赖，导致发布和故障排查困难。

## 七、本节达标标准

- 能用 map 表达有向图。
- 能用 DFS 检测依赖环。
- 能把图用于迁移、任务、权限和服务依赖分析。
- 能说明迁移工具、回滚方案和生产安全的重要性。
