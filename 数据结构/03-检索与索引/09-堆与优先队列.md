# 9. 堆与优先队列

本节目标：你能用堆实现延迟任务和优先级任务调度。

简短引入：普通队列按进入顺序处理，优先队列按优先级处理。后端里的延迟任务、重试任务、风控任务常常不是谁先来谁先做，而是谁更急谁先做。

## 一、为什么需要它

比如订单 30 分钟未支付要关闭，失败消息 5 秒后重试，高风险日志要优先处理。堆可以快速拿到“当前最应该处理”的任务。

```text
优先队列只解决调度顺序，不解决任务持久化和失败重试记录。
```

`main.go`：

```go
package main

import (
	"container/heap"
	"fmt"
)

type Task struct {
	Name  string
	RunAt int64
}

type TaskHeap []Task

func (h TaskHeap) Len() int           { return len(h) }
func (h TaskHeap) Less(i, j int) bool { return h[i].RunAt < h[j].RunAt }
func (h TaskHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *TaskHeap) Push(x any)        { *h = append(*h, x.(Task)) }
func (h *TaskHeap) Pop() any {
	old := *h
	n := len(old)
	x := old[n-1]
	*h = old[:n-1]
	return x
}

func main() {
	h := &TaskHeap{}
	heap.Init(h)
	heap.Push(h, Task{Name: "close unpaid order", RunAt: 30})
	heap.Push(h, Task{Name: "retry message", RunAt: 5})
	fmt.Println(heap.Pop(h))
}
```

## 三、关键参数/语法/代码结构

- `Less` 决定优先级。这里 `RunAt` 越小越早执行。
- `heap.Push` 和 `heap.Pop` 必须配合 `heap.Init` 使用。
- `Pop` 取出的是当前最小元素，因为 `Less` 写成了小顶堆。
- 堆适合频繁取最小或最大，不适合按任意字段查询。

## 四、真实后端场景示例

延迟关闭订单时，可以把 `RunAt` 设置为订单过期时间。worker 每次看堆顶，如果还没到时间就等待；到了就取出执行。生产中任务应存在数据库或消息系统里，内存堆只是调度加速。

## 五、注意点

重试任务要限制次数，并记录最后错误。否则一个永久失败的任务会反复占用资源。延迟任务还要考虑服务重启后的恢复。

## 六、常见误区

- 误区一：把堆当普通排序列表用。堆只保证堆顶是最优先，不保证整体有序。
- 误区二：任务只放内存。服务重启后延迟任务全部丢失。
- 误区三：重试没有上限。故障时会放大压力。
- 误区四：执行任务不做幂等。重复执行可能造成重复关闭或重复通知。
