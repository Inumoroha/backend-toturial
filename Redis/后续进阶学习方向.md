# Redis 后续进阶学习方向

完成当前 Redis 学习路线后，说明你已经具备了 Redis 的核心使用能力。

后续进阶不建议继续只背命令，而应该围绕真实系统问题继续深入：高并发、数据一致性、可观测性、架构取舍、源码原理和生产治理。

---

## 一、把第 13 阶段项目真正做完

优先级最高的是把短链接项目完整实现。

建议完成：

- `POST /links` 创建短链接。
- `GET /:code` 跳转短链接。
- `GET /links/:code` 查询详情。
- `PATCH /links/:code` 更新短链接。
- `GET /links/:code/stats` 查询统计。
- Redis 缓存短链接详情。
- Redis 空值缓存防止缓存穿透。
- Redis 记录访问计数。
- Redis 创建接口限流。
- 更新短链接后删除缓存。
- Redis 故障时能降级。

完成标准：

- 能独立写出代码。
- 能解释每个 Redis key 的用途和 TTL。
- 能模拟 Redis 不可用，并说明接口如何处理。
- 能说明哪些数据可以丢，哪些数据必须落数据库。

---

## 二、缓存一致性专题

缓存一致性是后端开发中长期会遇到的问题。

建议继续学习：

- Cache-Aside 的边界。
- 先更新数据库再删除缓存。
- 延迟双删。
- binlog / CDC 驱动缓存失效。
- 消息队列异步删除缓存。
- 本地缓存和 Redis 多级缓存一致性。
- 热点 key 更新时的并发读写问题。

练习项目：

- 商品详情缓存。
- 用户资料缓存。
- 文章详情缓存。
- 配置中心本地缓存。

---

## 三、高并发与热点 key 治理

当业务访问量变大，Redis 常见瓶颈会从“会不会用”变成“扛不扛得住”。

建议继续学习：

- 热点 key 识别。
- 热点 key 拆分。
- 本地缓存削峰。
- 请求合并。
- SingleFlight 防止重复回源。
- 缓存预热。
- 缓存过期时间抖动。
- 大 key 拆分。
- 慢命令治理。

练习项目：

- 秒杀商品详情页。
- 热门文章排行榜。
- 活动页配置缓存。

---

## 四、分布式锁深入

基础阶段已经学过分布式锁，但真实项目里还要继续理解它的边界。

建议继续学习：

- 锁过期时间如何设置。
- 业务未执行完锁过期怎么办。
- 看门狗续期机制。
- 锁误删问题。
- Lua 解锁。
- 锁粒度设计。
- 分布式锁和数据库事务的关系。
- Redlock 的争议和适用边界。

练习项目：

- 防止重复提交订单。
- 定时任务单实例执行。
- 用户积分变更串行化。

---

## 五、限流与风控能力

限流不只是 Redis 命令练习，它是系统保护能力。

建议继续学习：

- 固定窗口限流。
- 滑动窗口限流。
- 令牌桶。
- 漏桶。
- Lua + ZSet 实现滑动窗口。
- 用户维度、IP 维度、接口维度限流。
- 黑名单和灰名单。
- 限流后的降级响应设计。

练习项目：

- 登录限流。
- 短信验证码限流。
- API 网关限流。
- 短链接创建限流升级版。

---

## 六、Redis Stream 与异步系统

Redis Stream 可以作为轻量消息流使用。

建议继续学习：

- `XADD` 写入消息。
- `XREAD` 读取消息。
- Consumer Group。
- `XACK` 确认消息。
- Pending List。
- 消息重试。
- 死信处理。
- Stream 裁剪。
- Stream 与 Kafka / RabbitMQ 的区别。

练习项目：

- 短链接访问日志异步消费。
- 异步统计访问来源。
- 简单订单事件流。
- 延迟任务的替代方案对比。

---

## 七、排行榜与实时统计

ZSet 是 Redis 很有代表性的高级结构。

建议继续学习：

- `ZADD`、`ZINCRBY`、`ZREVRANGE`。
- 日榜、周榜、月榜设计。
- 榜单合并。
- 分数衰减。
- 排名分页。
- 大榜单内存控制。
- 排行榜和数据库落库策略。

练习项目：

- 热门短链接 Top 100。
- 文章热榜。
- 用户积分排行榜。
- 游戏分数榜。

## 十、Redis 源码与底层原理

如果想进一步变强，可以逐步读 Redis 原理。

建议继续学习：

- Redis 单线程事件循环。
- I/O 多路复用。
- SDS 字符串。
- Dict 哈希表。
- Ziplist / Listpack。
- Quicklist。
- Skiplist。
- 过期 key 删除策略。
- 内存淘汰策略。
- AOF 重写。
- RDB fork 与 COW。

学习目标不是立刻读完整源码，而是能解释常见现象背后的原因。

例如：

- 为什么大 key 删除会卡。
- 为什么 fork 时内存会抖。
- 为什么 `KEYS` 危险。
- 为什么 Pipeline 能提升吞吐。
- 为什么 Lua 太慢会阻塞 Redis。

## 十一、推荐进阶项目

建议按下面顺序做项目：

1. 短链接缓存与限流服务。
2. 文章系统缓存与排行榜。
3. 秒杀库存预扣与限流。
4. 登录风控与验证码限流。
5. Feed 流点赞、收藏、计数系统。
6. Redis Stream 异步访问日志系统。
7. 多级缓存商品详情服务。
8. Redis Cluster 高可用实验环境。

每个项目都要写：

- key 设计文档。
- TTL 设计。
- 降级策略。
- 数据一致性说明。
- 压测结果。
- 故障演练记录。

## 十二、学习完成判断标准

如果你能回答下面问题，说明 Redis 已经进入比较扎实的阶段：

1. 什么数据适合放 Redis，什么数据不适合？
2. Redis 故障时哪些接口可以降级？
3. 缓存和数据库不一致时如何定位？
4. 如何发现热点 key 和大 key？
5. 如何设计一个不会永久封禁用户的限流 key？
6. Redis Cluster 下为什么有些多 key 操作会失败？
7. AOF、RDB 和备份有什么区别？
8. Lua 为什么要控制执行时间？
9. Pipeline 解决什么问题，不解决什么问题？
10. 如果 Redis 命中率突然下降，应该查什么？

## 十三、建议学习节奏

建议分三轮进阶：

第一轮：项目落地

- 完成短链接项目。
- 完成文章缓存项目。
- 完成限流和排行榜练习。

第二轮：生产能力

- 搭监控。
- 做故障演练。
- 做高可用实验。
- 写运行手册。

第三轮：原理深入

- 学 Redis 数据结构实现。
- 学事件循环和持久化原理。
- 学 Cluster 和复制细节。
- 结合线上案例复盘。

---

## 十四、最后建议

Redis 进阶不要追求“学完所有命令”。

真正重要的是：

- 能把 Redis 放在正确的位置。
- 能知道 Redis 能解决什么问题。
- 能知道 Redis 不能承担什么职责。
- 能在故障时保护业务。
- 能在高并发下做取舍。

如果只能选一个后续动作，优先把短链接项目完整做出来。
