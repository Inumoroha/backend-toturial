# 4. 危险命令限制与配置文件管理

Redis 中有一些命令非常强大，也非常危险。

生产环境要限制危险命令，并把 Redis 配置纳入工程化管理，而不是靠手工修改。

学完这一节后，你应该能够：

- 识别常见危险命令。
- 使用 ACL 限制危险命令。
- 理解命令重命名的作用和局限。
- 管理 Redis 配置文件。
- 整理 Redis 运行配置说明。

---

## 一、常见危险命令

危险命令包括：

```text
FLUSHALL
FLUSHDB
KEYS
CONFIG
SHUTDOWN
SAVE
DEBUG
MODULE
EVAL
SCRIPT
```

不同命令危险点不同：

- 清空数据。
- 阻塞 Redis。
- 修改配置。
- 执行脚本。
- 关闭服务。
- 加载模块。

生产中要按角色限制。

---

## 二、FLUSHALL 和 FLUSHDB

```redis
FLUSHALL
FLUSHDB
```

风险：

- 清空所有库或当前库。
- 误操作影响巨大。

业务应用绝不应该有这些权限。

管理员也应该谨慎使用，最好通过审批和审计。

---

## 三、KEYS

```redis
KEYS *
```

风险：

- 全库扫描。
- key 多时阻塞 Redis。

生产排查应该使用：

```redis
SCAN
```

业务应用不应该使用 `KEYS`。

---

## 四、CONFIG

```redis
CONFIG SET ...
CONFIG GET ...
```

风险：

- 修改运行配置。
- 暴露敏感信息。
- 影响持久化、安全和性能。

业务应用不应该有 `CONFIG` 权限。

配置变更应该通过配置文件和发布流程。

---

## 五、SAVE 和 SHUTDOWN

`SAVE`：

```redis
SAVE
```

会阻塞 Redis 主进程生成 RDB。

`SHUTDOWN`：

```redis
SHUTDOWN
```

会关闭 Redis。

这类命令只应由受控运维流程使用。

---

## 六、用 ACL 限制命令

示例：

```redis
ACL SETUSER app on >app-password ~app:* +@read +@write -flushall -flushdb -keys -config -shutdown -save -debug -module
```

含义：

- 允许常规读写。
- 禁止危险命令。
- 只允许访问 `app:*` key。

建议为每个服务创建独立用户。

---

## 七、命令重命名

Redis 支持命令重命名：

```conf
rename-command FLUSHALL ""
rename-command FLUSHDB ""
rename-command CONFIG ""
```

空字符串表示禁用命令。

也可以改名：

```conf
rename-command FLUSHALL very-dangerous-flushall
```

但 Redis 新版本和 ACL 更推荐使用权限控制。

命令重命名可能影响运维工具兼容性。

---

## 八、配置文件管理

Redis 配置应该文件化。

例如：

```text
redis/
  redis.conf
  users.acl
  docker-compose.yml
  README.md
```

`redis.conf` 中管理：

- bind。
- protected-mode。
- requirepass 或 aclfile。
- maxmemory。
- maxmemory-policy。
- appendonly。
- slowlog。
- rename-command。

不要在生产中长期依赖手工 `CONFIG SET`。

---

## 九、配置变更流程

建议流程：

```text
1. 修改配置文件。
2. 代码评审。
3. 测试环境验证。
4. 记录变更影响。
5. 低峰发布。
6. 观察监控。
7. 保留回滚方案。
```

Redis 配置变更可能影响全业务。

要像应用发布一样认真。

## 十一、常见错误

### 1. 业务用户有管理员权限

误操作风险极大。

### 2. 生产使用 KEYS 排查

可能阻塞 Redis。

### 3. 手工 CONFIG SET 后没有记录

重启后配置丢失，或无人知道差异。

### 4. 禁用命令后运维工具失效

命令限制要评估兼容性。

### 5. 没有配置说明文档

故障时没人知道实例用途和策略。

## 十三、本节小结

这一节你学习了危险命令限制和配置管理。

你需要记住：

- `FLUSHALL`、`FLUSHDB`、`KEYS`、`CONFIG`、`SHUTDOWN` 等命令生产要限制。
- ACL 是限制命令和 key 权限的重要方式。
- 命令重命名可以作为补充，但要注意兼容性。
- Redis 配置应文件化、版本化、可审计。
- 每个 Redis 实例都应该有运行配置说明。
