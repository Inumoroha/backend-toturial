# 7. 备份、恢复与演练流程

Redis 支持 RDB 和 AOF 持久化，但持久化不等于备份。

备份要能抵抗误删、磁盘损坏、机器故障和人为事故。更重要的是，备份必须能恢复。

学完这一节后，你应该能够：

- 区分持久化和备份。
- 设计 Redis 备份策略。
- 保存和管理 RDB/AOF 文件。
- 设计恢复流程。
- 做恢复演练。

---

## 一、持久化不是备份

持久化文件通常在 Redis 本机磁盘上。

如果发生：

- 机器损坏。
- 磁盘损坏。
- 数据目录误删。
- `FLUSHALL` 后持久化覆盖。
- 勒索或入侵。

本机持久化可能救不了你。

备份应该保存到独立位置。

---

## 二、备份什么

可备份：

- RDB 文件。
- AOF 文件。
- Redis 配置文件。
- ACL 文件。
- Docker Compose 文件。
- 运行说明文档。

只有数据文件不够。

没有配置和密码策略，也很难完整恢复服务。

---

## 三、备份频率

备份频率取决于：

- 数据重要程度。
- 允许丢失数据量。
- 数据变化频率。
- 存储成本。
- 恢复目标。

例如：

```text
纯缓存：每天备份甚至不备份也可能接受
会话状态：小时级备份
重要队列：更频繁，并考虑数据库或 MQ 替代
```

核心业务数据不应该只靠 Redis 备份。

---

## 四、备份保存位置

建议：

- 独立机器。
- 对象存储。
- 备份系统。
- 跨可用区。
- 加密存储。

不要只把备份放在 Redis 同一台机器上。

也要设置保留周期：

```text
最近 7 天每日备份
最近 4 周每周备份
最近 3 个月每月备份
```

---

## 五、备份 RDB

手动生成：

```redis
BGSAVE
```

确认成功：

```redis
INFO persistence
```

复制文件：

```bash
cp /data/dump.rdb /backup/redis/dump-20260705.rdb
```

生产中备份脚本要：

- 检查 BGSAVE 状态。
- 校验文件存在。
- 上传远程存储。
- 记录日志。
- 告警失败。

---

## 六、备份 AOF

AOF 开启时，要备份相关 AOF 文件。

Redis 版本不同，AOF 可能是单文件，也可能是目录结构。

备份前要确认：

- AOF 文件路径。
- 是否正在重写。
- 文件是否完整。

如果有条件，可以优先使用 RDB 做离线备份，AOF 用于更细恢复。

---

## 七、恢复流程

恢复 RDB 的基本思路：

```text
1. 停止 Redis。
2. 备份当前数据目录。
3. 放入要恢复的 dump.rdb。
4. 检查 redis.conf 中 dir 和 dbfilename。
5. 启动 Redis。
6. 验证关键 key。
7. 恢复应用流量。
```

不要直接覆盖生产数据目录。

先保留现场。

---

## 八、恢复演练

演练比备份更重要。

演练内容：

- 从备份文件启动一个临时 Redis。
- 验证数据能加载。
- 检查关键 key。
- 记录恢复耗时。
- 记录操作步骤。
- 发现权限、路径、版本问题。

演练环境不要连生产应用。

---

## 九、RPO 和 RTO

RPO：

```text
Recovery Point Objective，最多允许丢多少数据。
```

RTO：

```text
Recovery Time Objective，最多允许多久恢复服务。
```

例如：

```text
RPO = 5 分钟
RTO = 30 分钟
```

备份和恢复方案要围绕 RPO/RTO 设计。

---

## 十、误删后的处理

如果误执行：

```redis
FLUSHALL
```

第一时间：

```text
1. 停止写入或隔离 Redis。
2. 保留当前数据目录。
3. 确认 AOF 是否已经记录 FLUSHALL。
4. 找最近可用备份。
5. 在临时实例恢复验证。
6. 决定是否回切。
```

误删恢复非常依赖预案和演练。

---

## 十一、常见错误

### 1. 从不做恢复演练

备份文件可能不可用。

### 2. 只备份数据，不备份配置

恢复后安全、内存、持久化策略可能不一致。

### 3. 备份和源数据放同一台机器

机器故障时一起丢。

### 4. 没有 RPO/RTO

不知道备份频率和恢复速度是否满足业务。

### 5. 恢复时覆盖现场

丢失排查和二次恢复机会。

- 持久化不是备份。

- 数据文件、配置文件、ACL 和运行文档都要备份。
- 恢复流程必须演练。
- RPO 和 RTO 决定备份频率和恢复方案。
