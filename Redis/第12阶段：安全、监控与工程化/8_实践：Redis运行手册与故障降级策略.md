# 8. 实践：Redis 运行手册与故障降级策略

这一节把第 12 阶段串起来，产出一份 Redis 运行手册。

运行手册不是形式主义。它能让你在 Redis 出问题时知道谁负责、看哪里、怎么止血、怎么恢复。

学完这一节后，你应该能够：

- 编写 Redis 运行配置说明。
- 整理安全检查清单。
- 整理监控和告警清单。
- 设计备份恢复流程。
- 记录 Redis 故障时的降级策略。

## 七、故障处理：Redis 不可用

现象：

```text
应用 Redis timeout / connection refused。
```

处理：

```text
1. 查看 Redis 是否存活。
2. 查看网络和安全组。
3. 查看连接数和 CPU/内存。
4. 查看近期变更。
5. 启用业务降级。
6. 如有高可用，确认是否切换。
7. 恢复后观察错误率和命中率。
```

不要一上来重启。

先保留现场和指标。

---

## 八、故障处理：内存满

现象：

```text
OOM command not allowed
evicted_keys 持续增长
used_memory 接近 maxmemory
```

处理：

```text
1. 判断淘汰策略。
2. 查看 keyspace 和 TTL。
3. 查找 Big Key。
4. 暂停异常写入或批任务。
5. 清理无用 key，优先 UNLINK。
6. 临时扩容。
7. 修复 key 设计和 TTL。
```

如果有重要状态被淘汰，要启动数据恢复和业务核对。

---

## 九、故障处理：命中率下降

处理：

```text
1. 查看 hits/misses。
2. 查看 expired_keys 和 evicted_keys。
3. 查看是否发布变更导致 key 命名变化。
4. 查看数据库 QPS。
5. 启用回源限流。
6. 预热热点缓存。
7. 修复缓存写入逻辑。
```

命中率下降的真正风险往往是数据库被打爆。

## 十、业务降级策略表

示例：

| 业务能力 | Redis 故障策略 |
| --- | --- |
| 短链接详情缓存 | 查数据库，限流保护 |
| 短链接访问计数 | 临时丢弃或本地缓冲 |
| 创建接口限流 | 保守拒绝或使用本地限流 |
| 排行榜 | 返回旧榜或空榜 |
| 短信限流 | 拒绝发送 |
| 分布式锁 | 返回稍后重试，不盲目放行 |
| Stream 任务 | 暂停消费并告警 |

每个项目都应该有自己的表。

## 十二、常见错误

### 1. 运行手册只写部署命令

还需要安全、监控、备份和故障处理。

### 2. 降级策略太笼统

“Redis 故障时降级”不够，要写每个业务怎么降级。

### 3. 没有负责人

故障时没人决策。

### 4. 没有演练记录

恢复流程可能不可用。

### 5. 手册写完不更新

过期文档会误导排查。

## 十四、本节小结

这一节完成了第 12 阶段的实践闭环。

你需要记住：

- Redis 是生产基础设施，需要运行手册。
- 手册要覆盖实例、安全、配置、监控、备份、故障和降级。
- 故障处理要先止血，再定位，再复盘。
- 降级策略要按业务能力设计，不能一刀切。
- 运行手册要持续更新。

学完第 12 阶段，你已经能够把 Redis 当作生产组件来管理。下一阶段会进入项目落地：短链接缓存与限流服务。
