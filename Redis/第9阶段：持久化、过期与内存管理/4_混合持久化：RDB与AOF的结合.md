# 4. 混合持久化：RDB 与 AOF 的结合

RDB 恢复快，但可能丢较多数据。

AOF 数据更安全，但文件可能大，恢复时重放命令较慢。

混合持久化试图结合两者的优点。

学完这一节后，你应该能够：

- 理解混合持久化是什么。
- 知道它和普通 AOF 的关系。
- 理解为什么混合持久化恢复更快。
- 知道生产中如何看待混合持久化。
- 理解它仍然不能替代核心数据库。

---

## 一、混合持久化是什么

混合持久化是 AOF 重写时的一种文件格式。

开启后，AOF 重写生成的新文件大致由两部分组成：

```text
前半部分：RDB 格式的数据快照
后半部分：重写期间新增的 AOF 命令
```

也就是：

```text
RDB 快照 + AOF 增量
```

Redis 重启时可以更快加载 RDB 部分，再应用少量增量命令。

---

## 二、开启配置

配置：

```conf
aof-use-rdb-preamble yes
```

通常需要同时开启 AOF：

```conf
appendonly yes
```

很多 Redis 版本默认已经开启混合持久化。

实际以你的 Redis 配置为准。

查看：

```redis
CONFIG GET aof-use-rdb-preamble
```

---

## 三、为什么恢复更快

普通 AOF 恢复：

```text
从头到尾重放大量写命令。
```

混合持久化恢复：

```text
先加载 RDB 快照，再重放少量增量命令。
```

RDB 是紧凑二进制格式，加载通常比重放大量命令更快。

所以混合持久化可以缩短 Redis 重启恢复时间。

---

## 四、混合持久化和 AOF 重写

混合持久化不是第三套独立持久化。

它发生在 AOF 重写过程中。

执行：

```redis
BGREWRITEAOF
```

如果开启了 `aof-use-rdb-preamble yes`，重写出的 AOF 文件就会带 RDB 前导部分。

所以它可以理解为：

```text
AOF 的一种优化格式。
```

---

## 五、优点

优点：

- 恢复速度更快。
- 文件通常更紧凑。
- 保留 AOF 较好的数据安全性。
- 减少重放大量历史命令的成本。

如果你开启 AOF，通常建议了解混合持久化。

---

## 六、缺点和限制

限制：

- 文件不再是纯文本命令日志，前半部分是 RDB 格式。
- 仍然依赖 AOF 刷盘策略。
- 仍然可能丢最近数据。
- AOF 重写仍然有 fork、COW、磁盘 IO 成本。
- 不能解决 Redis 作为唯一数据库的业务风险。

混合持久化只是优化恢复速度和文件格式，不是强一致保障。

---

## 七、RDB、AOF、混合持久化对比

| 方式 | 数据安全 | 恢复速度 | 文件特点 |
| --- | --- | --- | --- |
| RDB | 较弱 | 快 | 紧凑快照 |
| AOF | 较好 | 可能较慢 | 写命令日志 |
| 混合持久化 | 较好 | 较快 | RDB + AOF 增量 |

一般经验：

```text
缓存为主：RDB 可能足够
需要更好恢复：AOF everysec
开启 AOF 后：考虑混合持久化
核心业务数据：仍放数据库
```

---

## 八、生产配置思路

常见组合：

```conf
appendonly yes
appendfsync everysec
aof-use-rdb-preamble yes
auto-aof-rewrite-percentage 100
auto-aof-rewrite-min-size 64mb
```

再配合 RDB 备份：

```conf
save 900 1
save 300 10
save 60 10000
```

是否开启以及参数如何设置，要看业务和 Redis 负载。

---

## 九、备份仍然重要

持久化文件在本机磁盘。

如果机器磁盘损坏、误删文件、误执行危险命令：

```text
本地持久化也可能救不了你。
```

生产还需要：

- 定期备份 RDB/AOF 文件。
- 备份上传到独立存储。
- 恢复演练。
- 限制危险命令。
- 监控持久化状态。

持久化不是备份的替代品。

---

## 十、常见错误

### 1. 以为混合持久化等于不丢数据

它仍然受 AOF 刷盘策略影响。

### 2. 以为混合持久化是单独第三种机制

它是 AOF 重写的一种格式。

### 3. 开启 AOF 后不关注重写

文件增长和重写失败仍然会发生。

### 4. 没有备份

本机持久化不能抵抗所有故障。

### 5. 不做恢复演练

没有演练过的备份，经常在故障时不可用。

## 十二、本节小结

这一节你学习了混合持久化。

你需要记住：

- 混合持久化是 AOF 重写的一种格式。
- 它把 RDB 快照和 AOF 增量结合起来。
- 它可以提升恢复速度，同时保留 AOF 的数据安全优势。
- 它仍然可能丢数据，仍然需要备份和恢复演练。
- 核心业务数据仍然应该保存在数据库中。
