# 0. 安全、服务治理与生产能力总览

本阶段目标：让 gRPC 服务从 demo 走向可上线的基本形态。

真实服务不只是能响应请求，还要能安全通信、暴露健康状态、支持调试、优雅退出、设置 keepalive、接入观测系统。本阶段关注这些生产环境基础能力。

---

## 一、本阶段要解决的问题

- 服务间通信是否加密？
- 调度系统如何判断服务是否健康？
- 开发人员如何查看服务暴露了哪些 RPC？
- 服务发布时如何不中断正在处理的请求？
- 长连接如何保持稳定？
- 出问题时如何通过日志、指标、trace 定位？

---

## 二、能力清单

- TLS：保护传输链路。
- mTLS：客户端和服务端互相验证身份。
- Health Check：让调度系统判断服务是否健康。
- Reflection：方便 grpcurl 等工具调试。
- Graceful Shutdown：优雅关闭。
- Service Config：配置重试、负载均衡等行为。
- OpenTelemetry：指标、日志、链路追踪。

---

## 三、学习顺序

1. 先实现 health check 和 reflection。
2. 加入优雅关闭。
3. 学习 TLS/mTLS。
4. 理解服务发现、负载均衡和重试直觉。
5. 接入基础日志、指标和 trace。

---

## 四、常见误区

- 生产环境无脑开启 reflection：可能暴露服务结构，需要按环境控制。
- 没有优雅关闭：发布时可能中断正在处理的请求。
- TLS 证书管理混乱：证书过期会直接影响服务调用。
- 只写业务日志，不记录 method、code、duration：排查 gRPC 问题会很困难。

---

## 五、完成标准

- 服务具备基本健康检查。
- 能通过 reflection 调试。
- 退出时能执行 GracefulStop。
- 知道 TLS、mTLS、服务发现、重试和观测性的基本作用。

## 七、本阶段为什么是生产分水岭

前面阶段已经能写 gRPC 服务、处理错误、加拦截器。但这些还只能说明服务“能跑”。生产环境还会继续追问：

```text
服务之间的流量是否加密？
服务是否能被健康检查识别？
服务发布时会不会中断正在处理的请求？
客户端如何发现多个服务实例？
调用失败是否能安全重试？
服务慢了、错了、不可用了，如何观测？
```

第 7 阶段就是从“功能可用”走向“工程可上线”的一层。它不一定涉及复杂业务，但会决定服务在真实环境里是否稳定。

## 十、完整代码：生产化 server 骨架

```go
func main() {
    lis, err := net.Listen("tcp", ":50051")
    if err != nil {
        log.Fatalf("listen: %v", err)
    }

    server := grpc.NewServer(
        grpc.ChainUnaryInterceptor(
            middleware.RecoveryInterceptor,
            middleware.UnaryLoggingInterceptor,
            middleware.AuthInterceptor,
        ),
    )

    userv1.RegisterUserServiceServer(server, newUserServer())

    healthServer := health.NewServer()
    grpc_health_v1.RegisterHealthServer(server, healthServer)
    healthServer.SetServingStatus("", grpc_health_v1.HealthCheckResponse_SERVING)

    reflection.Register(server)

    go func() {
        log.Println("gRPC server listening on :50051")
        if err := server.Serve(lis); err != nil {
            log.Printf("serve stopped: %v", err)
        }
    }()

    waitForShutdown(server, healthServer)
}
```

这个骨架包含：业务服务、健康检查、反射、拦截器、优雅关闭入口。

## 十三、常见错误排查

### 1. grpcurl list 失败

如果没有开启 reflection，`grpcurl list` 会失败。开发环境可以开启：

```go
reflection.Register(server)
```

### 2. health check 返回 UNKNOWN

检查是否设置了：

```go
healthServer.SetServingStatus("", grpc_health_v1.HealthCheckResponse_SERVING)
```

### 3. Ctrl+C 后请求被硬断

检查是否使用 `GracefulStop()`，而不是直接进程退出。

### 4. TLS 连接失败

检查证书域名、CA、server name 是否匹配。
