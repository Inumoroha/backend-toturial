# 4. 超时、请求体大小与 504 排查

本节目标：掌握 Nginx 代理 Go 服务时的超时配置、请求体大小限制，并能复现和解释 504。

---

## 一、三类代理超时

```nginx
location /api/ {
    proxy_connect_timeout 5s;
    proxy_send_timeout 30s;
    proxy_read_timeout 30s;

    proxy_pass http://127.0.0.1:8080;
}
```

解释：

- `proxy_connect_timeout`：Nginx 连接 Go 的超时时间。
- `proxy_send_timeout`：Nginx 向 Go 发送请求的超时时间。
- `proxy_read_timeout`：Nginx 等待 Go 响应的超时时间。

最常见导致 504 的是 `proxy_read_timeout`。

---

## 二、复现 504

Go 慢接口：

```bash
curl "http://127.0.0.1:8080/api/slow?seconds=40"
```

如果 Nginx：

```nginx
proxy_read_timeout 30s;
```

通过 Nginx 请求：

```bash
curl -i "http://api.local/api/slow?seconds=40"
```

预期：

```text
504 Gateway Timeout
```

error log：

```text
upstream timed out while reading response header from upstream
```

---

## 三、不要盲目调大超时

如果接口应该很快返回，却出现 504，要先查 Go：

- 数据库慢查询。
- 外部 API 慢。
- goroutine 阻塞。
- 连接池耗尽。
- 锁竞争。

只有业务确实需要长时间等待，才考虑调大超时。

---

## 四、请求体大小限制

```nginx
server {
    client_max_body_size 10m;
}
```

超过会返回：

```text
413 Request Entity Too Large
```

上传接口单独配置：

```nginx
location /api/upload {
    client_max_body_size 100m;
    proxy_pass http://127.0.0.1:8080;
}
```

---

## 五、测试请求体大小

创建文件：

```bash
dd if=/dev/zero of=/tmp/20m.bin bs=1M count=20
```

上传：

```bash
curl -X POST --data-binary @/tmp/20m.bin http://api.local/api/upload
```

如果普通 API 是 10MB、上传 API 是 100MB，上传接口应该成功，普通接口应该被限制。

---

## 六、本节练习

1. 设置 `proxy_read_timeout 5s`。
2. 请求 `/api/slow?seconds=10`，复现 504。
3. 查看 error log。
4. 设置 `client_max_body_size 10m`。
5. 给 `/api/upload` 单独设置 100m。
6. 上传 20MB 文件验证。
