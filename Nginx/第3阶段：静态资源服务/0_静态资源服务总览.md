# 0. 静态资源服务总览

本阶段目标：使用 Nginx 部署静态页面、前端构建产物、图片、CSS、JS，并能排查 403、404、缓存和 MIME 问题。

Go 后端工程师也需要掌握静态资源服务，因为真实项目里经常会有：

- 前端 SPA 页面。
- 管理后台。
- API 文档。
- 上传文件访问。
- 静态图片、CSS、JS。

---

## 一、本阶段你要掌握什么

- `root`。
- `alias`。
- `index`。
- `try_files`。
- SPA 前端路由。
- 静态资源缓存。
- gzip 压缩。
- MIME 类型。
- 文件权限。
- 403、404 排查。

## 三、静态资源请求链路

```text
浏览器请求 /assets/app.js
  |
  v
Nginx location /assets/
  |
  v
根据 root 或 alias 计算磁盘路径
  |
  v
读取文件
  |
  v
返回 Content-Type、Cache-Control、文件内容
```

如果文件找不到，通常返回 404。如果文件存在但 Nginx 没权限读，通常返回 403。

---

## 四、本阶段验收

完成后你应该能：

1. 部署一个静态 HTML 页面。
2. 正确解释 `root` 和 `alias` 的路径拼接。
3. 配置前端 SPA 兜底到 `index.html`。
4. 给 `/assets/` 配置缓存。
5. 开启 gzip。
6. 使用 `curl -I` 检查响应头。
7. 使用 `namei -l` 排查权限问题。
