# 1. 需求分析与接口设计

本节目标：明确最终项目要提供哪些接口、每个接口用于验证什么 Nginx 能力，以及如何用 curl 验收。

很多项目教程一上来就写代码，但后端工程更好的习惯是先定义接口和验收标准。

---

## 一、项目角色划分

本项目有两个核心角色：

```text
Nginx：入口层、网关层、静态资源层。
Go：业务 API 层。
```

Nginx 负责：

- HTTPS。
- 静态页面。
- 反向代理。
- 负载均衡。
- 日志。
- 限流。
- 上传大小限制。

Go 负责：

- 返回 API 数据。
- 打印请求头。
- 模拟慢请求。
- 接收上传请求。
- 输出实例端口。

## 二、Go API 接口设计

### 1. 健康检查

```http
GET /health
```

响应：

```text
ok
```

用途：

- 验证 Go 服务是否活着。
- 后续可用于 upstream 健康检查或部署验证。

### 2. ping

```http
GET /api/ping
```

响应：

```json
{
  "message": "pong",
  "port": "8080"
}
```

用途：

- 验证 Nginx 到 Go 的代理是否成功。
- 验证负载均衡是否打到不同端口。

### 3. headers

```http
GET /api/headers
```

响应：

```json
{
  "remote_addr": "127.0.0.1:53122",
  "host": "app.local",
  "x_request_id": "xxx",
  "x_real_ip": "127.0.0.1",
  "x_forwarded_for": "127.0.0.1",
  "x_forwarded_proto": "https"
}
```

用途：

- 验证代理头。
- 验证请求 ID。
- 验证 HTTPS 场景下协议传递。

### 4. slow

```http
GET /api/slow?seconds=5
```

响应：

```json
{
  "message": "slept 5 seconds",
  "port": "8080"
}
```

用途：

- 模拟慢接口。
- 制造 504。
- 学习 `$request_time` 和 `$upstream_response_time`。

### 5. upload

```http
POST /api/upload
Content-Type: application/octet-stream
```

响应：

```json
{
  "message": "upload received",
  "bytes": 1048576,
  "port": "8080"
}
```

用途：

- 验证 `client_max_body_size`。
- 对比普通 API 和上传 API 的不同限制。

## 三、Nginx 路由设计

### 静态首页

```text
GET /
```

由 Nginx 直接返回 `static/index.html`。

### 静态资源

```text
GET /assets/style.css
```

由 Nginx 直接返回，并带缓存头。

### API

```text
/api/
```

由 Nginx 转发到 Go upstream。

### 上传

```text
/api/upload
```

仍然转发到 Go，但单独设置更大的请求体限制。

### Nginx 健康检查

```text
GET /health
```

可以由 Nginx 直接返回：

```nginx
location = /health {
    return 200 "ok\n";
}
```

## 四、验收命令设计

静态首页：

```bash
curl -k https://app.local/
```

静态资源缓存：

```bash
curl -k -I https://app.local/assets/style.css
```

API：

```bash
curl -k https://app.local/api/ping
```

代理头：

```bash
curl -k https://app.local/api/headers
```

负载均衡：

```bash
for i in {1..12}; do curl -k -s https://app.local/api/ping; echo; done
```

慢请求：

```bash
curl -k "https://app.local/api/slow?seconds=40"
```

上传：

```bash
dd if=/dev/zero of=/tmp/20m.bin bs=1M count=20
curl -k -X POST --data-binary @/tmp/20m.bin https://app.local/api/upload
```

限流：

```bash
for i in {1..100}; do curl -k -s -o /dev/null -w "%{http_code}\n" https://app.local/api/ping; done
```
