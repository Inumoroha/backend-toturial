# 3. 静态页面与目录结构

本节目标：为最终项目准备清晰的目录结构和静态页面，让 Nginx 同时承担静态资源服务和 API 网关入口。

---

## 一、创建项目结构

建议结构：

```text
nginx-gateway-demo/
  app/
    go.mod
    main.go
  static/
    index.html
    assets/
      style.css
  nginx/
    nginx.conf
    conf.d/
  scripts/
```

创建目录：

```bash
mkdir -p nginx-gateway-demo/{app,static/assets,nginx/conf.d,scripts}
```

如果你的 shell 不支持大括号写法，就分别创建。

---

## 二、创建首页

`static/index.html`：

```html
<!doctype html>
<html>
<head>
  <meta charset="utf-8">
  <title>Nginx Gateway Demo</title>
  <link rel="stylesheet" href="/assets/style.css">
</head>
<body>
  <main>
    <h1>Nginx Gateway Demo</h1>
    <p>静态页面由 Nginx 返回，API 请求由 Nginx 转发到 Go 多实例服务。</p>
    <ul>
      <li><code>/api/ping</code>：验证 Go API 代理。</li>
      <li><code>/api/headers</code>：验证代理头。</li>
      <li><code>/api/slow</code>：验证超时排查。</li>
      <li><code>/api/upload</code>：验证上传大小限制。</li>
    </ul>
  </main>
</body>
</html>
```

## 三、创建样式

`static/assets/style.css`：

```css
body {

  background: #f6f8fa;

  margin-bottom: 16px;
}

  background: #eaeef2;

  border-radius: 4px;

```

---

## 四、静态资源 Nginx 配置目标

后面 Nginx 要做到：

```text
GET /              -> static/index.html
GET /assets/style.css -> static/assets/style.css
GET /unknown/path  -> static/index.html
```

这是 SPA 风格配置，便于前端路由。

---

## 五、静态资源缓存目标

`/assets/` 应带缓存头：

```text
Cache-Control: public, immutable
Expires: ...
```

验证命令：

```bash
curl -I http://app.local/assets/style.css
```
