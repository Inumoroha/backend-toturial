# 0. 网关能力与流量治理总览

本阶段目标：理解 Nginx 作为轻量网关时能做什么，不能做什么，并掌握限流、连接限制、CORS、缓存和上传大小控制。

Nginx 不只是反向代理。它也可以在 Go 服务前面做一些通用保护：

```text
Client
  |
  v
Nginx 网关层
  |-- 限流
  |-- CORS
  |-- 上传大小限制
  |-- 静态缓存
  |-- 代理缓存
  |-- IP 访问控制
  v
Go 业务服务
```

---

## 一、本阶段你要掌握什么

- `limit_req` 请求限流。
- `limit_conn` 连接限制。
- `limit_req_status 429`。
- CORS 响应头。
- OPTIONS 预检请求。
- `client_max_body_size`。
- `proxy_cache`。
- `proxy_no_cache`。
- IP allow/deny。
- Basic Auth。
- 哪些逻辑适合 Nginx，哪些必须放 Go。

## 三、Nginx 适合做什么

适合放 Nginx：

- 单 IP 粗粒度限流。
- 上传大小限制。
- 静态资源缓存。
- 部分只读接口短缓存。
- 管理后台 Basic Auth。
- 内网 IP 白名单。
- CORS 统一处理。

适合放 Go：

- 用户级限流。
- 租户级限流。
- API Key 限流。
- 复杂权限判断。
- 订单幂等。
- 支付验签。
- 业务缓存一致性。

完成后你应该能：

1. 给 `/api/` 配置每秒请求限制。

3. 给 `/api/upload` 单独放大请求体。
4. 给指定域名配置 CORS。
5. 判断某个缓存能不能放在 Nginx。
