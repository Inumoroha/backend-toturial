# 4. 常用命令与第一次配置修改

本节目标：完成第一次 Nginx 配置修改，并掌握“修改、检查、重载、验证、看日志”的标准流程。

这一节非常重要。后面所有 Nginx 学习都会反复使用这个流程。

```text
修改配置
-> nginx -t
-> reload
-> curl 验证
-> 看 access.log / error.log
```

---

## 一、准备一个练习 server

建议在系统安装环境中创建：

```text
/etc/nginx/conf.d/hello.conf
```

内容：

```nginx
server {
    listen 80;
    server_name hello.local;

    location / {
        return 200 "hello nginx\n";
    }
}
```

如果你在 Docker 中练习，可以先进入容器看配置，但更推荐后续使用挂载配置的方式。初学阶段先用系统安装更直观。

---

## 二、配置本地域名

Linux / WSL：

```bash
sudo vim /etc/hosts
```

加入：

```text
127.0.0.1 hello.local
```

Windows 浏览器访问时，需要修改 Windows hosts 文件：

```text
C:\Windows\System32\drivers\etc\hosts
```

也加入：

```text
127.0.0.1 hello.local
```

如果不想改 hosts，可以用 Host 头测试：

```bash
curl -H "Host: hello.local" http://127.0.0.1
```

---

## 三、检查配置

执行：

```bash
sudo nginx -t
```

如果看到：

```text
syntax is ok
test is successful
```

说明语法没问题。

如果报错，要先修复，不要 reload。

---

## 四、重载配置

```bash
sudo systemctl reload nginx
```

或者：

```bash
sudo nginx -s reload
```

推荐系统服务环境使用：

```bash
sudo systemctl reload nginx
```

---

## 五、验证结果

```bash
curl http://hello.local
```

期望输出：

```text
hello nginx
```

如果没有配置 hosts：

```bash
curl -H "Host: hello.local" http://127.0.0.1
```

---

## 六、查看完整配置是否加载

```bash
sudo nginx -T | grep -A 10 hello.local
```

如果能看到你写的配置，说明它被加载了。

如果看不到，检查：

- 文件是否以 `.conf` 结尾。
- 文件是否放在被 include 的目录。
- 主配置里是否有 `include /etc/nginx/conf.d/*.conf;`。
- 是否修改的是正确服务器。

---

## 七、故意制造错误

把配置中的分号删掉：

```nginx
return 200 "hello nginx\n"
```

执行：

```bash
sudo nginx -t
```

你会看到类似：

```text
unexpected "}" in /etc/nginx/conf.d/hello.conf:6
```

这说明 Nginx 解析配置失败。

修复：

```nginx
return 200 "hello nginx\n";
```

再次检查：

```bash
sudo nginx -t
```

---

## 八、常用命令清单

查看版本：

```bash
nginx -v
```

检查配置：

```bash
sudo nginx -t
```

打印完整配置：

```bash
sudo nginx -T
```

重载：

```bash
sudo systemctl reload nginx
```

重启：

```bash
sudo systemctl restart nginx
```

查看状态：

```bash
sudo systemctl status nginx
```

查看访问日志：

```bash
sudo tail -f /var/log/nginx/access.log
```

查看错误日志：

```bash
sudo tail -f /var/log/nginx/error.log
```

查看端口：

```bash
sudo ss -lntp | grep nginx
```

## 九、本节练习

1. 新建 `hello.conf`。
2. 配置 `hello.local`。
3. 使用 `nginx -t` 检查配置。
4. reload Nginx。
5. 使用 `curl` 访问。
6. 删除分号制造错误。
7. 使用错误信息定位问题。
8. 修复并重新 reload。
