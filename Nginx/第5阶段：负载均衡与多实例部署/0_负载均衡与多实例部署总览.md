# 0. 负载均衡与多实例部署总览

本阶段目标：使用 Nginx 把请求分发到多个 Go 服务实例，并理解失败、重试、会话和幂等问题。

单个 Go 实例：

```text
Nginx -> Go :8080
```

多个 Go 实例：

```text
Nginx -> Go :8080
      -> Go :8081
      -> Go :8082
```

Nginx 通过 `upstream` 管理这些后端实例。

---

## 一、本阶段你要掌握什么

- `upstream`。
- 默认轮询。
- `weight` 权重。
- `least_conn` 最少连接。
- `ip_hash`。
- `max_fails` 与 `fail_timeout`。
- `proxy_next_upstream`。
- 多实例无状态。
- 重试与幂等。
- Go 服务优雅关闭。

## 三、为什么要多实例

多实例可以：

- 提高吞吐。
- 避免单进程故障影响全部服务。
- 支持滚动发布。
- 更好利用多核或多机器资源。

但它也带来问题：

- 请求可能落到不同实例。
- 内存 session 失效。
- 重试可能导致重复写入。
- 发布时要处理正在执行的请求。

4. 停掉一个实例，观察 Nginx 行为。
