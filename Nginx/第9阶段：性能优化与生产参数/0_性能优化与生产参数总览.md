# 0. 性能优化与生产参数总览

本阶段目标：理解 Nginx 的性能模型，知道如何通过日志、压测和系统指标判断瓶颈，而不是盲目复制所谓“高性能配置”。

Nginx 性能调优的核心不是调大所有参数，而是先回答：

```text
瓶颈在哪里？
Nginx？
Go 服务？
数据库？
系统连接数？
CPU？
磁盘 I/O？
```

---

## 一、本阶段你要掌握什么

- master/worker 模型。
- `worker_processes`。
- `worker_connections`。
- 文件描述符限制。
- `keepalive_timeout`。
- upstream keepalive。
- gzip 对 CPU 的影响。
- `sendfile`。
- 压测工具。
- Nginx 日志耗时分析。
- Go pprof 配合定位。

## 三、调优基本原则

1. 先有基线。
2. 再压测复现。
3. 每次只改一个变量。
4. 修改后再次压测。
5. 结合 access log、error log、系统指标和 Go 日志判断。
6. 不要用 Nginx 参数掩盖业务慢查询。

---

## 四、本阶段验收

完成后你应该能：

1. 查看 Nginx worker 进程。
2. 查看文件描述符限制。
3. 配置 upstream keepalive。
4. 对比 Go 直连和 Nginx 代理性能。
5. 根据 `$request_time` 和 `$upstream_response_time` 判断慢在哪里。
