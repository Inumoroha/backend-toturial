# 3. 压测、观察与瓶颈定位

本节目标：学习如何通过压测、日志和系统指标判断瓶颈在哪里，而不是凭感觉修改 Nginx 配置。

---

## 一、压测前先建立基线

记录环境：

```bash
nginx -v
go version
ulimit -n
nproc
free -h
```

记录配置：

```bash
sudo nginx -T > /tmp/nginx-full.conf
```

这样后续调参时能知道自己改了什么。

---

## 二、压测工具

可以使用：

- `hey`
- `wrk`
- `ab`

示例使用 `hey`：

```bash
hey -n 10000 -c 100 http://127.0.0.1:8080/api/ping
hey -n 10000 -c 100 http://api.local/api/ping
```

第一条压 Go 直连，第二条压 Nginx 代理。

---

## 三、对比 Go 直连和 Nginx 代理

如果 Go 直连慢，Nginx 代理也慢：

```text
瓶颈大概率在 Go 或 Go 的依赖。
```

如果 Go 直连快，Nginx 代理慢：

```text
需要重点看 Nginx、TLS、gzip、日志、连接复用。
```

---

## 四、观察 Nginx 日志

自定义日志中关注：

```text
rt=$request_time
urt=$upstream_response_time
uct=$upstream_connect_time
```

示例：

```text
rt=0.005 urt=0.004
```

说明 Go 响应很快。

示例：

```text
rt=3.002 urt=3.001
```

说明慢主要在 Go。

示例：

```text
uct=1.200
```

说明连接 Go 就花了较长时间。

---

## 五、观察系统指标

CPU：

```bash
top
```

连接：

```bash
ss -s
```

端口连接：

```bash
ss -ant | grep ':80' | wc -l
ss -ant | grep ':8080' | wc -l
```

文件描述符：

```bash
cat /proc/$(pgrep -o nginx)/limits | grep "open files"
```

---

## 六、Go pprof 配合分析

如果怀疑 Go CPU 高，可以在 Go 服务中加入：

```go
import _ "net/http/pprof"
```

启动本地调试端口：

```go
go func() {
    log.Println(http.ListenAndServe("127.0.0.1:6060", nil))
}()
```

采样：

```bash
go tool pprof http://127.0.0.1:6060/debug/pprof/profile?seconds=30
```

注意：pprof 不要暴露公网。

---

## 七、调优原则

1. 先复现。
2. 再定位。
3. 每次只改一个变量。
4. 修改后重新压测。
5. 记录结果。
6. 不要用调大 Nginx 超时掩盖业务慢。

---

## 八、本节练习

1. 压测 Go 直连。
2. 压测 Nginx 代理。
3. 记录 QPS、平均延迟、P95、P99。
4. 打开 upstream keepalive，再压一次。
5. 请求慢接口，观察 Nginx 日志耗时。
6. 如果 Go 慢，用 pprof 找原因。
