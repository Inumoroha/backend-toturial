# Nginx 学习路线

本路线面向 Go 后端工程师。学习目标不是“会复制 Nginx 配置”，而是能够理解 Nginx 在后端系统中的位置，并能独立完成反向代理、静态资源服务、负载均衡、HTTPS、日志排障、限流、缓存、性能调优和生产部署。

这套路线上手方式参考同目录 PostgreSQL 教程的写法：每个阶段放在独立目录中，每篇文档只解决一个明确问题，尽量给出可运行命令、配置解释、验证方式、常见错误和阶段练习。

---

## 一、为什么 Go 后端工程师必须学 Nginx

Go 服务本身可以直接监听端口，例如：

```text
Go API -> :8080
```

但真实生产环境通常不会让用户直接访问 Go 进程，而是这样：

```text
浏览器 / App / curl
        |
        | HTTP / HTTPS
        v
      Nginx
        |
        | HTTP
        v
      Go API
```

Nginx 常承担这些职责：

- 对外监听 `80`、`443`。
- 给 Go 服务提供 HTTPS 入口。
- 把请求转发到一个或多个 Go 实例。
- 托管前端静态页面。
- 记录统一访问日志。
- 处理基础限流和上传大小限制。
- 在 Go 服务不可用时返回明确错误。
- 帮助排查 `502`、`503`、`504`。

所以学习 Nginx 的核心不是背指令，而是理解请求进入系统后的路径。

## 二、阶段安排

### 第 1 阶段：环境搭建与 Nginx 直觉

目标：把 Nginx 跑起来，知道它监听哪个端口、配置在哪里、日志在哪里、如何验证修改是否生效。

建议学习：

1. 使用 Docker 启动 Nginx。
2. 使用本机或 Linux 服务安装 Nginx。
3. 理解 Nginx、端口、配置文件、日志文件之间的关系。
4. 掌握 `nginx -t`、`nginx -T`、reload、日志查看。

### 第 2 阶段：Nginx 配置基础与请求匹配

目标：看懂 `nginx.conf`，理解 `main`、`events`、`http`、`server`、`location`。

建议学习：

1. 配置上下文。
2. `include` 机制。
3. `server_name` 匹配。
4. `location` 匹配。
5. `return`、`rewrite`、变量。

### 第 3 阶段：静态资源服务

目标：用 Nginx 部署前端页面和静态文件。

建议学习：

1. `root` 与 `alias`。
2. `index` 与 `try_files`。
3. SPA 前端路由。
4. 缓存头。
5. gzip。
6. 权限、MIME、403、404 排查。

### 第 4 阶段：反向代理 Go 后端服务

目标：把 Nginx 放在 Go 服务前面，掌握 Go 后端最常用的 Nginx 配置。

建议学习：

1. 准备 Go HTTP 服务。
2. `proxy_pass` 基础。
3. `proxy_pass` 路径规则。
4. 代理头与真实 IP。
5. 超时配置。
6. 请求体大小限制。
7. WebSocket 代理。

### 第 5 阶段：负载均衡与多实例部署

目标：使用 Nginx 把流量分发到多个 Go 实例。

建议学习：

1. `upstream`。
2. 轮询、权重、最少连接、IP Hash。
3. 后端失败处理。
4. 重试与幂等。
5. 多实例会话策略。
6. Go 服务优雅关闭。

### 第 6 阶段：HTTPS 与安全基础

目标：让 Nginx 对外提供 HTTPS，并理解证书、TLS 终止和基础安全配置。

建议学习：

1. 自签名证书。
2. Let's Encrypt 和 Certbot。
3. HTTP 跳转 HTTPS。
4. `X-Forwarded-Proto`。
5. 安全响应头。
6. Basic Auth、IP 访问控制。
7. 证书续期。

### 第 7 阶段：日志监控与故障排查

目标：能通过日志排查常见线上问题。

建议学习：

1. `access.log`。
2. `error.log`。
3. 自定义日志格式。
4. upstream 耗时字段。
5. 请求 ID 关联 Nginx 与 Go 日志。
6. `502`、`503`、`504` 排查。
7. `499`、`413`、`403`、`404` 分析。

### 第 8 阶段：网关能力与流量治理

目标：掌握 Nginx 作为轻量网关时常见能力。

建议学习：

1. `limit_req` 请求限流。
2. `limit_conn` 连接限制。
3. CORS。
4. `proxy_cache`。
5. 上传大小限制。
6. 管理后台保护。
7. Nginx 与 Go 的职责边界。

### 第 9 阶段：性能优化与生产参数

目标：理解 Nginx 性能模型，知道如何压测、观察和调参。

建议学习：

1. master/worker。
2. `worker_processes`、`worker_connections`。
3. 文件描述符限制。
4. keep-alive。
5. upstream 连接复用。
6. gzip 与 CPU。
7. 压测与瓶颈定位。
8. Go pprof 配合分析。

### 第 10 阶段：项目落地：Go 服务 Nginx 网关

目标：完成一个真实可运行的小项目，把前面知识串起来。

最终项目包含：

- Go API 三实例。
- Nginx upstream 负载均衡。
- 静态首页。
- HTTPS。
- HTTP 自动跳转 HTTPS。
- 代理头。
- 请求 ID。
- 日志。
- 限流。
- 上传接口。
- 慢接口制造 504。
- 停服务制造 502。
- systemd 或 Docker Compose 部署。

## 三、推荐学习方式

每篇文档都按下面方式学习：

1. 先读目标，知道本节解决什么问题。
2. 复制最小命令或配置跑起来。
3. 用 `curl` 或日志验证结果。
4. 故意制造一个错误。
5. 按排查步骤修复它。
6. 最后回答本节复盘问题。

只看文档不动手，Nginx 很难真正学会。Nginx 的熟练感来自“配置、请求、日志、错误、修复”之间反复来回。
