# 0. 日志监控与故障排查总览

本阶段目标：学会用 Nginx 日志定位问题，尤其是 Go 后端常见的 `502`、`503`、`504`。

排障不是猜。排障要按链路看证据：

```text
客户端现象
-> Nginx access.log
-> Nginx error.log
-> Go 应用日志
-> 系统端口和进程
-> 数据库或外部依赖
```

---

## 一、本阶段你要掌握什么

- access log。
- error log。
- 自定义日志格式。
- `$request_time`。
- `$upstream_response_time`。
- `$upstream_addr`。
- `$request_id`。
- 502 排查。
- 503 排查。
- 504 排查。
- 499、413、403、404 的含义。

## 三、为什么日志是第一入口

当用户说“接口访问不了”时，你不能马上改配置。

先问：

- Nginx 有没有收到请求？
- 返回状态码是什么？
- 请求有没有到 Go？
- upstream 是哪个实例？
- 耗时在哪里？
- error log 有没有连接失败或超时？

这些问题大多能从日志里找到答案。

## 四、本阶段验收

完成后你应该能：

1. 自定义 access log。
2. 记录 upstream 地址和耗时。
3. 使用请求 ID 串联 Nginx 和 Go 日志。
4. 停掉 Go 服务制造 502。
5. 慢接口制造 504。
6. 根据日志解释问题原因。
