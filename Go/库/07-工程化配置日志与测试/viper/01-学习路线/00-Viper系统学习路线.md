# Viper 系统学习路线：面向 Go 后端工程师

## 学习目标

学完这套路线后，你应该能够在真实 Go 后端项目中独立设计配置系统，包括本地配置、环境变量覆盖、结构体绑定、配置校验、热更新、测试隔离、生产安全与可回滚发布。

```text
配置不是“把参数读出来”这么简单，而是让服务在不同环境中稳定、可控、可追踪地启动和运行。
```

## 一、学习前准备

你需要具备：

- Go 基础语法：结构体、方法、包、错误处理。
- Go module 基础：知道 `go mod init`、`go get`、`go run` 的作用。
- 后端项目直觉：知道数据库、Redis、HTTP 服务、日志、迁移、JWT 等组件通常需要配置。
- 基本命令行能力：能在 Windows PowerShell 或 Linux/macOS 终端中执行命令。

建议目录：

```text
viper/
├── 01-学习路线/
└── 02-实践教程/
```

## 二、学习阶段

### 第 1 阶段：建立配置直觉

目标：理解后端项目为什么需要配置管理，能用 Viper 读取一个 YAML 配置文件。

对应教程：

- `02-实践教程/01-从一个配置文件启动服务.md`

你要掌握：

- 配置文件在后端项目中的位置。
- `SetConfigName`、`SetConfigType`、`AddConfigPath`、`ReadInConfig` 的基本关系。
- 为什么配置读取失败时通常要让服务启动失败。

### 第 2 阶段：设计配置结构

目标：能设计适合后端项目的 YAML 层级，并区分开发、测试、生产环境。

对应教程：

- `02-实践教程/02-配置文件结构与多环境管理.md`

你要掌握：

- 按业务组件组织配置。
- 使用 `config.dev.yaml`、`config.test.yaml`、`config.prod.yaml`。
- 避免把生产密码提交到仓库。

### 第 3 阶段：默认值、必填项和类型安全

目标：能为配置设置合理默认值，并对关键配置做缺失检查。

对应教程：

- `02-实践教程/03-默认值必填项与类型转换.md`

你要掌握：

- `SetDefault` 的使用边界。
- `GetString`、`GetInt`、`GetDuration` 的常见用法。
- 哪些配置必须启动时检查。

### 第 4 阶段：绑定结构体

目标：能把配置绑定到结构体，并在业务代码中传递强类型配置。

对应教程：

- `02-实践教程/04-结构体绑定与配置校验.md`

你要掌握：

- `Unmarshal` 的基本用法。
- `mapstructure` 标签的作用。
- 配置校验应该靠近启动流程完成。

### 第 5 阶段：覆盖优先级

目标：理解环境变量和命令行参数如何覆盖配置文件，适配容器和 CI/CD。

对应教程：

- `02-实践教程/05-环境变量与命令行覆盖.md`

你要掌握：

- 为什么生产环境常用环境变量注入敏感信息。
- `AutomaticEnv`、`SetEnvPrefix`、`SetEnvKeyReplacer`。
- 命令行参数适合控制一次性启动行为，不适合承载复杂配置。

### 第 6 阶段：配置热更新

目标：理解哪些配置适合热更新，哪些配置必须重启服务。

对应教程：

- `02-实践教程/06-配置热更新与运行期风险.md`

你要掌握：

- `WatchConfig` 和 `OnConfigChange`。
- 日志级别这类配置可以热更新。
- 数据库连接地址、迁移开关这类配置通常不建议热更新。

### 第 7 阶段：在项目中组织配置包

目标：能把 Viper 封装成项目内部的 `config` 包，而不是在业务代码里到处调用 Viper。

对应教程：

- `02-实践教程/07-在Go后端项目中组织配置包.md`

你要掌握：

- 初始化阶段读取配置。
- 业务层依赖结构体，不直接依赖 Viper。
- 配置对象只读传递，降低隐藏依赖。

### 第 8 阶段：测试与本地开发

目标：能为配置读取逻辑写测试，并让不同测试互不污染。

对应教程：

- `02-实践教程/08-测试配置与本地开发隔离.md`

你要掌握：

- 测试中使用临时目录和临时配置文件。
- 每个测试创建独立 Viper 实例。
- 不让测试依赖本机环境变量和真实生产配置。

### 第 9 阶段：生产安全

目标：能在生产项目中安全处理密码、密钥、开关和可回滚发布。

对应教程：

- `02-实践教程/09-生产配置安全与可回滚发布.md`

你要掌握：

- 配置文件、环境变量、密钥管理平台各自适合放什么。
- 功能开关的默认值应保守。
- 配置变更要可审计、可回滚。

### 第 10 阶段：综合项目落地

目标：用 Viper 支撑一个短链接服务的配置系统。

对应教程：

- `02-实践教程/10-短链接服务配置实战.md`

你要掌握：

- HTTP、数据库、Redis、日志、JWT、限流、迁移开关的配置设计。
- 启动时校验关键配置。
- 给后续 Gin、GORM、Redis、Zap 等组件提供配置入口。

## 三、推荐学习顺序

建议按顺序学习，不要跳过前面的工程直觉：

1. 先能读取一个配置文件。
2. 再学会设计配置文件结构。
3. 再处理默认值、必填项和类型转换。
4. 再把配置绑定到结构体。
5. 再学习环境变量和命令行覆盖。
6. 再学习热更新。
7. 再把配置封装到项目结构中。
8. 再写配置测试。
9. 最后补齐生产安全和综合项目。

## 四、练习建议

每学完一篇，建议你做一个小改动：

- 给配置增加一个字段，例如 `log.level`。
- 故意删掉一个必填项，观察程序是否启动失败。
- 用环境变量覆盖配置文件中的数据库地址。
- 写一个测试验证配置读取结果。
- 把配置结构迁移到一个独立 `internal/config` 包。

```text
配置系统的练习重点不是“记住 Viper 方法名”，而是训练你判断哪些配置能默认、哪些必须校验、哪些不能热更新。
```

## 五、最终达标标准

完成整套教程后，你应该能够：

- 独立为 Go 后端项目设计 `config.yaml`。
- 根据环境加载不同配置文件。
- 使用 Viper 读取配置并绑定到结构体。
- 对数据库、Redis、JWT、日志等关键配置做启动校验。
- 使用环境变量覆盖敏感配置。
- 判断配置热更新的使用边界。
- 为配置读取逻辑编写稳定测试。
- 在生产发布中做到配置可审计、可回滚、默认值保守。
