# 10. 短链接服务配置实战

本节目标：学完后你能为一个短链接 Go 后端服务设计完整配置，并用 Viper 加载、覆盖和校验。

## 简短引入

这一节把前面的内容串起来。我们不实现完整短链接业务，而是专注配置系统：HTTP 服务、数据库、Redis、日志、JWT、限流、迁移开关都从配置进入项目。

真实项目中，配置系统越早整理清楚，后续接入 Gin、GORM、Redis、Zap、迁移工具时越顺。

## 一、为什么需要它

短链接服务常见能力：

- 创建短链接。
- 查询短链接。
- 跳转到原始链接。
- 登录后台管理。
- 限制接口频率。
- 记录访问日志。

对应需要的配置：

- HTTP 地址和超时。
- PostgreSQL DSN 和连接池。
- Redis 地址和缓存 TTL。
- JWT 密钥和过期时间。
- 日志级别。
- 限流参数。
- 数据库迁移开关。

```text
配置设计要围绕组件启动和运行风险，而不是把所有变量随手塞进 YAML。
```

创建 `internal/config/config.go`：

```go
package config

import (
	"errors"
	"strings"
	"time"

	"github.com/spf13/viper"
)

type Config struct {
	App       App       `mapstructure:"app"`
	Server    Server    `mapstructure:"server"`
	Database  Database  `mapstructure:"database"`
	Redis     Redis     `mapstructure:"redis"`
	JWT       JWT       `mapstructure:"jwt"`
	Log       Log       `mapstructure:"log"`
	RateLimit RateLimit `mapstructure:"rate_limit"`
}

type App struct {
	Name string `mapstructure:"name"`
	Env  string `mapstructure:"env"`
}

type Server struct {
	Addr         string        `mapstructure:"addr"`
	ReadTimeout  time.Duration `mapstructure:"read_timeout"`
	WriteTimeout time.Duration `mapstructure:"write_timeout"`
}

type Database struct {
	DSN          string `mapstructure:"dsn"`
	MaxOpenConns int    `mapstructure:"max_open_conns"`
	MaxIdleConns int    `mapstructure:"max_idle_conns"`
	AutoMigrate  bool   `mapstructure:"auto_migrate"`
}

type Redis struct {
	Addr     string        `mapstructure:"addr"`
	Password string        `mapstructure:"password"`
	DB       int           `mapstructure:"db"`
	CacheTTL time.Duration `mapstructure:"cache_ttl"`
}

type JWT struct {
	Secret string        `mapstructure:"secret"`
	Expire time.Duration `mapstructure:"expire"`
}

type Log struct {
	Level string `mapstructure:"level"`
}

type RateLimit struct {
	Enabled           bool `mapstructure:"enabled"`
	RequestsPerMinute int  `mapstructure:"requests_per_minute"`
}

func Load(env string, paths ...string) (*Config, error) {
	v := viper.New()
	setDefaults(v)

	v.SetConfigName("config." + env)
	v.SetConfigType("yaml")
	for _, path := range paths {
		v.AddConfigPath(path)
	}

	v.SetEnvPrefix("SHORTLINK")
	v.SetEnvKeyReplacer(strings.NewReplacer(".", "_"))
	v.AutomaticEnv()
	if err := bindEnvs(v,
		"server.addr",
		"server.read_timeout",
		"server.write_timeout",
		"database.dsn",
		"database.max_open_conns",
		"database.max_idle_conns",
		"database.auto_migrate",
		"redis.addr",
		"redis.password",
		"redis.db",
		"redis.cache_ttl",
		"jwt.secret",
		"jwt.expire",
		"log.level",
		"rate_limit.enabled",
		"rate_limit.requests_per_minute",
	); err != nil {
		return nil, err
	}

	if err := v.ReadInConfig(); err != nil {
		return nil, err
	}

	var cfg Config
	if err := v.Unmarshal(&cfg); err != nil {
		return nil, err
	}

	if err := cfg.Validate(); err != nil {
		return nil, err
	}

	return &cfg, nil
}

func bindEnvs(v *viper.Viper, keys ...string) error {
	for _, key := range keys {
		if err := v.BindEnv(key); err != nil {
			return err
		}
	}
	return nil
}

func setDefaults(v *viper.Viper) {
	v.SetDefault("server.read_timeout", "3s")
	v.SetDefault("server.write_timeout", "3s")
	v.SetDefault("database.max_open_conns", 10)
	v.SetDefault("database.max_idle_conns", 2)
	v.SetDefault("database.auto_migrate", false)
	v.SetDefault("redis.db", 0)
	v.SetDefault("redis.cache_ttl", "10m")
	v.SetDefault("log.level", "info")
	v.SetDefault("rate_limit.enabled", true)
	v.SetDefault("rate_limit.requests_per_minute", 60)
}

func (c Config) Validate() error {
	if c.App.Name == "" {
		return errors.New("app.name is required")
	}
	if c.App.Env == "" {
		return errors.New("app.env is required")
	}
	if c.Server.Addr == "" {
		return errors.New("server.addr is required")
	}
	if c.Database.DSN == "" {
		return errors.New("database.dsn is required")
	}
	if c.Redis.Addr == "" {
		return errors.New("redis.addr is required")
	}
	if c.JWT.Secret == "" {
		return errors.New("jwt.secret is required")
	}
	if c.RateLimit.Enabled && c.RateLimit.RequestsPerMinute <= 0 {
		return errors.New("rate_limit.requests_per_minute must be positive")
	}
	if c.App.Env == "prod" && c.Database.AutoMigrate {
		return errors.New("database.auto_migrate must be false in prod")
	}
	return nil
}
```

创建 `main.go`：

```go
package main

import (
	"flag"
	"log"

	"shortlink-config-demo/internal/config"
)

func main() {
	env := flag.String("env", "dev", "runtime environment")
	configPath := flag.String("config", "./configs", "config directory")
	flag.Parse()

	cfg, err := config.Load(*env, *configPath)
	if err != nil {
		log.Fatalf("load config failed: %v", err)
	}

	log.Printf("start %s env=%s addr=%s log=%s",
		cfg.App.Name,
		cfg.App.Env,
		cfg.Server.Addr,
		cfg.Log.Level,
	)
	log.Printf("db_conns=%d redis=%s cache_ttl=%s rate_limit=%d/min",
		cfg.Database.MaxOpenConns,
		cfg.Redis.Addr,
		cfg.Redis.CacheTTL,
		cfg.RateLimit.RequestsPerMinute,
	)
}
```

## 三、关键参数/语法/代码结构

`Config` 按组件分组，和后端项目启动顺序一致。

`setDefaults` 集中设置低风险默认值。数据库 DSN、JWT 密钥这类关键配置没有默认生产值。

`Validate` 集中处理启动前校验。它不需要负责连接数据库，只负责判断配置是否合理。

`SHORTLINK_DATABASE_DSN` 可以覆盖 `database.dsn`，适合容器部署时注入生产值。

示例中显式绑定了常见可覆盖 key。这样环境变量不仅能被 `GetString` 读到，也能进入 `Unmarshal` 后的结构体。

## 五、注意点

开发环境可以开启 `auto_migrate` 方便练习，但生产环境应该关闭，并使用迁移工具管理数据库结构。

数据库连接池参数不是越大越好。连接数会占用数据库资源，应该根据数据库规格、服务实例数和业务压力评估。

限流配置需要和真实网关、负载均衡配合。如果服务有多个实例，本地内存限流只能限制单实例，不能代表全局限流。

JWT 密钥开发环境可以用示例值，生产环境必须由安全渠道注入，并定期轮换。

## 六、常见误区

误区一：把所有配置都设计成字符串。

时间、布尔、整数应该使用合适类型。这样组件初始化时更少出错。

误区二：配置结构为了“通用”设计得过度抽象。

初学阶段按组件分组就足够。不要提前设计复杂的动态配置中心接口。

误区三：生产也沿用开发配置。

开发配置通常包含弱密码、自动迁移、宽松日志，不适合生产。生产应该有单独文件和注入策略。

误区四：数据库迁移和业务服务启动强绑定。

真实项目中迁移要有明确工具、顺序和回滚方案，不能依赖每个服务实例启动时自动执行。
