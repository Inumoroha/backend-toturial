学习建议：

- 每篇都新建一个小目录敲一遍示例，不要只看。
- 先保证测试能稳定运行，再追求覆盖更多分支。
- 单元测试、集成测试、benchmark、fuzz 各有边界，不要混用。
- 遇到数据库、迁移、事务、生产配置相关内容时，优先采取保守做法。
