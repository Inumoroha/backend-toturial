# 04. 布尔值与配置开关解析

本节目标：学完后，你能用 `ParseBool` 解析环境变量和功能开关，并为生产环境设置安全默认值。

简短引入：后端服务经常通过环境变量控制行为，比如是否开启调试日志、是否启用新功能、是否允许迁移任务执行。这些值来自字符串，需要转换成布尔值。

## 一、为什么需要它

可以理解为：配置文件和环境变量都只是文本，程序启动时要把它们变成明确的类型。

常见场景是：

- `ENABLE_SIGNUP=false`
- `DEBUG=true`
- `RUN_MIGRATION=0`
- `USE_MOCK_PAYMENT=1`

`strconv.ParseBool` 支持多种写法，比如 `true`、`false`、`1`、`0`、`t`、`f`。

```text
生产环境的开关默认值要保守，不能因为配置缺失就开启高风险行为。
```

## 二、基本用法

Windows PowerShell：

```powershell
@'
package main

import (
	"fmt"
	"os"
	"strconv"
)

func main() {
	os.Setenv("ENABLE_SIGNUP", "false")

	enabled, err := strconv.ParseBool(os.Getenv("ENABLE_SIGNUP"))
	if err != nil {
		fmt.Println("ENABLE_SIGNUP 配置不合法:", err)
		return
	}

	fmt.Println("是否开放注册:", enabled)
}
'@ | Set-Content -Encoding UTF8 main.go
go run main.go
```

Linux/macOS：

```bash
cat > main.go <<'EOF'
```

## 三、关键参数/语法/代码结构

`strconv.ParseBool(s)`：把字符串转成 `bool`。

它能接受的常见值：

- 真：`1`、`t`、`T`、`true`、`TRUE`、`True`
- 假：`0`、`f`、`F`、`false`、`FALSE`、`False`

`strconv.FormatBool(b)`：把布尔值转成 `"true"` 或 `"false"`，常用于日志和状态输出。

## 四、真实后端场景示例

下面实现一个配置读取函数，缺失时使用默认值。

```go
package main

import (
	"fmt"
	"os"
	"strconv"
)

func boolEnv(key string, defaultValue bool) (bool, error) {
	raw, ok := os.LookupEnv(key)
	if !ok || raw == "" {
		return defaultValue, nil
	}

	value, err := strconv.ParseBool(raw)
	if err != nil {
		return false, fmt.Errorf("%s 必须是布尔值", key)
	}
	return value, nil
}

func main() {
	allowMigration, err := boolEnv("RUN_MIGRATION", false)
	if err != nil {
		fmt.Println("配置错误:", err)
		return
	}

	fmt.Println("是否允许运行迁移:", strconv.FormatBool(allowMigration))
}
```

迁移开关默认是 `false`，这是保守做法。真实项目里，数据库迁移通常由专门工具和发布流程控制，不建议服务启动时自动做破坏性迁移。

## 五、注意点

功能开关不要只看“能不能解析”，还要看默认值是否安全。

例如：

- `DEBUG` 默认应为 `false`
- `RUN_MIGRATION` 默认应为 `false`
- `ENABLE_PAYMENT_MOCK` 生产环境默认应为 `false`
- `ENABLE_SIGNUP` 要看业务策略，不能随意默认开启

```text
配置缺失时，不要默认开启高风险功能。
```

## 六、常见误区

- 误区一：用字符串直接判断。  
  `if os.Getenv("DEBUG") != ""` 会把 `"false"` 也当成开启。

- 误区二：配置错误时继续启动。  
  生产服务遇到关键配置错误，通常应该启动失败，让问题尽早暴露。

- 误区三：所有开关默认 `true`。  
  这会让缺失配置变成隐性风险，尤其是调试、迁移、支付 mock 这类功能。

## 七、本节达标标准

- 能用 `ParseBool` 解析配置开关。
- 能用 `LookupEnv` 区分配置缺失和配置为空。
- 能为不同开关选择保守默认值。
- 能用 `FormatBool` 输出清晰日志。
- 知道生产环境高风险开关不应默认开启。
