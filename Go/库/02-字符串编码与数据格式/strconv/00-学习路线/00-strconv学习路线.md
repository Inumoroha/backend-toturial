# 00. strconv 学习路线

本路线面向正在学习 Go 后端开发的同学。这里的主题是 `strconv`，也就是 Go 标准库里负责“字符串和基础类型互相转换”的包。

在真实后端项目中，`strconv` 常见于这些位置：

- HTTP 请求参数：`page=1`、`limit=20`、`user_id=1001`
- 配置解析：`FEATURE_LOGIN=true`、`PORT=8080`
- 数据库字段转换：字符串 ID、整数字段、迁移脚本
- 日志和响应拼接：把数字、布尔值格式化成文本
- 短链接、权限位、库存、金额等业务边界处理

```text
后端项目里，大多数外部输入一开始都是字符串；进入业务逻辑前，必须先转换、校验、处理错误。
```

## 一、学习目标

学完这套教程，你应该能够：

- 正确把字符串转换成 `int`、`int64`、`uint64`、`bool`、`float64`
- 理解 `base`、`bitSize`、`FormatFloat` 等常用参数的业务意义
- 能在 HTTP 参数、环境变量、日志、迁移脚本中稳妥使用 `strconv`
- 遇到转换失败、越界、精度问题时，知道如何处理
- 不把 `strconv` 当成安全转义工具，不用它替代参数化查询、JSON 编码、HTML 转义

## 二、学习顺序

### 第一阶段：基础转换能力

先掌握后端最常用的字符串和基础类型转换。

1. `01-基础转换/01-字符串与整数转换.md`
   - 学会用 `Atoi`、`Itoa` 处理常见整数参数。
2. `01-基础转换/02-ParseInt的base和bitSize.md`
   - 学会用 `ParseInt`、`ParseUint` 处理进制、ID 和越界。
3. `01-基础转换/03-浮点数与金额精度边界.md`
   - 学会用 `ParseFloat`、`FormatFloat`，并知道金额为什么通常不用浮点数保存。
4. `01-基础转换/04-布尔值与配置开关解析.md`
   - 学会解析环境变量和配置开关。

### 第二阶段：后端常见场景

把基础转换放进真实项目里看。

5. `02-后端场景/05-Format系列生成响应和日志.md`
   - 学会用 `FormatInt`、`FormatBool`、`FormatFloat` 生成可控文本。
6. `02-后端场景/06-Quote与Unquote处理可见文本.md`
   - 学会在日志、迁移脚本中安全显示特殊字符，同时明确它不是 SQL/HTML 安全工具。
7. `02-后端场景/07-Append系列减少临时字符串.md`
   - 学会在高频日志、字节缓冲场景中使用 `AppendInt` 等函数。
8. `02-后端场景/08-转换错误处理与输入校验.md`
   - 学会识别 `strconv.NumError`，把底层错误转换成清晰的业务错误。

### 第三阶段：小型项目实战

把多个函数组合成后端可复用代码。

9. `03-项目实战/09-分页权限库存参数解析.md`
   - 实现一个请求参数解析器，处理分页、权限、库存等字段。
10. `03-项目实战/10-短链接与迁移脚本中的进制转换.md`
    - 用进制转换做短链接编码，并把迁移脚本的风险讲清楚。

## 三、建议学习方法

每篇都建议按下面的节奏学习：

1. 先读“为什么需要它”，建立业务直觉。
2. 把“基本用法”的代码复制运行。
3. 修改输入，比如改成空字符串、超大数字、非法字符，观察错误。
4. 再看“真实后端场景示例”，理解它在项目里的位置。
5. 最后对照“本节达标标准”自查。

## 四、运行环境建议

确认本机能运行 Go：

Windows PowerShell：

```powershell
go version
```

Linux/macOS：

```bash
go version
```

如果当前目录没有 `go.mod`，可以在练习目录初始化：

Windows PowerShell：

```powershell
go mod init strconv-practice
```

Linux/macOS：

```bash
go mod init strconv-practice
```

真实项目中，通常会在已有项目模块里练习，不需要每个小例子都单独建模块。这里单独初始化只是为了让你能复制运行。

## 五、学习中的保守原则

```text
strconv 只负责类型转换，不负责业务合法性、安全转义、权限判断和数据库防注入。
```

真实后端项目中要额外注意：

- SQL 参数必须使用参数化查询，不要把用户输入拼进 SQL。
- 金额建议用“分”这样的整数保存，不要用 `float64` 保存账务金额。
- 数据迁移要可回滚，转换失败要记录并中断或进入人工处理队列。
- 解析请求参数时，要设置最大值、最小值和默认值。
- 日志中要记录足够上下文，但不要泄露密码、Token、身份证等敏感数据。
