# 04. 读取配置文件

本节目标：学完后你能用 `os.ReadFile` 读取小型配置文件，并清楚它适合什么场景。

简短引入：配置文件常用于本地开发、单机工具和简单部署。即使未来使用配置中心，你也需要理解文件读取失败时应该怎么处理。

## 一、为什么需要它

环境变量适合少量配置，但当配置项变多时，文件更容易阅读和版本管理。常见场景是读取 `config.json`、迁移 SQL、权限策略文件。

`os.ReadFile` 可以一次性把整个文件读进内存。它适合小文件，不适合超大日志、导入数据和视频文件。

```text
配置文件可以一次性读取，用户上传和大批量导入通常不要一次性读完。
```

## 二、基本用法

先创建配置文件。

Windows PowerShell：

```powershell
Set-Content -Path config.json -Value '{"env":"dev","port":8080}'
go run .
```

Linux/macOS：

```bash
printf '{"env":"dev","port":8080}\n' > config.json
go run .
```

`main.go`：

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	data, err := os.ReadFile("config.json")
	if err != nil {
		fmt.Println("read config:", err)
		return
	}

	fmt.Println(string(data))
}
```

## 三、关键代码结构

`os.ReadFile(path)` 返回 `[]byte` 和 `error`。

真实项目中一般不要只打印 `err` 后继续运行。配置读不到通常应该启动失败，因为服务不知道该连哪个数据库、监听哪个端口。

如果你要解析 JSON，可以接上 `encoding/json`：

```go
type Config struct {
	Env  string `json:"env"`
	Port int    `json:"port"`
}
```

## 四、真实后端场景示例

下面读取一个简化配置，并做基本校验：

```go
package main

import (
	"encoding/json"
	"fmt"
	"os"
)

type Config struct {
	Env  string `json:"env"`
	Port int    `json:"port"`
}

func main() {
	cfg, err := loadConfig("config.json")
	if err != nil {
		fmt.Println("config error:", err)
		os.Exit(1)
	}

	fmt.Printf("start with env=%s port=%d\n", cfg.Env, cfg.Port)
}

func loadConfig(path string) (Config, error) {
	data, err := os.ReadFile(path)
	if err != nil {
		return Config{}, fmt.Errorf("read config file: %w", err)
	}

	var cfg Config
	if err := json.Unmarshal(data, &cfg); err != nil {
		return Config{}, fmt.Errorf("parse config json: %w", err)
	}

	if cfg.Port <= 0 {
		return Config{}, fmt.Errorf("port must be positive")
	}

	return cfg, nil
}
```

这里的重点不是 JSON 本身，而是“读取、解析、校验”三个步骤都可能失败，每一步都要返回清晰错误。

## 五、注意点

文件读取路径要明确。开发时可以默认 `./config.json`，生产环境建议用环境变量传入配置路径。

配置文件不适合直接提交密钥。真实项目中常用配置文件放非敏感结构，密钥由环境变量或密钥管理系统注入。

如果配置文件可能被人工修改，要让错误信息说明是哪一步失败，而不是只写 `invalid config`。

## 六、常见误区

误区一：用 `os.ReadFile` 读取大文件。这样可能造成内存暴涨。

误区二：解析失败后使用零值继续运行。端口是 `0`、数据库地址是空字符串都会导致更隐蔽的问题。

误区三：把配置错误当成普通日志。配置错误通常应该阻止服务启动。

## 七、本节达标标准

- 能用 `os.ReadFile` 读取小文件。
- 能说明它不适合大文件。
- 能把读取错误、解析错误、校验错误分开处理。
- 知道生产配置路径应由部署环境明确指定。
