# 05. 写入日志和导出文件

本节目标：学完后你能用 `os.WriteFile` 写出简单结果文件，并理解覆盖写入的风险。

简短引入：后端项目经常要写文件，例如导出订单 CSV、生成报表、写入一次性任务结果。写文件看起来简单，但覆盖、权限和失败清理都要小心。

## 一、为什么需要它

可以理解为：写文件是把程序中的数据落到磁盘。常见场景包括订单导出、用户批量导入失败报告、短链接统计快照。

`os.WriteFile` 会一次性写入整个内容。如果目标文件已存在，它会覆盖原内容。这个特性很方便，也很危险。

```text
覆盖写入之前，要确认这个文件是否允许被覆盖。
```

## 二、基本用法

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	data := []byte("order_id,total\n1001,59.90\n")

	if err := os.WriteFile("orders.csv", data, 0644); err != nil {
		fmt.Println("write export:", err)
		return
	}

	fmt.Println("exported orders.csv")
}
```

Windows PowerShell：

```powershell
go run .
Get-Content .\orders.csv
```

Linux/macOS：

```bash
go run .
cat ./orders.csv
```

## 三、关键参数

`os.WriteFile(name, data, perm)` 有三个参数。

- `name`：目标路径。
- `data`：要写入的字节。
- `perm`：创建新文件时的权限，例如 `0644` 表示所有者可读写，其他用户只读。

注意：如果文件已经存在，权限参数通常不会重新修改已有文件权限。

## 四、真实后端场景示例

下面模拟导出订单结果：

```go
package main

import (
	"fmt"
	"os"
	"path/filepath"
	"strings"
)

func main() {
	dir := filepath.Join("data", "exports")
	if err := os.MkdirAll(dir, 0755); err != nil {
		fmt.Println("create export dir:", err)
		return
	}

	rows := []string{
		"order_id,total,status",
		"1001,59.90,paid",
		"1002,21.00,pending",
	}

	path := filepath.Join(dir, "orders.csv")
	if err := os.WriteFile(path, []byte(strings.Join(rows, "\n")+"\n"), 0644); err != nil {
		fmt.Println("write export:", err)
		return
	}

	fmt.Println("export file:", path)
}
```

这里先创建目录再写文件。真实项目中导出文件路径通常不能由用户完整指定，最多由用户选择导出类型、日期范围，最终路径由服务端生成。

## 五、注意点

`os.WriteFile` 适合小文件或中等大小结果。如果导出几十万行订单，应使用 `os.Create` 加 `bufio.Writer` 分批写入。

如果写的是重要文件，例如配置或任务状态，直接覆盖会有风险。更保守的做法是先写临时文件，再 `os.Rename` 替换，后面会单独学习。

对于用户可下载文件，要限制目录边界和文件名，不要允许用户传入任意路径。

## 六、常见误区

误区一：不知道 `os.WriteFile` 会覆盖已有文件，导致日志或报表被覆盖。

误区二：把权限写成 `0777`。后端服务生成的文件通常不应该给所有人写权限。

误区三：导出大文件也一次性拼成字符串。数据量上来后会浪费内存。

## 七、本节达标标准

- 能用 `os.WriteFile` 写出文件。
- 能解释 `0644` 的基本含义。
- 知道覆盖写入的风险。
- 能在写文件前创建必要目录。
