# Go os 包后端学习路线

适用对象：已经会写基础 Go 语法，准备进入后端工程实践的学习者。

学习目标：不是把 `os` 包每个函数背下来，而是能在真实项目中安全地处理配置、日志、文件、目录、临时数据、进程信息和退出流程。

```text
学习 os 包时，不要只问“这个函数怎么用”，要问“这个操作在生产项目里失败了怎么办、能不能回滚、会不会误删数据”。
```

## 一、学习顺序

### 1. 进程环境与路径

先理解一个后端程序运行时能从操作系统拿到什么信息：命令行参数、环境变量、当前工作目录、用户目录、可执行文件位置。

对应文章：

- `01. 命令行参数与进程信息`
- `02. 环境变量与配置读取`
- `03. 工作目录与项目路径`

学完应能做到：

- 用环境变量区分开发、测试、生产配置。
- 知道为什么后端服务不要依赖“你在终端哪个目录启动”。
- 能把配置文件、日志文件、上传文件放到稳定的位置。

### 2. 文件读写

后端项目经常要读配置、写日志、导入数据、生成导出文件。这里重点学习 `os.ReadFile`、`os.WriteFile`、`os.Open`、`os.OpenFile`、`File.Read`、`File.Write`。

对应文章：

- `04. 读取配置文件`
- `05. 写入日志和导出文件`
- `06. 大文件流式读写`
- `07. OpenFile 与文件打开模式`

学完应能做到：

- 区分一次性读取和流式读取，知道大文件要分批处理。
- 正确关闭文件，避免句柄泄漏。
- 理解追加、覆盖、创建、只读、读写这些模式在项目里的风险。

### 3. 目录与临时文件

真实项目不会只有一个文件。迁移脚本、用户上传、导出任务、缓存文件通常都需要目录管理和临时文件。

对应文章：

- `08. 创建目录与项目数据目录`
- `09. 读取目录与迁移文件`
- `10. 临时文件与上传缓存`

学完应能做到：

- 启动服务时创建必要目录。
- 按名称顺序读取迁移文件。
- 用临时文件承接中间结果，失败时能清理。

### 4. 文件状态、错误和安全操作

文件系统操作失败很常见。重点不是消灭错误，而是判断错误属于“不存在”“权限不足”“路径非法”还是“未知失败”。

对应文章：

- `11. Stat 与存在性判断`
- `12. Rename、Remove 与可回滚文件替换`
- `13. 文件权限、时间和安全边界`

学完应能做到：

- 不用“先判断存在再操作”的脆弱流程处理并发场景。
- 用 `errors.Is` 或 `os.IsNotExist` 做清晰的错误分支。
- 对删除、覆盖、权限修改保持保守。

### 5. 进程信号与退出

后端服务不是只要能启动就够了，还要能安全退出。这里学习 `os.Args`、`os.Exit`、`os.Interrupt`、`signal.Notify`、`syscall.SIGTERM` 等配合方式。

对应文章：

- `14. 命令行参数与运维入口`
- `15. 优雅退出与信号处理`

学完应能做到：

- 写一个简单的命令行入口。
- 收到 Ctrl+C 或容器停止信号时先清理资源，并在超时内退出。
- 知道 `os.Exit` 会跳过 `defer`，不能乱用。

### 6. 综合项目

把前面内容合起来，做一个小型“配置 + 日志 + 迁移文件 + 安全退出”的后端工具。

对应文章：

- `16. 综合项目：后端服务启动器`

学完应能做到：

- 组织一个能落地的小工具。
- 在启动、运行、退出三个阶段都考虑错误处理。
- 为以后学习数据库迁移、日志库、配置库打基础。

## 二、建议学习节奏

第一轮：每节先复制运行最小示例，不追求改造。

第二轮：把示例中的业务名替换成自己的项目，例如 `orders`、`users`、`articles`。

第三轮：补充失败场景，例如文件不存在、权限不足、目录不存在、并发执行。

```text
能处理失败路径，才算真正学会文件和系统交互。
```

## 三、最终练习方向

建议最后完成一个小项目：`backend-runner`。

它应具备：

- 从环境变量读取运行环境和数据目录，并打印解析后的绝对路径。
- 启动时创建日志目录、上传目录、迁移目录。
- 读取迁移文件并按顺序打印 dry-run 执行计划。
- 将运行日志追加写入文件，并处理关闭文件时的错误。
- 收到 Ctrl+C 或 SIGTERM 时写入关闭日志，并使用超时控制退出。

这个项目不连接数据库，但它模拟了真实后端服务启动时的文件系统职责。后面学习 SQL、迁移工具、日志框架、容器部署时，会更容易理解它们为什么这么设计。
