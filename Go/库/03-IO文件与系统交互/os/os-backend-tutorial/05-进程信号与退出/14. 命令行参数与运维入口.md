# 14. 命令行参数与运维入口

本节目标：学完后你能写一个支持 `serve`、`migrate`、`version` 的后端命令入口。

简短引入：一个后端程序不仅要处理 HTTP 请求，也经常要执行运维动作。把这些动作做成明确命令，比在代码里临时改开关更安全。

## 一、为什么需要它

常见场景是：

- `app serve` 启动服务。
- `app migrate` 执行数据库迁移。
- `app version` 打印版本，便于排查部署。

真实项目中通常会用 CLI 库，但先用标准库写一遍，有助于理解命令入口的边界。

```text
生产运维动作必须显式触发，不能隐藏在普通启动流程里。
```

## 二、基本用法

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	if len(os.Args) < 2 {
		usage()
		return
	}

	switch os.Args[1] {
	case "serve":
		fmt.Println("serve api")
	case "migrate":
		fmt.Println("run migrations")
	case "version":
		fmt.Println("app version dev")
	default:
		usage()
	}
}

func usage() {
	fmt.Println("usage: app serve|migrate|version")
}
```

Windows PowerShell：

```powershell
go run . version
go run . migrate
```

Linux/macOS：

```bash
go run . version
go run . migrate
```

## 三、关键代码结构

入口函数不要塞满业务逻辑。更好的结构是：解析命令，调用对应函数，统一处理错误和退出码。

`os.Exit(code)` 可以设置退出码，但它会直接结束程序，不会执行 `defer`。

因此通常在 `main` 里最后使用它，而不是在业务函数深处随意调用。

## 四、真实后端场景示例

下面把入口改成更适合扩展的结构：

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	if err := run(os.Args); err != nil {
		fmt.Fprintln(os.Stderr, "error:", err)
		os.Exit(1)
	}
}

func run(args []string) error {
	if len(args) < 2 {
		return fmt.Errorf("usage: app serve|migrate|version")
	}

	switch args[1] {
	case "serve":
		return serve()
	case "migrate":
		return migrate()
	case "version":
		fmt.Println("app version dev")
		return nil
	default:
		return fmt.Errorf("unknown command: %s", args[1])
	}
}

func serve() error {
	fmt.Println("starting api server")
	return nil
}

func migrate() error {
	fmt.Println("checking migration files")
	return nil
}
```

这种写法便于单元测试 `run(args)`，也便于将来接入配置加载、日志初始化和数据库连接。

## 五、注意点

迁移命令不要自动连接生产库执行危险操作。真实项目中应显示当前环境、数据库地址摘要、待执行版本，并要求由部署流程控制。

如果命令参数越来越复杂，不要继续手写解析。可以使用标准库 `flag` 或 CLI 框架，但原则仍然是命令明确、错误清晰、退出码可用。

## 六、常见误区

误区一：在业务函数里随处 `os.Exit`。这样会跳过 `defer`，资源可能无法释放。

误区二：启动服务时顺便自动迁移生产数据库。迁移失败会拖垮正常启动流程。

误区三：错误只打印到标准输出。运维脚本通常更容易从标准错误和退出码判断失败。

## 七、本节达标标准

- 能写出 `serve`、`migrate`、`version` 入口。
- 能用 `run(args) error` 组织命令逻辑。
- 知道 `os.Exit` 会跳过 `defer`。
- 知道迁移命令应显式、可审计、可回滚。
