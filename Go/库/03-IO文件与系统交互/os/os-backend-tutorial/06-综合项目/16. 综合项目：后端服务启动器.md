# 16. 综合项目：后端服务启动器

本节目标：学完后你能把环境变量、目录准备、日志追加、迁移 dry-run、信号处理和退出超时串成一个小工具。

简短引入：前面每节都只讲一个点。本节把它们组合起来，做一个贴近真实后端启动流程的 `backend-runner`。它不连接数据库，但会模拟服务启动时最常见的系统交互。

## 一、为什么需要它

真实后端服务启动时通常会做这些事：

- 读取运行环境和数据目录，并打印解析后的绝对路径。
- 准备日志、上传、临时目录。
- 检查迁移文件，先 dry-run 再执行。
- 写启动日志和停止日志。
- 监听 Ctrl+C 和 SIGTERM，并在超时内清理资源。

这些动作看起来普通，但如果做得粗糙，生产上就会出现配置错、目录不可写、迁移顺序错、日志丢失、退出不干净等问题。

```text
后端工程不是只把接口写通，还要让程序能被稳定地启动、观察和停止。
```

## 二、基本用法

创建一个空目录，放入下面的 `main.go`。

Windows PowerShell：

```powershell
New-Item -ItemType Directory -Force -Path migrations | Out-Null
Set-Content migrations\001_create_users.sql "CREATE TABLE users(id INT);"
$env:APP_ENV="dev"
$env:DATA_DIR="./data"
go run . migrate --dry-run
go run . serve
```

Linux/macOS：

```bash
mkdir -p migrations
printf 'CREATE TABLE users(id INT);\n' > migrations/001_create_users.sql
APP_ENV=dev DATA_DIR=./data go run . migrate --dry-run
APP_ENV=dev DATA_DIR=./data go run . serve
```

## 三、关键代码结构

下面代码刻意保持简单，不引入第三方库：

```go
package main

import (
	"context"
	"fmt"
	"os"
	"os/signal"
	"path/filepath"
	"strings"
	"syscall"
	"time"
)

type Config struct {
	Env          string
	DataDir      string
	MigrationDir string
}

func main() {
	if err := run(os.Args); err != nil {
		fmt.Fprintln(os.Stderr, "error:", err)
		os.Exit(1)
	}
}

func run(args []string) error {
	if len(args) < 2 {
		return fmt.Errorf("usage: backend-runner serve|migrate [--dry-run]|version")
	}

	cfg, err := loadConfig()
	if err != nil {
		return err
	}

	switch args[1] {
	case "serve":
		return serve(cfg)
	case "migrate":
		dryRun := len(args) >= 3 && args[2] == "--dry-run"
		return printMigrationPlan(cfg.MigrationDir, dryRun)
	case "version":
		fmt.Println("backend-runner dev")
		return nil
	default:
		return fmt.Errorf("unknown command: %s", args[1])
	}
}

func loadConfig() (Config, error) {
	dataDir, err := filepath.Abs(getEnv("DATA_DIR", "./data"))
	if err != nil {
		return Config{}, fmt.Errorf("resolve DATA_DIR: %w", err)
	}

	migrationDir, err := filepath.Abs(getEnv("MIGRATION_DIR", "./migrations"))
	if err != nil {
		return Config{}, fmt.Errorf("resolve MIGRATION_DIR: %w", err)
	}

	return Config{
		Env:          getEnv("APP_ENV", "dev"),
		DataDir:      dataDir,
		MigrationDir: migrationDir,
	}, nil
}

func serve(cfg Config) error {
	fmt.Println("env:", cfg.Env)
	fmt.Println("data dir:", cfg.DataDir)
	fmt.Println("migration dir:", cfg.MigrationDir)

	if err := prepareDirs(cfg.DataDir); err != nil {
		return err
	}
	if err := appendLog(cfg.DataDir, "service starting env="+cfg.Env); err != nil {
		return err
	}
	if err := printMigrationPlan(cfg.MigrationDir, true); err != nil {
		return err
	}

	stop := make(chan os.Signal, 1)
	signal.Notify(stop, os.Interrupt, syscall.SIGTERM)
	defer signal.Stop(stop)

	fmt.Println("service running, press Ctrl+C to stop")
	sig := <-stop
	fmt.Println("received signal:", sig)

	ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
	defer cancel()
	if err := shutdown(ctx, cfg); err != nil {
		return err
	}

	fmt.Println("service stopped")
	return nil
}

func shutdown(ctx context.Context, cfg Config) error {
	done := make(chan error, 1)
	go func() {
		if err := appendLog(cfg.DataDir, "service stopped"); err != nil {
			done <- err
			return
		}
		time.Sleep(1 * time.Second)
		done <- nil
	}()

	select {
	case err := <-done:
		return err
	case <-ctx.Done():
		return fmt.Errorf("shutdown timeout: %w", ctx.Err())
	}
}

func prepareDirs(dataDir string) error {
	for _, name := range []string{"logs", "uploads", "tmp", "exports"} {
		path := filepath.Join(dataDir, name)
		if err := os.MkdirAll(path, 0755); err != nil {
			return fmt.Errorf("create dir %s: %w", path, err)
		}
	}
	return nil
}

func appendLog(dataDir, message string) error {
	path := filepath.Join(dataDir, "logs", "app.log")
	file, err := os.OpenFile(path, os.O_CREATE|os.O_WRONLY|os.O_APPEND, 0644)
	if err != nil {
		return fmt.Errorf("open log file: %w", err)
	}

	closeFile := true
	defer func() {
		if closeFile {
			_ = file.Close()
		}
	}()

	line := fmt.Sprintf("%s %s\n", time.Now().Format(time.RFC3339), message)
	if _, err := file.WriteString(line); err != nil {
		return fmt.Errorf("write log file: %w", err)
	}
	if err := file.Close(); err != nil {
		closeFile = false
		return fmt.Errorf("close log file: %w", err)
	}
	closeFile = false
	return nil
}

func printMigrationPlan(dir string, dryRun bool) error {
	entries, err := os.ReadDir(dir)
	if err != nil {
		return fmt.Errorf("read migration dir: %w", err)
	}

	mode := "plan"
	if dryRun {
		mode = "dry-run"
	}
	fmt.Println("migration", mode+":")

	count := 0
	for _, entry := range entries {
		if entry.IsDir() || !strings.HasSuffix(entry.Name(), ".sql") {
			continue
		}
		count++
		fmt.Println(" -", entry.Name())
	}
	if count == 0 {
		fmt.Println(" - no migration files")
	}

	if !dryRun {
		fmt.Println("execute migrations here with version table, transaction, and rollback plan")
	}
	return nil
}

func getEnv(key, fallback string) string {
	value, ok := os.LookupEnv(key)
	if !ok || value == "" {
		return fallback
	}
	return value
}
```

## 四、真实后端场景示例

这个小工具对应真实项目里的几个模块：

- `loadConfig` 对应配置加载，并把路径解析成绝对路径。
- `prepareDirs` 对应启动自检。
- `appendLog` 对应日志落盘，并检查关闭文件时的错误。
- `printMigrationPlan` 对应迁移 dry-run。
- `signal.Notify` 对应 Ctrl+C 和 SIGTERM 的优雅退出。
- `shutdown` 对应退出清理和超时控制。

如果继续扩展，下一步不是把所有逻辑塞进 `main.go`，而是拆成 `config`、`storage`、`migrate`、`server` 几个包。

## 五、注意点

这个示例没有真正执行 SQL，因为真实迁移应该由迁移工具管理版本、事务和回滚。这里的 `--dry-run` 可以理解为“先打印计划，让人确认会发生什么”。

日志示例也只是教学用。真实生产环境通常会使用日志库，并配置日志轮转、结构化字段和采集系统。

上传目录只做了创建，没有做完整上传处理。真实上传要生成文件 ID、限制大小、校验类型，并避免用户输入直接决定路径。

退出超时要小于部署平台的强制停止时间。比如平台 30 秒后强杀进程，应用内部可以设置 20 到 25 秒清理窗口。

```text
教程里的代码用于建立工程直觉，生产代码还要补齐鉴权、参数校验、事务、监控和回滚。
```

## 六、常见误区

误区一：所有启动逻辑都写在 `main` 里。这样难测试，也难复用。

误区二：服务启动时遇到目录或配置错误仍继续运行。后面业务请求会更难排查。

误区三：迁移命令没有 dry-run，上来就改生产数据库。迁移应该先能看计划，再由部署流程执行。

误区四：只监听 Ctrl+C，不监听 SIGTERM。容器停止和滚动发布通常不会只靠 Ctrl+C。

误区五：认为本地文件可以替代数据库事务。订单、库存、支付这类核心状态应放在数据库里，并通过事务保护。

## 七、本节达标标准

- 能独立运行 `backend-runner migrate --dry-run` 和 `backend-runner serve`。
- 能通过环境变量修改运行环境和数据目录，并看到解析后的绝对路径。
- 能创建项目目录并追加日志。
- 能扫描迁移文件并说明真实迁移还需要版本表、事务和回滚策略。
- 能处理 Ctrl+C 和 SIGTERM，并在退出超时内写入关闭日志。
