# 09. 读取目录与迁移文件

本节目标：学完后你能读取目录文件列表，并按名称顺序处理迁移脚本。

简短引入：数据库迁移、静态模板、批量导入文件都需要读取目录。对后端开发者来说，重点不是“列出文件”，而是按明确规则处理文件。

## 一、为什么需要它

迁移文件通常会命名为 `001_create_users.sql`、`002_create_orders.sql`。程序读取目录后要按顺序执行，不能依赖操作系统返回的任意顺序。

Go 的 `os.ReadDir` 返回目录项，并且会按文件名排序。即便如此，你仍应该在业务上约定清楚命名规则。

```text
迁移文件必须有稳定顺序，不能靠“看起来差不多”的文件名。
```

## 二、基本用法

先准备文件。

Windows PowerShell：

```powershell
New-Item -ItemType Directory -Force -Path migrations | Out-Null
Set-Content migrations\001_create_users.sql "CREATE TABLE users(id INT);"
Set-Content migrations\002_create_orders.sql "CREATE TABLE orders(id INT);"
go run .
```

Linux/macOS：

```bash
mkdir -p migrations
printf 'CREATE TABLE users(id INT);\n' > migrations/001_create_users.sql
printf 'CREATE TABLE orders(id INT);\n' > migrations/002_create_orders.sql
go run .
```

`main.go`：

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	entries, err := os.ReadDir("migrations")
	if err != nil {
		fmt.Println("read migrations:", err)
		return
	}

	for _, entry := range entries {
		if entry.IsDir() {
			continue
		}
		fmt.Println(entry.Name())
	}
}
```

## 三、关键代码结构

`os.ReadDir(path)` 返回 `[]DirEntry`。

`entry.Name()` 是文件名，不包含目录前缀。

`entry.IsDir()` 用来判断是不是目录。

如果需要更多信息，比如大小、修改时间，可以调用 `entry.Info()`。

## 四、真实后端场景示例

下面模拟打印迁移执行计划：

```go
package main

import (
	"fmt"
	"os"
	"path/filepath"
	"strings"
)

func main() {
	if err := printMigrationPlan("migrations"); err != nil {
		fmt.Println("migration plan error:", err)
		os.Exit(1)
	}
}

func printMigrationPlan(dir string) error {
	entries, err := os.ReadDir(dir)
	if err != nil {
		return fmt.Errorf("read migration dir: %w", err)
	}

	for _, entry := range entries {
		if entry.IsDir() || !strings.HasSuffix(entry.Name(), ".sql") {
			continue
		}

		path := filepath.Join(dir, entry.Name())
		data, err := os.ReadFile(path)
		if err != nil {
			return fmt.Errorf("read migration %s: %w", entry.Name(), err)
		}

		fmt.Printf("will run %s, size=%d bytes\n", entry.Name(), len(data))
	}

	return nil
}
```

真实项目中执行 SQL 时要使用迁移工具记录已执行版本，不能每次启动都把所有 SQL 再跑一遍。

```text
数据库迁移要有版本记录和事务边界，不能靠目录扫描直接重复执行。
```

## 五、注意点

迁移文件命名建议用固定宽度编号，例如 `001`、`002`，不要混用 `1`、`10`、`2`。按字符串排序时，`10` 可能排在 `2` 前面。

读取目录时要过滤文件类型，不要把临时文件、备份文件、编辑器缓存当成迁移脚本。

执行迁移前要明确失败策略：失败时回滚事务，还是停止在当前版本等待人工处理。

## 六、常见误区

误区一：目录里有什么就执行什么。这样容易把 `.bak` 或临时文件也执行了。

误区二：迁移脚本没有版本表。程序无法知道哪些脚本已经执行。

误区三：不考虑事务边界。部分执行成功、部分失败会让数据库处于不一致状态。

## 七、本节达标标准

- 能用 `os.ReadDir` 列出目录项。
- 能过滤目录和非目标扩展名文件。
- 能按迁移文件命名规则理解执行顺序。
- 知道真实迁移需要版本记录、事务和回滚策略。
