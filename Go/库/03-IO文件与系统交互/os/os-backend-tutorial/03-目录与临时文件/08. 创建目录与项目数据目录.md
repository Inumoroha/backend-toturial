# 08. 创建目录与项目数据目录

本节目标：学完后你能在服务启动时创建日志、上传和导出目录。

简短引入：后端服务运行时经常需要写文件。如果目录不存在，写文件会失败。启动阶段检查和创建目录，是比运行时临时补救更清晰的做法。

## 一、为什么需要它

可以理解为：目录是后端服务存放本地状态的边界。常见目录包括 `logs`、`uploads`、`exports`、`tmp`、`migrations`。

真实项目中通常会把这些目录放在统一的数据目录下，例如 `DATA_DIR=/var/lib/myapp`。

```text
服务启动时应该尽早发现目录不可写，而不是等用户上传时才失败。
```

## 二、基本用法

```go
package main

import (
	"fmt"
	"os"
	"path/filepath"
)

func main() {
	dir := filepath.Join("data", "uploads")
	if err := os.MkdirAll(dir, 0755); err != nil {
		fmt.Println("create dir:", err)
		return
	}

	fmt.Println("ready:", dir)
}
```

Windows PowerShell：

```powershell
go run .
Get-ChildItem .\data
```

Linux/macOS：

```bash
go run .
ls ./data
```

## 三、关键参数

`os.Mkdir(name, perm)` 只创建一级目录，如果父目录不存在会失败。

`os.MkdirAll(path, perm)` 会创建整条路径，父目录不存在也会补齐。后端启动时更常用它。

权限 `0755` 表示所有者可读写执行，其他用户可读执行。对目录来说，“执行”可以理解为允许进入目录。

## 四、真实后端场景示例

下面模拟服务启动时准备本地目录：

```go
package main

import (
	"fmt"
	"os"
	"path/filepath"
)

func main() {
	dataDir := os.Getenv("DATA_DIR")
	if dataDir == "" {
		dataDir = "./data"
	}

	for _, name := range []string{"logs", "uploads", "exports", "tmp"} {
		path := filepath.Join(dataDir, name)
		if err := os.MkdirAll(path, 0755); err != nil {
			fmt.Println("prepare dir:", path, err)
			os.Exit(1)
		}
	}

	fmt.Println("data dirs ready:", dataDir)
}
```

这个例子把目录准备放在启动阶段。真实项目中如果目录创建失败，通常要直接启动失败，因为继续运行会让后面的业务请求不断出错。

## 五、注意点

不要把上传目录放在源码目录里。源码目录适合代码和配置模板，不适合运行时数据。

不要给目录设置过宽权限，例如 `0777`。如果确实需要跨用户写入，应该先确认部署用户、容器挂载权限和安全边界。

如果服务部署在容器里，要确认数据目录是否挂载到持久卷。否则容器重建后文件可能丢失。

## 六、常见误区

误区一：只创建文件，不创建父目录。结果 `open data/logs/app.log: no such file or directory`。

误区二：用 `Mkdir` 创建多级目录。父目录不存在时它会失败。

误区三：开发时写到 `./data`，生产也默认写到当前目录。生产路径应该明确配置。

## 七、本节达标标准

- 能用 `os.MkdirAll` 创建多级目录。
- 能解释目录权限中执行位的基本含义。
- 能在启动阶段准备日志、上传、导出目录。
- 知道运行时数据不应该随意写进源码目录。
