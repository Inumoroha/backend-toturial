# 13. 文件权限、时间和安全边界

本节目标：学完后你能设置基本文件权限，并知道什么时候不要随意改权限。

简短引入：权限不是 Linux 运维专属知识。后端服务写出的日志、导出文件、上传文件，都涉及谁能读、谁能写、谁能执行。

## 一、为什么需要它

可以理解为：文件权限是操作系统层面的第一道访问控制。应用层有登录和权限系统，但本地文件如果权限过宽，仍可能被不该访问的人读到。

`os.Chmod` 可以修改权限，`os.Chtimes` 可以修改访问时间和修改时间。后端中更常见的是创建文件时选择合适权限，而不是事后到处改。

```text
权限应该默认收紧，只有明确需要时再放开。
```

## 二、基本用法

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	if err := os.WriteFile("secret.txt", []byte("token example\n"), 0600); err != nil {
		fmt.Println("write secret:", err)
		return
	}

	if err := os.Chmod("secret.txt", 0600); err != nil {
		fmt.Println("chmod secret:", err)
		return
	}

	fmt.Println("secret file created")
}
```

Windows PowerShell：

```powershell
go run .
Get-Item .\secret.txt
```

Linux/macOS：

```bash
go run .
ls -l ./secret.txt
```

## 三、关键参数

常见权限：

- `0600`：只有文件所有者可读写，适合密钥、私有配置。
- `0644`：所有者可读写，其他用户只读，适合普通导出文件。
- `0755`：目录或可执行文件常用。

在 Windows 上，Unix 风格权限的行为会有差异。写跨平台代码时，不要把权限控制完全建立在 `Chmod` 上。

## 四、真实后端场景示例

下面模拟写入本地私有配置：

```go
package main

import (
	"fmt"
	"os"
	"path/filepath"
)

func main() {
	if err := saveLocalSecret("dev-token-123"); err != nil {
		fmt.Println("save local secret:", err)
	}
}

func saveLocalSecret(token string) error {
	dir := filepath.Join("data", "private")
	if err := os.MkdirAll(dir, 0700); err != nil {
		return fmt.Errorf("create private dir: %w", err)
	}

	path := filepath.Join(dir, "token.txt")
	if err := os.WriteFile(path, []byte(token+"\n"), 0600); err != nil {
		return fmt.Errorf("write token file: %w", err)
	}

	return nil
}
```

真实生产项目中密钥通常不建议长期放在普通文件里，更推荐使用环境变量、密钥管理系统或平台挂载的密钥文件。

## 五、注意点

不要为了省事把文件和目录权限都设成 `0777`。这会让同一机器上的其他用户也能写入，带来篡改风险。

`os.Chown` 在很多环境需要管理员权限，跨平台行为也不一致。应用程序里一般不应该依赖它来修复部署权限。

文件时间可以用于缓存失效、静态文件刷新判断，但不要把它当成强一致业务依据。订单、库存、支付状态应该以数据库事务和业务字段为准。

## 六、常见误区

误区一：认为应用层鉴权足够，忽略文件权限。日志和导出文件也可能包含敏感信息。

误区二：遇到权限错误就改成 `0777`。这只是掩盖部署问题。

误区三：用文件修改时间判断业务顺序。真实业务顺序应该由数据库字段、事务和日志记录保证。

## 七、本节达标标准

- 能选择 `0600`、`0644`、`0755` 这类常见权限。
- 能说明为什么不要使用 `0777`。
- 能理解 Windows 和 Linux 权限行为可能不同。
- 知道敏感文件应默认收紧权限。
