# 11. Stat 与存在性判断

本节目标：学完后你能判断文件是否存在、是否是目录，并用清晰错误分支处理失败。

简短引入：后端服务经常要判断配置文件、迁移目录、上传文件是否存在。这里真正重要的是：判断只是辅助，最终操作仍然可能失败。

## 一、为什么需要它

可以理解为：`Stat` 是询问操作系统“这个路径现在是什么状态”。它能告诉你文件大小、是否目录、修改时间等。

但文件系统是会变化的。你刚判断文件存在，下一瞬间它可能被删除。所以真实项目中不能只依赖提前判断。

```text
先检查再操作不能替代操作时的错误处理。
```

## 二、基本用法

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	info, err := os.Stat("config.json")
	if err != nil {
		if os.IsNotExist(err) {
			fmt.Println("config file does not exist")
			return
		}
		fmt.Println("stat config:", err)
		return
	}

	fmt.Println("name:", info.Name())
	fmt.Println("size:", info.Size())
	fmt.Println("is dir:", info.IsDir())
}
```

Windows PowerShell：

```powershell
Set-Content config.json "{}"
go run .
```

Linux/macOS：

```bash
printf '{}\n' > config.json
go run .
```

## 三、关键代码结构

`os.Stat(path)` 会跟随符号链接。

`os.Lstat(path)` 不跟随符号链接。初学阶段先知道有这个区别即可，涉及上传目录和安全边界时要更谨慎。

`os.IsNotExist(err)` 判断是否不存在。

`os.IsPermission(err)` 判断是否权限不足。

## 四、真实后端场景示例

下面在启动时检查迁移目录：

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	if err := ensureMigrationDir("migrations"); err != nil {
		fmt.Println("startup check failed:", err)
		os.Exit(1)
	}

	fmt.Println("migration dir is ready")
}

func ensureMigrationDir(path string) error {
	info, err := os.Stat(path)
	if err != nil {
		if os.IsNotExist(err) {
			return fmt.Errorf("migration dir missing: %s", path)
		}
		if os.IsPermission(err) {
			return fmt.Errorf("no permission to access migration dir: %s", path)
		}
		return fmt.Errorf("stat migration dir: %w", err)
	}

	if !info.IsDir() {
		return fmt.Errorf("migration path is not a directory: %s", path)
	}

	return nil
}
```

这个检查适合启动阶段。真正读取迁移文件时仍要处理 `ReadDir` 和 `ReadFile` 的错误。

## 五、注意点

如果你只是想读取文件，通常可以直接 `os.ReadFile`，然后根据错误处理“不存在”或“权限不足”。不要为了读取文件先写一大段 `Stat`。

如果你要创建文件并且不能覆盖旧文件，应该用 `OpenFile` 的 `O_CREATE|O_EXCL`，而不是先 `Stat` 再创建。因为并发时两个进程可能同时判断“不存在”。

## 六、常见误区

误区一：先 `Stat` 不存在，再 `WriteFile` 创建，以为这样不会覆盖。并发场景下仍然可能出问题。

误区二：只处理“不存在”，忽略权限错误。生产上权限错误很常见。

误区三：用 `Stat` 判断路径安全。路径安全还涉及清洗、边界目录、符号链接等问题。

## 七、本节达标标准

- 能用 `os.Stat` 获取文件信息。
- 能判断文件是否不存在、是否权限不足。
- 能检查路径是否目录。
- 知道检查不能替代最终操作的错误处理。
