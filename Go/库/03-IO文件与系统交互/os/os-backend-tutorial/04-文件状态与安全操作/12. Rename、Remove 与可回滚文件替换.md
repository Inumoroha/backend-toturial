# 12. Rename、Remove 与可回滚文件替换

本节目标：学完后你能安全地替换结果文件，并理解删除操作的边界。

简短引入：后端任务常常要更新本地文件，例如生成最新库存快照、刷新短链接黑名单、写入任务状态。直接覆盖有风险，先写临时文件再替换更稳妥。

## 一、为什么需要它

`os.Rename` 可以把文件移动或改名。常见做法是先写 `xxx.tmp`，全部成功后再改名为 `xxx.json`。

`os.Remove` 删除文件或空目录，`os.RemoveAll` 递归删除目录。后者非常危险，必须限制路径边界。

```text
递归删除只能用于程序明确拥有的目录，不能用于用户传入的任意路径。
```

## 二、基本用法

```go
package main

import (
	"fmt"
	"os"
)

func main() {
	if err := os.WriteFile("status.tmp", []byte("ok\n"), 0644); err != nil {
		fmt.Println("write temp:", err)
		return
	}

	if err := os.Rename("status.tmp", "status.txt"); err != nil {
		fmt.Println("rename:", err)
		return
	}

	fmt.Println("status updated")
}
```

Windows PowerShell：

```powershell
go run .
Get-Content .\status.txt
```

Linux/macOS：

```bash
go run .
cat ./status.txt
```

## 三、关键代码结构

`os.Rename(old, new)` 用于重命名或移动。

`os.Remove(path)` 删除单个文件或空目录。

`os.RemoveAll(path)` 递归删除。它适合清理程序创建的临时目录，不适合直接处理用户输入。

同一目录下写临时文件再 `Rename`，通常比跨目录移动更可靠。

## 四、真实后端场景示例

下面模拟安全更新库存快照：

```go
package main

import (
	"fmt"
	"os"
	"path/filepath"
)

func main() {
	data := []byte(`{"sku_1001":20,"sku_1002":0}` + "\n")
	if err := replaceFile(filepath.Join("data", "inventory.json"), data); err != nil {
		fmt.Println("replace inventory:", err)
	}
}

func replaceFile(finalPath string, data []byte) error {
	dir := filepath.Dir(finalPath)
	if err := os.MkdirAll(dir, 0755); err != nil {
		return fmt.Errorf("create dir: %w", err)
	}

	tmp, err := os.CreateTemp(dir, ".replace-*.tmp")
	if err != nil {
		return fmt.Errorf("create temp file: %w", err)
	}
	tmpName := tmp.Name()
	defer os.Remove(tmpName)

	if _, err := tmp.Write(data); err != nil {
		tmp.Close()
		return fmt.Errorf("write temp file: %w", err)
	}
	if err := tmp.Close(); err != nil {
		return fmt.Errorf("close temp file: %w", err)
	}

	if err := os.Rename(tmpName, finalPath); err != nil {
		return fmt.Errorf("rename temp file: %w", err)
	}

	return nil
}
```

这个流程比 `os.WriteFile(finalPath, data, 0644)` 更保守。失败时旧文件通常还在，成功时新内容一次性替换到位。

## 五、注意点

Windows 上如果目标文件正在被其他进程打开，`Rename` 可能失败。真实项目要记录错误并允许重试。

如果数据很重要，只靠写文件还不够，可能需要 `Sync`、备份文件、版本号和回滚流程。初学阶段先掌握“临时文件 + 替换”的基本形态。

删除文件时要打印或记录被删除的绝对路径，避免配置错误时删错目录。

## 六、常见误区

误区一：用 `RemoveAll` 清理用户传入路径。路径一旦被构造错误，可能删除重要数据。

误区二：直接覆盖状态文件。程序中断时可能留下半截内容。

误区三：跨磁盘创建临时文件再 `Rename`。这可能失败，最好在目标目录里创建临时文件。

## 七、本节达标标准

- 能用 `os.Rename` 替换文件。
- 能解释为什么先写临时文件更安全。
- 能区分 `Remove` 和 `RemoveAll`。
- 知道递归删除必须限制在程序拥有的目录内。
