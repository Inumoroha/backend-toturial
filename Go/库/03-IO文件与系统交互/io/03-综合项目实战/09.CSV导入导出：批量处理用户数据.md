# 9. CSV 导入导出：批量处理用户数据

本节目标：学完后，你能用 `encoding/csv` 读取和生成 CSV，并处理导入时的错误行。

## 简短引入

后台系统经常有批量导入导出：导入用户、导出订单、调整库存、迁移权限。CSV 是最常见的交换格式之一。

不要用 `strings.Split(line, ",")` 解析 CSV。真实 CSV 可能包含引号、逗号、换行和空字段。

```text
结构化格式优先使用标准解析器，不要靠手写字符串切分碰运气。
```

## 一、为什么需要它

假设运营要批量创建用户，上传 `users.csv`：

```text
email,name
alice@example.com,Alice
bob@example.com,Bob
bad-email,Tom
```

后端要读取文件、校验字段、导入合法数据、记录错误行。真实项目中还要配合数据库事务和幂等设计。

## 二、基本用法

准备 CSV：

Windows PowerShell：

```powershell
@"
email,name
alice@example.com,Alice
bob@example.com,Bob
"@ | Set-Content .\users.csv
```

Linux/macOS：

```bash
cat > users.csv <<'EOF'
email,name
alice@example.com,Alice
bob@example.com,Bob
EOF
```

新建 `main.go`：

```go
package main

import (
	"encoding/csv"
	"fmt"
	"os"
)

func main() {
	file, err := os.Open("users.csv")
	if err != nil {
		panic(err)
	}
	defer file.Close()

	reader := csv.NewReader(file)
	records, err := reader.ReadAll()
	if err != nil {
		panic(err)
	}

	for _, record := range records {
		fmt.Println(record)
	}
}
```

运行：

```powershell
go run .\main.go
```

Linux/macOS：

```bash
go run ./main.go
```

## 三、关键参数/语法/代码结构

`csv.NewReader(file)` 创建 CSV 读取器。它会处理引号和逗号等细节。

`ReadAll()` 一次性读取全部记录，只适合小文件。大文件应该循环 `Read()`。

`csv.NewWriter(file)` 用于写 CSV。写完要调用 `Flush()`，并检查 `writer.Error()`。

## 四、真实后端场景示例

下面是一版更贴近真实导入的写法：逐行读取，校验邮箱，错误行写入报告。

```go
package main

import (
	"encoding/csv"
	"fmt"
	"io"
	"os"
	"strings"
)

type User struct {
	Email string
	Name  string
}

func ImportUsers(inputPath, errorPath string) ([]User, error) {
	input, err := os.Open(inputPath)
	if err != nil {
		return nil, err
	}
	defer input.Close()

	errorFile, err := os.Create(errorPath)
	if err != nil {
		return nil, err
	}
	defer errorFile.Close()

	reader := csv.NewReader(input)
	errorWriter := csv.NewWriter(errorFile)

	if err := errorWriter.Write([]string{"line", "reason"}); err != nil {
		return nil, err
	}

	header, err := reader.Read()
	if err != nil {
		return nil, err
	}
	if len(header) != 2 || header[0] != "email" || header[1] != "name" {
		return nil, fmt.Errorf("invalid csv header")
	}

	var users []User
	lineNo := 1
	for {
		lineNo++
		record, err := reader.Read()
		if err == io.EOF {
			break
		}
		if err != nil {
			_ = errorWriter.Write([]string{fmt.Sprint(lineNo), err.Error()})
			continue
		}
		if len(record) != 2 {
			_ = errorWriter.Write([]string{fmt.Sprint(lineNo), "invalid column count"})
			continue
		}
		if !strings.Contains(record[0], "@") {
			_ = errorWriter.Write([]string{fmt.Sprint(lineNo), "invalid email"})
			continue
		}

		users = append(users, User{Email: record[0], Name: record[1]})
	}

	errorWriter.Flush()
	if err := errorWriter.Error(); err != nil {
		return nil, err
	}
	return users, nil
}

func main() {
	users, err := ImportUsers("users.csv", "users_errors.csv")
	if err != nil {
		panic(err)
	}
	fmt.Printf("valid users: %+v\n", users)
}
```

真实项目中，`users` 可能会分批写入数据库。写入时要考虑唯一键冲突、事务边界和失败回滚。

## 五、注意点

CSV 导入不是只要解析成功就能入库。还要做字段校验、权限校验、重复数据处理和错误报告。

大批量导入时不要把所有行都放到内存里。可以按批次收集，例如每 500 行写一次数据库。

数据库写入要使用参数化查询，并明确事务边界。导入任务通常要决定“全部成功才提交”还是“允许部分成功”。

```text
批量导入前先定义失败策略：全量回滚，还是部分成功并输出错误报告。
```

## 六、生产版改造建议

如果 CSV 导入要进入真实项目，可以继续增强：

- 上传后先创建导入任务，异步处理，不要让 HTTP 请求一直等待。
- 错误报告按任务 ID 命名，并提供下载入口。
- 对导入文件做大小限制、行数限制和表头校验。
- 数据库写入使用批量插入，并控制每批事务大小。
- 用唯一键或导入批次号实现幂等，避免重复上传造成重复数据。
- 导入完成后记录成功数、失败数、耗时和操作人。

```text
批量导入要先设计失败策略和幂等策略，再写解析代码。
```

## 七、常见误区

误区一：用 `strings.Split` 解析 CSV。遇到引号和逗号会解析错误。

误区二：导入失败只返回一条错误。运营更需要知道哪些行失败、为什么失败。

误区三：一次性开启超大事务。数据量大时可能锁表、占用连接、影响线上服务。

误区四：没有幂等设计。用户重复上传同一个文件时可能造成重复数据。

## 八、小练习

把用户导入改成“库存调整导入”。

CSV 表头：

```text
sku,delta,reason
```

要求：

- `sku` 不能为空。
- `delta` 要能解析为整数，允许正数和负数。
- `reason` 不能为空。
- 错误行写入 `stock_adjust_errors.csv`。

思考：真实扣减库存时，为什么不能只靠 CSV 解析结果直接更新数据库？

## 九、本节达标标准

- 能用 `encoding/csv` 读取 CSV。
- 能逐行处理 CSV 并识别 EOF。
- 能输出错误行报告。
- 知道导入数据库前要做校验、事务和幂等设计。
- 知道不应该用简单字符串切分解析 CSV。
