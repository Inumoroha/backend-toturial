# Go 后端工程师 IO 系统学习路线

这套路线上手目标很明确：不是把 `io`、`os`、`bufio`、`net/http` 的所有 API 背下来，而是能在真实后端项目里安全地处理文件、请求体、响应流、日志、导入导出、批处理和资源关闭。

```text
后端里的 IO，本质上是在和“不可靠的外部世界”打交道：磁盘可能满，请求可能中断，文件可能很大，用户输入可能有风险。
```

## 一、学习目标

学完后，你应该能够：

- 理解 Go 中 `io.Reader`、`io.Writer`、`io.Closer` 的常见组合方式。
- 能写出可运行、可关闭、可处理错误的文件读写代码。
- 能处理大文件，不把所有数据一次性读进内存。
- 能在 HTTP 接口里安全读取请求体、限制大小、处理上传文件。
- 能写业务日志、审计日志、导入导出文件，并理解生产风险。
- 能用临时文件、原子替换、备份和回滚思路降低线上事故概率。
- 能围绕错误处理、资源释放、超时取消做保守工程设计。

## 二、学习阶段

### 阶段 1：建立 IO 直觉

对应章节：

- `01-IO基础能力/01.Reader与Writer：理解Go后端IO的统一入口.md`
- `01-IO基础能力/02.文件读写：保存用户资料和业务数据.md`

重点是理解：很多东西都可以当成“读数据的来源”或“写数据的目标”。文件、网络请求、内存字符串、压缩流、HTTP 响应体，在 Go 里经常都通过 `Reader` 和 `Writer` 串起来。

### 阶段 2：掌握文件系统操作

对应章节：

- `01-IO基础能力/03.目录与路径：管理项目里的上传和导出文件.md`
- `01-IO基础能力/04.缓冲IO：逐行处理日志和大文件.md`

重点是：真实项目里不能只会 `ReadFile`。日志、导入文件、迁移脚本、上传文件经常很大，需要按块或逐行处理。

### 阶段 3：进入后端业务场景

对应章节：

- `02-后端项目场景/05.HTTP请求体：安全读取JSON接口输入.md`
- `02-后端项目场景/06.文件上传：处理头像和附件.md`
- `02-后端项目场景/07.日志写入：记录业务行为与排障信息.md`
- `02-后端项目场景/08.流式复制：下载导出文件和代理响应.md`
- `04-专题增强/11.IO错误处理与资源关闭：写出可靠的后端IO代码.md`

重点是：IO 代码一旦放到接口里，就要考虑大小限制、超时、错误返回、用户输入、权限边界和资源释放。

### 阶段 4：综合项目落地

对应章节：

- `03-综合项目实战/09.CSV导入导出：批量处理用户数据.md`
- `03-综合项目实战/10.综合项目：订单审计日志归档工具.md`

重点是把前面的能力串起来：读取、校验、写入、临时文件、可回滚、日志、错误处理和命令行参数。

## 三、建议练习顺序

1. 先阅读每章的“为什么需要它”和“基本用法”。
2. 把每章代码复制到临时目录里运行，不要只看。
3. 每章至少改一个业务字段，例如把用户改成订单、文章、库存。
4. 每章练习完，对照“本节达标标准”检查。
5. 最后做第 10 章综合项目，把散点能力连成一个小工具。
6. 再阅读第 11 章专题，把错误处理和资源关闭习惯补齐。

## 四、推荐本地练习目录

Windows PowerShell：

```powershell
mkdir go-io-practice
cd go-io-practice
go mod init go-io-practice
```

Linux/macOS：

```bash
mkdir go-io-practice
cd go-io-practice
go mod init go-io-practice
```

真实项目中通常不会把所有练习代码塞进一个 `main.go`。建议每章新建一个目录，例如：

```text
go-io-practice/
  01-reader-writer/
    main.go
  02-file-read-write/
    main.go
  03-path-dir/
    main.go
```

## 五、学习时最重要的保守原则

```text
凡是来自用户、网络、文件、环境变量、命令行参数的数据，都先当成不可信输入处理。
```

```text
凡是打开的文件、响应体、请求体、压缩流，都要明确谁负责关闭。
```

```text
凡是可能很大的数据，都优先考虑流式处理，不要默认一次性读入内存。
```

```text
凡是会覆盖生产数据的写入，都要考虑临时文件、备份、原子替换和可回滚。
```

## 六、最终项目能力图

学完以后，你可以尝试实现一个后端常见的小工具：

输入：

- 一个订单审计日志目录。
- 多个按日期命名的日志文件。
- 一个归档输出目录。

处理：

- 按行读取日志。
- 校验 JSON。
- 过滤指定日期或订单状态。
- 写入临时文件。
- 成功后原子替换为正式归档文件。
- 失败时保留错误报告，不破坏原文件。

输出：

- 归档文件。
- 错误日志。
- 处理统计信息。

这个项目不大，但会覆盖后端 IO 的核心能力。
