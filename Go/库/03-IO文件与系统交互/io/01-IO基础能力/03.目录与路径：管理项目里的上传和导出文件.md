# 3. 目录与路径：管理项目里的上传和导出文件

本节目标：学完后，你能安全创建目录、拼接路径，并管理项目中的上传和导出文件。

## 简短引入

后端项目经常需要按业务分类存文件。例如头像放在 `uploads/avatar`，订单导出放在 `exports/order`，任务报告放在 `reports/job`。

路径处理如果随手拼字符串，代码在 Windows 和 Linux 上都容易出问题，甚至留下安全风险。

## 一、为什么需要它

真实项目中常见目录设计：

```text
data/
  uploads/
    avatar/
  exports/
    users/
  reports/
    migration/
```

这些目录可能在第一次启动时不存在，也可能部署到容器或服务器后权限不同。后端代码要能明确创建目录，并在失败时给出错误。

## 二、基本用法

新建 `main.go`：

```go
package main

import (
	"fmt"
	"os"
	"path/filepath"
)

func main() {
	dir := filepath.Join("data", "exports", "users")
	if err := os.MkdirAll(dir, 0755); err != nil {
		panic(err)
	}

	path := filepath.Join(dir, "users_20260706.csv")
	if err := os.WriteFile(path, []byte("id,name\n1001,Alice\n"), 0644); err != nil {
		panic(err)
	}

	fmt.Println(path)
}
```

运行：

Windows PowerShell：

```powershell
go run .\main.go
Get-ChildItem -Recurse .\data
```

Linux/macOS：

```bash
go run ./main.go
find ./data -type f
```

## 三、关键参数/语法/代码结构

`filepath.Join(...)` 会按当前系统规则拼接路径。Windows 使用反斜杠，Linux/macOS 使用斜杠。

`os.MkdirAll(dir, 0755)` 会递归创建目录。如果目录已经存在，不会报错。

`filepath.Clean(path)` 可以清理路径里的多余分隔符和 `.`。但它不是安全校验的全部，不能只靠它阻止恶意路径。

## 四、真实后端场景示例

下面模拟“为用户导出文件生成安全路径”。文件名不直接使用用户输入，而是由服务端生成。

```go
package main

import (
	"fmt"
	"os"
	"path/filepath"
	"time"
)

func BuildUserExportPath(baseDir string, userID int) (string, error) {
	dir := filepath.Join(baseDir, "exports", "users")
	if err := os.MkdirAll(dir, 0755); err != nil {
		return "", err
	}

	filename := fmt.Sprintf("user_%d_%s.csv", userID, time.Now().Format("20060102_150405"))
	return filepath.Join(dir, filename), nil
}

func main() {
	path, err := BuildUserExportPath("data", 1001)
	if err != nil {
		panic(err)
	}

	if err := os.WriteFile(path, []byte("id,name\n1001,Alice\n"), 0644); err != nil {
		panic(err)
	}

	fmt.Println("created:", path)
}
```

真实项目中，这个路径通常会存到数据库里，供前端查询导出任务状态。

## 五、注意点

路径分隔符不要手写。比如不要写 `"data/exports/users"` 后再到处拼接，跨平台项目更容易出问题。

上传文件名不要直接使用原始文件名。原始文件名可以作为展示字段保存，但磁盘上的存储名建议由服务端生成。

生产环境里还要考虑磁盘空间、目录权限、容器挂载目录和备份策略。

```text
用户可控的路径片段，不能直接参与最终落盘路径。
```

## 六、常见误区

误区一：用字符串 `+ "/" +` 拼路径。这样在 Windows 上容易出问题，也不利于维护。

误区二：只创建文件，不创建上级目录。部署到新环境时经常失败。

误区三：把上传原文件名当存储文件名。重名、特殊字符、路径穿越都会带来麻烦。

误区四：把导出文件永久堆在服务器上。真实项目通常需要过期清理或对象存储生命周期策略。

## 七、小练习

为“文章封面上传”设计一个本地存储路径。

要求：

- 基础目录为 `data/uploads/article-cover`。
- 文件名由 `articleID` 和当前时间组成。
- 使用 `filepath.Join` 拼接路径。
- 使用 `os.MkdirAll` 确保目录存在。

完成后思考：原始文件名应该保存在哪里，是落盘路径，还是数据库展示字段？

## 八、本节达标标准

- 能用 `filepath.Join` 拼接路径。
- 能用 `os.MkdirAll` 创建多级目录。
- 知道不要直接使用用户传入的文件名作为落盘路径。
- 能为上传或导出场景设计基础目录结构。
