# 4. 缓冲 IO：逐行处理日志和大文件

本节目标：学完后，你能用 `bufio` 逐行读取日志或大文件，避免一次性占用大量内存。

## 简短引入

后端项目里的日志、CSV、导入文件经常很大。用 `os.ReadFile` 一次性读取虽然简单，但文件变大后会让服务内存压力明显增加。

`bufio` 可以理解为“带缓冲的读写工具”。它让你按行或按块处理数据，更适合后端批处理。

## 一、为什么需要它

常见场景是：运营上传一个用户导入 CSV，里面有几十万行。你不应该把整个文件读到内存再处理，而应该一行一行读、校验、写入数据库或生成错误报告。

```text
大文件处理的基本思路是：边读、边处理、边释放。
```

## 二、基本用法

先准备一个日志文件。

Windows PowerShell：

```powershell
@"
1001 login
1002 update_profile
1003 logout
"@ | Set-Content .\audit.log
```

Linux/macOS：

```bash
cat > audit.log <<'EOF'
1001 login
1002 update_profile
1003 logout
EOF
```

新建 `main.go`：

```go
package main

import (
	"bufio"
	"fmt"
	"os"
)

func main() {
	file, err := os.Open("audit.log")
	if err != nil {
		panic(err)
	}
	defer file.Close()

	scanner := bufio.NewScanner(file)
	for scanner.Scan() {
		line := scanner.Text()
		fmt.Println("line:", line)
	}

	if err := scanner.Err(); err != nil {
		panic(err)
	}
}
```

运行：

```powershell
go run .\main.go
```

Linux/macOS：

```bash
go run ./main.go
```

## 三、关键参数/语法/代码结构

`os.Open` 打开文件用于读取。打开后要 `defer file.Close()`。

`bufio.NewScanner(file)` 创建逐行扫描器。默认按行读取。

`scanner.Scan()` 每次读取下一行。返回 `false` 可能是读完了，也可能是出错了，所以循环后要检查 `scanner.Err()`。

`scanner.Text()` 获取当前行的字符串。

## 四、真实后端场景示例

下面模拟从日志中统计用户行为次数：

```go
package main

import (
	"bufio"
	"fmt"
	"os"
	"strings"
)

func CountAction(path string, action string) (int, error) {
	file, err := os.Open(path)
	if err != nil {
		return 0, err
	}
	defer file.Close()

	count := 0
	scanner := bufio.NewScanner(file)
	for scanner.Scan() {
		line := scanner.Text()
		fields := strings.Fields(line)
		if len(fields) != 2 {
			continue
		}
		if fields[1] == action {
			count++
		}
	}

	if err := scanner.Err(); err != nil {
		return 0, err
	}
	return count, nil
}

func main() {
	count, err := CountAction("audit.log", "login")
	if err != nil {
		panic(err)
	}
	fmt.Println("login count:", count)
}
```

真实项目中可以把 `continue` 的异常行写入错误报告，而不是静默跳过。

## 五、注意点

`Scanner` 默认单行最大 token 有限制，特别长的一行可能报错。处理超长 JSON 行或特殊日志时，要调整 buffer，或者改用 `bufio.Reader`。

逐行处理不代表没有错误。文件中间可能有坏数据，磁盘读取可能失败，权限可能变化。

写入数据库时不要每读一行就提交一次事务。真实项目通常会按批次处理，并控制事务边界。

## 六、常见误区

误区一：只写 `for scanner.Scan()`，不检查 `scanner.Err()`。这样读取错误会被吞掉。

误区二：认为逐行读取一定适合所有文件。二进制文件、超长行文件、复杂 CSV 需要更合适的解析方式。

误区三：在循环里频繁打开和关闭文件。文件应该在循环外打开，处理完成后关闭。

误区四：导入数据时一行失败就让整个程序崩溃。真实项目通常要记录失败行，并明确是否允许部分成功。

## 七、小练习

把日志统计程序改成统计“库存扣减失败”的次数。

准备一个 `stock.log`：

```text
sku_1001 deduct_ok
sku_1002 deduct_failed
sku_1003 deduct_failed
```

要求：

- 逐行读取文件。
- 统计第二列等于 `deduct_failed` 的行数。
- 对列数不等于 2 的行，不要崩溃，跳过或记录错误。

## 八、本节达标标准

- 能用 `bufio.Scanner` 逐行读取文件。
- 知道读取后要检查 `scanner.Err()`。
- 知道 `os.Open` 后要关闭文件。
- 能解释为什么大文件不适合 `os.ReadFile`。
- 能写一个简单的日志统计程序。
