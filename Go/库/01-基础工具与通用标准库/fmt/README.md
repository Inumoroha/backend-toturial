# Go fmt 后端实践教程

这套文档面向想成为 Go 后端工程师的学习者。`fmt` 看起来只是打印字符串，但在真实项目里，它会出现在调试、错误包装、HTTP 输出、CLI 工具、迁移脚本、日志上下文、数据脱敏和小型运维工具中。

建议按顺序学习：

1. [学习路线](00-学习路线/01-fmt-系统学习路线.md)
2. [基础输出与格式化](01-基础输出与格式化)
3. [后端场景实践](02-后端场景实践)
4. [错误处理与调试](03-错误处理与调试)
5. [输入解析与自定义格式](04-输入解析与自定义格式)
6. [综合项目](05-综合项目)

学习方式：

- 每节先复制最小示例运行，再读解释。
- 不要只记格式化动词，要思考它在项目中解决什么问题。
- 遇到 SQL、迁移、日志、错误处理相关示例时，优先理解边界和风险。
- 每节末尾对照“本节达标标准”自查。
