# 01. fmt 系统学习路线

本节目标：明确学习 `fmt` 的顺序，知道每个阶段要解决的后端开发问题。

简短引入：`fmt` 是 Go 后端项目中最常见的基础包之一。它不只是 `fmt.Println`，还负责把业务数据变成可读文本、构造错误、向 `io.Writer` 输出内容、解析简单输入，以及定义对象在日志和调试中的展示方式。

## 一、为什么需要系统学习 fmt

真实后端项目中，`fmt` 常见于这些位置：

- 本地调试：临时看变量、结构体、类型和值。
- 后台任务：打印导入、迁移、修复脚本的执行结果。
- 错误处理：用 `fmt.Errorf` 补充业务上下文，并保留原始错误。
- HTTP 和 CLI：把结果写到 `http.ResponseWriter`、文件、缓冲区或命令行。
- 日志辅助：生成短消息、脱敏展示、格式化金额和数量。

```text
fmt 不是日志库、不是模板引擎、不是 SQL 构造器。它适合做小而清晰的文本格式化。
```

## 二、学习阶段

### 第一阶段：能正确输出

目标是区分 `Print`、`Println`、`Printf`，知道什么时候用默认格式，什么时候用格式化动词。

对应文章：

- `01-基础输出与格式化/01-用 Println 输出可读的后端状态.md`
- `01-基础输出与格式化/02-用 Printf 控制业务字段格式.md`

### 第二阶段：能安全构造字符串

目标是掌握 `Sprintf` 的使用边界：可以构造提示、Key、URL 片段、日志消息，但不能拼 SQL。

对应文章：

- `02-后端场景实践/03-用 Sprintf 构造业务文本.md`
- `02-后端场景实践/04-用 Fprintf 写入 HTTP 响应和文件.md`

### 第三阶段：能处理错误和调试

目标是会用 `fmt.Errorf("%w")` 包装错误，并用 `%+v`、`%#v`、`%T` 定位问题。

对应文章：

- `03-错误处理与调试/05-用 Errorf 给错误补充上下文.md`
- `03-错误处理与调试/06-用格式化动词调试结构体.md`

### 第四阶段：能写小工具

目标是能用 `Sscan`、`Fscan` 处理简单输入，知道它们适合脚本和学习，不适合复杂协议解析。

对应文章：

### 第五阶段：能控制对象展示方式

目标是理解 `Stringer` 和 `Formatter` 的适用场景，尤其是脱敏展示和调试输出。

### 第六阶段：能落到项目里

目标是把 `fmt` 放进一个小型后端运维项目中，串起输入解析、错误处理、输出和安全边界。

对应文章：

- `05-综合项目/11-订单修复命令行工具.md`

## 三、学习建议

每节建议按这个顺序完成：

1. 复制代码运行。
2. 改一个业务字段，再运行。
3. 故意写错一个格式化动词，观察输出。
4. 对照“常见误区”检查自己是否踩坑。
5. 用“本节达标标准”判断是否可以进入下一节。

## 四、最终能力

学完后，你应该能做到：

- 看懂项目中大多数 `fmt` 用法。
- 在 CLI、HTTP、调试、错误处理场景中合理使用 `fmt`。
- 区分格式化字符串和安全边界，尤其不把用户输入拼进 SQL。
- 为后续学习 `log/slog`、`errors`、`io`、`net/http`、数据库访问打好基础。
