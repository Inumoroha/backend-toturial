学习建议：

- 每篇文章只关注一个主题，不急着一次性掌握所有概念。
- 先复制运行最小示例，再阅读解释。
- 看到 `errors.Is`、`errors.As`、`fmt.Errorf("%w")` 时，多想一想它们在 handler、service、repository 三层分别解决什么问题。
- 真正写项目时，优先保证错误可定位、响应不泄露内部信息、业务错误可被稳定识别。

```text
后端错误处理的目标不是把错误藏起来，而是让内部能定位、外部能理解、系统能安全恢复。
```
