# 12. 短链接项目综合实践

本节目标：学完后，你能够把 `errors.New`、`fmt.Errorf("%w")`、`errors.Is`、`errors.As` 和统一错误响应组合到一个小型后端项目思路里。

最后我们用短链接服务把前面的内容串起来。短链接项目足够小，但包含很多真实后端问题：参数校验、code 冲突、链接不存在、链接过期、数据库错误、统一响应、日志记录。

## 一、为什么需要它

一个短链接服务通常有两个核心接口：

- 创建短链接：提交目标 URL，生成 code。
- 访问短链接：根据 code 查目标 URL 并重定向。

这两个接口会遇到不同错误：

```text
目标 URL 不合法 -> 400
code 已存在 -> 409
短链接不存在 -> 404
短链接已过期 -> 410
数据库失败 -> 500
```

这些错误如果不统一建模，接口会很快变乱。

## 二、基本用法

下面是一份可直接运行的内存版示例。复制到 `main.go`。

```go
package main

import (
    "encoding/json"
    "errors"
    "fmt"
    "net/http"
    "strings"
)

var (
    ErrShortLinkNotFound = errors.New("short link not found")
    ErrShortLinkConflict = errors.New("short link conflict")
    ErrShortLinkExpired  = errors.New("short link expired")
)

type ValidationError struct {
    Field string
    Msg   string
}

func (e *ValidationError) Error() string {
    return e.Field + ": " + e.Msg
}

type ShortLink struct {
    Code   string
    Target string
    Expired bool
}

var store = map[string]ShortLink{
    "go":      {Code: "go", Target: "https://go.dev"},
    "expired": {Code: "expired", Target: "https://example.com", Expired: true},
}

func createShortLink(code, target string) error {
    if code == "" {
        return &ValidationError{Field: "code", Msg: "cannot be empty"}
    }
    if !strings.HasPrefix(target, "https://") {
        return &ValidationError{Field: "target", Msg: "must start with https://"}
    }
    if _, ok := store[code]; ok {
        return ErrShortLinkConflict
    }

    store[code] = ShortLink{Code: code, Target: target}
    return nil
}

func resolveShortLink(code string) (string, error) {
    link, ok := store[code]
    if !ok {
        return "", ErrShortLinkNotFound
    }
    if link.Expired {
        return "", ErrShortLinkExpired
    }

    return link.Target, nil
}

type ErrorResponse struct {
    Code    string `json:"code"`
    Message string `json:"message"`
}

func mapError(err error) (int, ErrorResponse) {
    var validationErr *ValidationError
    switch {
    case errors.As(err, &validationErr):
        return 400, ErrorResponse{Code: "VALIDATION_ERROR", Message: validationErr.Field + " " + validationErr.Msg}
    case errors.Is(err, ErrShortLinkConflict):
        return 409, ErrorResponse{Code: "SHORT_LINK_CONFLICT", Message: "short link already exists"}
    case errors.Is(err, ErrShortLinkNotFound):
        return 404, ErrorResponse{Code: "SHORT_LINK_NOT_FOUND", Message: "short link not found"}
    case errors.Is(err, ErrShortLinkExpired):
        return 410, ErrorResponse{Code: "SHORT_LINK_EXPIRED", Message: "short link expired"}
    default:
        return 500, ErrorResponse{Code: "INTERNAL_ERROR", Message: "internal server error"}
    }
}

func writeError(w http.ResponseWriter, err error) {
    status, body := mapError(err)
    fmt.Println("log:", err)

    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(status)
    _ = json.NewEncoder(w).Encode(body)
}

func main() {
    http.HandleFunc("/create", func(w http.ResponseWriter, r *http.Request) {
        code := r.URL.Query().Get("code")
        target := r.URL.Query().Get("target")

        if err := createShortLink(code, target); err != nil {
            writeError(w, fmt.Errorf("create short link: %w", err))
            return
        }

        w.WriteHeader(http.StatusCreated)
        _, _ = w.Write([]byte("created"))
    })

    http.HandleFunc("/r/", func(w http.ResponseWriter, r *http.Request) {
        code := strings.TrimPrefix(r.URL.Path, "/r/")

        target, err := resolveShortLink(code)
        if err != nil {
            writeError(w, fmt.Errorf("resolve short link %q: %w", code, err))
            return
        }

        http.Redirect(w, r, target, http.StatusFound)
    })

    fmt.Println("listen on :8080")
    _ = http.ListenAndServe(":8080", nil)
}
```

运行命令：

Windows PowerShell：

```powershell

```

Linux/macOS：

```bash

```

另开终端测试：


## 三、关键代码结构

业务错误：

```go
var ErrShortLinkNotFound = errors.New("short link not found")
```

这些错误用 `errors.Is` 判断。

校验错误：

```go
type ValidationError struct {
    Field string
    Msg   string
}
```

它携带字段信息，用 `errors.As` 提取。

错误包装：

```go
fmt.Errorf("resolve short link %q: %w", code, err)
```

它让日志知道是解析哪个短链接失败。

统一响应：

```go
mapError(err)
```

它把内部错误转换成 HTTP 状态码和 JSON 错误码。

## 四、真实后端场景示例

如果把内存 map 换成数据库，repository 层大致会变成：

```go
func FindShortLink(ctx context.Context, db *sql.DB, code string) (ShortLink, error) {
    const query = `SELECT code, target, expired FROM short_links WHERE code = ?`

    var link ShortLink
    if err := db.QueryRowContext(ctx, query, code).Scan(&link.Code, &link.Target, &link.Expired); err != nil {
        if errors.Is(err, sql.ErrNoRows) {
            return ShortLink{}, ErrShortLinkNotFound
        }

        return ShortLink{}, fmt.Errorf("query short link by code: %w", err)
    }

    return link, nil
}
```

这里仍然要使用参数化查询。`code` 是用户输入，不能拼进 SQL。

创建短链接时，如果数据库有唯一索引保护 `code`，插入冲突应该转换成 `ErrShortLinkConflict`。不同数据库驱动的唯一冲突错误类型不同，真实项目中要按驱动文档判断，不要简单依赖错误字符串。

数据库表结构建议通过迁移工具管理，例如：

```sql
CREATE TABLE short_links (
    code VARCHAR(64) PRIMARY KEY,
    target TEXT NOT NULL,
    expired BOOLEAN NOT NULL DEFAULT FALSE
);
```

上线迁移前要考虑：

- `code` 的唯一索引会增加写入成本。
- `target` 是否需要长度限制。
- 删除字段或改类型是否有回滚方案。
- 大表加索引是否会影响生产写入。

## 五、注意点

短链接服务看起来简单，但生产环境要保守处理：

- 创建短链接要校验目标 URL，避免开放重定向风险。
- code 要限制长度和字符集，避免日志污染和路由异常。
- 数据库查询必须参数化。
- code 唯一性要靠数据库约束兜底。
- 过期链接可以返回 410，也可以按产品要求返回 404。
- 内部错误进入日志，外部响应保持稳定。
- 批量创建短链接时，要先决定是否允许部分成功。

```text
业务校验可以在应用层做，数据一致性必须让数据库约束兜底。
```

## 六、常见误区

误区一：只在应用层检查 code 是否存在。

并发创建时，两个请求可能同时通过检查。数据库唯一约束才是最后防线。

误区二：把数据库唯一冲突直接返回给用户。

用户只需要知道短链接已存在，不需要看到数据库错误细节。

误区三：target 不校验。

短链接跳转是安全敏感功能。至少要限制协议，真实项目中还可能需要域名黑白名单。

误区四：没有统一错误响应。

一个接口返回字符串，一个接口返回 JSON，会让前端和调用方很难处理。
