# 01. 时间类型与当前时间

本节目标：学完后，你能在 Go 后端代码里获取当前时间、创建固定时间，并知道什么时候应该传递 `time.Time`。

简短引入：后端项目里几乎所有业务都会碰到时间，比如用户注册时间、订单创建时间、文章发布时间、短链接过期时间。第一步不是记住很多函数，而是先把“某一个时间点”和“一段时间”区分开。

## 一、为什么需要它

在真实项目中，`created_at`、`updated_at`、`expired_at` 通常表示某一个明确时间点。Go 里一般用 `time.Time` 表示这种值。

可以理解为：

- `time.Time`：一个具体时间点，例如“订单在 2026-07-06 10:30:00 创建”。
- `time.Duration`：一段时间长度，例如“订单 30 分钟未支付就关闭”。

```text
表示时间点用 time.Time，表示时间长度用 time.Duration。
```

## 二、基本用法

新建一个文件 `main.go`：

```go
package main

import (
	"fmt"
	"time"
)

func main() {
	now := time.Now()
	createdAt := time.Date(2026, 7, 6, 10, 30, 0, 0, time.Local)

	fmt.Println("now:", now)
	fmt.Println("createdAt:", createdAt)
}
```

Windows PowerShell：

```powershell
go run .\main.go
```

Linux/macOS：

```bash
go run ./main.go
```

这个例子在项目里的作用很常见：创建订单时写入当前时间，或者在测试数据里构造一个固定创建时间。

## 三、关键参数/语法/代码结构

`time.Now()` 返回当前系统时间。真实项目中常见用途是生成创建时间、更新时间、日志时间。

`time.Date(year, month, day, hour, min, sec, nsec, loc)` 用来创建固定时间。它适合测试、初始化活动时间、构造迁移脚本里的固定时间。

`time.Local` 表示本地时区。开发机器和服务器的本地时区可能不同，所以生产项目里不要随意依赖它。

## 四、真实后端场景示例

下面模拟创建用户时记录注册时间：

```go
package main

import (
	"fmt"
	"time"
)

type User struct {
	ID        int64
	Email     string
	CreatedAt time.Time
}

func NewUser(id int64, email string) User {
	return User{
		ID:        id,
		Email:     email,
		CreatedAt: time.Now(),
	}
}

func main() {
	user := NewUser(1, "dev@example.com")
	fmt.Printf("%+v\n", user)
}
```

真实项目中，这个 `CreatedAt` 通常会继续写入数据库。这里先不引入数据库，是为了先看清楚时间值在业务对象里的位置。

## 五、注意点

不要把时间先转成字符串再在业务里传来传去。字符串适合接口展示、日志输出、配置读取，不适合作为内部业务计算的主要类型。

```text
业务内部优先传 time.Time，只有在输入输出边界再转字符串。
```

还有一个容易忽略的问题：`time.Now()` 依赖系统时间。如果服务器时间被错误配置，订单过期、会员到期、日志排序都可能出问题。生产环境通常要保证机器时间同步。

## 六、常见误区

- 把 `created_at` 定义成字符串：后面排序、比较、时区处理都会变麻烦。
- 在每一层代码里都调用 `time.Now()`：同一个业务动作可能出现多个略有差异的时间点。
- 过早使用 `time.Local`：本地开发没问题，上服务器后时区可能不一致。

## 七、本节达标标准

- 能说清楚 `time.Time` 和 `time.Duration` 的区别。
- 能用 `time.Now()` 获取当前时间。
- 能用 `time.Date()` 构造固定时间。
- 知道业务内部不要优先使用字符串保存时间。
