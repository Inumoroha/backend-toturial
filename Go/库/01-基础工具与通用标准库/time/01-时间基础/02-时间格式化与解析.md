# 02. 时间格式化与解析

本节目标：学完后，你能把 `time.Time` 格式化成接口返回值，也能把用户传入的时间字符串解析成可计算的时间。

简短引入：后端接口经常要接收和返回时间，例如文章发布时间、订单查询起止时间、活动开始时间。Go 的格式化方式和很多语言不同，它不是 `yyyy-MM-dd`，而是用固定参考时间。

## 一、为什么需要它

接口层面对的是 JSON 和字符串，业务层面对的是 `time.Time`。格式化和解析就是这两层之间的转换。

常见场景是：

- 返回用户注册时间：`time.Time` 转字符串。
- 查询订单列表：把 `start_time`、`end_time` 字符串转成 `time.Time`。
- 读取配置：把活动开始时间字符串转成 `time.Time`。

```text
时间字符串只应该停留在输入输出边界，进入业务逻辑前要尽早解析。
```

## 二、基本用法

```go
package main

import (
	"fmt"
	"time"
)

func main() {
	layout := "2006-01-02 15:04:05"

	now := time.Date(2026, 7, 6, 10, 30, 0, 0, time.Local)
	fmt.Println(now.Format(layout))

	t, err := time.ParseInLocation(layout, "2026-07-06 10:30:00", time.Local)
	if err != nil {
		fmt.Println("parse time:", err)
		return
	}
	fmt.Println(t)
}
```

Windows PowerShell：

```powershell
go run .\main.go
```

Linux/macOS：

```bash
go run ./main.go
```

## 三、关键参数/语法/代码结构

Go 的布局字符串必须使用这个参考时间：

```text
2006-01-02 15:04:05
```

可以理解为 Go 用这个固定时间告诉程序：“年放哪里、月放哪里、日放哪里、小时放哪里”。

`Format(layout)`：把时间转成字符串，常用于接口返回和日志。

`time.Parse(layout, value)`：按布局解析字符串，默认按 UTC 处理没有时区的值。

`time.ParseInLocation(layout, value, loc)`：按指定时区解析没有时区的值。后端业务里更常用，因为用户输入的 `2026-07-06 10:30:00` 通常是某个业务时区下的时间。

## 四、真实后端场景示例

下面模拟查询订单列表时解析起止时间：

```go
package main

import (
	"fmt"
	"time"
)

type OrderQuery struct {
	StartTime time.Time
	EndTime   time.Time
}

func ParseOrderQuery(start, end string) (OrderQuery, error) {
	layout := "2006-01-02 15:04:05"

	startTime, err := time.ParseInLocation(layout, start, time.Local)
	if err != nil {
		return OrderQuery{}, fmt.Errorf("start_time 格式错误: %w", err)
	}

	endTime, err := time.ParseInLocation(layout, end, time.Local)
	if err != nil {
		return OrderQuery{}, fmt.Errorf("end_time 格式错误: %w", err)
	}

	if !startTime.Before(endTime) {
		return OrderQuery{}, fmt.Errorf("start_time 必须早于 end_time")
	}

	return OrderQuery{StartTime: startTime, EndTime: endTime}, nil
}

func main() {
	query, err := ParseOrderQuery("2026-07-01 00:00:00", "2026-07-06 23:59:59")
	if err != nil {
		fmt.Println(err)
		return
	}
	fmt.Printf("%+v\n", query)
}
```

真实项目中，这一步通常发生在 handler 或 request DTO 校验层。解析成功后再传给 service 或 repository。

## 五、注意点

接口返回时间格式要统一。一个项目里不要有的接口返回 `2026-07-06 10:30:00`，有的返回 `2026/07/06`，还有的返回 Unix 时间戳。

对外 API 推荐优先使用带时区的格式，例如 RFC3339：

```go
time.Now().Format(time.RFC3339)
```

如果是内部管理后台，使用 `2006-01-02 15:04:05` 也可以，但要明确它对应的业务时区。

## 六、常见误区

- 写成 `yyyy-MM-dd HH:mm:ss`：这是其他语言常见写法，在 Go 里不会按预期工作。
- 忽略解析错误：用户输入错了时间格式，后面查询条件就不可信。
- 用 `time.Parse` 解析本地业务时间：没有时区的字符串会按 UTC 解析，可能比预期差 8 小时。

## 七、本节达标标准

- 能使用 Go 的参考时间布局格式化时间。
- 能用 `ParseInLocation` 解析本地业务时间。
- 能对起止时间做基本校验。
- 知道接口时间格式必须统一。
