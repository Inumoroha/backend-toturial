# Go time 标准库学习路线

面向目标：想成为 Go 后端工程师，并希望系统掌握真实项目里的时间处理能力。

学习重点不是背 API，而是知道在用户、订单、库存、日志、定时任务、超时控制这些场景里，时间应该怎么表示、怎么计算、怎么存储、怎么避免线上问题。

```text
后端项目中的时间处理，优先追求清晰、统一、可测试、可回滚。
```

## 一、学习阶段

### 01. 时间基础

目标：先建立 Go 里时间值的直觉，能正确获取当前时间、格式化输出、解析用户输入。

- `01. 时间类型与当前时间`：理解 `time.Time`、`time.Now()`、创建时间。
- `02. 时间格式化与解析`：掌握 Go 特殊布局字符串，处理接口入参和日志输出。
- `03. 时区与 UTC`：知道服务端为什么常用 UTC 存储，展示时再转用户时区。

### 02. 时间计算与业务规则

目标：能写出订单超时、会员到期、库存锁定、活动时间窗等业务逻辑。

- `04. Duration 与时间加减`：使用 `time.Duration` 表示一段时间。
- `05. 截断、比较与时间窗口`：处理按天统计、活动开始结束、分页筛选。
- `06. 定时器与周期任务`：使用 `Timer`、`Ticker` 做延迟任务和轮询。

### 03. 后端工程实践

目标：把时间处理放进 HTTP、数据库、日志、测试这些真实工程环境里。

- `07. 请求超时与上下文截止时间`：避免接口无限等待。
- `08. 数据库中的时间字段`：统一 `created_at`、`updated_at`、过期时间。
- `09. 可测试的时间代码`：不要让测试依赖真实系统时间。

### 04. 综合项目

目标：通过一个短链接过期项目，把前面的知识串起来。

- `10. 短链接过期服务`：实现创建短链接、设置过期时间、访问时判断是否失效。

## 二、建议学习顺序

1. 先完成 `01-时间基础`，能看懂和写出时间值。
2. 再完成 `02-时间计算与业务规则`，把时间放进业务判断。
3. 然后完成 `03-后端工程实践`，理解服务端时间处理的边界。
4. 最后做 `04-综合项目`，把过期时间、数据库字段、超时和测试合在一起。

## 三、学习时的保守原则

```text
数据库里存储时间，优先统一为 UTC；面向用户展示时，再按用户时区转换。
```

```text
超时、过期、截止时间都要有明确边界，不要依赖“差不多”的后台轮询。
```

```text
业务代码不要直接到处调用 time.Now()，需要测试的逻辑要能注入当前时间。
```

## 四、最终达标标准

学完这套教程后，你应该能够：

- 在 Go 项目中正确使用 `time.Time` 和 `time.Duration`。
- 能解析接口传入的时间字符串，并给出清晰的错误处理。
- 能解释 UTC、Local、业务时区之间的关系。
- 能实现订单超时、会员到期、短链接过期这类常见逻辑。
- 能用 `context` 控制请求和外部调用超时。
- 能为时间相关业务写稳定测试。
- 能给数据库时间字段设计基本约束和索引建议。
