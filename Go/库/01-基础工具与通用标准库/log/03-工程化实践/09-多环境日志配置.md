# 09. 多环境日志配置

本节目标：学完后，你能根据开发、测试、生产环境选择不同日志格式和级别。

## 简短引入

开发环境需要方便人看，生产环境需要方便机器采集。开发时你可能希望看到 Debug 和文本格式；生产中则更关心 Info 以上、JSON、稳定字段和可检索性。

## 一、为什么需要它

同一套日志配置不适合所有环境：

- 开发环境：关注调试效率，可以更详细。
- 测试环境：关注复现问题，接近生产。
- 生产环境：关注稳定、成本、安全和可检索。

```text
生产日志默认应该克制：够排查、可检索、不泄露、不爆量。
```

## 二、基本用法

用环境变量控制日志模式。

Windows PowerShell：

```powershell
$env:APP_ENV="dev"
go run .
```

Linux/macOS：

```bash
APP_ENV=dev go run .
```

示例代码：

```go
package main

import (
	"log/slog"
	"os"
)

func newLogger() *slog.Logger {
	env := os.Getenv("APP_ENV")

	if env == "dev" {
		return slog.New(slog.NewTextHandler(os.Stdout, &slog.HandlerOptions{
			Level: slog.LevelDebug,
		}))
	}

	return slog.New(slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{
		Level: slog.LevelInfo,
	}))
}

func main() {
	logger := newLogger()

	logger.Debug("debug database config loaded")
	logger.Info("server started", "env", os.Getenv("APP_ENV"))
}
```

## 三、关键参数/语法/代码结构

`APP_ENV` 是常见的环境变量名，也可以用 `GO_ENV`、`ENV`，但团队内要统一。

`slog.HandlerOptions{Level: slog.LevelDebug}` 表示输出 Debug 及以上级别。生产中通常使用 `slog.LevelInfo`，减少日志量和敏感信息风险。

开发环境使用 `TextHandler`，阅读方便；生产环境使用 `JSONHandler`，采集方便。

## 四、真实后端场景示例

下面模拟服务启动时的配置日志。

```go
package main

import (
	"log/slog"
	"os"
)

func newLogger() *slog.Logger {
	if os.Getenv("APP_ENV") == "dev" {
		return slog.New(slog.NewTextHandler(os.Stdout, &slog.HandlerOptions{Level: slog.LevelDebug}))
	}
	return slog.New(slog.NewJSONHandler(os.Stdout, &slog.HandlerOptions{Level: slog.LevelInfo}))
}

func main() {
	logger := newLogger()

	env := os.Getenv("APP_ENV")
	if env == "" {
		env = "prod"
	}

	logger.Info("application starting",
		"env", env,
		"http_addr", ":8080",
	)

	logger.Debug("debug detail for local development",
		"cache_enabled", true,
	)
}
```

生产环境不会输出 Debug，开发环境会输出。这样既保留调试便利，又控制生产风险。

## 五、注意点

环境变量是配置入口，不应该把生产配置写死在代码里。

不要在启动日志中打印数据库密码、AccessKey、Token。可以打印配置是否存在、连接目标的非敏感部分，比如主机名和数据库名。

日志级别调整要谨慎。临时打开 Debug 排查问题时，要有关闭计划，否则日志成本和泄露风险都会增加。

## 六、常见误区

误区一：开发和生产完全一样。  
开发需要可读性，生产需要可采集性，两者重点不同。

误区二：生产默认 Debug。  
这会造成日志量暴涨，也容易暴露内部细节。

误区三：启动时打印完整配置。  
配置里经常有密码和密钥，不能为了方便排查全部输出。

## 七、本节达标标准

- 能用环境变量控制日志格式和级别。
- 能解释 TextHandler 和 JSONHandler 的使用场景。
- 知道生产环境不应默认输出 Debug。
- 能避免在启动日志中打印敏感配置。
