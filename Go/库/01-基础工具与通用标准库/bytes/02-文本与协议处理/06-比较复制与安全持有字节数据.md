# 06. 比较、复制与安全持有字节数据

本节目标：学完后你能用 `bytes.Equal`、`bytes.Compare`、`bytes.Clone` 处理字节比较和数据持有，避免切片被意外修改。

## 简短引入

`[]byte` 是切片，它经常共享底层数组。共享本身不是问题，但如果你把一段字节保存起来，后面又有人修改了原始切片，就可能出现“缓存内容变了”“签名原文变了”“日志字段串了”的问题。

## 一、为什么需要它

后端里常见这些场景：

- 缓存一份请求体用于签名校验
- 保存上传文件的头部特征
- 比较两个摘要是否相同
- 对 key 做排序或范围判断

这时要区分“只是读一下”和“我要长期持有”。

```text
只要要跨函数、跨 goroutine、跨请求阶段保存 []byte，就优先考虑复制。
```

## 二、基本用法

```go
package main

import (
	"bytes"
	"fmt"
)

func main() {
	a := []byte("order:1001")
	b := []byte("order:1001")

	fmt.Println(bytes.Equal(a, b))

	saved := bytes.Clone(a)
	a[0] = 'X'

	fmt.Println(string(a))
	fmt.Println(string(saved))
}
```

## 三、关键参数和代码结构

`bytes.Equal(a, b)`

判断两个字节切片内容是否相同。不要用 `a == b`，切片不能这样比较。

`bytes.Compare(a, b)`

按字典序比较，返回 `-1`、`0`、`1`。常见于排序、范围比较，但业务排序通常更建议转成明确字段后排序。

`bytes.Clone(src)`

复制一份新的字节切片。适合缓存、异步处理、跨生命周期保存。

## 四、真实后端场景示例

保存请求体摘要前，复制原始数据，避免后续复用缓冲区影响签名。

```go
package main

import (
	"bytes"
	"crypto/sha256"
	"fmt"
)

type AuditEvent struct {
	BodyDigest []byte
}

func NewAuditEvent(body []byte) AuditEvent {
	sum := sha256.Sum256(body)
	return AuditEvent{
		BodyDigest: bytes.Clone(sum[:]),
	}
}

func main() {
	body := []byte(`{"order_id":"A1001","total":99}`)
	event := NewAuditEvent(body)

	body[0] = 'X'

	fmt.Printf("%x\n", event.BodyDigest)
}
```

这里复制的是摘要，不是完整请求体。真实项目中通常不要长期保存完整请求体，尤其是包含个人信息或支付信息时。

## 五、注意点

`bytes.Buffer.Bytes()` 返回的是当前缓冲区内容的字节视图，不是独立副本。

如果你要保存结果，应该复制：

```go
data := bytes.Clone(buf.Bytes())
```

如果只是马上写入响应或文件，可以不复制。

## 六、常见误区

误区一：把 `[]byte` 当成不可变数据。

`[]byte` 可以被修改，也可能和其他切片共享底层数组。

误区二：把 `Buffer.Bytes()` 放进全局缓存。

后续 `Reset` 或继续写入可能影响你以为已经保存的数据。

误区三：比较密码、token 时只考虑 `bytes.Equal`。

安全敏感比较可能需要常量时间比较，例如 `crypto/subtle`。本节只讲普通字节比较。

## 七、本节达标标准

- 能用 `bytes.Equal` 比较字节内容
- 能用 `bytes.Clone` 保存独立副本
- 知道 `Buffer.Bytes()` 不是长期安全持有的副本
- 能识别跨生命周期持有 `[]byte` 的风险
