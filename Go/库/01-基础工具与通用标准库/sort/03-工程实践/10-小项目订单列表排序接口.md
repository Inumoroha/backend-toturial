# 10. 小项目：订单列表排序接口

本节目标：学完后，你能够实现一个可运行的订单列表接口，支持排序、分页、参数校验和清晰错误返回。

简短引入：前面学的是单点能力，这一节把它们串成后端开发中常见的订单列表接口。

## 一、为什么需要它

后台订单列表通常会有这些需求：

- 按创建时间、金额、状态排序。
- 支持升序和降序。
- 支持分页。
- 非法参数返回 400。

真实项目里，数据量大时应优先让数据库排序分页。这一节用内存数据模拟，是为了把接口参数、排序规则和分页边界讲清楚。

## 二、基本用法

创建 `main.go`：

```go
package main

import (
	"encoding/json"
	"errors"
	"fmt"
	"log"
	"net/http"
	"sort"
	"strconv"
	"time"
)

type Order struct {
	ID        int       `json:"id"`
	Amount    int       `json:"amount"`
	Status    string    `json:"status"`
	CreatedAt time.Time `json:"created_at"`
}

var orders = []Order{
	{ID: 1, Amount: 300, Status: "paid", CreatedAt: time.Date(2026, 1, 1, 10, 0, 0, 0, time.UTC)},
	{ID: 2, Amount: 100, Status: "pending", CreatedAt: time.Date(2026, 1, 2, 10, 0, 0, 0, time.UTC)},
	{ID: 3, Amount: 300, Status: "paid", CreatedAt: time.Date(2026, 1, 3, 10, 0, 0, 0, time.UTC)},
	{ID: 4, Amount: 800, Status: "cancelled", CreatedAt: time.Date(2026, 1, 4, 10, 0, 0, 0, time.UTC)},
}

func main() {
	http.HandleFunc("/orders", listOrders)
	log.Println("listen on http://localhost:8080/orders")
	log.Fatal(http.ListenAndServe(":8080", nil))
}

func listOrders(w http.ResponseWriter, r *http.Request) {
	sortField := defaultString(r.URL.Query().Get("sort"), "created_at")
	direction := defaultString(r.URL.Query().Get("order"), "desc")
	page, pageSize, err := parsePage(r)
	if err != nil {
		http.Error(w, err.Error(), http.StatusBadRequest)
		return
	}

	result := append([]Order(nil), orders...)
	if err := sortOrderList(result, sortField, direction); err != nil {
		http.Error(w, err.Error(), http.StatusBadRequest)
		return
	}

	result = paginate(result, page, pageSize)

	w.Header().Set("Content-Type", "application/json")
	_ = json.NewEncoder(w).Encode(result)
}

func sortOrderList(list []Order, field, direction string) error {
	if direction != "asc" && direction != "desc" {
		return fmt.Errorf("unsupported order: %s", direction)
	}

	allowed := map[string]bool{
		"created_at": true,
		"amount":     true,
		"status":     true,
	}
	if !allowed[field] {
		return fmt.Errorf("unsupported sort field: %s", field)
	}

	desc := direction == "desc"
	sort.Slice(list, func(i, j int) bool {
		cmp := compareOrders(list[i], list[j], field)
		if cmp == 0 {
			cmp = compareInt(list[i].ID, list[j].ID)
		}
		if desc {
			return cmp > 0
		}
		return cmp < 0
	})
	return nil
}

func compareOrders(a, b Order, field string) int {
	switch field {
	case "created_at":
		if a.CreatedAt.Before(b.CreatedAt) {
			return -1
		}
		if a.CreatedAt.After(b.CreatedAt) {
			return 1
		}
		return 0
	case "amount":
		return compareInt(a.Amount, b.Amount)
	case "status":
		if a.Status < b.Status {
			return -1
		}
		if a.Status > b.Status {
			return 1
		}
		return 0
	default:
		return 0
	}
}

func compareInt(a, b int) int {
	switch {
	case a < b:
		return -1
	case a > b:
		return 1
	default:
		return 0
	}
}

func parsePage(r *http.Request) (int, int, error) {
	page, err := parsePositiveInt(r.URL.Query().Get("page"), 1)
	if err != nil {
		return 0, 0, errors.New("page must be a positive integer")
	}
	pageSize, err := parsePositiveInt(r.URL.Query().Get("page_size"), 20)
	if err != nil {
		return 0, 0, errors.New("page_size must be a positive integer")
	}
	if pageSize > 100 {
		return 0, 0, errors.New("page_size must be <= 100")
	}
	return page, pageSize, nil
}

func parsePositiveInt(raw string, fallback int) (int, error) {
	if raw == "" {
		return fallback, nil
	}
	n, err := strconv.Atoi(raw)
	if err != nil || n <= 0 {
		return 0, errors.New("invalid positive integer")
	}
	return n, nil
}

func paginate(list []Order, page, pageSize int) []Order {
	start := (page - 1) * pageSize
	if start >= len(list) {
		return []Order{}
	}
	end := start + pageSize
	if end > len(list) {
		end = len(list)
	}
	return list[start:end]
}

func defaultString(value, fallback string) string {
	if value == "" {
		return fallback
	}
	return value
}
```

运行服务：

```powershell
go run .\main.go
```

Linux/macOS：

```bash

```

请求示例：

```powershell

```

```bash

```

## 三、关键参数/语法/代码结构

这个接口分成几块：

- `listOrders`：读取参数、复制数据、调用排序和分页。
- `sortOrderList`：校验排序字段和方向。
- `compareOrders`：集中维护字段比较规则。
- `parsePage`：校验分页参数。
- `paginate`：根据页码切片。

复制数据这一行很重要：

```go
result := append([]Order(nil), orders...)
```

它避免每次请求都修改全局 `orders` 的顺序。

## 四、真实后端场景示例

如果换成数据库，`sortOrderList` 的职责会变成生成受控 `ORDER BY`：

```go
var columns = map[string]string{
	"created_at": "created_at",
	"amount":     "amount",
	"status":     "status",
}
```

然后只允许从 `columns` 里取字段名。`LIMIT`、`OFFSET`、业务过滤条件继续使用参数化查询。

真实项目中还要考虑事务边界：列表查询通常不需要长事务；如果列表和统计总数必须强一致，要明确事务隔离级别和性能成本。数据库迁移增加索引时，要使用迁移工具，并准备回滚方案。

## 五、注意点

这个小项目适合学习接口排序流程，不适合直接照搬到大数据量生产接口。

生产环境建议：

- 大表排序分页优先使用数据库 `ORDER BY` 和合适索引。
- 排序字段保持白名单。
- `page_size` 必须有上限。
- 重要列表使用唯一字段做兜底排序。
- 错误信息对调用方清楚，但不要暴露内部 SQL。

## 六、常见误区

- 误区一：每次请求直接排序全局变量。并发服务中这会造成结果互相影响。
- 误区二：没有参数校验，非法字段也走默认排序。调用方会以为参数生效了。
- 误区三：把内存示例当生产方案。真实订单表通常应该数据库排序分页。
- 误区四：忽略错误处理。后端接口应该明确告诉调用方参数哪里错了。

## 七、本节达标标准

- 能运行 `/orders` 接口并传入排序分页参数。
- 能解释排序字段白名单的作用。
- 能说明为什么请求中要复制全局数据再排序。
- 能把这个内存示例迁移成数据库 `ORDER BY` 思路。
- 能识别生产环境中索引、迁移、错误处理和可回滚的重要性。
