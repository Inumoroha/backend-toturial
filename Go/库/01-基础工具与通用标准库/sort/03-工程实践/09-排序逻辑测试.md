# 09. 排序逻辑测试

本节目标：学完后，你能够为排序函数写表格测试，覆盖升序、降序、重复字段和空列表。

简短引入：排序逻辑很容易“看起来对”，但在相同字段、空数据、非法参数时出错。测试能帮你守住接口行为。

## 一、为什么需要它

后端接口一旦暴露排序参数，前端、运营后台、甚至第三方调用方都会依赖结果顺序。一次小改动如果把降序改成升序，可能不会编译报错，但业务会出问题。

表格测试适合这种输入输出明确的逻辑。

## 二、基本用法

创建 `order_sort_test.go`：

```go
package main

import (
	"reflect"
	"sort"
	"testing"
)

type Order struct {
	ID     int
	Amount int
}

func sortOrdersByAmountDesc(orders []Order) {
	sort.Slice(orders, func(i, j int) bool {
		if orders[i].Amount != orders[j].Amount {
			return orders[i].Amount > orders[j].Amount
		}
		return orders[i].ID > orders[j].ID
	})
}

func TestSortOrdersByAmountDesc(t *testing.T) {
	orders := []Order{
		{ID: 1, Amount: 100},
		{ID: 2, Amount: 300},
		{ID: 3, Amount: 300},
	}

	sortOrdersByAmountDesc(orders)

	got := []int{orders[0].ID, orders[1].ID, orders[2].ID}
	want := []int{3, 2, 1}
	if !reflect.DeepEqual(got, want) {
		t.Fatalf("got %v, want %v", got, want)
	}
}
```

## 三、关键参数/语法/代码结构

更推荐表格测试：

```go
func TestSortOrders(t *testing.T) {
	tests := []struct {
		name string
		in   []Order
		want []int
	}{
		{
			name: "amount desc with id desc fallback",
			in:   []Order{{ID: 1, Amount: 100}, {ID: 2, Amount: 300}, {ID: 3, Amount: 300}},
			want: []int{3, 2, 1},
		},
		{
			name: "empty list",
			in:   []Order{},
			want: []int{},
		},
	}

	for _, tt := range tests {
		t.Run(tt.name, func(t *testing.T) {
			sortOrdersByAmountDesc(tt.in)
			got := make([]int, 0, len(tt.in))
			for _, order := range tt.in {
				got = append(got, order.ID)
			}
			if !reflect.DeepEqual(got, tt.want) {
				t.Fatalf("got %v, want %v", got, tt.want)
			}
		})
	}
}
```

## 四、真实后端场景示例

订单列表排序至少建议覆盖：

- 正常金额降序。
- 金额相同按 ID 降序。
- 空列表。
- 单条数据。
- 非法排序字段返回错误。
- 非法排序方向返回错误。

这些测试不是为了追求数量，而是覆盖容易在线上产生误解的边界。

```text
排序测试优先验证业务顺序，不要只验证函数有没有运行。
```

## 五、注意点

如果排序函数会原地修改输入，测试用例之间不要复用同一个切片。否则一个用例排序后的结果会影响下一个用例。

可以在每个测试里复制输入：

```go
orders := append([]Order(nil), tt.in...)
```

生产项目里，排序测试还应该配合接口测试，确认 HTTP 参数到排序规则的映射没有错。

## 六、常见误区

- 误区一：只测不同字段，不测相同字段。相同字段才是分页和稳定性的高风险点。
- 误区二：测试用例复用切片，导致用例互相污染。
- 误区三：只测成功，不测非法参数。真实接口必须给调用方明确错误。

## 七、本节达标标准

- 能为排序函数写表格测试。
- 能覆盖空列表、重复值、二级排序。
- 能理解原地排序对测试数据的影响。
- 能把排序测试和接口参数测试联系起来。
