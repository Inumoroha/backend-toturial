# 02. 基础类型排序与原地修改

本节目标：学完后，你能够对整数、字符串、浮点数切片排序，并知道什么时候需要复制切片。

简短引入：基础类型排序常用于配置权重、标签名、日志级别、统计结果等简单列表。

## 一、为什么需要它

后端项目里不一定每次排序都是结构体。常见场景是：

- 对配置中的端口、权重、优先级排序。
- 对用户标签按字典序展示。
- 对统计结果里的金额或耗时排序。

基础类型排序是最容易上手的部分，但也最容易忽略“原地修改”。

## 二、基本用法

创建 `main.go`：

```go
package main

import (
	"fmt"
	"sort"
)

func main() {
	ids := []int{42, 7, 18}
	names := []string{"writer", "admin", "reader"}
	costs := []float64{19.9, 3.5, 100.0}

	sort.Ints(ids)
	sort.Strings(names)
	sort.Float64s(costs)

	fmt.Println(ids)
	fmt.Println(names)
	fmt.Println(costs)
}
```

Windows PowerShell：

```powershell
go run .\main.go
```

Linux/macOS：

```bash
go run ./main.go
```

## 三、关键参数/语法/代码结构

常用函数：

- `sort.Ints([]int)`：整数升序。
- `sort.Strings([]string)`：字符串按字典序升序。
- `sort.Float64s([]float64)`：浮点数升序。
- `sort.IntsAreSorted([]int)`：判断整数切片是否已有序。

这些函数都直接修改传入切片。判断是否有序不会修改数据。

如果业务语义是“拿一份排序后的副本”，就显式复制：

```go
func sortedCopy(ids []int) []int {
	cp := append([]int(nil), ids...)
	sort.Ints(cp)
	return cp
}
```

## 四、真实后端场景示例

权限系统里，角色名要按字典序展示，避免前端每次自己排序：

```go
package main

import (
	"fmt"
	"sort"
)

func main() {
	rolesFromDB := []string{"editor", "admin", "viewer"}

	rolesForResponse := append([]string(nil), rolesFromDB...)
	sort.Strings(rolesForResponse)

	fmt.Println(rolesForResponse)
}
```

如果这是一个响应 DTO 的临时数据，直接排序也可以。如果这个切片还会被用于审计日志、权限判断或缓存写回，建议复制。

## 五、注意点

基础类型排序只有升序函数。如果要降序，可以使用 `sort.Slice`：

```go
sort.Slice(ids, func(i, j int) bool {
	return ids[i] > ids[j]
})
```

真实项目中要让排序方向来自受控参数，例如只允许 `asc` 和 `desc`。不要让用户传任意表达式。

```text
排序方向是业务选项，不是让用户传一段代码或 SQL。
```

## 六、常见误区

- 误区一：把排序后的副本和原切片混用，导致后续逻辑读到被改过的顺序。
- 误区二：浮点数金额直接排序并用于严肃财务计算。金额建议用整数分或定点数表达，排序只是展示层动作。
- 误区三：每次响应都排序配置。配置不常变化时，可以在加载配置时排序一次。

## 七、本节达标标准

- 能使用 `sort.Ints`、`sort.Strings`、`sort.Float64s`。
- 能用复制保护原切片顺序。
- 能用 `sort.Slice` 实现基础类型降序。
- 能说明基础类型排序在后端中的常见用途。
