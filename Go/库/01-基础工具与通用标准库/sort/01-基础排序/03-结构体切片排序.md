# 03. 结构体切片排序

本节目标：学完后，你能够使用 `sort.Slice` 按结构体字段排序，例如按订单金额或文章发布时间排序。

简短引入：后端项目中最常见的排序对象不是 `[]int`，而是 `[]Order`、`[]User`、`[]Article` 这样的结构体切片。

## 一、为什么需要它

接口返回的数据通常有多个字段。用户不会说“把结构体排序”，而是说：

- 订单按金额从高到低。
- 文章按发布时间从新到旧。
- 用户按注册时间排序。

`sort.Slice` 可以理解为：你告诉 Go “任意两个元素谁应该排在前面”，Go 负责完成排序过程。

## 二、基本用法

创建 `main.go`：

```go
package main

import (
	"fmt"
	"sort"
)

type Order struct {
	ID     int
	Amount int
}

func main() {
	orders := []Order{
		{ID: 1, Amount: 300},
		{ID: 2, Amount: 100},
		{ID: 3, Amount: 500},
	}

	sort.Slice(orders, func(i, j int) bool {
		return orders[i].Amount > orders[j].Amount
	})

	fmt.Println(orders)
}
```

Windows PowerShell：

```powershell
go run .\main.go
```

Linux/macOS：

```bash
go run ./main.go
```

## 三、关键参数/语法/代码结构

`sort.Slice(orders, less)` 有两个关键点：

- 第一个参数是要排序的切片。
- 第二个参数是比较函数，返回 `true` 表示 `i` 位置的元素应该排在 `j` 前面。

升序：

```go
return orders[i].Amount < orders[j].Amount
```

降序：

```go
return orders[i].Amount > orders[j].Amount
```

比较函数里不要修改切片、不要发请求、不要查数据库。它会被调用很多次，应该保持简单、确定、无副作用。

## 四、真实后端场景示例

文章接口从缓存中取出热门文章后，按发布时间倒序返回：

```go
package main

import (
	"fmt"
	"sort"
	"time"
)

type Article struct {
	ID        int
	Title     string
	CreatedAt time.Time
}

func main() {
	articles := []Article{
		{ID: 1, Title: "Go 入门", CreatedAt: time.Date(2026, 1, 2, 9, 0, 0, 0, time.UTC)},
		{ID: 2, Title: "订单接口", CreatedAt: time.Date(2026, 1, 3, 9, 0, 0, 0, time.UTC)},
		{ID: 3, Title: "缓存实践", CreatedAt: time.Date(2026, 1, 1, 9, 0, 0, 0, time.UTC)},
	}

	sort.Slice(articles, func(i, j int) bool {
		return articles[i].CreatedAt.After(articles[j].CreatedAt)
	})

	for _, article := range articles {
		fmt.Println(article.ID, article.Title)
	}
}
```

这里使用 `After` 比直接比较时间字符串更可靠。真实项目中，时间字段尽量用 `time.Time`，不要用格式化后的字符串参与业务排序。

## 五、注意点

如果两个元素的排序字段相同，`sort.Slice` 不保证它们保留原来的相对顺序。对订单、日志、消息列表这类用户敏感数据，通常需要补一个二级排序字段，例如 `ID`。

```text
比较函数要能形成清楚的先后关系，相等时要想好是否需要二级排序。
```

## 六、常见误区

- 误区一：比较函数写反，导致升序降序相反。建议用小样本先打印验证。
- 误区二：在比较函数里查数据库。排序函数会多次调用，性能和稳定性都会变差。
- 误区三：按时间字符串排序。字符串格式不统一时结果会错，业务代码优先用 `time.Time`。

## 七、本节达标标准

- 能使用 `sort.Slice` 排序结构体切片。
- 能写出升序和降序比较函数。
- 能解释比较函数返回 `true` 的含义。
- 能识别比较函数中的副作用风险。
