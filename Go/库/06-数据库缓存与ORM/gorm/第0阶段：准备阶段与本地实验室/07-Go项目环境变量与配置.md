# 07 Go 项目环境变量与配置

本节目标：把数据库连接字符串从代码中抽出来，使用环境变量管理配置。真实后端项目不应该把密码、端口、数据库名硬编码在业务代码里。

---

## 一、为什么不要硬编码 DSN

入门示例常写：

```go
dsn := "root:root123456@tcp(127.0.0.1:3306)/gorm_study?charset=utf8mb4&parseTime=True&loc=Local"
```

这适合学习第一天，但不适合长期项目。

问题：

- 密码暴露在代码里。
- 开发库、测试库、生产库不好切换。
- 多人协作时每个人的配置可能不同。
- 代码提交到 Git 后容易泄露敏感信息。

---

## 二、使用环境变量

PowerShell 设置环境变量：

```powershell
$env:DB_DSN="root:root123456@tcp(127.0.0.1:3306)/gorm_study?charset=utf8mb4&parseTime=True&loc=Local"
```

运行：

```powershell
go run .
```

注意：这样设置只对当前终端窗口有效。关闭窗口后会失效。

---

## 三、在 Go 中读取环境变量

```go
package config

import (
    "fmt"
    "os"
)

type Config struct {
    DBDSN string
}

func Load() (Config, error) {
    dsn := os.Getenv("DB_DSN")
    if dsn == "" {
        return Config{}, fmt.Errorf("DB_DSN is required")
    }

    return Config{
        DBDSN: dsn,
    }, nil
}
```

文件位置：

```text
internal/config/config.go
```

---

## 四、在 main 中使用配置

```go
func main() {
    cfg, err := config.Load()
    if err != nil {
        log.Fatal(err)
    }

    db, err := gorm.Open(mysql.Open(cfg.DBDSN), &gorm.Config{})
    if err != nil {
        log.Fatal(err)
    }

    _ = db
}
```

这样代码不再关心具体密码是多少，只关心环境里有没有 `DB_DSN`。

## 五、使用 .env 文件

学习阶段也可以使用 `.env` 文件。安装：

```powershell
go get github.com/joho/godotenv
```

创建：

```text
.env
```

内容：

```text
DB_DSN=root:root123456@tcp(127.0.0.1:3306)/gorm_study?charset=utf8mb4&parseTime=True&loc=Local
APP_PORT=8080
APP_ENV=development
```

读取：

```go
func Load() (Config, error) {
    _ = godotenv.Load()

    dsn := os.Getenv("DB_DSN")
    if dsn == "" {
        return Config{}, fmt.Errorf("DB_DSN is required")
    }

    return Config{DBDSN: dsn}, nil
}
```

注意：`.env` 不应该提交到公开仓库。可以提交 `.env.example`。

---

## 六、.env.example

创建：

```text
.env.example
```

内容：

```text
DB_DSN=root:your_password@tcp(127.0.0.1:3306)/gorm_study?charset=utf8mb4&parseTime=True&loc=Local
APP_PORT=8080
APP_ENV=development
```

它告诉别人需要哪些配置，但不暴露真实密码。

---

## 七、配置结构扩展

后续项目可以扩展：

```go
type Config struct {
    App AppConfig
    DB  DBConfig
}

type AppConfig struct {
    Port string
    Env  string
}

type DBConfig struct {
    DSN          string
    MaxOpenConns int
    MaxIdleConns int
}
```

不要一开始就引入很复杂的配置框架。先把环境变量这条线学清楚。

---

## 八、常见错误

### 1. 环境变量只在当前终端生效

你在一个 PowerShell 窗口里设置了 `$env:DB_DSN`，然后在另一个窗口运行程序，是读不到的。

### 2. .env 文件提交到 Git

`.env` 里可能有密码，不应该提交。应该加入 `.gitignore`。

### 3. 配置缺失时继续运行

错误：

```go
dsn := os.Getenv("DB_DSN")
db, _ := gorm.Open(mysql.Open(dsn), &gorm.Config{})
```

如果 `dsn` 是空字符串，错误会变得难排查。应该启动时就明确失败。

## 九、本节达标标准

学完本节后，你应该能够做到：

- 使用 PowerShell 设置 `DB_DSN`。
- 在 Go 中读取环境变量。
- 配置缺失时让程序明确报错。
- 使用 `.env` 管理本地学习配置。
- 创建 `.env.example`。
- 理解为什么密码不应该硬编码在代码里。
