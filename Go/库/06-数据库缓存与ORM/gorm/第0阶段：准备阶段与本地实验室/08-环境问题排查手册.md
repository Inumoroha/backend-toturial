# 08 环境问题排查手册

本节目标：整理学习 GORM 前最常见的环境问题。以后遇到连接失败、表不存在、乱码、端口冲突，不要靠猜，按清单逐项排查。

---

## 一、排查顺序

遇到问题时，按这个顺序：

```text
1. Go 命令是否可用
2. MySQL 容器是否运行
3. 端口是否正确
4. 用户名密码是否正确
5. 数据库名是否存在
6. DSN 参数是否完整
7. GORM 是否连接到预期数据库
8. 表是否迁移成功
9. SQL 日志是否符合预期
```

不要一上来就改代码。很多问题其实是数据库没启动或连错库。

---

## 二、Go 命令不可用

检查：

```powershell
go version
```

如果提示找不到 `go`，说明 Go 没装好或 PATH 没配置。

解决：

- 重新安装 Go。
- 重新打开终端。
- 检查环境变量 PATH。

---

## 三、MySQL 容器没启动

检查：

```powershell
docker ps
```

如果看不到 `gorm-mysql`，再看所有容器：

```powershell
docker ps -a
```

启动：

```powershell
docker start gorm-mysql
```

如果使用 Compose：

```powershell
docker compose up -d
```

---

## 四、端口不对

如果 Compose 配置：

```yaml
ports:
  - "3307:3306"
```

那么 DSN 必须写：

```text
127.0.0.1:3307
```

不是 `3306`。

检查端口占用：

```powershell
netstat -ano | Select-String ":3306"
```

---

## 五、账号密码错误

错误信息类似：

```text
Access denied for user 'root'
```

排查：

- DSN 里的用户名是否是 `root`。
- DSN 里的密码是否是 `root123456`。
- Docker volume 是否已经初始化过旧密码。

如果忘记密码，学习环境可以删除 volume 重建。但要知道数据会丢失：

```powershell
docker compose down -v
docker compose up -d
```

---

## 六、数据库不存在

错误：

```text
Unknown database 'gorm_study'
```

进入 MySQL：

```powershell
docker exec -it gorm-mysql mysql -uroot -proot123456
```

创建：

```sql
CREATE DATABASE gorm_study DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

---

## 七、时间字段扫描问题

如果遇到时间字段扫描错误，检查 DSN 是否包含：

```text
parseTime=True
```

推荐完整 DSN：

```text
root:root123456@tcp(127.0.0.1:3306)/gorm_study?charset=utf8mb4&parseTime=True&loc=Local
```

---

## 八、中文乱码

检查数据库字符集：

```sql
SHOW CREATE DATABASE gorm_study;
```

检查表字符集：

```sql
SHOW CREATE TABLE users;
```

DSN 中也要有：

```text
charset=utf8mb4
```

---

## 九、表不存在

错误：

```text
Table 'gorm_study.users' doesn't exist
```

可能原因：

- 忘记执行 `AutoMigrate`。
- 连接到错误数据库。
- 模型没有被传入迁移。

检查当前数据库：

```sql
SELECT DATABASE();
SHOW TABLES;
```

---

## 十、GORM 日志太少

学习阶段建议开启：

```go
Logger: logger.Default.LogMode(logger.Info)
```

这样可以看到 SQL。否则你很难判断 GORM 到底执行了什么。

---

## 十一、本节达标标准

学完本节后，你应该能够做到：

- 按固定顺序排查环境问题。
- 判断 MySQL 容器是否运行。
- 判断端口和 DSN 是否匹配。
- 处理数据库不存在问题。
- 处理 `parseTime` 和字符集问题。
- 知道表不存在时先检查迁移和连接库。
