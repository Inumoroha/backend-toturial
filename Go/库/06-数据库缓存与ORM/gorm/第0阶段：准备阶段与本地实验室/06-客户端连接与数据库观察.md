# 06 使用客户端连接并观察数据库

本节目标：学会使用图形化客户端和命令行客户端观察数据库。GORM 写得对不对，不能只看 Go 代码，还要看数据库里真实生成的表、字段、索引和数据。

---

## 一、为什么要会观察数据库

GORM 初学者常见问题：

```text
以为表创建了，其实连错库了。
以为索引创建了，其实没有。
以为删除了数据，其实是软删除。
以为字段名是 UserID，其实数据库里是 user_id。
以为更新了 0 值，其实 GORM 忽略了。
```

这些问题只看代码很难发现，必须打开数据库检查。

## 二、推荐工具

任选一个：

- DBeaver
- DataGrip
- Navicat
- MySQL Workbench

如果你喜欢命令行，可以直接使用 MySQL CLI。

## 三、图形化客户端连接参数

```text
Host: 127.0.0.1
Port: 3306
User: root
Password: root123456
Database: gorm_study
```

如果 Docker Compose 端口写的是 `3307:3306`，这里端口要改成：

```text
3307
```

---

## 四、命令行连接

如果 MySQL 在 Docker 容器里：

```powershell
docker exec -it gorm-mysql mysql -uroot -proot123456 gorm_study
```

进入后执行：

```sql
SELECT DATABASE();
```

预期输出：

```text
gorm_study
```

这一步非常重要，它能确认你当前连接的是学习库。

---

## 五、常用观察命令

查看表：

```sql
SHOW TABLES;
```

查看表结构：

```sql
DESC users;
```

查看建表 SQL：

```sql
SHOW CREATE TABLE users;
```

查看索引：

```sql
SHOW INDEX FROM users;
```

查询最近数据：

```sql
SELECT *
FROM users
ORDER BY id DESC
LIMIT 10;
```

查看当前连接：

```sql
SHOW PROCESSLIST;
```

## 六、观察 AutoMigrate 的结果

假设 Go 模型：

```go
type User struct {
    ID        uint
    Email     string `gorm:"size:128;not null;uniqueIndex"`
    CreatedAt time.Time
}
```

执行：

```go
db.AutoMigrate(&User{})
```

然后在数据库里执行：

```sql
SHOW CREATE TABLE users;
SHOW INDEX FROM users;
```

你要检查：

- 表名是不是 `users`。
- 字段名是不是 `email`。
- `email` 是否 `NOT NULL`。
- `email` 是否有唯一索引。
- `created_at` 是否存在。

---

## 七、观察软删除

如果模型有：

```go
DeletedAt gorm.DeletedAt `gorm:"index"`
```

执行：

```go
db.Delete(&user)
```

数据库里不要只看：

```sql
SELECT * FROM users;
```

更应该看：

```sql
SELECT id, email, deleted_at
FROM users
ORDER BY id DESC;
```

软删除的本质是更新 `deleted_at`，不是物理删除。

---

## 八、观察零值更新

如果执行：

```go
db.Model(&user).Updates(User{
    Age: 0,
})
```

然后发现数据库里 `age` 没有变，不要急着怀疑数据库。GORM 用结构体更新时默认忽略零值。

你可以在数据库里确认：

```sql
SELECT id, age
FROM users
WHERE id = 1;
```

如果要更新 0，使用：

```go
db.Model(&user).Updates(map[string]any{
    "age": 0,
})
```

## 九、常见错误

### 1. 连接错数据库

很多人开了两个库：

```text
gorm_study
gorm_study_test
```

结果 Go 代码连的是测试库，客户端看的是开发库。遇到“表怎么没有变化”，先执行：

```sql
SELECT DATABASE();
```

### 2. 看不到软删除数据

GORM 普通查询会自动加：

```sql
deleted_at IS NULL
```

数据库客户端直接查表可以看到软删除数据。要理解两者差异。

### 3. 字段名理解错误

Go 字段：

```go
PasswordHash
```

数据库列名默认是：

```text
password_hash
```

## 十、本节达标标准

学完本节后，你应该能够做到：

- 使用图形化客户端连接 MySQL。
- 使用 MySQL CLI 连接学习库。
- 查看表、字段、索引和建表 SQL。
- 验证 GORM `AutoMigrate` 的结果。
- 看懂软删除在数据库中的表现。
- 用数据库查询验证 GORM 更新是否生效。
