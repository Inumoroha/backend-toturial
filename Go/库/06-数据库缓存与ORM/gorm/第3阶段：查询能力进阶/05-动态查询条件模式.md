# 05 动态查询条件模式

本节目标：掌握真实项目中最常见的动态查询写法。文章列表、订单列表、后台筛选都不是固定条件，而是根据请求参数动态拼装查询。

---

## 一、动态查询场景

文章列表可能支持：

```text
分类筛选
状态筛选
关键词搜索
作者筛选
时间范围
分页
排序
```

如果每种组合都写一个函数，代码会爆炸。更好的方式是从一个基础 query 开始，按条件逐步追加。

---

## 二、查询参数结构

```go
type ArticleQuery struct {
    CategoryID uint
    UserID     uint
    Status     string
    Keyword    string
    StartTime  *time.Time
    EndTime    *time.Time
    Page       int
    PageSize   int
}
```

时间用指针，是为了区分：

```text
没有传时间
传了时间零值
```

---

## 三、基础 query

```go
query := db.Model(&Article{})
```

不要一开始就执行 `Find`。GORM 链式调用是在构建查询，最后 `Find`、`First`、`Scan` 才真正执行。

---

## 四、逐步追加条件

```go
if q.CategoryID > 0 {
    query = query.Where("category_id = ?", q.CategoryID)
}

if q.UserID > 0 {
    query = query.Where("user_id = ?", q.UserID)
}

if q.Status != "" {
    query = query.Where("status = ?", q.Status)
}

if q.Keyword != "" {
    query = query.Where("title LIKE ?", "%"+q.Keyword+"%")
}

if q.StartTime != nil {
    query = query.Where("created_at >= ?", *q.StartTime)
}

if q.EndTime != nil {
    query = query.Where("created_at < ?", *q.EndTime)
}
```

---

## 五、执行 Count 和列表

```go
var total int64
if err := query.Count(&total).Error; err != nil {
    return nil, 0, err
}

page, pageSize := normalizePagination(q.Page, q.PageSize)
offset := (page - 1) * pageSize

var articles []Article
err := query.Order("created_at DESC").
    Limit(pageSize).
    Offset(offset).
    Find(&articles).Error
```

---

## 六、避免 query 污染

复杂情况下，`Count` 和 `Find` 复用同一个 query 可能带来链式状态影响。

可以抽函数：

```go
func buildArticleQuery(db *gorm.DB, q ArticleQuery) *gorm.DB {
    query := db.Model(&Article{})
    // append conditions
    return query
}
```

使用：

```go
countQuery := buildArticleQuery(db, q)
listQuery := buildArticleQuery(db, q)
```

这种写法更稳。

## 七、排序字段白名单

如果排序来自前端：

```text
sort=created_at_desc
```

不要直接拼：

```go
Order(req.Sort)
```

应该使用白名单：

```go
func resolveArticleOrder(sort string) string {
    switch sort {
    case "view_count_desc":
        return "view_count DESC"
    case "created_at_asc":
        return "created_at ASC"
    default:
        return "created_at DESC"
    }
}
```

## 八、常见错误

### 1. 忘记重新赋值 query

错误：

```go
query.Where("status = ?", "published")
```

正确：

```go
query = query.Where("status = ?", "published")
```

### 2. 直接拼接关键词

错误：

```go
Where("title LIKE '%" + keyword + "%'")
```

正确：

```go
Where("title LIKE ?", "%"+keyword+"%")
```

### 3. 前端排序直接进入 Order

这可能导致 SQL 注入。

## 九、本节达标标准

学完本节后，你应该能够做到：

- 使用结构体表达查询参数。
- 根据参数动态追加 `Where`。
- 分开处理 Count 和列表查询。
- 对排序字段做白名单。
- 避免字符串拼接 SQL 条件。
