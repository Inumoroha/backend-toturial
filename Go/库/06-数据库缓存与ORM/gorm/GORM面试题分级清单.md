# GORM 面试题分级清单

本节目标：把 `GORM面试题与参考答案.md` 中的面试题按难度拆成初级、中级、高级三个层次。你可以用这份清单判断自己现在处在哪个水平，也可以按层级进行阶段复习。

使用方式：

```text
初级：确认会用 GORM 完成基础业务
中级：确认能解释 SQL、事务、关联、性能问题
高级：确认能做工程设计、线上排查、取舍判断和项目表达
```

达标建议：

- 初级题能答出 80%，说明可以进入项目练习。
- 中级题能答出 70%，说明可以准备普通 Go 后端面试。
- 高级题能答出 60%，说明你不只是会写 API，而是具备工程化理解。

---

## 一、初级面试题清单

初级题重点考察：你是否知道 GORM 是什么，是否能用它完成单表 CRUD、条件查询、分页、软删除和基础模型定义。

### 1. GORM 是什么？

考察点：

- ORM 的概念。
- GORM 和数据库表之间的映射关系。
- GORM 不是 SQL 的替代品。

达标回答：

GORM 是 Go 语言常用 ORM 库，可以把结构体映射为数据库表，把字段映射为数据库列，用 Go 代码完成增删改查、关联、事务和迁移等操作。它能减少重复 CRUD 代码，但仍然需要理解底层 SQL。

追问方向：

- ORM 有什么优缺点？
- 什么时候不用 ORM？

---

### 2. GORM 和原生 SQL 相比有什么优缺点？

考察点：

- 开发效率。
- SQL 可控性。
- 复杂查询取舍。

达标回答：

普通 CRUD、分页、简单关联适合用 GORM；复杂统计、窗口函数、CTE、强性能优化查询适合直接写 SQL。GORM 提升效率，但不能让开发者忽略 SQL。

---

### 3. GORM 默认表名和字段名如何生成？

考察点：

- 结构体名转复数蛇形表名。
- 字段名转蛇形列名。
- `ID` 默认主键。

达标回答：

`BlogArticle` 默认映射为 `blog_articles` 表，`UserID` 默认映射为 `user_id` 字段。可以通过 `TableName()` 或 tag 自定义。

---

### 4. `gorm.Model` 包含哪些字段？

考察点：

- 主键。
- 创建时间。
- 更新时间。
- 软删除字段。

达标回答：

`gorm.Model` 包含 `ID`、`CreatedAt`、`UpdatedAt`、`DeletedAt`。学习阶段可以用，但真实项目中为了可读性和响应控制，也可以显式声明字段。

---

### 5. 如何定义主键？

考察点：

- 默认主键。
- `primaryKey` tag。
- 自定义主键类型。

达标回答：

字段名为 `ID` 时默认是主键，也可以使用 `gorm:"primaryKey"` 显式指定。自增 ID、雪花 ID、UUID 都可以作为主键方案。

---

### 6. 如何设置普通索引和唯一索引？

考察点：

- `index`。
- `uniqueIndex`。
- 联合索引。

达标回答：

邮箱、用户名、订单号适合唯一索引；外键字段和高频查询字段适合普通索引；多条件组合查询可以设计联合索引。

---

### 7. 什么是软删除？

考察点：

- `gorm.DeletedAt`。
- 普通查询自动过滤。
- `Delete` 不一定是物理删除。

达标回答：

模型中包含 `gorm.DeletedAt` 后，`Delete` 默认更新 `deleted_at` 字段，普通查询自动追加 `deleted_at IS NULL`。

---

### 8. `Create` 插入后主键会回填吗？

考察点：

- 指针传参。
- 数据库自增主键。

达标回答：

会。只要传入结构体指针，插入成功后 GORM 会把自增主键回填到结构体中。

---

### 9. `First`、`Take`、`Last`、`Find` 有什么区别？

考察点：

- 单条查询。
- 多条查询。
- 排序行为。
- `record not found`。

达标回答：

`First` 按主键升序取第一条，`Last` 按主键降序取最后一条，`Take` 不指定排序取一条，`Find` 查询多条。查询单条不存在时通常会返回 `ErrRecordNotFound`。

---

### 10. 如何处理 `record not found`？

考察点：

- 错误判断。
- HTTP 404 转换。

达标回答：

用 `errors.Is(err, gorm.ErrRecordNotFound)` 判断，然后转换成业务错误或 404 响应。不要把不存在和数据库故障混在一起。

---

### 11. `Save` 和 `Updates` 有什么区别？

考察点：

- 全字段保存。
- 局部更新。
- 零值问题。

达标回答：

`Save` 更接近全量保存，可能更新所有字段；`Updates` 更适合局部更新。真实项目中更推荐明确使用 `Updates`、`Select` 或 `map` 控制更新字段。

---

### 12. 结构体更新零值为什么不生效？

考察点：

- Go 零值。
- GORM 默认忽略零值。

达标回答：

使用结构体执行 `Updates` 时，空字符串、0、false 默认会被忽略。可以改用 `map` 或 `Select` 明确更新字段。

---

### 13. 如何写条件查询？

考察点：

- 参数绑定。
- 防 SQL 注入。

达标回答：

使用 `Where("email = ?", email)`，不要拼接用户输入。多个条件可以链式追加，也可以根据请求参数动态构造。

---

### 14. 如何实现分页？

考察点：

- `Limit`。
- `Offset`。
- 参数校验。

达标回答：

分页通常使用 `Limit(pageSize).Offset((page - 1) * pageSize)`。必须限制最大 `pageSize`，避免一次查出过多数据。

---

### 15. 如何只查询部分字段？

考察点：

- `Select`。
- 列表接口字段裁剪。

达标回答：

使用 `Select("id", "title", "created_at")`。列表页不要把正文、大 JSON、长文本字段都查出来。

---

### 16. 如何执行原生 SQL？

考察点：

- `Raw`。
- `Exec`。
- `Scan`。

达标回答：

查询类 SQL 用 `Raw(...).Scan(&result)`，写操作 SQL 可以用 `Exec`。复杂统计或数据库特定语法可以直接写 SQL。

---

### 17. `Scan` 和 `Find` 有什么区别？

考察点：

- 模型查询。
- 自定义结果结构。

达标回答：

`Find` 常用于查询模型或模型切片，`Scan` 常用于把查询结果扫描到自定义结构体，比如统计结果 DTO。

---

### 18. 如何打印 GORM 生成的 SQL？

考察点：

- Debug。
- Logger。

达标回答：

开发时可以用 `db.Debug()` 或配置 GORM Logger。生产环境应该重点记录慢 SQL 和错误 SQL。

---

### 19. Model 和 DTO 为什么要分离？

考察点：

- 数据库结构。
- 接口结构。
- 敏感字段。

达标回答：

Model 表示数据库结构，DTO 表示请求和响应结构。分离可以避免暴露密码哈希、内部状态等字段，也能让接口结构不被数据库表结构绑死。

---

### 20. GORM 项目通常如何分层？

考察点：

- Model。
- Repository。
- Service。
- Handler。

达标回答：

Model 定义数据库结构，Repository 封装数据访问，Service 处理业务规则和事务，Handler 负责 HTTP 请求响应。

---

## 二、中级面试题清单

中级题重点考察：你是否能解释关联、事务、SQL 安全、连接池、性能排查和项目中的常见坑。

### 1. GORM 支持哪些关联关系？

考察点：

- belongs to。
- has one。
- has many。
- many2many。

达标回答：

GORM 支持一对一、一对多、多对多等关联。比如文章属于用户，用户有多篇文章，文章和标签是多对多。

---

### 2. `Preload` 是什么？

考察点：

- 预加载。
- 分批查询。
- 避免 N+1。

达标回答：

`Preload` 用来预加载关联数据。它通常会先查询主表，再用关联外键批量查询关联表，适合文章详情、列表附带分类等场景。

---

### 3. `Preload` 和 `Joins` 有什么区别？

考察点：

- 多 SQL。
- JOIN SQL。
- 结果重复。

达标回答：

`Preload` 通常用多条 SQL 查询关联数据，结果映射更方便；`Joins` 生成 JOIN SQL，适合按关联表字段筛选或排序，但一对多 JOIN 容易造成主记录重复。

---

### 4. 什么是 N+1 查询？

考察点：

- 循环查关联。
- SQL 条数膨胀。

达标回答：

先查 N 条主记录，再循环为每条记录查一次关联，就会产生 1+N 条 SQL。解决方式是 `Preload`、JOIN、批量查询或缓存。

---

### 5. GORM 如何使用事务？

考察点：

- `db.Transaction`。
- `tx`。
- 错误回滚。

达标回答：

推荐使用 `db.Transaction(func(tx *gorm.DB) error { ... })`。函数返回错误时自动回滚，返回 nil 时自动提交。

---

### 6. 事务内部为什么不能使用外部 `db`？

考察点：

- 事务连接。
- 一致性。

达标回答：

事务内部必须使用 `tx`，否则部分操作会跑到事务外，导致回滚不完整，破坏一致性。

---

### 7. 如何安全扣减库存？

考察点：

- 条件更新。
- `RowsAffected`。
- 事务。

达标回答：

使用 `WHERE stock >= ?` 的条件更新，并检查 `RowsAffected`。订单创建、明细写入和库存扣减要放在同一个事务中。

---

### 8. 悲观锁怎么写？

考察点：

- `FOR UPDATE`。
- 事务内使用。

达标回答：

使用 `Clauses(clause.Locking{Strength: "UPDATE"})`，并且必须放在事务中。适合竞争强、需要串行化修改的场景。

---

### 9. 乐观锁怎么实现？

考察点：

- version 字段。
- 条件更新。

达标回答：

表中增加 `version` 字段，更新时带上 `WHERE id = ? AND version = ?`，同时把版本号加一。`RowsAffected=0` 表示数据已被别人修改。

---

### 10. GORM 默认事务是什么？

考察点：

- 写操作默认事务。
- 性能取舍。

达标回答：

GORM 对单条写操作默认开启事务，以保证写入安全。性能敏感场景可以关闭默认事务，但必须理解一致性风险。

---

### 11. 如何排查 GORM 慢查询？

考察点：

- SQL 日志。
- SQL 条数。
- EXPLAIN。
- 索引。

达标回答：

先打开 SQL 日志，看 SQL 条数和耗时，再复制慢 SQL 到数据库执行 `EXPLAIN`，检查索引、扫描行数、排序、回表和是否存在 N+1。

---

### 12. 如何配置连接池？

考察点：

- `db.DB()`。
- `SetMaxOpenConns`。
- `SetMaxIdleConns`。
- `SetConnMaxLifetime`。

达标回答：

通过 `sqlDB, _ := db.DB()` 获取底层连接池，再设置最大打开连接数、最大空闲连接数和连接生命周期。

---

### 13. 如何避免 SQL 注入？

考察点：

- 参数绑定。
- 排序白名单。

达标回答：

条件值用占位符传参，不拼接用户输入。`Order`、`Table`、`Select`、`Raw` 中涉及用户输入时要使用白名单。

---

### 14. 软删除和唯一索引冲突怎么处理？

考察点：

- 软删除数据仍存在。
- 唯一约束仍生效。

达标回答：

可以选择恢复账号、禁止重复注册、物理删除、联合唯一索引，或者软删除时改写唯一字段。具体取决于业务规则。

---

### 15. `RowsAffected` 有什么用？

考察点：

- 判断更新是否生效。
- 库存扣减。
- 乐观锁。

达标回答：

`RowsAffected` 表示受影响行数。条件更新库存或乐观锁更新时，必须检查它来判断是否成功。

---

### 16. `AutoMigrate` 在生产环境中可以直接用吗？

考察点：

- 自动迁移边界。
- migration。

达标回答：

本地和测试环境可以用，生产环境更推荐使用版本化 migration。生产 DDL 要可审计、可回滚、可评审。

---

### 17. `Unscoped` 是什么？

考察点：

- 查询软删除数据。
- 物理删除。

达标回答：

`Unscoped` 可以绕过软删除过滤，也可以执行物理删除。它权限较大，普通业务接口应谨慎使用。

---

### 18. `Session` 有什么用？

考察点：

- 局部配置。
- `DryRun`。
- `PrepareStmt`。

达标回答：

`Session` 用于创建带局部配置的 GORM 会话，不影响全局 `db`。常用于调试 SQL、局部启用预编译、跳过默认事务等。

---

### 19. `FindInBatches` 适合什么场景？

考察点：

- 大数据批处理。
- 内存控制。

达标回答：

适合批量修复数据、导出任务、离线任务，避免一次性加载大量记录到内存。

---

### 20. Repository 如何支持事务？

考察点：

- 传入 `*gorm.DB`。
- Repository 不自己开启事务。

达标回答：

Repository 方法接收 `*gorm.DB` 或者 Repository 可以基于 `tx` 构造。事务边界放在 Service 层，由 Service 决定多个 Repository 操作是否在同一事务中。

---

## 三、高级面试题清单

高级题重点考察：你是否能做架构取舍、线上问题排查、项目复盘，以及是否理解 ORM 背后的数据库行为。

### 1. 你如何评价 ORM？

考察点：

- 工具边界。
- SQL 能力。
- 工程效率。

达标回答：

ORM 是提高开发效率的工具，不是 SQL 的替代品。普通业务用 GORM 提升效率，复杂统计和性能瓶颈直接写 SQL。关键查询必须看生成 SQL 和执行计划。

---

### 2. 如果一个 GORM 接口很慢，你怎么排查？

考察点：

- 观测。
- SQL。
- 数据库执行计划。
- 业务接口设计。

达标回答：

先确认接口耗时分布，再看 GORM SQL 日志，检查 SQL 条数、是否 N+1、是否查了大字段、是否分页。然后用 `EXPLAIN` 看索引和扫描行数，最后根据结果优化查询、索引、缓存或接口设计。

---

### 3. 大分页如何优化？

考察点：

- Offset 性能。
- 游标分页。

达标回答：

`OFFSET` 很大时数据库需要跳过大量记录。可以改为基于 `id` 或 `created_at` 的游标分页，例如 `WHERE id < last_id ORDER BY id DESC LIMIT ?`。

---

### 4. 如何设计文章列表接口？

考察点：

- 字段裁剪。
- 动态查询。
- 分页。
- 索引。

达标回答：

列表接口只查询必要字段，支持分类、状态、关键词、时间范围等筛选，分页限制最大条数，排序字段使用白名单，并为高频条件设计联合索引。

---

### 5. 如何设计文章详情接口？

考察点：

- 主体查询。
- 关联加载。
- N+1。

达标回答：

文章详情查询文章主体，并按需要 `Preload` 作者、分类、标签、评论。评论可能分页加载，不一定一次性全部加载。

---

### 6. 创建文章并绑定标签如何保证一致性？

考察点：

- 事务。
- 多对多。
- Association Mode。

达标回答：

创建文章、校验标签、绑定标签关系放在同一个事务中。任何一步失败都回滚，避免文章已创建但标签关系缺失。

---

### 7. 如何设计订单下单流程？

考察点：

- 订单主表。
- 订单明细。
- 库存扣减。
- 事务。

达标回答：

在事务中创建订单、创建订单明细、扣减库存。库存扣减使用条件更新和 `RowsAffected` 防止超卖。

---

### 8. 如何防止重复支付？

考察点：

- 唯一支付单号。
- 状态机。
- 幂等。

达标回答：

支付回调使用支付单号唯一索引防重复，订单状态更新使用条件更新，例如只允许 `pending -> paid`。重复回调返回成功但不重复执行业务副作用。

---

### 9. 读写分离场景下 GORM 怎么处理？

考察点：

- Database Resolver。
- 主从延迟。
- 一致性读。

达标回答：

可以使用 GORM Database Resolver 做读写分离。但刚写完马上读的重要场景要走主库，避免主从延迟导致读不到新数据。

---

### 10. 生产环境如何做数据库迁移？

考察点：

- migration。
- 回滚。
- 灰度。
- 大表 DDL。

达标回答：

使用版本化 migration 文件管理结构变更。执行前备份和评审，评估锁表风险，必要时拆分变更或使用在线 DDL 工具。GORM 模型和数据库结构要保持同步。

---

### 11. Hook 适合做什么？不适合做什么？

考察点：

- 模型生命周期。
- 隐式副作用。

达标回答：

Hook 适合做创建前默认值、更新时间、简单校验等和模型强相关的逻辑。不适合写复杂业务流程、跨服务调用、发送消息等副作用。

---

### 12. Scope 的价值是什么？

考察点：

- 查询条件复用。
- 可组合。

达标回答：

Scope 可以封装可复用查询条件，比如分页、状态过滤、租户隔离。它让 Repository 查询逻辑更清晰。

---

### 13. 多租户系统中 GORM 查询要注意什么？

考察点：

- tenant_id。
- 数据隔离。
- Scope。

达标回答：

所有业务表应带 `tenant_id`，查询和写入必须带租户条件。可以用 Scope 或 Repository 统一注入租户过滤，避免漏条件造成越权访问。

---

### 14. 如何做 GORM 测试？

考察点：

- Repository 单测。
- 事务回滚。
- 测试数据库。

达标回答：

Repository 测试要连接真实测试数据库或容器数据库，验证 SQL 行为、唯一约束、事务回滚、软删除和关联查询。复杂 Service 要测试成功和失败回滚路径。

---

### 15. 如何把 GORM 项目讲成简历亮点？

考察点：

- 不写空话。
- 业务 + 技术 + 结果。

达标回答：

不要写“使用 GORM 完成增删改查”，而要写清楚模型关系、事务一致性、性能优化、索引设计和项目结果。

---

## 四、分级复习路线

### 第 1 轮：初级清单

目标：

- 能独立写出模型、连接数据库、完成 CRUD。
- 能解释常见 API 的差异。
