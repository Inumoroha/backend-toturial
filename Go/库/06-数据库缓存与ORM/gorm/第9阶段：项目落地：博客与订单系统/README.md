## 本阶段重点

- 从需求拆模型。
- 从模型拆接口。
- 从接口拆 Repository 和 Service。
- 关键写操作加事务。
- 列表接口分页。
- 详情接口预加载。
- 观察 SQL 并优化。

## 项目完成标准

一个项目不只是“能跑”，还应该具备：

- 清晰目录结构。
- 可读模型定义。
- 基础错误处理。
- 必要事务。
- 基础测试。
- README 说明如何运行。
- SQL 日志可开启。
- 接口可以用 Apifox 或 Postman 调试。
