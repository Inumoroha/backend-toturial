## 3. GORM 高频面试题

### GORM 的默认约定是什么？

结构体名默认映射为复数蛇形表名，字段名默认映射为蛇形列名，`ID` 默认是主键。

### `First`、`Take`、`Find` 有什么区别？

- `First`：按主键升序取第一条，查不到返回 `ErrRecordNotFound`。
- `Take`：不指定排序取一条，查不到返回 `ErrRecordNotFound`。
- `Find`：查询多条，查不到通常返回空切片。

### `Save` 和 `Updates` 有什么区别？

`Save` 倾向于保存整个模型，可能更新所有字段。`Updates` 更适合按需更新多个字段。真实项目里更推荐明确字段更新，避免误更新。

### 结构体更新为什么零值不生效？

GORM 使用结构体执行 `Updates` 时，默认忽略零值字段。要更新零值，可以使用 `map` 或 `Select`。

### 如何实现软删除？

模型中加入：

```go
DeletedAt gorm.DeletedAt `gorm:"index"`
```

调用 `Delete` 时，GORM 会更新 `deleted_at`，普通查询会自动排除软删除数据。

### `Preload` 和 `Joins` 有什么区别？

`Preload` 用额外 SQL 加载关联，再组装结果。`Joins` 使用 SQL JOIN 查询。普通关联加载优先 `Preload`，按关联字段过滤或排序时可考虑 `Joins`。

### 如何使用事务？

优先使用闭包事务：

```go
db.Transaction(func(tx *gorm.DB) error {
    // use tx
    return nil
})
```

事务内部必须使用 `tx`，错误要 return 出去。

### 如何打印 SQL？

单次查询用 `Debug`，全局配置用 GORM logger。

### 什么时候使用原生 SQL？

复杂统计、复杂联表、窗口函数、CTE、性能优化场景可以使用原生 SQL。简单 CRUD 和普通分页优先 GORM API。

## 4. 简历项目描述示例

博客系统：

```text
基于 Go、Gin、GORM、MySQL 实现博客 API，包含用户认证、文章管理、分类标签、评论管理等模块。使用 GORM 设计一对多和多对多关系，通过 Preload 优化文章详情关联查询，使用事务保证文章编辑与标签替换的一致性，并通过索引和分页优化文章列表查询。
```

订单系统：

```text
基于 Go、GORM、MySQL 实现订单系统核心流程，包括商品查询、下单、库存扣减、订单取消和支付模拟。下单流程使用 GORM 事务保证订单、订单明细和库存更新一致，通过条件更新和 RowsAffected 判断避免库存超扣，并设计订单状态流转防止重复支付和非法取消。
```
