# Go net 后端工程师学习路线

这套路线路面向想做 Go 后端工程师的学习者。目标不是把 `net` 包所有 API 背下来，而是理解真实服务如何建立连接、处理请求、控制超时、定位网络问题，并知道什么时候该使用更高层的 `net/http`、RPC 框架或消息队列。

```text
学习 net 的核心不是“手写一个 HTTP 框架”，而是理解后端服务在网络层如何接入、如何失败、如何保护自己。
```

## 一、学习目标

学完后你应该能够：

- 使用 `net.Listen`、`net.Dial`、`net.Conn` 编写最小 TCP 服务和客户端。
- 理解 IP、端口、连接、监听、读写、关闭这些后端日常概念。
- 给网络读写设置合理的超时，避免服务被慢连接拖死。
- 在 TCP 字节流上设计简单业务协议，知道粘包和半包为什么会出现。
- 用 goroutine 安全处理多连接，并能做基础连接数控制和优雅退出。
- 使用 DNS 查询、UDP、Unix Socket 解决常见后端工程问题。
- 知道网络程序上线前要检查哪些风险：超时、日志、指标、限流、错误处理、可回滚。

## 二、前置知识

建议先具备：

- Go 基础语法：变量、函数、结构体、接口、错误处理。
- goroutine 和 channel 的基础用法。
- 能使用命令行运行 `go run`、`go test`。
- 对 HTTP 有直觉：请求、响应、状态码即可，不需要深入协议细节。

## 三、阶段路线

### 阶段 1：建立网络编程直觉

对应目录：`01-基础入门`

1. 认识 `net`：后端服务的网络入口  
   明确 `net` 在 Go 后端里的位置，知道什么时候直接用它。

2. 地址、端口与 `net.Addr`  
   学会用 `JoinHostPort`、`SplitHostPort`、`ResolveTCPAddr` 处理服务地址，避免字符串拼接踩坑。

3. `net.Conn`：一次 TCP 连接的生命周期  
   理解连接的读、写、关闭，以及后端为什么必须考虑连接生命周期。

4. `net.Listen`：写一个最小 TCP 服务  
   学会监听端口、接收连接、处理错误。

### 阶段 2：面向后端业务的网络实践

对应目录：`02-后端网络实践`

5. 并发处理连接：每个连接一个 goroutine  
   让服务能同时处理多个客户端，并知道连接泄漏的风险。

6. 超时与取消：不要让慢连接拖垮服务  
   使用 `SetDeadline`、`SetReadDeadline`、`Dialer.DialContext` 控制等待时间。

7. TCP 字节流协议：解决粘包和半包  
   用换行协议和长度前缀协议承载用户、订单、短链接等业务命令。

8. DNS 与服务发现：从域名找到服务地址  
   使用 `Resolver` 和 `LookupHost`，理解 DNS 缓存、超时和故障降级。

9. UDP：适合日志、探测和低成本通知的场景  
   使用 `ListenPacket` 或 `ListenUDP`，理解 UDP 不保证送达的边界。

### 阶段 3：工程化、排障和上线意识

对应目录：`03-工程化与排障`

10. Unix Socket 与本机服务通信  
    用 Unix Socket 做本机 sidecar、代理、运维工具通信，并注意 Windows 差异。

11. 网络错误处理与日志：让问题可定位  
    识别超时、临时错误、连接关闭，记录足够但不过量的日志。

12. 连接治理与优雅退出：让服务可上线  
    控制连接数、关闭监听、等待连接处理完成，避免发布时直接中断请求。

13. 综合项目：订单通知 TCP 网关  
    把前面的能力串成一个小项目：接收订单通知、校验、响应、超时保护、日志和优雅退出。

## 四、建议学习节奏

- 第 1 天：完成第 1 到 4 节，能写出 TCP 服务和客户端。
- 第 2 天：完成第 5 到 7 节，理解并发、超时和协议边界。
- 第 3 天：完成第 8 到 10 节，知道 DNS、UDP、Unix Socket 的适用场景。
- 第 4 天：完成第 11 到 13 节，把综合项目跑起来，并尝试改造成自己的业务场景。

## 五、练习方式

每节都建议做到三件事：

- 先复制最小示例跑通。
- 再改一个业务字段，例如把 `user_id` 改成 `order_id`。
- 最后故意制造一个错误，例如端口被占用、客户端不发送换行、服务端超时，观察日志。

```text
后端工程师的网络能力，来自“跑通正常路径”和“见过失败路径”。
```

## 六、最终达标标准

完成整套教程后，你应该能够：

- 独立写一个 TCP 服务并说明它的连接处理流程。
- 给客户端和服务端都设置超时，并解释每个超时保护的是什么。
- 设计一个简单文本协议或长度前缀协议，避免把 TCP 当成消息队列。
- 判断 TCP、UDP、Unix Socket、HTTP 在项目中的适用边界。
- 看到 `connection reset by peer`、`i/o timeout`、`use of closed network connection` 时知道从哪里排查。
- 在发布服务前列出网络层检查清单：端口、超时、日志、连接数、优雅退出、回滚方案。
