# 3. 从单文件重构到分层项目

本节目标：把前面写在 `main.go` 中的用户 CRUD，一步一步拆成 model、repository、service、handler、router。重点不是一次写出完美结构，而是学会安全重构顺序。

---

## 一、为什么不要一次性全拆

很多人重构时会犯一个错误：

```text
一口气创建很多目录。
一口气移动很多代码。
一口气修改很多 import。
最后项目无法编译，不知道哪里错了。
```

更好的方式是：

```text
每次只移动一小块。
每移动一步就 go test 或 go run。
```

重构不是炫技，而是控制风险。

---

## 二、重构前的准备

先确认当前单文件接口是可运行的。

至少测试：

```http
GET http://localhost:8080/api/v1/users
POST http://localhost:8080/api/v1/users
GET http://localhost:8080/api/v1/users/1
```

然后提交一次 Git：

```bash
git add .
git commit -m "user crud before layering"
```

这样如果重构失败，你至少有一个清晰的回退点。

---

## 三、推荐重构顺序

建议按下面顺序：

```text
1. 抽 model
2. 抽 response
3. 抽 repository
4. 抽 service
5. 抽 handler
6. 抽 router
7. 精简 main
```

为什么这样排？

- model 依赖最少，最容易抽。
- response 也比较独立。
- repository 可以先封装内存 map。
- service 再调用 repository。
- handler 最后改成调用 service。
- router 最后接管路由注册。
- main 最终只做组装。

## 四、第 1 步：抽 model

创建：

```text
internal/model/user.go
```

代码：

```go
package model

import "time"

type User struct {
    ID        uint      `json:"id"`
    Username  string    `json:"username"`
    Email     string    `json:"email"`
    Age       int       `json:"age"`
    Status    string    `json:"status"`
    CreatedAt time.Time `json:"created_at"`
}
```

然后把原来 `main.go` 中的 `User` 删除，改成使用：

```go
model.User
```

导入：

```go
import "gin-layer-demo/internal/model"
```

运行：

```bash
go run ./cmd/server
```

如果你还没创建 `cmd/server`，可以先在根目录 `main.go` 中验证。

---

## 五、第 2 步：抽 response

创建：

```text
internal/response/response.go
internal/response/code.go
```

把第3阶段的统一响应搬进来：

```go
package response

import (
    "net/http"

    "github.com/gin-gonic/gin"
)

type Body struct {
    Code    int         `json:"code"`
    Message string      `json:"message"`
    Data    interface{} `json:"data,omitempty"`
}

func OK(c *gin.Context, data interface{}) {
    c.JSON(http.StatusOK, Body{Code: CodeOK, Message: "ok", Data: data})
}

func Created(c *gin.Context, data interface{}) {
    c.JSON(http.StatusCreated, Body{Code: CodeOK, Message: "ok", Data: data})
}

func Fail(c *gin.Context, status int, code int, message string) {
    c.JSON(status, Body{Code: code, Message: message})
}
```

错误码：

```go
package response

const (
    CodeOK             = 0
    CodeInvalidRequest = 40001
    CodeNotFound       = 40401
    CodeConflict       = 40901
    CodeInternalError  = 50001
)
```

然后把 handler 里的 `c.JSON` 逐步替换为：

```go
response.OK(c, data)
response.Fail(c, http.StatusBadRequest, response.CodeInvalidRequest, "invalid request")
```

## 六、第 3 步：抽 repository

创建：

```text
internal/repository/user_repository.go
```

代码：

```go
package repository

import (
    "errors"
    "strings"
    "time"

    "gin-layer-demo/internal/model"
)

var ErrUserNotFound = errors.New("user not found")
var ErrEmailAlreadyExists = errors.New("email already exists")

type UserRepository struct {
    users  map[uint]model.User
    nextID uint
}

func NewUserRepository() *UserRepository {
    return &UserRepository{
        users: map[uint]model.User{
            1: {ID: 1, Username: "alice", Email: "alice@example.com", Age: 20, Status: "active", CreatedAt: time.Now()},
            2: {ID: 2, Username: "bob", Email: "bob@example.com", Age: 22, Status: "disabled", CreatedAt: time.Now()},
        },
        nextID: 3,
    }
}
```

先只把数据放进 repository，不急着一次实现所有方法。

---

## 七、repository 方法

继续补方法：

```go
func (r *UserRepository) GetByID(id uint) (model.User, error) {
    user, ok := r.users[id]
    if !ok {
        return model.User{}, ErrUserNotFound
    }
    return user, nil
}

func (r *UserRepository) Create(user model.User) (model.User, error) {
    for _, item := range r.users {
        if item.Email == user.Email {
            return model.User{}, ErrEmailAlreadyExists
        }
    }

    user.ID = r.nextID
    user.CreatedAt = time.Now()
    r.users[user.ID] = user
    r.nextID++

    return user, nil
}

func (r *UserRepository) List(status, keyword string) []model.User {
    result := make([]model.User, 0)
    keyword = strings.ToLower(keyword)

    for _, user := range r.users {
        if status != "" && user.Status != status {
            continue
        }
        if keyword != "" {
            username := strings.ToLower(user.Username)
            email := strings.ToLower(user.Email)
            if !strings.Contains(username, keyword) && !strings.Contains(email, keyword) {
                continue
            }
        }
        result = append(result, user)
    }

    return result
}
```

repository 现在负责内存数据访问。后面换 GORM 时，这些方法内部会变，但上层调用方式尽量不变。

## 八、第 4 步：抽 service

创建：

```text
internal/service/user_service.go
```

代码：

```go
package service

import (
    "errors"

    "gin-layer-demo/internal/model"
    "gin-layer-demo/internal/repository"
)

var ErrUserNotFound = errors.New("user not found")
var ErrEmailAlreadyExists = errors.New("email already exists")
var ErrInvalidUserStatus = errors.New("invalid user status")

type UserService struct {
    repo *repository.UserRepository
}

func NewUserService(repo *repository.UserRepository) *UserService {
    return &UserService{repo: repo}
}
```

service 可以把 repository 错误转换成 service 自己的错误，也可以直接向上透传。入门阶段两种都能接受，但要保持清楚。

---

## 九、service 方法

```go
type CreateUserInput struct {
    Username string
    Email    string
    Age      int
    Status   string
}

func (s *UserService) Create(input CreateUserInput) (model.User, error) {
    status := input.Status
    if status == "" {
        status = "active"
    }
    if !isValidStatus(status) {
        return model.User{}, ErrInvalidUserStatus
    }

    user, err := s.repo.Create(model.User{
        Username: input.Username,
        Email:    input.Email,
        Age:      input.Age,
        Status:   status,
    })
    if errors.Is(err, repository.ErrEmailAlreadyExists) {
        return model.User{}, ErrEmailAlreadyExists
    }
    if err != nil {
        return model.User{}, err
    }

    return user, nil
}

func (s *UserService) Get(id uint) (model.User, error) {
    user, err := s.repo.GetByID(id)
    if errors.Is(err, repository.ErrUserNotFound) {
        return model.User{}, ErrUserNotFound
    }
    return user, err
}

func isValidStatus(status string) bool {
    return status == "active" || status == "disabled"
}
```

注意默认状态和状态合法性放在 service，因为它们是业务规则。

## 十、第 5 步：抽 handler

创建：

```text
internal/handler/user_handler.go
```

handler 结构：

```go
package handler

import (
    "errors"
    "net/http"
    "strconv"

    "gin-layer-demo/internal/response"
    "gin-layer-demo/internal/service"

    "github.com/gin-gonic/gin"
)

type UserHandler struct {
    service *service.UserService
}

func NewUserHandler(service *service.UserService) *UserHandler {
    return &UserHandler{service: service}
}
```

请求 DTO：

```go
type CreateUserRequest struct {
    Username string `json:"username" binding:"required,min=2,max=32"`
    Email    string `json:"email" binding:"required,email"`
    Age      int    `json:"age" binding:"gte=1,lte=120"`
    Status   string `json:"status" binding:"omitempty,oneof=active disabled"`
}
```

handler 示例：

```go
func (h *UserHandler) Create(c *gin.Context) {
    var req CreateUserRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.Fail(c, http.StatusBadRequest, response.CodeInvalidRequest, "invalid request")
        return
    }

    user, err := h.service.Create(service.CreateUserInput{
        Username: req.Username,
        Email:    req.Email,
        Age:      req.Age,
        Status:   req.Status,
    })
    if errors.Is(err, service.ErrEmailAlreadyExists) {
        response.Fail(c, http.StatusConflict, response.CodeConflict, "email already exists")
        return
    }
    if errors.Is(err, service.ErrInvalidUserStatus) {
        response.Fail(c, http.StatusBadRequest, response.CodeInvalidRequest, "invalid status")
        return
    }
    if err != nil {
        response.Fail(c, http.StatusInternalServerError, response.CodeInternalError, "internal server error")
        return
    }

    response.Created(c, user)
}
```

handler 负责把 HTTP 请求转换成 service input，再把 service error 转换成 HTTP 响应。

## 十一、第 6 步：抽 router

创建：

```text
internal/router/router.go
```

代码：

```go
package router

import (
    "gin-layer-demo/internal/handler"

    "github.com/gin-gonic/gin"
)

func New(userHandler *handler.UserHandler) *gin.Engine {
    r := gin.Default()

    api := r.Group("/api/v1")
    {
        api.GET("/users", userHandler.List)
        api.POST("/users", userHandler.Create)
        api.GET("/users/:id", userHandler.Get)
        api.PUT("/users/:id", userHandler.Update)
        api.PATCH("/users/:id/status", userHandler.UpdateStatus)
        api.DELETE("/users/:id", userHandler.Delete)
    }

    return r
}
```

如果某些 handler 方法还没实现，可以先只注册已经实现的路由。

## 十二、第 7 步：精简 main

`cmd/server/main.go`：

```go
package main

import (
    "log"

    "gin-layer-demo/internal/handler"
    "gin-layer-demo/internal/repository"
    "gin-layer-demo/internal/router"
    "gin-layer-demo/internal/service"
)

func main() {
    userRepo := repository.NewUserRepository()
    userService := service.NewUserService(userRepo)
    userHandler := handler.NewUserHandler(userService)

    r := router.New(userHandler)

    if err := r.Run(":8080"); err != nil {
        log.Fatal(err)
    }
}
```

现在 main 的职责非常清楚：

```text
创建依赖。
组装依赖。
启动服务。
```

如果还没有测试，也执行：

## 十四、常见问题

### 1. import path 不对

检查 `go.mod` 中的 module 名。

如果是：

```go
module gin-layer-demo
```

导入路径就应该是：

```go
"gin-layer-demo/internal/model"
```

### 2. 出现 import cycle

例如：

```text
handler imports service
service imports handler
```

说明依赖方向错了。

service 不应该导入 handler。

### 3. 移动代码后方法找不到

检查：

- 方法名是否首字母大写。
- 包名是否正确。
- 构造函数是否返回正确类型。
- router 是否使用了新的 handler 方法。

## 十五、本节练习

把你当前的用户 CRUD 按顺序拆成：

```text
model
response
repository
service
handler
router
main
```

要求：

- 每拆一步都运行一次。
- 每拆一步都至少测试一个接口。
- 最终 `main.go` 不再包含具体 handler 逻辑。
- handler 不再直接访问 users map。
