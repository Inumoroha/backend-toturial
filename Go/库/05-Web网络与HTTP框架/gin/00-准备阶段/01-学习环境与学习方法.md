# 00-01 学习环境与学习方法

本阶段目标：把学习 Gin 需要的环境、工具和心态准备好。Gin 只是 Web 框架，真正的后端能力还包括 HTTP、数据库、工程结构、测试、部署和排错能力。

## 一、你需要准备什么

### 1. Go 环境

建议安装 Go 1.22 或更高版本。安装完成后检查：

```bash
go version
go env GOPATH
go env GOPROXY
```

国内网络环境下可以设置 Go 代理：

```bash
go env -w GOPROXY=https://goproxy.cn,direct
```

### 2. 编辑器

推荐使用 VS Code 或 GoLand。

如果使用 VS Code，建议安装：

- Go 插件
- GitLens
- REST Client
- Docker 插件
- YAML 插件

### 3. 接口测试工具

至少准备一个接口测试工具：

- Postman
- Apifox
- Insomnia
- VS Code REST Client

初学阶段推荐 Apifox 或 Postman，因为图形界面更直观。

### 4. 数据库和缓存

后续教程会用到：

- MySQL 或 PostgreSQL，二选一即可。
- Redis。
- Docker Desktop，用于统一启动依赖。

如果你还没有安装数据库，可以先继续学习前几个阶段。Gin 路由、参数、响应、中间件这些内容不依赖数据库。

## 二、建议的学习方式

### 1. 先跑通，再理解，再重构

学习后端最容易卡在“我想一次写得很优雅”。不要这样。

每个功能都按三步来：

1. 先把功能跑通。
2. 再理解每行代码为什么这样写。
3. 最后再重构目录结构、抽公共方法、补测试。

这比一开始就追求完美架构更有效。

### 2. 每一阶段都要有可运行代码

不要只看教程。后端开发的很多问题只有运行后才会出现，例如：

- 路由没匹配上。
- 参数没有绑定成功。
- JSON 字段名和结构体 tag 不一致。
- 数据库连接失败。
- token 过期或解析失败。
- 跨域请求失败。

每个阶段都要留下一个可以运行的小项目。

### 3. 写学习笔记

建议每学完一节，都记录：

- 今天学了什么。
- 哪个概念最难。
- 哪个错误排查了多久。
- 如果明天重写一次，我会怎么写。

工程能力往往不是靠“看懂”形成的，而是靠重复排错和复盘形成的。

## 三、统一项目约定

后续教程默认使用下面的项目风格。

### 1. 模块名

示例模块名：

```bash
go mod init gin-learning
```

你也可以使用自己的模块名，例如：

```bash
go mod init github.com/yourname/gin-learning
```

### 2. API 前缀

统一使用：

```text
/api/v1
```

例如：

```text
GET /api/v1/users
POST /api/v1/users
```

### 3. 响应格式

后续会逐步封装统一响应格式：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

### 4. 错误格式

错误响应示例：

```json
{
  "code": 40001,
  "message": "invalid request"
}
```

### 5. 学习项目

前半段用“用户管理 API”练习基础能力。

后半段用“任务管理系统”做综合实战。

## 四、阶段练习

完成准备阶段后，请做下面几件事：

1. 安装 Go 并确认 `go version` 正常输出。
2. 创建一个空目录 `gin-learning`。
3. 执行 `go mod init gin-learning`。
4. 安装一个接口测试工具。
5. 准备一个笔记文件，记录学习过程。

## 五、验收清单

完成本阶段后，你应该能回答：

- Go 模块是用来做什么的？
- 为什么学习 Gin 之前要理解 HTTP？
- 为什么每个阶段都要保留可运行代码？
- API 前缀 `/api/v1` 有什么意义？
- 统一响应格式解决了什么问题？

如果这些问题你能说清楚，就可以进入下一阶段。
