# 06-02 认证模块完整实现步骤

本节目标：把注册、登录、JWT、中间件串成完整流程。上一节讲了概念，这一节按开发顺序走一遍。

## 一、认证流程总览

注册流程：

```text
请求注册 -> 参数校验 -> 检查邮箱是否存在 -> 密码加密 -> 保存用户 -> 返回用户信息
```

登录流程：

```text
请求登录 -> 参数校验 -> 根据邮箱查用户 -> 校验密码 -> 生成 JWT -> 返回 token
```

访问私有接口：

```text
读取 Authorization -> 解析 token -> 写入 gin.Context -> handler 读取 user_id -> 查询数据
```

## 二、请求结构体

```go
type RegisterRequest struct {
    Username string `json:"username" binding:"required,min=2,max=32"`
    Email    string `json:"email" binding:"required,email"`
    Password string `json:"password" binding:"required,min=8,max=64"`
}

type LoginRequest struct {
    Email    string `json:"email" binding:"required,email"`
    Password string `json:"password" binding:"required"`
}
```

密码长度必须限制。太短不安全，太长也可能带来无意义的计算成本。

## 三、AuthService

```go
type AuthService struct {
    userRepo *repository.UserRepository
    jwtSecret string
}

func NewAuthService(userRepo *repository.UserRepository, jwtSecret string) *AuthService {
    return &AuthService{userRepo: userRepo, jwtSecret: jwtSecret}
}
```

注册：

```go
func (s *AuthService) Register(input RegisterInput) (*model.User, error) {
    _, err := s.userRepo.GetByEmail(input.Email)
    if err == nil {
        return nil, ErrEmailAlreadyExists
    }
    if err != nil && !errors.Is(err, repository.ErrUserNotFound) {
        return nil, err
    }

    hash, err := HashPassword(input.Password)
    if err != nil {
        return nil, err
    }

    user := &model.User{
        Username: input.Username,
        Email: input.Email,
        PasswordHash: hash,
        Role: "user",
    }

    if err := s.userRepo.Create(user); err != nil {
        return nil, err
    }
    return user, nil
}
```

登录：

```go
func (s *AuthService) Login(input LoginInput) (string, error) {
    user, err := s.userRepo.GetByEmail(input.Email)
    if errors.Is(err, repository.ErrUserNotFound) {
        return "", ErrInvalidCredentials
    }
    if err != nil {
        return "", err
    }

    if !CheckPassword(input.Password, user.PasswordHash) {
        return "", ErrInvalidCredentials
    }

    return GenerateToken(user.ID, user.Email, user.Role, s.jwtSecret)
}
```

登录失败时建议统一返回“邮箱或密码错误”，不要告诉攻击者到底是邮箱不存在还是密码不对。

## 四、AuthHandler

```go
func (h *AuthHandler) Register(c *gin.Context) {
    var req RegisterRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.Fail(c, http.StatusBadRequest, 40001, "invalid request")
        return
    }

    user, err := h.service.Register(service.RegisterInput{
        Username: req.Username,
        Email: req.Email,
        Password: req.Password,
    })
    if errors.Is(err, service.ErrEmailAlreadyExists) {
        response.Fail(c, http.StatusConflict, 40901, "email already exists")
        return
    }
    if err != nil {
        response.Fail(c, http.StatusInternalServerError, 50001, "internal server error")
        return
    }

    response.Created(c, user)
}
```

登录：

```go
func (h *AuthHandler) Login(c *gin.Context) {
    var req LoginRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.Fail(c, http.StatusBadRequest, 40001, "invalid request")
        return
    }

    token, err := h.service.Login(service.LoginInput{
        Email: req.Email,
        Password: req.Password,
    })
    if errors.Is(err, service.ErrInvalidCredentials) {
        response.Fail(c, http.StatusUnauthorized, 40101, "invalid email or password")
        return
    }
    if err != nil {
        response.Fail(c, http.StatusInternalServerError, 50001, "internal server error")
        return
    }

    response.OK(c, gin.H{"token": token})
}
```

## 五、认证中间件要设置 user_id

中间件解析 token 后：

```go
c.Set("user_id", claims.UserID)
```

handler 读取：

```go
userID := c.GetUint("user_id")
```

如果你用的是 `c.Set("user_id", uint(...))`，`c.GetUint` 可以正常读取。类型要保持一致。

## 六、测试顺序

按这个顺序测：

1. 注册成功。
2. 重复注册失败。
3. 密码太短注册失败。
4. 登录成功。
5. 登录密码错误失败。
6. 不带 token 访问 `/profile` 失败。
7. 带 token 访问 `/profile` 成功。
8. 修改 token 中一个字符，再访问失败。

## 七、常见坑

- 保存了明文密码。
- 返回用户信息时把 `password_hash` 返回了。
- JWT secret 写死在代码里。
- token 没设置过期时间。
- Authorization 少了 `Bearer ` 前缀。
- 中间件没有 `c.Abort()`，导致失败后继续执行 handler。
