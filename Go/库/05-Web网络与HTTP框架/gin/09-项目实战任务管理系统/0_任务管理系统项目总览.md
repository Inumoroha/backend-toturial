# 0. 任务管理系统项目总览

本阶段目标：完成一个可以作为 Go 后端入门作品的 Gin 任务管理系统。这个项目会把前面学到的路由、参数校验、项目分层、GORM、JWT、Redis、测试和 Docker 部署串起来。

项目不追求复杂页面，也不做前端。我们只做后端 API。

---

## 一、项目要解决什么问题

用户可以注册账号，登录后管理自己的任务。

核心功能：

```text
注册
登录
获取个人信息
创建任务
查询任务列表
查询任务详情
更新任务
修改任务状态
删除任务
```

后端重点不只是“功能能跑”，还要训练：

- API 设计。
- 分层结构。
- 参数校验。
- 统一响应。
- 错误处理。
- 数据库操作。
- 登录认证。
- 权限边界。
- 测试和部署。

职责边界：

- `cmd/server`：程序入口和依赖组装。

## 九、主线数据流

创建任务的数据流：

```text
HTTP POST /api/v1/tasks
  ↓
JWT 中间件解析当前用户
  ↓
handler 绑定 JSON 并校验参数
  ↓
service 设置默认状态并检查业务规则
  ↓
repository 写入 tasks 表
  ↓
handler 返回统一响应
```

查询任务详情的数据流：

```text
HTTP GET /api/v1/tasks/:id
  ↓
JWT 中间件解析当前用户
  ↓
handler 解析路径参数
  ↓
service 调用 repository
  ↓
repository 使用 id + user_id 查询
  ↓
返回任务或 404
```

注意这里的 `user_id`。它是权限边界的核心。

## 融合补充

+### 09-01 需求、接口与数据模型

#### 第一版范围

个人任务管理系统支持注册、登录、查看个人信息以及创建、查询、修改、完成和删除自己的任务。管理员可扩展查看全部任务。第一版不做团队、评论、复杂搜索和附件审核，先完成可靠的核心闭环。

业务规则：邮箱唯一；密码仅存哈希；普通用户只能操作自己的任务；状态只能是 `todo`、`doing`、`done`；进入 `done` 时写完成时间；列表分页、过滤和排序均受限。

| 方法 | 路径 | 认证 | 含义 |
| --- | --- | --- | --- |
| POST | `/api/v1/auth/register` | 否 | 注册 |
| POST | `/api/v1/auth/login` | 否 | 登录，返回 JWT |
| GET | `/api/v1/profile` | 是 | 当前用户 |
| POST | `/api/v1/tasks` | 是 | 创建任务 |
| GET | `/api/v1/tasks` | 是 | 分页查询任务 |
| GET | `/api/v1/tasks/:id` | 是 | 查看详情 |
| PATCH | `/api/v1/tasks/:id` | 是 | 修改内容 |
| PATCH | `/api/v1/tasks/:id/status` | 是 | 变更状态 |
| DELETE | `/api/v1/tasks/:id` | 是 | 软删除 |

```go
type TaskStatus string
const (
    TaskTodo TaskStatus = "todo"
    TaskDoing TaskStatus = "doing"
    TaskDone TaskStatus = "done"
)

type Task struct {
    ID uint `gorm:"primaryKey" json:"id"`
    OwnerID uint `gorm:"not null;index" json:"owner_id"`
    Title string `gorm:"size:120;not null" json:"title"`
    Description string `gorm:"type:text" json:"description"`
    Status TaskStatus `gorm:"size:16;not null;index" json:"status"`
    DueAt *time.Time `json:"due_at"`
    CompletedAt *time.Time `json:"completed_at"`
    CreatedAt time.Time `json:"created_at"`
    UpdatedAt time.Time `json:"updated_at"`
    DeletedAt gorm.DeletedAt `gorm:"index" json:"-"`
}
```

常见查询是“某用户按状态、创建时间列任务”，可从 `(owner_id, status, created_at)` 复合索引开始；上线前用真实数据量与 `EXPLAIN` 验证。索引来自查询模式，而不是越多越好。

**验收：**接口表能直接转成 OpenAPI 文档；每条规则都有明确的责任层与测试场景。

+### 09-02 核心实现、测试、部署与项目验收

#### 路由与 Handler

```go
api := r.Group("/api/v1")
api.POST("/auth/register", authHandler.Register)
api.POST("/auth/login", authHandler.Login)

protected := api.Group("")
protected.Use(middleware.Auth(cfg.JWT.Secret))
protected.GET("/profile", profileHandler.Get)
protected.POST("/tasks", taskHandler.Create)
protected.GET("/tasks", taskHandler.List)
protected.GET("/tasks/:id", taskHandler.Get)
protected.PATCH("/tasks/:id", taskHandler.Update)
protected.PATCH("/tasks/:id/status", taskHandler.ChangeStatus)
protected.DELETE("/tasks/:id", taskHandler.Delete)
```

Handler 只做绑定、从 Context 获取 `user_id`、调用 Service 和映射错误。客户端不提交 owner ID；Service 读取任务后判定所有权，Repository 只处理存取。

```go
func (s *TaskService) ChangeStatus(ctx context.Context, userID, taskID uint, next domain.TaskStatus) (*domain.Task, error) {
    task, err := s.repo.FindByID(ctx, taskID)
    if err != nil { return nil, err }
    if task.OwnerID != userID { return nil, domain.ErrForbidden }
    if !validTransition(task.Status, next) { return nil, domain.ErrInvalidState }
    now := time.Now()
    task.Status = next
    if next == domain.TaskDone { task.CompletedAt = &now } else { task.CompletedAt = nil }
    if err := s.repo.Save(ctx, task); err != nil { return nil, err }
    return task, nil
}
```

#### 最小验收流程

```bash
curl -X POST http://localhost:8080/api/v1/auth/register -H 'Content-Type: application/json' -d '{"name":"Ada","email":"ada@example.com","password":"secure-password"}'
curl -X POST http://localhost:8080/api/v1/auth/login -H 'Content-Type: application/json' -d '{"email":"ada@example.com","password":"secure-password"}'
curl -X POST http://localhost:8080/api/v1/tasks -H 'Authorization: Bearer TOKEN' -H 'Content-Type: application/json' -d '{"title":"finish gin tutorial"}'
curl 'http://localhost:8080/api/v1/tasks?page=1&page_size=20' -H 'Authorization: Bearer TOKEN'
```

自动化测试至少覆盖：重复注册、错误密码、无 token、过期 token、访问他人任务、非法状态、分页边界、事务失败回滚、Redis 不可用降级。执行 `go test ./...`、`go test -race ./...` 后，再通过 `docker compose up --build` 做一次从 Nginx 到数据库的完整验收。

#### 完成定义

- 空数据库执行迁移后可启动；重启不会丢失数据。
- 所有端点在文档中说明认证、请求、响应、状态码和错误码。
- 非法输入不 panic，内部错误不泄露细节，日志可通过 request ID 关联。
- 缓存、限流、上传与关闭逻辑均有超时和失败处理。
- 密钥不在仓库；备份、迁移、监控和告警策略已记录。
