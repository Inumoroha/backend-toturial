# 7. 接口测试请求集合

本节目标：给出第3阶段所有关键接口的测试请求。你可以把这些请求复制到项目根目录的 `requests.http` 文件中，用 VS Code REST Client、Apifox 或 Postman 逐个验证。

接口测试请求是学习后端时非常重要的资产。它不仅能验证接口是否可用，还能帮你记录每个接口应该如何调用。

---

## 一、基础地址

本阶段默认服务地址：

```text
http://localhost:8080
```

API 前缀：

```text
/api/v1
```

如果你把端口改成了 8081，下面请求中的端口也要同步修改。

---

## 二、查询用户列表

```http
### list users default
GET http://localhost:8080/api/v1/users
```

期望：

```text
HTTP 200
code = 0
data.items 是数组
data.total 是数字
```

```http

GET http://localhost:8080/api/v1/users?page=1&page_size=10
```

```http
### invalid page
GET http://localhost:8080/api/v1/users?page=0&page_size=10
```


## 七、创建用户参数错误

用户名太短：

```http
### create user invalid username
POST http://localhost:8080/api/v1/users
Content-Type: application/json

{
  "username": "a",
  "email": "new@example.com",
  "age": 20
}
```

邮箱格式错误：

```http
### create user invalid email
POST http://localhost:8080/api/v1/users
Content-Type: application/json

{
  "username": "newuser",
  "email": "not-email",
  "age": 20
}
```

年龄非法：

```http
### create user invalid age
POST http://localhost:8080/api/v1/users
Content-Type: application/json

{
  "username": "newuser",
  "email": "new@example.com",
  "age": 200
}
```

状态非法：

```http
### create user invalid status
POST http://localhost:8080/api/v1/users
Content-Type: application/json

{
  "username": "newuser",
  "email": "new@example.com",
  "age": 20,
  "status": "unknown"
}
```

期望：

```text
HTTP 400
code = 40001
```

## 十三、404 和 405

不存在路由：

```http
### route not found
GET http://localhost:8080/api/v1/not-exists
```

方法不允许：

```http
### method not allowed
POST http://localhost:8080/api/v1/users/1
```

前提是你没有注册 `POST /api/v1/users/:id`，并且启用了：

```go
r.HandleMethodNotAllowed = true
```

---

## 十四、建议的测试顺序

建议按下面顺序测试：

1. 查询默认用户列表。
2. 创建一个新用户。
3. 查询新用户详情。
4. 用非法参数创建用户。
5. 重复邮箱创建用户。
6. 修改用户状态。
7. 用状态筛选列表。
8. 删除用户。
9. 查询已删除用户。
10. 测试 404 和 405。

按这个顺序测试，能覆盖本阶段大部分关键行为。

---

## 十五、测试记录模板

建议每次测试后记录结果：

```text
接口：
请求：
预期状态码：
实际状态码：
预期 code：
实际 code：
是否通过：
备注：
```

示例：

```text
接口：创建用户参数错误
请求：POST /api/v1/users，username = a
预期状态码：400
实际状态码：400
预期 code：40001
实际 code：40001
是否通过：是
备注：字段级错误中包含 username
```

这样做的好处是：当接口行为变化时，你能清楚知道是有意调整，还是重构引入的问题。

---

## 十六、本节验收标准

完成本请求集合后，请确认：

- [ ] 成功创建用户返回 `201` 或团队约定的成功状态码。
- [ ] 参数错误返回 `400` 和稳定业务错误码。
- [ ] 邮箱重复返回 `409`。
- [ ] 用户不存在返回 `404`。
- [ ] 列表接口返回分页结构。
- [ ] 非法分页、非法状态都能被拦截。
- [ ] 404 路由不存在返回统一 JSON。
- [ ] 405 方法不允许返回统一 JSON。
- [ ] 每个失败请求都不是 Gin 默认文本响应。

如果这些请求全部通过，第3阶段的路由、参数绑定和统一响应才算真正闭环。
