# 03-02 校验错误细化与错误码规范

本节目标：把“参数错误”讲得更细。真实项目中，简单返回 `invalid request` 可以用，但学习阶段你要知道如何定位具体字段错误，也要知道什么时候应该隐藏细节。

## 一、为什么要细化校验错误

如果用户提交：

```json
{
  "username": "",
  "email": "abc",
  "age": 200
}
```

只返回：

```json
{
  "code": 40001,
  "message": "invalid request"
}
```

前端和调用者不知道到底错在哪里。

开发和调试阶段可以返回字段级错误：

```json
{
  "code": 40001,
  "message": "invalid request",
  "data": {
    "username": "required",
    "email": "email",
    "age": "lte"
  }
}
```

## 二、解析 validator 错误

Gin 使用 validator 做结构体校验。可以这样处理：

```go
func FormatValidationError(err error) map[string]string {
    result := make(map[string]string)

    var validationErrors validator.ValidationErrors
    if errors.As(err, &validationErrors) {
        for _, fieldError := range validationErrors {
            field := fieldError.Field()
            tag := fieldError.Tag()
            result[field] = tag
        }
        return result
    }

    result["body"] = "invalid json"
    return result
}
```

需要导入：

```go
import (
    "errors"

    "github.com/go-playground/validator/v10"
)
```

## 三、字段名转换问题

上面的 `fieldError.Field()` 返回 Go 字段名，例如 `Username`，不是 JSON 字段名 `username`。

学习阶段可以先接受这个结果。进阶时可以通过反射读取 `json` tag，把字段名转换成前端更熟悉的形式。

示例目标：

```json
{
  "username": "required",
  "email": "email"
}
```

## 四、统一错误响应可以带 data

把响应结构调整为：

```go
type Body struct {
    Code    int         `json:"code"`
    Message string      `json:"message"`
    Data    interface{} `json:"data,omitempty"`
}
```

增加：

```go
func FailWithData(c *gin.Context, httpStatus int, code int, message string, data interface{}) {
    c.JSON(httpStatus, Body{
        Code:    code,
        Message: message,
        Data:    data,
    })
}
```

handler 中使用：

```go
if err := c.ShouldBindJSON(&req); err != nil {
    response.FailWithData(c, http.StatusBadRequest, 40001, "invalid request", response.FormatValidationError(err))
    return
}
```

## 五、什么时候不返回详细错误

不是所有错误都应该详细返回。

可以详细返回：

- 参数缺失。
- 字段格式不对。
- 分页参数非法。

不要详细返回：

- 数据库 SQL 错误。
- 文件系统路径错误。
- 密码校验内部逻辑。
- token 签名错误细节。

对外统一返回：

```json
{
  "code": 50001,
  "message": "internal server error"
}
```

详细错误写入日志。

## 六、错误码分层建议

可以按模块划分错误码：

```text
400xx 通用请求错误
401xx 认证错误
403xx 权限错误
404xx 资源不存在
409xx 冲突错误
429xx 限流错误
500xx 服务错误
```

更大项目可以按业务模块：

```text
10001 用户名不能为空
10002 邮箱已存在
20001 任务不存在
20002 任务状态非法
```

初学阶段不必设计太复杂，但要保证一个原则：同一种错误不要今天返回 40001，明天返回 50001。
