# 04-01 项目结构与分层重构

本阶段目标：把所有代码从 `main.go` 中拆出来，形成更接近真实项目的结构。你要开始从“写接口”进入“写工程”。

## 一、为什么需要分层

如果所有逻辑都写在 handler 里，代码会很快变成这样：

- 解析参数。
- 校验参数。
- 查数据库。
- 判断业务规则。
- 组装响应。
- 记录日志。
- 处理错误。

一个函数几百行，后面很难维护。

分层的目的不是为了显得高级，而是为了让每一层职责清楚：

- handler 负责 HTTP。
- service 负责业务。
- repository 负责数据访问。
- model 负责数据结构。
- response 负责响应格式。

## 二、推荐目录结构

```text
gin-learning/
  cmd/
    server/
      main.go
  internal/
    handler/
      user_handler.go
    service/
      user_service.go
    repository/
      user_repository.go
    model/
      user.go
    response/
      response.go
    router/
      router.go
  go.mod
```

`internal` 是 Go 的特殊目录。放在里面的包不能被项目外部直接导入，适合存放业务代码。

## 三、model 层

`internal/model/user.go`

```go
package model

type User struct {
    ID       int    `json:"id"`
    Username string `json:"username"`
    Email    string `json:"email"`
    Age      int    `json:"age"`
    Status   string `json:"status"`
}
```

## 四、repository 层

`internal/repository/user_repository.go`

```go
package repository

import (
    "errors"

    "gin-learning/internal/model"
)

var ErrUserNotFound = errors.New("user not found")

type UserRepository struct {
    users  map[int]model.User
    nextID int
}

func NewUserRepository() *UserRepository {
    return &UserRepository{
        users: map[int]model.User{
            1: {ID: 1, Username: "alice", Email: "alice@example.com", Age: 20, Status: "active"},
            2: {ID: 2, Username: "bob", Email: "bob@example.com", Age: 22, Status: "active"},
        },
        nextID: 3,
    }
}

func (r *UserRepository) List() []model.User {
    result := make([]model.User, 0, len(r.users))
    for _, user := range r.users {
        result = append(result, user)
    }
    return result
}

func (r *UserRepository) GetByID(id int) (model.User, error) {
    user, ok := r.users[id]
    if !ok {
        return model.User{}, ErrUserNotFound
    }
    return user, nil
}

func (r *UserRepository) Create(user model.User) model.User {
    user.ID = r.nextID
    r.users[user.ID] = user
    r.nextID++
    return user
}
```

repository 只关心数据如何保存和读取，不关心 HTTP 请求。

## 五、service 层

`internal/service/user_service.go`

```go
package service

import (
    "gin-learning/internal/model"
    "gin-learning/internal/repository"
)

type CreateUserInput struct {
    Username string
    Email    string
    Age      int
}

type UserService struct {
    repo *repository.UserRepository
}

func NewUserService(repo *repository.UserRepository) *UserService {
    return &UserService{repo: repo}
}

func (s *UserService) ListUsers() []model.User {
    return s.repo.List()
}

func (s *UserService) GetUser(id int) (model.User, error) {
    return s.repo.GetByID(id)
}

func (s *UserService) CreateUser(input CreateUserInput) model.User {
    user := model.User{
        Username: input.Username,
        Email:    input.Email,
        Age:      input.Age,
        Status:   "active",
    }
    return s.repo.Create(user)
}
```

service 放业务规则。例如新用户默认状态为 `active`，这属于业务逻辑，不应该放在 handler。

## 六、handler 层

`internal/handler/user_handler.go`

```go
package handler

import (
    "errors"
    "net/http"
    "strconv"

    "gin-learning/internal/repository"
    "gin-learning/internal/response"
    "gin-learning/internal/service"

    "github.com/gin-gonic/gin"
)

type UserHandler struct {
    service *service.UserService
}

func NewUserHandler(service *service.UserService) *UserHandler {
    return &UserHandler{service: service}
}

type CreateUserRequest struct {
    Username string `json:"username" binding:"required,min=2,max=32"`
    Email    string `json:"email" binding:"required,email"`
    Age      int    `json:"age" binding:"gte=1,lte=120"`
}

func (h *UserHandler) List(c *gin.Context) {
    users := h.service.ListUsers()
    response.OK(c, users)
}

func (h *UserHandler) Get(c *gin.Context) {
    id, err := strconv.Atoi(c.Param("id"))
    if err != nil {
        response.Fail(c, http.StatusBadRequest, 40001, "invalid user id")
        return
    }

    user, err := h.service.GetUser(id)
    if errors.Is(err, repository.ErrUserNotFound) {
        response.Fail(c, http.StatusNotFound, 40401, "user not found")
        return
    }
    if err != nil {
        response.Fail(c, http.StatusInternalServerError, 50001, "internal server error")
        return
    }

    response.OK(c, user)
}

func (h *UserHandler) Create(c *gin.Context) {
    var req CreateUserRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.Fail(c, http.StatusBadRequest, 40001, "invalid request")
        return
    }

    user := h.service.CreateUser(service.CreateUserInput{
        Username: req.Username,
        Email:    req.Email,
        Age:      req.Age,
    })
    response.Created(c, user)
}
```

handler 只做 HTTP 相关工作：

- 从路径、查询参数、请求体中取数据。
- 调用 service。
- 把结果转换为 HTTP 响应。

## 七、router 层

`internal/router/router.go`

```go
package router

import (
    "gin-learning/internal/handler"

    "github.com/gin-gonic/gin"
)

func New(userHandler *handler.UserHandler) *gin.Engine {
    r := gin.Default()

    api := r.Group("/api/v1")
    {
        api.GET("/users", userHandler.List)
        api.GET("/users/:id", userHandler.Get)
        api.POST("/users", userHandler.Create)
    }

    return r
}
```

## 八、main 入口

`cmd/server/main.go`

```go
package main

import (
    "log"

    "gin-learning/internal/handler"
    "gin-learning/internal/repository"
    "gin-learning/internal/router"
    "gin-learning/internal/service"
)

func main() {
    userRepo := repository.NewUserRepository()
    userService := service.NewUserService(userRepo)
    userHandler := handler.NewUserHandler(userService)

    r := router.New(userHandler)

    log.Println("server started at http://localhost:8080")
    if err := r.Run(":8080"); err != nil {
        log.Fatal(err)
    }
}
```

运行：

```bash
go run ./cmd/server
```

## 九、依赖方向

推荐依赖方向：

```text
main -> router -> handler -> service -> repository -> model
```

不要反过来。例如 repository 不应该导入 handler，model 不应该导入 service。

如果出现循环依赖，通常说明职责划分出了问题。

## 十、阶段练习

继续补全：

- `PUT /api/v1/users/:id`
- `DELETE /api/v1/users/:id`
- `PATCH /api/v1/users/:id/status`

要求：

- handler 不直接操作 map。
- service 负责业务判断。
- repository 负责数据读写。
- 所有接口使用统一响应。
