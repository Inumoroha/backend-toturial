# Gin 系统学习路线图

目标：从 Go 基础能力出发，系统掌握 Gin 框架，并逐步具备独立开发、测试、部署和维护 Go 后端服务的能力。

适合对象：已经开始学习 Go，想成为 Go 后端工程师，并希望围绕 Gin 构建完整 Web 后端能力的人。

建议周期：8 到 12 周。每天 1 到 2 小时可以按 12 周推进；每天 3 小时以上可以压缩到 8 周。

---

## 第一阶段：Go 后端基础补齐

建议时间：1 到 2 周

这一阶段不要急着写 Gin 项目，先把 Go 后端开发会频繁用到的基础打牢。

### 必学内容

- Go 项目结构：`go.mod`、包管理、模块拆分
- 基础语法：变量、函数、结构体、接口、方法
- 错误处理：`error`、自定义错误、错误包装
- 指针和值传递
- 切片、map、结构体组合
- JSON 序列化与反序列化
- 文件读取、环境变量读取
- 时间处理：`time.Time`、时间格式化、时区
- 并发基础：goroutine、channel、context
- HTTP 基础：请求、响应、Header、状态码、RESTful API

### 练习任务

- 写一个命令行程序，读取 JSON 文件并解析成结构体
- 使用 Go 标准库 `net/http` 写一个简单 HTTP 服务
- 实现 3 个接口：
  - `GET /ping`
  - `GET /users`
  - `POST /users`

### 阶段目标

完成后你应该能够理解：

- Go 项目如何组织
- HTTP 服务的基本工作方式
- 请求数据如何进入后端，后端如何返回 JSON
- 为什么 Gin 是对标准 HTTP 开发体验的增强

---

## 第二阶段：Gin 入门与核心用法

建议时间：1 周

这一阶段重点是掌握 Gin 的基本开发模式，不追求架构复杂度。

### 必学内容

- 安装 Gin
- 创建 Gin 服务
- 路由注册
- 路由分组
- 路径参数：`/users/:id`
- 查询参数：`?page=1&pageSize=10`
- 表单参数
- JSON 请求体绑定
- 返回 JSON 响应
- HTTP 状态码设计
- Gin 上下文：`*gin.Context`
- Gin 的启动方式：`router.Run()`

### 示例项目

做一个简单的用户管理 API。

接口建议：

- `GET /api/v1/users`
- `GET /api/v1/users/:id`
- `POST /api/v1/users`
- `PUT /api/v1/users/:id`
- `DELETE /api/v1/users/:id`

暂时可以使用内存 map 保存数据，不急着连接数据库。

### 阶段目标

完成后你应该能够：

- 使用 Gin 写出完整 CRUD 接口
- 区分路径参数、查询参数和请求体参数
- 正确返回 JSON 和状态码
- 理解 `gin.Context` 在一次请求中的作用

## 第三阶段：请求校验、响应规范与错误处理

建议时间：1 周

真实后端项目不能只把接口跑通，还要让接口稳定、清晰、容易维护。

### 必学内容

- `ShouldBindJSON`
- `ShouldBindQuery`
- `binding` 标签
- 参数校验规则：
  - `required`
  - `min`
  - `max`
  - `email`
  - `oneof`
- 统一响应结构
- 统一错误结构
- 业务错误与系统错误区分
- 错误码设计
- 参数校验失败的响应处理

### 推荐响应结构

```go
type Response struct {
    Code    int         `json:"code"`
    Message string      `json:"message"`
    Data    interface{} `json:"data,omitempty"`
}
```

### 练习任务

给第二阶段的用户管理 API 增加：

- 创建用户时校验用户名不能为空
- 邮箱必须符合邮箱格式
- 年龄必须在合理范围内
- 所有接口返回统一格式
- 所有错误返回统一格式

### 阶段目标

完成后你应该能够：

- 写出可维护的 API 响应格式
- 对请求参数进行基础校验
- 避免在每个 handler 里重复写大量错误处理逻辑

## 第四阶段：项目结构与分层设计

建议时间：1 周

这一阶段开始从“能写接口”过渡到“能写项目”。

### 推荐项目结构

```text
gin-demo/
  cmd/
    server/
      main.go
  internal/
    config/
    handler/
    service/
    repository/
    middleware/
    model/
    response/
    router/
  pkg/
  go.mod
```

### 分层职责

- `handler`：处理 HTTP 请求和响应
- `service`：处理业务逻辑
- `repository`：处理数据访问
- `model`：定义数据模型
- `middleware`：中间件
- `router`：路由注册
- `config`：配置加载
- `response`：统一响应封装

### 必学内容

- 为什么不要把所有代码写进 `main.go`
- handler、service、repository 如何协作
- 依赖注入的基本思想
- 如何避免循环依赖
- 配置文件与环境变量管理

### 练习任务

重构用户管理 API：

- 把路由注册移动到 `router`
- 把 HTTP 逻辑移动到 `handler`
- 把业务逻辑移动到 `service`
- 把数据访问移动到 `repository`

### 阶段目标

完成后你应该能够：

- 搭建一个清晰的 Gin 项目骨架
- 理解后端常见分层方式
- 写出更容易扩展和测试的代码

## 第五阶段：数据库与 ORM

建议时间：1 到 2 周

Gin 本身不负责数据库操作。Go 后端工程师必须掌握数据库集成能力。

### 推荐技术

- MySQL 或 PostgreSQL
- GORM
- 数据库迁移工具可后续再学，初期可以先手写建表 SQL

### 必学内容

- 数据库连接配置
- GORM 模型定义
- 创建记录
- 查询记录
- 更新记录
- 删除记录
- 分页查询
- 条件查询
- 事务
- 数据库连接池配置
- 软删除
- 唯一索引

### 练习任务

把用户管理 API 从内存存储改为数据库存储。

用户表字段建议：

```text
id
username
email
password_hash
status
created_at
updated_at
deleted_at
```

接口建议：

- 创建用户
- 查询用户列表，支持分页
- 根据 ID 查询用户
- 更新用户状态
- 删除用户

### 阶段目标

完成后你应该能够：

- 在 Gin 项目中集成数据库
- 使用 GORM 完成常见 CRUD
- 理解 repository 层存在的意义
- 写出基础分页和条件查询

## 第六阶段：中间件与认证授权

建议时间：1 到 2 周

中间件是 Gin 非常重要的能力，也是后端工程师必须熟练掌握的部分。

### 必学内容

- Gin 中间件的执行流程
- 全局中间件
- 路由组中间件
- 自定义中间件
- 日志中间件
- 请求耗时统计
- 跨域 CORS
- JWT 认证
- 登录接口
- 密码加密：bcrypt
- 用户身份注入到上下文
- 需要登录的接口保护
- 简单角色权限控制

### 练习任务

给项目增加认证模块：

- `POST /api/v1/auth/register`
- `POST /api/v1/auth/login`
- `GET /api/v1/profile`
- `PUT /api/v1/profile`

要求：

- 密码不能明文存储
- 登录成功返回 JWT
- 访问个人信息必须携带 token
- token 无效或过期时返回统一错误

### 阶段目标

完成后你应该能够：

- 编写和使用 Gin 中间件
- 实现常见登录认证流程
- 理解无状态认证的基本原理
- 给接口增加基础权限保护

## 学习时的代码规范建议

- 每个 handler 尽量只做请求解析和响应返回
- 业务逻辑放在 service
- 数据库操作放在 repository
- 不要在 handler 里直接写 SQL 或复杂业务逻辑
- 错误处理要统一
- 响应格式要统一
- 密码必须加密存储
- 配置不要硬编码
- 日志不要只用 `fmt.Println`
- 每完成一个功能就补充测试
- 每个项目都写 README
