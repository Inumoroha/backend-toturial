# 10-01 项目实战：任务管理系统需求与接口设计

本阶段目标：用前面学到的能力完成一个真实程度较高的任务管理系统 API。项目不追求功能花哨，而是训练完整后端开发流程。

## 一、项目目标

实现一个多人使用的任务管理系统，用户可以注册、登录、管理自己的任务。

核心能力：

- 用户注册和登录。
- JWT 认证。
- 创建任务。
- 查询任务列表。
- 查询任务详情。
- 更新任务。
- 删除任务。
- 按状态筛选任务。
- 分页查询。
- Redis 缓存用户信息。
- Docker Compose 部署。

## 二、业务规则

### 用户规则

- 邮箱唯一。
- 密码长度至少 8 位。
- 密码必须加密保存。
- 登录成功返回 JWT。
- 未登录不能访问任务接口。

### 任务规则

- 任务只能属于一个用户。
- 用户只能查看和修改自己的任务。
- 任务标题必填。
- 任务状态只能是：
  - `todo`
  - `doing`
  - `done`
- 删除任务使用软删除。
- 列表按创建时间倒序。

## 三、接口清单

### 认证接口

```text
POST /api/v1/auth/register
POST /api/v1/auth/login
```

### 用户接口

```text
GET /api/v1/profile
PUT /api/v1/profile
```

### 任务接口

```text
POST   /api/v1/tasks
GET    /api/v1/tasks
GET    /api/v1/tasks/:id
PUT    /api/v1/tasks/:id
PATCH  /api/v1/tasks/:id/status
DELETE /api/v1/tasks/:id
```

## 四、请求响应设计

### 注册

请求：

```json
{
  "username": "alice",
  "email": "alice@example.com",
  "password": "12345678"
}
```

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": 1,
    "username": "alice",
    "email": "alice@example.com"
  }
}
```

### 登录

请求：

```json
{
  "email": "alice@example.com",
  "password": "12345678"
}
```

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "token": "jwt-token"
  }
}
```

### 创建任务

请求头：

```text
Authorization: Bearer jwt-token
```

请求体：

```json
{
  "title": "学习 Gin 中间件",
  "description": "完成认证中间件和角色权限控制",
  "status": "todo"
}
```

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "id": 1,
    "title": "学习 Gin 中间件",
    "description": "完成认证中间件和角色权限控制",
    "status": "todo"
  }
}
```

### 查询任务列表

请求：

```text
GET /api/v1/tasks?page=1&page_size=10&status=todo
```

响应：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "items": [],
    "total": 0,
    "page": 1,
    "page_size": 10
  }
}
```

## 五、错误码设计

```text
0      成功
40001  参数错误
40101  未登录或 token 无效
40301  无权限访问资源
40401  资源不存在
40901  资源冲突，例如邮箱已存在
42901  请求过于频繁
50001  系统内部错误
```

## 七、实现顺序

不要一口气写完整项目。建议按顺序推进：

1. 创建项目和目录。
2. 接入配置。
3. 连接数据库。
4. 定义 User 和 Task 模型。
5. 完成统一响应。
6. 完成注册。
7. 完成登录和 JWT。
8. 完成认证中间件。
9. 完成任务创建。
10. 完成任务列表和详情。
11. 完成任务更新和删除。
12. 增加 Redis 缓存。
13. 增加测试。
14. 增加 Docker 部署。

每完成一步就运行一次，不要攒到最后一起调。
