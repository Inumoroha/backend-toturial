# 10-04 项目实战：从零实现步骤清单

本节目标：给你一份可以真正照着做的项目实施清单。前面几节讲了设计和核心代码，这一节按开发当天的顺序安排任务。

## 第 1 步：创建项目

```bash
mkdir task-api
cd task-api
go mod init task-api
```

创建目录：

```text
cmd/server
configs
internal/config
internal/database
internal/handler
internal/middleware
internal/model
internal/repository
internal/response
internal/router
internal/service
```

验收：

- `go test ./...` 能运行。
- `go run ./cmd/server` 能启动一个 `/healthz`。

## 第 2 步：统一响应

实现：

```text
internal/response/response.go
```

至少包含：

- `OK`
- `Created`
- `Fail`
- `FailWithData`
- `PageResult`

验收：

- `/healthz` 返回统一格式。
- 404 返回统一格式。

## 第 3 步：配置加载

实现：

```text
configs/config.yaml
internal/config/config.go
```

配置项：

- server port
- database dsn
- redis addr
- jwt secret
- jwt expire hours

验收：

- 端口从配置读取。
- 改配置后服务端口变化。

## 第 4 步：数据库连接和模型

安装：

```bash
go get gorm.io/gorm
go get gorm.io/driver/mysql
```

实现：

- `internal/database/database.go`
- `internal/model/user.go`
- `internal/model/task.go`

验收：

- 服务启动后能连接数据库。
- `AutoMigrate` 能创建 users 和 tasks 表。

## 第 5 步：注册功能

实现：

- `UserRepository.Create`
- `UserRepository.GetByEmail`
- `AuthService.Register`
- `AuthHandler.Register`
- 路由 `POST /api/v1/auth/register`

验收：

- 正常注册成功。
- 邮箱重复返回 409。
- 密码太短返回 400。
- 响应里没有 `password_hash`。

## 第 6 步：登录和 JWT

实现：

- bcrypt 加密校验。
- `GenerateToken`
- `ParseToken`
- `AuthService.Login`
- `AuthHandler.Login`

验收：

- 正确邮箱密码返回 token。
- 错误密码返回 401。
- token 有过期时间。

## 第 7 步：认证中间件

实现：

- 读取 Authorization。
- 校验 Bearer 格式。
- 解析 JWT。
- 写入 `user_id`、`email`、`role`。

验收：

- 不带 token 访问 `/profile` 返回 401。
- 带正确 token 返回用户信息。
- 修改 token 一个字符后返回 401。

## 第 8 步：任务创建和查询

实现：

- `TaskRepository.Create`
- `TaskRepository.GetByIDAndUserID`
- `TaskRepository.ListByUserID`
- `TaskService.Create`
- `TaskService.Get`
- `TaskService.List`
- `TaskHandler.Create`
- `TaskHandler.Get`
- `TaskHandler.List`

验收：

- 登录用户能创建任务。
- 标题为空返回 400。
- 用户只能查询自己的任务。
- 列表支持分页和状态筛选。

## 第 9 步：任务更新和删除

实现：

- `PUT /api/v1/tasks/:id`
- `PATCH /api/v1/tasks/:id/status`
- `DELETE /api/v1/tasks/:id`

验收：

- 修改标题和描述成功。
- 非法状态返回 400。
- 删除别人的任务返回 404 或 403。
- 删除后列表不再出现。

## 第 10 步：Redis 缓存和限流

安装：

```bash
go get github.com/redis/go-redis/v9
```

实现：

- Redis 初始化。
- 缓存用户 profile。
- 更新 profile 后删除缓存。
- 登录失败次数限制。

验收：

- Redis 不启动时，核心数据库查询仍可用。
- 连续登录失败超过限制返回 429。

## 第 11 步：日志和测试

实现：

- 请求日志中间件。
- service 单元测试。
- handler 测试。
- middleware 测试。

最低测试清单：

- 注册成功。
- 邮箱重复。
- 登录失败。
- 未登录访问任务失败。
- 创建任务成功。
- 查询别人任务失败。

验收：

```bash
go test ./...
```

全部通过。

## 第 12 步：Docker 部署

实现：

- Dockerfile。
- docker-compose.yml。
- app、mysql、redis 三个服务。
- 上传目录 volume。

验收：

```bash
docker compose up -d
docker compose ps
```

接口验证：

```text
GET /healthz
POST /api/v1/auth/register
POST /api/v1/auth/login
POST /api/v1/tasks
GET /api/v1/tasks
```

## 最终交付清单

项目完成时应该有：

- 完整源码。
- README。
- 配置示例。
- Dockerfile。
- docker-compose.yml。
- Makefile。
- requests.http 或接口测试集合。
- 至少 6 个关键测试。
- Swagger 文档或接口说明。
