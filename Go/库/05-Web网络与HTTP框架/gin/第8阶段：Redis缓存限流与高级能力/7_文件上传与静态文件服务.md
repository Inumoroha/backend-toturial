# 7. 文件上传与静态文件服务

本节目标：实现头像上传接口，并通过 Gin 提供静态文件访问。

文件上传看起来简单，但它属于安全边界功能。你必须限制大小、类型、保存路径和文件名，不能完全相信客户端传来的内容。

---

## 一、接口设计

头像上传接口：

```text
POST /api/v1/profile/avatar
```

要求：

- 必须登录。
- 表单字段名为 `file`。
- 最大 2MB。
- 只允许 jpg、jpeg、png、webp。
- 保存到 `uploads/avatars`。
- 返回可访问 URL。

响应示例：

```json
{
  "code": "OK",
  "message": "success",
  "data": {
    "avatar_url": "/static/avatars/1_1720180000000000000.png"
  }
}
```

---

## 二、配置上传参数

配置文件：

```yaml
upload:
  avatar_dir: "uploads/avatars"
  max_avatar_size_mb: 2
```

结构体：

```go
type UploadConfig struct {
    AvatarDir       string `mapstructure:"avatar_dir"`
    MaxAvatarSizeMB int64  `mapstructure:"max_avatar_size_mb"`
}
```

总配置增加：

```go
Upload UploadConfig `mapstructure:"upload"`
```

---

## 三、注册静态文件路由

在路由初始化中：

```go
r.Static("/static/avatars", cfg.Upload.AvatarDir)
```

含义：

```text
浏览器访问 /static/avatars/a.png
实际读取 uploads/avatars/a.png
```

注意：静态文件路由只负责访问，不负责上传安全。上传安全必须在 handler 中处理。

---

## 四、确保目录存在

启动时确保目录存在：

```go
if err := os.MkdirAll(cfg.Upload.AvatarDir, 0755); err != nil {
    logger.Fatal("create avatar dir failed", zap.Error(err))
}
```

不要等用户第一次上传时才发现目录不存在。

## 五、handler 实现

```go
func (h *UserHandler) UploadAvatar(c *gin.Context) {
    userID := middleware.GetCurrentUserID(c)

    file, err := c.FormFile("file")
    if err != nil {
        response.Fail(c, http.StatusBadRequest, "FILE_REQUIRED", "请选择上传文件")
        return
    }

    maxSize := h.uploadCfg.MaxAvatarSizeMB * 1024 * 1024
    if file.Size > maxSize {
        response.Fail(c, http.StatusBadRequest, "FILE_TOO_LARGE", "文件过大")
        return
    }

    ext := strings.ToLower(filepath.Ext(file.Filename))
    if !isAllowedAvatarExt(ext) {
        response.Fail(c, http.StatusBadRequest, "INVALID_FILE_TYPE", "不支持的文件类型")
        return
    }

    newName := fmt.Sprintf("%d_%d%s", userID, time.Now().UnixNano(), ext)
    savePath := filepath.Join(h.uploadCfg.AvatarDir, newName)

    if err := c.SaveUploadedFile(file, savePath); err != nil {
        response.Fail(c, http.StatusInternalServerError, "SAVE_FILE_FAILED", "保存文件失败")
        return
    }

    avatarURL := "/static/avatars/" + newName
    if err := h.userService.UpdateAvatar(c.Request.Context(), userID, avatarURL); err != nil {
        response.Fail(c, http.StatusInternalServerError, "UPDATE_AVATAR_FAILED", "更新头像失败")
        return
    }

    response.OK(c, gin.H{"avatar_url": avatarURL})
}
```

辅助函数：

```go
func isAllowedAvatarExt(ext string) bool {
    switch ext {
    case ".jpg", ".jpeg", ".png", ".webp":
        return true
    default:
        return false
    }
}
```

## 六、为什么要生成新文件名

不要直接使用：

```go
file.Filename
```

原因：

- 可能重复覆盖。
- 可能包含奇怪字符。
- 可能暴露用户本地文件名。
- 可能带来路径安全问题。

生成文件名时至少包含：

- 用户 ID。
- 时间戳或 UUID。
- 经过校验的扩展名。

## 七、service 更新头像

service：

```go
func (s *UserService) UpdateAvatar(ctx context.Context, userID uint, avatarURL string) error {
    if avatarURL == "" {
        return ErrInvalidAvatarURL
    }

    if err := s.repo.UpdateAvatar(ctx, userID, avatarURL); err != nil {
        return err
    }

    if err := s.cache.DeleteUserDetail(ctx, userID); err != nil {
        s.log.Warn("delete user cache failed",
            zap.Uint("user_id", userID),
            zap.Error(err),
        )
    }

    return nil
}
```

更新头像后删除用户详情缓存，避免接口继续返回旧头像。

## 八、使用 curl 测试上传

```bash
curl -X POST http://localhost:8080/api/v1/profile/avatar \
  -H "Authorization: Bearer your-token" \
  -F "file=@avatar.png"
```

成功后访问返回的 URL：

```bash
curl -I http://localhost:8080/static/avatars/your-file-name.png
```

应该返回：

```text
HTTP/1.1 200 OK
```

## 九、常见问题

### 1. 上传时报 file required

检查表单字段名是不是 `file`：

```bash
-F "file=@avatar.png"
```

### 2. 上传成功但访问 404

检查：

- 静态路由是否注册。
- 保存目录是否和静态路由目录一致。
- 返回 URL 是否拼写正确。

### 3. 大文件没有被拒绝

检查 handler 是否判断了 `file.Size`。只设置 `MaxMultipartMemory` 不等于完成所有限制。

## 十一、验收标准

完成后确认：

- 未登录上传返回 `401`。
- 未选择文件返回 `400`。
- 超过大小限制返回 `400`。
- 非图片扩展名返回 `400`。
- 上传成功后能通过静态 URL 访问。
- 用户详情接口返回新的头像地址。
