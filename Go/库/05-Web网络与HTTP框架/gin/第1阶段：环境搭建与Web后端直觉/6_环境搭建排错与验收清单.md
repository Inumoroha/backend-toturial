# 6. 环境搭建排错与验收清单

本节目标：把第1阶段所有环境问题、命令检查、HTTP 直觉和每日工作流整理成一份可以逐项核对的清单。

环境搭建阶段看起来简单，但它决定后面所有学习是否顺畅。很多初学者不是卡在 Gin 语法，而是卡在这些地方：

- `go version` 正常，但 VS Code 找不到 Go。
- `go run` 能跑，但浏览器访问不到。
- 端口被占用，却以为代码写错。
- 不知道从终端输出判断服务是否启动。
- 不会用 curl 或 REST Client 验证接口。
- 每天写完代码没有提交，后面改坏了回不去。

本节就是为了把这些基础问题一次性排干净。

---

## 一、Go 环境检查

执行：

```bash
go version
go env GOPATH
go env GOPROXY
```

你应该能看到 Go 版本，例如：

```text
go version go1.22.x windows/amd64
```

如果 `go version` 找不到命令，优先检查：

- Go 是否安装。
- Go 安装目录是否加入 PATH。
- 终端是否重启。
- VS Code 是否重启。

Windows PowerShell 检查 PATH：

```powershell
$env:Path -split ';' | Select-String -Pattern 'Go'
```

验收：

- [ ] `go version` 能输出版本。
- [ ] `go env GOPATH` 能输出路径。
- [ ] 新开一个终端后 Go 命令仍然可用。

---

## 二、项目目录检查

推荐每个练习项目都单独建目录：

```text
gin-learning/
├── go.mod
└── main.go
```

初始化：

```bash
mkdir gin-learning
cd gin-learning
go mod init gin-learning
```

检查：

```bash
ls
go list -m
```

不要把很多不相关的练习都塞进一个 `main.go`，否则后面会越来越乱。

验收：

- [ ] 项目根目录有 `go.mod`。
- [ ] `go list -m` 输出当前模块名。
- [ ] `main.go` 位于项目根目录或清晰的子目录。

## 三、标准库 HTTP 服务检查

最小服务：

```go
package main

import (
    "fmt"
    "net/http"
)

func main() {
    http.HandleFunc("/ping", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintln(w, "pong")
    })

    fmt.Println("server listening on :8080")
    if err := http.ListenAndServe(":8080", nil); err != nil {
        panic(err)
    }
}
```

启动：

```bash
go run main.go
```

访问：

```bash
curl http://localhost:8080/ping
```

期望：

```text
pong
```

如果访问不到，先确认终端是否还在运行，不要关闭服务终端后再访问。

## 四、端口占用排错

常见错误：

```text
bind: address already in use
```

说明 `8080` 已经被占用。

Windows 查看：

```powershell
netstat -ano | findstr :8080
```

根据 PID 查进程：

```powershell
Get-Process -Id 进程ID
```

也可以临时换端口：

```go
http.ListenAndServe(":8081", nil)
```

验收：

- [ ] 你知道端口被占用不是 Go 语法错误。
- [ ] 你能查出哪个进程占用了端口。
- [ ] 你能把服务切换到另一个端口验证。

## 五、HTTP 请求直觉检查

你需要能区分：

```text
URL：访问地址
Method：GET、POST、PUT、DELETE
Header：请求元信息
Body：请求体
Status Code：响应状态码
Response Body：响应体
```

用 curl 查看响应头：

```bash
curl -i http://localhost:8080/ping
```

你应该能看到：

```text
HTTP/1.1 200 OK
Content-Type: text/plain; charset=utf-8
```

这一步很重要。后面学习 Gin 时，所有参数绑定、统一响应、JWT、CORS，本质上都围绕 HTTP 请求和响应展开。

## 六、接口测试工具检查

推荐至少掌握一种：

- `curl`
- VS Code REST Client
- Apifox
- Postman

REST Client 示例 `requests.http`：

```http
### ping
GET http://localhost:8080/ping
```

验收：

- [ ] 能用浏览器访问 GET 接口。
- [ ] 能用 curl 访问接口。
- [ ] 能用 REST Client 或 Apifox 保存请求。
- [ ] 能看懂状态码和响应体。

## 七、Git 工作流检查

初始化：

```bash
git init
git status
```

建议每天至少提交一次可运行状态：

```bash
git add .
git commit -m "complete standard http server practice"
```

`.gitignore` 示例：

```gitignore
bin/
tmp/
*.exe
*.log
.env
```

验收：

- [ ] `git status` 能看懂哪些文件修改了。
- [ ] 每完成一个小节有一次提交。
- [ ] 不把 `.env`、编译产物、日志文件提交进去。

## 八、每日学习流程

建议每天按这个顺序：

```text
1. git status 看当前状态。
2. go test 或 go run 确认昨天代码能跑。
3. 学一个小节。
4. 手写代码，不只复制。
5. 用请求工具验证。
6. 记录错误和解决方法。
7. git commit 保存进度。
```

记录模板：

```md
## 今天学习

主题：

## 完成内容

- 

## 遇到的问题

现象：
原因：
解决：

## 明天继续

- 
```

## 九、常见问题

### 1. 浏览器访问 localhost 失败

检查服务是否启动、端口是否一致、终端是否报错。

### 2. curl 能访问，浏览器不行

检查浏览器地址是否写错，是否访问了旧端口。

### 3. 修改代码后没有生效

`go run` 启动的服务不会自动热重载。停止后重新运行。

### 4. VS Code 报错但命令行能跑

重启 VS Code，确认打开的是项目根目录，确认 Go 插件正常。
