# 8. 创建任务接口

本节目标：实现 `POST /api/v1/tasks`，让登录用户可以创建属于自己的任务。

---

## 一、创建任务流程

```text
认证中间件解析 token
→ handler 从 gin.Context 读取 user_id
→ 读取 JSON 请求体
→ service 校验任务状态
→ repository 写入 tasks 表
→ 返回创建后的任务
```

关键点：

```text
user_id 来自 token，不来自请求体。
```

如果允许前端传 `user_id`，用户就可能伪造别人的 ID 创建数据。

## 三、TaskService

`internal/service/task_service.go`：

```go
package service

import (
    "errors"

    "task-api/internal/model"
    "task-api/internal/repository"
)

var ErrInvalidTaskStatus = errors.New("invalid task status")

type TaskService struct {
    repo *repository.TaskRepository
}

func NewTaskService(repo *repository.TaskRepository) *TaskService {
    return &TaskService{repo: repo}
}

type CreateTaskInput struct {
    UserID      uint
    Title       string
    Description string
    Status      string
}

func (s *TaskService) Create(input CreateTaskInput) (*model.Task, error) {
    status := input.Status
    if status == "" {
        status = "todo"
    }
    if !isValidTaskStatus(status) {
        return nil, ErrInvalidTaskStatus
    }

    task := &model.Task{
        UserID:      input.UserID,
        Title:       input.Title,
        Description: input.Description,
        Status:      status,
    }

    if err := s.repo.Create(task); err != nil {
        return nil, err
    }

    return task, nil
}

func isValidTaskStatus(status string) bool {
    switch status {
    case "todo", "doing", "done":
        return true
    default:
        return false
    }
}
```

状态默认值和状态合法性属于业务规则，所以放在 service。

## 四、TaskHandler

`internal/handler/task_handler.go`：

```go
package handler

import (
    "errors"
    "net/http"

    "task-api/internal/response"
    "task-api/internal/service"

    "github.com/gin-gonic/gin"
)

type TaskHandler struct {
    service *service.TaskService
}

func NewTaskHandler(service *service.TaskService) *TaskHandler {
    return &TaskHandler{service: service}
}

type CreateTaskRequest struct {
    Title       string `json:"title" binding:"required,min=1,max=128"`
    Description string `json:"description" binding:"omitempty,max=2000"`
    Status      string `json:"status" binding:"omitempty,oneof=todo doing done"`
}

func (h *TaskHandler) Create(c *gin.Context) {
    userID := c.GetUint("user_id")

    var req CreateTaskRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        response.Fail(c, http.StatusBadRequest, response.CodeInvalidRequest, "invalid request")
        return
    }

    task, err := h.service.Create(service.CreateTaskInput{
        UserID:      userID,
        Title:       req.Title,
        Description: req.Description,
        Status:      req.Status,
    })
    if errors.Is(err, service.ErrInvalidTaskStatus) {
        response.Fail(c, http.StatusBadRequest, response.CodeInvalidRequest, "invalid task status")
        return
    }
    if err != nil {
        response.Fail(c, http.StatusInternalServerError, response.CodeInternalError, "internal server error")
        return
    }

    response.Created(c, task)
}
```

## 五、路由注册

把任务接口放进私有路由组：

```go
private.POST("/tasks", taskHandler.Create)
```

完整结构类似：

```go
private := api.Group("")
private.Use(middleware.Auth(cfg.JWT.Secret))

    private.GET("/profile", userHandler.Profile)
```

## 七、常见问题

### 1. 创建任务 user_id 是 0

检查：

- 认证中间件是否执行。
- token 是否正确。
- 是否使用 `c.GetUint("user_id")`。

### 2. 非法状态没有拦住

检查：

- handler binding tag 是否写了 `oneof=todo doing done`。
- service 是否也做了 `isValidTaskStatus`。

handler 和 service 都校验不是重复浪费。handler 保护 HTTP 入口，service 保护业务规则。
