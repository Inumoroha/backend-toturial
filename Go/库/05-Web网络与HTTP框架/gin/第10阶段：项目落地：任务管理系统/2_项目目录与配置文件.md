# 2. 项目目录与配置文件

本节目标：创建任务管理系统的项目目录，并加入配置文件加载能力。

---

## 一、创建项目

```bash
mkdir task-api
cd task-api
go mod init task-api
```

创建目录：

```text
cmd/server
configs
internal/config
internal/database
internal/handler
internal/middleware
internal/model
internal/repository
internal/response
internal/router
internal/service
```

PowerShell 可以逐个创建，也可以使用：

```powershell
$dirs = @(
  "cmd/server",
  "configs",
  "internal/config",
  "internal/database",
  "internal/handler",
  "internal/middleware",
  "internal/model",
  "internal/repository",
  "internal/response",
  "internal/router",
  "internal/service"
)
foreach ($d in $dirs) { New-Item -ItemType Directory -Force -Path $d }
```

---

## 二、目录职责

```text
cmd/server       程序入口。
configs          配置文件。
internal/config  配置加载。
internal/database 数据库和 Redis 初始化。
internal/handler HTTP 请求处理。
internal/middleware Gin 中间件。
internal/model   数据模型。
internal/repository 数据访问。
internal/response 统一响应。
internal/router  路由注册。
internal/service 业务逻辑。
```

`internal` 目录表示这些包只给当前项目内部使用，不希望被外部项目导入。

## 三、创建配置文件

`configs/config.yaml`：

```yaml
server:
  port: 8080

database:
  dsn: "root:password@tcp(127.0.0.1:3306)/task_api?charset=utf8mb4&parseTime=True&loc=Local"

redis:
  addr: "127.0.0.1:6379"
  password: ""
  db: 0

jwt:
  secret: "change-me-in-production"
  expire_hours: 24
```

注意：`secret` 这里只适合本地学习。真实项目不能把生产密钥提交到 Git。

## 四、安装 Viper

```bash
go get github.com/spf13/viper
```

创建 `internal/config/config.go`：

```go
package config

import "github.com/spf13/viper"

type Config struct {
    Server struct {
        Port int `mapstructure:"port"`
    } `mapstructure:"server"`

    Database struct {
        DSN string `mapstructure:"dsn"`
    } `mapstructure:"database"`

    Redis struct {
        Addr     string `mapstructure:"addr"`
        Password string `mapstructure:"password"`
        DB       int    `mapstructure:"db"`
    } `mapstructure:"redis"`

    JWT struct {
        Secret      string `mapstructure:"secret"`
        ExpireHours int    `mapstructure:"expire_hours"`
    } `mapstructure:"jwt"`
}

func Load(path string) (*Config, error) {
    viper.SetConfigFile(path)

    if err := viper.ReadInConfig(); err != nil {
        return nil, err
    }

    var cfg Config
    if err := viper.Unmarshal(&cfg); err != nil {
        return nil, err
    }

    return &cfg, nil
}
```

## 五、创建 main.go 验证配置

`cmd/server/main.go`：

```go
package main

import (
    "fmt"
    "log"

    "task-api/internal/config"
)

func main() {
    cfg, err := config.Load("configs/config.yaml")
    if err != nil {
        log.Fatal(err)
    }

    fmt.Printf("server port: %d\n", cfg.Server.Port)
}
```

运行：

```bash
go run ./cmd/server
```

应该输出：

```text
server port: 8080
```

## 六、常见问题

### 1. 找不到配置文件

如果报错找不到 `configs/config.yaml`，先确认你是在项目根目录执行：

```bash
go run ./cmd/server
```

不要进入 `cmd/server` 目录后再运行。

### 2. 配置字段读不到

检查：

- YAML 字段名是否和 `mapstructure` 对应。
- 结构体标签是否写错。
- YAML 缩进是否正确。
