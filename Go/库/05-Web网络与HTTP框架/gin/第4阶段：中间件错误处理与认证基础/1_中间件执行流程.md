# 1. 中间件执行流程

本节目标：理解 Gin 中间件的基本形态、注册方式、执行顺序，以及 `c.Next()` 和 `c.Abort()` 的作用。中间件学不清楚，后面的认证、日志、限流都会写得很别扭。

---

## 一、中间件是什么

Gin 中间件本质上是一个函数：

```go
func(c *gin.Context)
```

通常写成：

```go
func SomeMiddleware() gin.HandlerFunc {
    return func(c *gin.Context) {
        // 中间件逻辑
    }
}
```

它和普通 handler 很像，也能拿到 `*gin.Context`。

区别在于：

```text
handler 负责具体业务接口。
middleware 负责请求进入 handler 前后都可能需要执行的公共逻辑。
```

---

## 二、最小中间件

先写一个最小中间件：

```go
func Trace() gin.HandlerFunc {
    return func(c *gin.Context) {
        fmt.Println("before handler")
        c.Next()
        fmt.Println("after handler")
    }
}
```

注册：

```go
r := gin.Default()
r.Use(Trace())
```

完整示例：

```go
package main

import (
    "fmt"
    "net/http"

    "github.com/gin-gonic/gin"
)

func main() {
    r := gin.Default()
    r.Use(Trace())

    r.GET("/ping", func(c *gin.Context) {
        fmt.Println("handler")
        c.JSON(http.StatusOK, gin.H{"message": "pong"})
    })

    r.Run(":8080")
}

func Trace() gin.HandlerFunc {
    return func(c *gin.Context) {
        fmt.Println("before handler")
        c.Next()
        fmt.Println("after handler")
    }
}
```

访问：

```http
GET http://localhost:8080/ping
```

终端输出：

```text
before handler
handler
after handler
```

## 三、理解 c.Next()

`c.Next()` 的作用是继续执行后续中间件和最终 handler。

可以把请求链路想成这样：

```text
中间件 A before
  中间件 B before
    handler
  中间件 B after
中间件 A after
```

示例：

```go
r.Use(MiddlewareA())
r.Use(MiddlewareB())
```

如果 A 和 B 都写了 `c.Next()`，执行顺序就是：

```text
A before
B before
handler
B after
A after
```

这就是典型的洋葱模型。

---

## 四、多个中间件执行顺序

代码：

```go
func A() gin.HandlerFunc {
    return func(c *gin.Context) {
        fmt.Println("A before")
        c.Next()
        fmt.Println("A after")
    }
}

func B() gin.HandlerFunc {
    return func(c *gin.Context) {
        fmt.Println("B before")
        c.Next()
        fmt.Println("B after")
    }
}
```

注册：

```go
r.Use(A(), B())
```

handler：

```go
r.GET("/ping", func(c *gin.Context) {
    fmt.Println("handler")
    c.JSON(200, gin.H{"message": "pong"})
})
```

输出：

```text
A before
B before
handler
B after
A after
```

这对日志、耗时统计非常重要。因为你可以在 `c.Next()` 前记录开始时间，在 `c.Next()` 后计算耗时。

---

## 五、不写 c.Next() 会怎样

中间件中如果不写 `c.Next()`，Gin 仍然会继续执行后面的 handler，除非你调用 `c.Abort()`。

例如：

```go
func NoNext() gin.HandlerFunc {
    return func(c *gin.Context) {
        fmt.Println("middleware")
    }
}
```

这种写法不适合做后置逻辑，因为你没有明确分出 handler 前后的代码。

推荐中间件中需要包裹 handler 时写：

```go
start := time.Now()
c.Next()
latency := time.Since(start)
```

---

## 六、c.Abort() 的作用

`c.Abort()` 表示停止执行后续 handler。

认证失败时常见写法：

```go
func DemoAuth() gin.HandlerFunc {
    return func(c *gin.Context) {
        token := c.GetHeader("X-Demo-Token")
        if token == "" {
            c.JSON(http.StatusUnauthorized, gin.H{"message": "missing token"})
            c.Abort()
            return
        }

        c.Next()
    }
}
```

如果请求没有 `X-Demo-Token`，就返回 401，并且不会进入后面的业务 handler。

---

## 七、为什么 Abort 后还要 return

看这段错误示例：

```go
if token == "" {
    c.JSON(http.StatusUnauthorized, gin.H{"message": "missing token"})
    c.Abort()
}

fmt.Println("still running")
```

`c.Abort()` 会阻止后续 handler，但不会自动停止当前函数后面的代码。

所以推荐：

```go
if token == "" {
    c.JSON(http.StatusUnauthorized, gin.H{"message": "missing token"})
    c.Abort()
    return
}
```

这是认证中间件中非常重要的习惯。

## 八、全局中间件

全局中间件作用于所有路由：

```go
r.Use(RequestID())
r.Use(AccessLog())
```

适合：

```text
Request ID
请求日志
Recovery
CORS
```

这些能力通常所有接口都需要。

---

## 九、路由组中间件

路由组中间件只作用于某一组路由：

```go
api := r.Group("/api/v1")

private := api.Group("")
private.Use(DemoAuth())
{
    private.GET("/profile", profile)
}
```

适合：

```text
JWT 认证
管理员权限
某一组接口的限流
```

公开接口不要挂认证中间件：

```go
api.POST("/auth/login", login)
```

私有接口才挂认证中间件：

```go
private.GET("/profile", profile)
```

## 十、完整练习：Header Token 中间件

目标：实现一个简单中间件，要求请求携带：

```text
X-Demo-Token: secret
```

代码：

```go
func DemoTokenAuth() gin.HandlerFunc {
    return func(c *gin.Context) {
        token := c.GetHeader("X-Demo-Token")
        if token != "secret" {
            c.JSON(http.StatusUnauthorized, gin.H{
                "message": "invalid demo token",
            })
            c.Abort()
            return
        }

        c.Next()
    }
}
```

路由：

```go
private := r.Group("/private")
private.Use(DemoTokenAuth())
{
    private.GET("/ping", func(c *gin.Context) {
        c.JSON(http.StatusOK, gin.H{"message": "private pong"})
    })
}
```

测试失败：

```http
GET http://localhost:8080/private/ping
```

测试成功：

```http
GET http://localhost:8080/private/ping
X-Demo-Token: secret
```

## 十一、常见问题

### 1. 中间件没有生效

检查：

- 是否调用了 `r.Use(...)`。
- 是否挂在正确的路由组上。
- 请求路径是否属于这个路由组。

### 2. 认证失败后 handler 还执行

检查是否写了：

```go
c.Abort()
return
```

### 3. 日志顺序看起来反了

记住洋葱模型：

```text
先进入的中间件，最后退出。
```
