# 6. 源码阅读笔记模板

本节目标：给你一个固定模板，用来记录 `net/http` 源码阅读结果。源码阅读最忌讳“看了很多，但没有沉淀”。

---

## 一、类型阅读模板

每读一个核心类型，按下面格式记录：

```markdown
## 类型名

位置：

职责：

被谁创建：

被谁调用：

它调用谁：

我对它的理解：

相关问题：
```

例如：

```markdown
## http.Handler

职责：统一表示一个可以处理 HTTP 请求的对象。

被谁调用：Server、ServeMux、中间件。

我对它的理解：它是 net/http 的中心抽象，路由器、中间件和业务处理函数都围绕它组合。
```

---

## 二、流程阅读模板

每读一个流程，按下面格式记录：

```markdown
## 流程名

入口函数：

核心步骤：

关键类型：

异常分支：

我画出的调用链：

还没看懂的问题：
```

例如：

```text
ListenAndServe
-> Server.ListenAndServe
-> net.Listen
-> Server.Serve
-> 接收连接
-> 读取请求
-> Handler.ServeHTTP
```

---

## 三、问题清单

源码阅读时要保留问题，例如：

```markdown
- Server 在哪里为连接设置 deadline？
- ServeMux 如何保存路径参数？
- Handler 为 nil 时默认 mux 在哪里被选择？
- Client.Do 遇到 500 为什么不返回 error？
- Transport 的空闲连接池在哪里管理？
```

问题不是失败，它是下一轮阅读的入口。

---

## 四、阶段交付物

完成源码阶段后，建议留下三个文件：

```text
server_flow.md
client_flow.md
questions.md
```

这三个文件比“我看过源码”更有价值，因为它们能证明你把源码翻译成了自己的理解。
