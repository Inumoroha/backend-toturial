# 0. 源码阅读与原理总览

本阶段目标：通过阅读 `net/http` 核心源码，理解标准库的设计，而不是只会使用表层 API。

源码阅读不是为了背每一行实现，而是为了看清几个核心问题：

```text
为什么 Handler 是中心？
ServeMux 怎样把请求转给 Handler？
Server 如何从连接走到业务代码？
Client 和 Transport 如何分工？
```

读源码时要克制。先抓主流程，再看边界细节。初学者最容易一打开源码就陷入大量内部类型，最后忘了自己要找什么。

---

## 一、建议阅读顺序

1. `http.Handler`。
2. `http.HandlerFunc`。
3. `http.ServeMux`。
4. `http.Server`。
5. 请求连接处理流程。
6. `http.Client`。
7. `http.Transport`。

源码阅读不要求背诵实现，而是要看懂核心流程：

```text
连接进入
-> 解析请求
-> 找到 Handler
-> 执行 Handler
-> 写出响应
```

---

## 二、本阶段阅读方法

每读一个类型，都回答三个问题：

```text
它负责什么？
它被谁调用？
它又调用谁？
```

每读一个流程，都画一条调用链。不要只摘抄源码，要把源码翻译成自己的语言。

---

## 三、本阶段达标标准

完成本阶段后，你应该能：

- 解释 `HandlerFunc` 如何适配普通函数。
- 解释 `ServeMux` 为什么也是 Handler。
- 说出 `ListenAndServe` 的大致流程。
- 说明 Handler 为 nil 时为什么会使用默认 mux。
- 解释 Client、Transport、RoundTripper 的关系。
- 写一份自己的源码阅读笔记。
