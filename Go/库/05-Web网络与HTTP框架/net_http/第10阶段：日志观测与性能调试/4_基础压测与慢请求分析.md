# 4. 基础压测与慢请求分析

本节目标：学会用简单压测工具观察接口表现，并结合日志判断慢请求。

---

## 一、压测工具

常见工具：

```text
hey
wrk
ab
```

Go 学习阶段推荐 `hey`，安装简单，输出直观。

示例：

```bash
hey -n 1000 -c 50 http://localhost:8080/health
```

含义：

- `-n 1000`：总请求数 1000。
- `-c 50`：并发 50。

---

## 二、压测前要确认什么

压测前确认：

- 服务运行在本地或测试环境。
- 不要压生产环境。
- 日志不会刷爆磁盘。
- 接口是幂等或测试数据可清理。
- 如果有数据库，确认数据和连接池配置。

---

## 三、看哪些指标

关注：

- 平均响应时间。
- P95、P99 延迟。
- 每秒请求数。
- 错误率。
- 最大响应时间。

平均值不够。一个接口平均 20ms，但 P99 是 3s，用户仍然会感到卡。

---

## 四、结合日志分析慢请求

请求日志中有：

```text
duration_ms
status
path
request_id
```

可以筛选：

```text
duration_ms > 1000
status >= 500
```

然后用 request_id 查同一请求内的业务日志。

---

## 五、常见慢请求原因

常见原因：

- 数据库查询慢。
- 下游 HTTP 调用慢。
- 锁竞争。
- 请求体太大。
- 响应数据太大。
- 日志同步写太多。
- 单个 Handler 做了 CPU 密集计算。

排查时不要凭感觉，先看日志、pprof、数据库慢查询和压测结果。

---

## 六、本节检查点

请确认你能做到：

- 使用 `hey` 对接口做基础压测。
- 解释 `-n` 和 `-c`。
- 关注 P95、P99 而不只是平均值。
- 通过请求日志找到慢请求。
