# 2. 请求日志字段设计

本节目标：设计一条有用的 HTTP 请求日志。

---

## 一、请求日志应该包含什么

建议字段：

```text
request_id
method
path
query
status
duration
remote_addr
user_agent
```

如果有认证系统，还可以包含：

```text
user_id
tenant_id
```

但不要记录完整 Token、密码、身份证号等敏感信息。

---

## 二、用 slog 写日志中间件

```go
func logging(logger *slog.Logger) Middleware {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			start := time.Now()
			rec := &statusRecorder{ResponseWriter: w, status: http.StatusOK}

			next.ServeHTTP(rec, r)

			logger.Info("http request",
				"request_id", getRequestID(r),
				"method", r.Method,
				"path", r.URL.Path,
				"status", rec.status,
				"duration_ms", time.Since(start).Milliseconds(),
				"remote_addr", r.RemoteAddr,
				"user_agent", r.UserAgent(),
			)
		})
	}
}
```

---

## 三、错误日志和请求日志的关系

请求日志记录“这次请求的结果”。

错误日志记录“某个错误的细节”。

两者用 `request_id` 关联：

```go
logger.Error("create todo failed",
	"request_id", getRequestID(r),
	"error", err,
)
```

这样排查时可以先找到失败请求，再用 request_id 查同一请求下的详细错误。

---

## 四、不要记录过多 Body

请求 Body 可能包含敏感信息，也可能很大。

不建议默认记录完整 Body。

如果确实需要排查，可以：

- 只在开发环境开启。
- 限制最大长度。
- 做敏感字段脱敏。
- 只对特定接口临时开启。
