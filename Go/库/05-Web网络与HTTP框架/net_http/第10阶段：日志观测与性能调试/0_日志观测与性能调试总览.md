# 0. 日志观测与性能调试总览

本阶段目标：让服务不只是“能跑”，而是“出问题时能看见”。生产后端必须能通过日志、指标、pprof 和压测定位问题。

很多初学项目只能在成功时演示，一旦接口变慢、偶发 500、内存升高，就不知道从哪里看。本阶段要训练的是“排查入口”：

```text
先看请求日志
-> 找 request_id
-> 看错误日志
-> 看慢请求耗时
-> 必要时用 pprof
-> 用压测复现
```

---

## 一、本阶段学习顺序

1. 使用 `log/slog` 写结构化日志。
2. 请求日志字段设计。
3. 接入 `net/http/pprof`。
4. 使用 `hey` 做基础压测。
5. 分析慢请求。
6. 常见性能问题和排查路径。
7. 慢接口 pprof 排查练习。

---

## 二、本阶段核心能力

你要学会给服务留下这些线索：

- 每个请求有 Request ID。
- 请求日志里有状态码和耗时。
- 错误日志能和请求日志关联。
- pprof 不暴露公网。
- 压测时看 P95、P99，而不是只看平均值。

---

## 三、本阶段达标标准

完成本阶段后，你应该能：

- 用 `slog` 输出结构化日志。
- 设计一条有排查价值的请求日志。
- 单独启动 pprof 调试端口。
- 用 `hey` 对接口做基础压测。
- 根据日志定位慢接口。
- 描述一个接口变慢时的排查顺序。
