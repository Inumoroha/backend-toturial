# 5. 跳转与访问计数

本节目标：实现 `GET /{code}`，根据短码找到原始链接并返回重定向。

---

## 一、跳转 Handler 流程

```text
读取 code
-> 查询链接
-> 不存在返回 404
-> 增加访问次数
-> 返回 302 Location
```

---

## 二、返回重定向

```go
http.Redirect(w, r, link.OriginalURL, http.StatusFound)
```

`http.StatusFound` 是 302。

---

## 三、访问次数更新

内存版本：

```go
func (s *MemoryStore) IncrementVisit(ctx context.Context, code string) error {
	s.mu.Lock()
	defer s.mu.Unlock()

	link, ok := s.links[code]
	if !ok {
		return ErrNotFound
	}

	link.VisitCount++
	s.links[code] = link
	return nil
}
```

数据库版本中应该使用原子更新：

```sql
UPDATE links
SET visit_count = visit_count + 1
WHERE code = $1;
```

---

## 四、计数失败怎么办

学习阶段可以让计数失败返回 500。

真实项目中要根据业务决定：

- 计数失败是否仍然跳转？
- 是否异步记录访问日志？
- 是否允许计数最终一致？

短链接服务通常更重视跳转成功，统计可以异步化。
