# 12. 测试、观测与项目收尾

本节目标：补齐短链接服务的测试、日志、README 和复盘。

---

## 一、测试清单

至少覆盖：

```text
POST /links
  - 创建成功
  - URL 为空返回 400
  - URL scheme 非 http/https 返回 400

GET /{code}
  - 存在返回 302
  - 不存在返回 404
  - 跳转后访问次数增加

GET /links/{code}
  - 存在返回 200
  - 不存在返回 404

GET /links
  - 返回列表
  - page_size 超限被限制
```

---

## 二、Service 测试清单

```text
Create URL 为空 -> ErrInvalidURL
Create 非 http/https -> ErrInvalidURL
Create 成功 -> 返回 code
短码冲突 -> 重试生成
GetByCode 不存在 -> ErrNotFound
IncrementVisit 后 visit_count 增加
```

Service 测试可以通过假的 Store 控制冲突和错误，比只测 HTTP 更容易覆盖边界。

---

## 三、Handler 测试清单

```text
POST /links 成功 -> 201
POST /links 非法 JSON -> 400
POST /links 非法 URL -> 400
GET /{code} 成功 -> 302 且 Location 正确
GET /{code} 不存在 -> 404
GET /links/{code} 成功 -> 200
```

跳转接口测试重点是状态码和 `Location` Header。

---

## 四、日志字段

请求日志保留：

```text
request_id
method
path
status
duration_ms
```

业务日志可以记录：

```text
code
original_url_host
```

不要在日志中随意记录完整敏感 URL，真实项目里 URL 可能包含 token 或隐私参数。

---

## 五、README 内容

README 至少包含：

- 项目介绍。
- 如何启动。
- 接口列表。
- curl 示例。
