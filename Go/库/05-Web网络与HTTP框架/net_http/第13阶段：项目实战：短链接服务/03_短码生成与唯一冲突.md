# 3. 短码生成与唯一冲突

本节目标：实现短码生成，并理解唯一冲突必须由存储层兜底。

---

## 一、短码要求

短码通常希望：

- 足够短。
- URL 友好。
- 不容易重复。
- 不包含容易混淆字符也可以作为优化。

学习阶段可以使用字符集：

```go
const alphabet = "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789"
```

---

## 二、生成随机短码

```go
func generateCode(n int) string {
	b := make([]byte, n)
	for i := range b {
		b[i] = alphabet[rand.Intn(len(alphabet))]
	}
	return string(b)
}
```

这只是学习写法。真实项目要使用更合适的随机源，或使用基于 ID 的编码方案。

---

## 三、唯一性不能只靠代码判断

错误思路：

```text
先查 code 是否存在。
不存在再插入。
```

并发下两个请求可能同时认为 code 不存在，然后都插入。

正确思路：

```text
存储层用唯一约束兜底。
冲突时业务层重试生成。
```

内存 Store 中可以在锁内检查并插入。数据库版本中要用 UNIQUE 约束。

---

## 四、冲突重试

```go
for i := 0; i < 5; i++ {
	code := generateCode(6)
	link := Link{Code: code, OriginalURL: originalURL, CreatedAt: time.Now()}

	created, err := s.store.Create(ctx, link)
	if err == nil {
		return created, nil
	}
	if !errors.Is(err, ErrConflict) {
		return Link{}, err
	}
}

return Link{}, fmt.Errorf("generate unique code failed")
```
