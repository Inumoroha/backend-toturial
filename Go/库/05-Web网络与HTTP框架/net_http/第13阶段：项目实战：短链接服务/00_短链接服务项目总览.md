# 0. 短链接服务项目总览

本阶段目标：完成一个更接近真实后端场景的综合项目：短链接服务。它会训练路由、重定向、JSON API、并发安全、存储设计、统计和错误处理。

相比 Todo API，短链接服务多了几个更真实的后端问题：

- 短码必须唯一。
- 跳转接口返回的不是 JSON，而是重定向。
- 访问次数会被频繁更新。
- 原始 URL 必须校验。
- 将来很自然会接 PostgreSQL 和 Redis。

---

## 一、核心功能

核心功能：

```text
用户提交长链接
-> 服务生成短码
-> 用户访问短码
-> 服务重定向到原始链接
-> 服务记录访问次数
```

接口规划：

```text
POST /links
GET  /{code}
GET  /links/{code}
GET  /links
```

本项目会优先使用 `net/http` 标准库表达核心逻辑。后续可以扩展 PostgreSQL、Redis、限流、访问日志和后台管理能力。

## 三、最小闭环

最小闭环是：

```text
POST /links 创建短链接
-> GET /{code} 返回 302
-> 再查详情看到 visit_count 增加
```

只要这个闭环跑通，说明：

- 短码生成可用。
- 存储可用。
- 重定向可用。
- 访问计数可用。

## 四、达标标准

完成本阶段后，你应该能：

- 校验原始 URL。
- 生成短码并处理冲突。
- 使用 `http.Redirect` 返回 302。
- 并发安全地更新访问次数。
- 设计详情和列表接口。
- 写出核心测试和 curl 验证命令。
- 说明如果换成 PostgreSQL 和 Redis，哪些模块需要改。
