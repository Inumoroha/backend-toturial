# 1. 需求分析与接口设计

本节目标：定义短链接服务的功能边界和 API。

---

## 一、核心流程

```text
用户提交长链接
-> 服务生成短码
-> 保存短码和长链接映射
-> 用户访问短码
-> 服务返回 302 重定向
-> 服务记录访问次数
```

---

## 二、接口设计

```text
POST /links
GET  /{code}
GET  /links/{code}
GET  /links
```

说明：

- `POST /links`：创建短链接。
- `GET /{code}`：跳转到原始链接。
- `GET /links/{code}`：查询短链接详情。
- `GET /links`：分页查询短链接列表。

---

## 三、创建请求

```json
{
  "original_url": "https://example.com/posts/go-net-http"
}
```

响应：

```json
{
  "code": "aB39xQ",
  "original_url": "https://example.com/posts/go-net-http",
  "short_url": "http://localhost:8080/aB39xQ",
  "visit_count": 0
}
```

---

## 四、重定向响应

访问：

```text
GET /aB39xQ
```

响应：

```http
HTTP/1.1 302 Found
Location: https://example.com/posts/go-net-http
```
