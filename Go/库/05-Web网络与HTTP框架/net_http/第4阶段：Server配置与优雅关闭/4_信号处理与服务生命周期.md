# 4. 信号处理与服务生命周期

本节目标：学会捕获系统退出信号，把服务启动、运行、关闭串成完整生命周期。

---

## 一、常见退出信号

本地开发时，你通常按：

```text
Ctrl+C
```

这会向进程发送中断信号。

在容器或 Linux 服务中，停止服务时常见的是：

```text
SIGTERM
```

Go 中可以使用 `os/signal` 捕获这些信号。

---

## 二、基础信号监听

```go
ctx, stop := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)
defer stop()

<-ctx.Done()
log.Println("shutdown signal received")
```

需要导入：

```go
import (
	"context"
	"os"
	"os/signal"
	"syscall"
)
```

`signal.NotifyContext` 会返回一个 context。当收到指定信号时，这个 context 会被取消。

---

## 三、完整服务生命周期示例

```go
package main

import (
	"context"
	"fmt"
	"log"
	"net/http"
	"os"
	"os/signal"
	"syscall"
	"time"
)

func main() {
	mux := http.NewServeMux()
	mux.HandleFunc("GET /health", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "ok")
	})

	srv := &http.Server{
		Addr:              ":8080",
		Handler:           mux,
		ReadHeaderTimeout: 5 * time.Second,
		ReadTimeout:       10 * time.Second,
		WriteTimeout:      10 * time.Second,
		IdleTimeout:       60 * time.Second,
	}

	go func() {
		log.Println("server listening on :8080")
		if err := srv.ListenAndServe(); err != nil && err != http.ErrServerClosed {
			log.Fatalf("server error: %v", err)
		}
	}()

	ctx, stop := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)
	defer stop()

	<-ctx.Done()
	log.Println("shutdown signal received")

	shutdownCtx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
	defer cancel()

	if err := srv.Shutdown(shutdownCtx); err != nil {
		log.Printf("server shutdown error: %v", err)
	}

	log.Println("server stopped")
}
```

---

## 四、为什么启动服务要放 goroutine

因为：

```go
srv.ListenAndServe()
```

会阻塞。

如果你直接写：

```go
srv.ListenAndServe()
<-ctx.Done()
```

后面的等待信号代码永远不会执行，除非服务启动失败。

所以要用 goroutine 启动服务，让主 goroutine 负责等待退出信号。

---

## 五、日志应该记录什么

服务生命周期建议至少记录：

- 服务开始启动。
- 监听地址。
- 启动失败错误。
- 收到关闭信号。
- 开始优雅关闭。
- 关闭失败错误。
- 服务已停止。

这些日志在排查发布问题时很有用。

- 正确忽略 `http.ErrServerClosed`。
