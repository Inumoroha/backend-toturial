# 1. 开发环境与工具准备

本节目标：确认 Go、编辑器、终端和调试工具可用，并创建一个专门学习 `net/http` 的工作目录。

---

## 一、确认 Go 是否安装成功

在终端执行：

```bash
go version
```

如果能看到类似下面的输出，说明 Go 已经安装成功：

```text
go version go1.22.0 windows/amd64
```

版本不需要和示例完全一致，但建议使用 Go 1.22 或更新版本。原因是新版 `net/http.ServeMux` 支持更现代的路由模式，例如：

```go
mux.HandleFunc("GET /users/{id}", handler)
```

这会让标准库写 API 更舒服。

---

## 二、确认 GOPATH 和 module 思维

现代 Go 项目基本都使用 Go module。你不需要把项目放在 GOPATH 下，只要在项目根目录执行：

```bash
go mod init example.com/net-http-learning
```

就会生成：

```text
go.mod
```

`go.mod` 的作用是记录：

- 当前项目的 module 名称。
- 使用的 Go 版本。
- 项目依赖了哪些第三方包。

学习 `net/http` 的前期几乎不需要第三方依赖，因为我们会优先使用标准库。

---

## 三、推荐目录

你可以在当前 `net_http` 目录下创建一个 `playground` 目录，用来写练习代码：

```text
net_http/
  playground/
    go.mod
    main.go
```

创建命令：

```bash
mkdir playground
cd playground
go mod init example.com/net-http-playground
```

后面每个阶段的练习都可以先在这个目录里完成。等项目实战阶段，再单独创建完整项目。

---

## 四、编辑器建议

推荐使用 VS Code 或 GoLand。

如果使用 VS Code，建议安装：

- Go 扩展。
- Error Lens，可选，用来更明显地显示错误。
- REST Client，可选，可以在编辑器里发 HTTP 请求。

但即使你安装了图形化工具，也不要跳过 `curl`。`curl` 是后端工程师最基础、最稳定的接口调试工具。

---

## 五、确认 curl 是否可用

执行：

```bash
curl --version
```

如果能输出版本信息，说明可用。

在 Windows PowerShell 中，部分环境里 `curl` 可能是 `Invoke-WebRequest` 的别名。新版本 Windows 通常已经内置真正的 curl。如果你发现行为和教程不一致，可以执行：

```bash
curl.exe --version
```

后续命令如果遇到差异，就把 `curl` 换成 `curl.exe`。

---

## 六、准备一个最小 main.go

进入练习目录后，新建 `main.go`：

```go
package main

import "fmt"

func main() {
	fmt.Println("net/http learning")
}
```

运行：

```bash
go run .
```

看到：

```text
net/http learning
```

说明基本环境没问题。

---

## 七、常用命令清单

后续会反复使用这些命令：

```bash
go run .
go test ./...
go test -race ./...
go fmt ./...
go mod tidy
```

含义分别是：

- `go run .`：运行当前目录的 Go 程序。
- `go test ./...`：运行当前 module 下所有测试。
- `go test -race ./...`：开启 data race 检查。
- `go fmt ./...`：格式化代码。
- `go mod tidy`：整理依赖。

---

## 八、本节检查点

完成本节后，请确认：

- `go version` 能正常输出。
- `curl --version` 或 `curl.exe --version` 能正常输出。
- 已经创建练习目录。
- `go run .` 能成功运行一个最小 Go 程序。

这些准备很普通，但它们会让后面所有练习变得顺畅。
