# 0. 学习准备与 HTTP 直觉总览

本阶段目标：先把学习环境、HTTP 基础概念和调试工具准备好。`net/http` 是 Go 标准库对 HTTP 协议的工程化封装，如果完全不了解 HTTP，一上来就写 Handler 很容易只记住 API，而不知道每一行代码背后的意义。

学习 `net/http` 时，建议你始终把一个请求想象成下面这条链路：

```text
浏览器或客户端
-> 发送 HTTP 请求
-> Go 服务端接收请求
-> 路由找到对应 Handler
-> Handler 读取 Request
-> Handler 写入 ResponseWriter
-> 客户端收到 HTTP 响应
```

后面所有知识都可以放回这条链路中理解。

---

## 一、本阶段要解决什么问题

很多初学者学 Web 后端时会遇到这些困惑：

- 为什么接口返回了内容，但浏览器显示不对？
- 为什么 POST 请求服务端读不到 JSON？
- 为什么明明写了错误信息，客户端状态码还是 200？
- Header、Body、Query、Path 到底分别放什么？
- 为什么后端服务要配置超时？
- 为什么调接口时推荐先学会 `curl`？

本阶段不追求写复杂服务，而是先建立这些基本直觉。

## 三、本阶段最终能力

学完本阶段后，你应该能做到：

- 确认本机 Go 环境可用。
- 创建一个最小 Go module。
- 说清楚 HTTP 请求和响应的大致结构。
- 使用浏览器访问 GET 接口。
- 使用 `curl` 发送 GET、POST、查看 Header。
- 理解状态码、方法、Header、Body 的基本作用。
- 知道后续每个阶段应该如何练习。

---

## 四、学习建议

不要跳过工具准备。后端开发不是只写代码，也包括启动服务、查看日志、发送请求、定位错误。`net/http` 学得好不好，很大程度上取决于你是否能清楚看到一次请求从哪里来、到哪里去、在哪里失败。

本阶段建议花 0.5 到 1 天。看起来基础，但它会让后面的学习轻松很多。
