# Go net/http 教程目录

这套教程面向想成为 Go 后端工程师的学习者，目标不是只会写几个接口，而是系统理解 Go 标准库 `net/http` 的设计、工程用法、测试方式、生产配置和项目落地。

目录组织参考同仓库 PostgreSQL 教程风格：

- 每个阶段放在独立文件夹中。
- 文件名前缀使用 `0_`、`1_`、`2_` 表示建议阅读顺序。
- 每个阶段先读 `0_...总览.md`，再进入具体小节。
- 前面阶段重视基础直觉，中间阶段重视工程能力，后面阶段重视源码、性能和项目实战。

```text
-> 第13阶段：项目实战：短链接服务
```

## 学习方式

每一节都建议按下面的节奏走：

1. 先读目标，知道这一节解决什么问题。
2. 再读概念，建立直觉。
3. 手动敲代码，不要只复制。
4. 用 `curl` 或浏览器验证结果。
5. 故意制造一个错误，观察报错。
6. 最后完成阶段练习。

## 代码约定

教程中的命令默认在 PowerShell 中执行。Go 代码默认使用 Go 1.22 或更新版本，因为新版 `net/http.ServeMux` 已经支持类似 `GET /users/{id}` 的方法和路径模式。

如果你使用旧版本 Go，路由部分需要自己判断 `r.Method` 和 `r.URL.Path`，或者升级 Go 版本。

## 最终能力

学完后你应该能独立完成：

- 使用标准库编写 RESTful JSON API。
- 组织 Handler、Service、Middleware。
- 编写可测试的 HTTP 服务。
- 正确配置 `http.Server` 的超时和优雅关闭。
- 编写可靠的 HTTP Client。
- 处理上传、下载、Cookie、CORS、鉴权等常见后端能力。
- 用日志、压测和 pprof 定位问题。
- 阅读 `net/http` 核心源码，理解它的组合模型。
- 完成 Todo API 和短链接服务两个项目。
