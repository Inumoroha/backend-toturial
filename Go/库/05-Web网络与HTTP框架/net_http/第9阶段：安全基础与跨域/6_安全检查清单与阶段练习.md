# 6. 安全检查清单与阶段练习

本节目标：把本阶段安全点落到一个具体 HTTP 服务中。安全知识如果不变成检查项，很容易学完就忘。

---

## 一、基础安全检查清单

对任意 Go HTTP 服务，先检查：

```text
Server 是否配置 ReadHeaderTimeout。
Server 是否配置 ReadTimeout、WriteTimeout、IdleTimeout。
JSON 请求体是否限制大小。
上传接口是否单独限制大小。
错误响应是否避免暴露内部错误。
Cookie 是否设置 HttpOnly。
生产 HTTPS Cookie 是否设置 Secure。
CORS 是否使用白名单。
鉴权失败是否返回 401。
无权限是否返回 403。
日志是否避免打印完整 Token。
是否理解 RemoteAddr 和代理 Header 的边界。
```

---

## 二、给 Todo API 做安全改造

建议你在 Todo API 中完成：

1. 所有 JSON Body 限制为 1MB。
2. 写操作需要 `Authorization: Bearer dev-token`。
3. CORS 只允许 `http://localhost:5173`。
4. recovery 只返回统一 `internal server error`。
5. 日志中不打印 `Authorization`。

---

## 三、验证命令

未授权：

```bash
curl -i -X POST http://localhost:8080/todos `
  -H "Content-Type: application/json" `
  -d "{\"title\":\"learn\"}"
```

授权：

```bash
curl -i -X POST http://localhost:8080/todos `
  -H "Content-Type: application/json" `
  -H "Authorization: Bearer dev-token" `
  -d "{\"title\":\"learn\"}"
```

CORS 预检：

```bash
curl -i -X OPTIONS http://localhost:8080/todos `
  -H "Origin: http://localhost:5173" `
  -H "Access-Control-Request-Method: POST"
```

## 四、阶段复盘

完成后请写下：

- 哪些安全点是在 Handler 中处理的？
- 哪些安全点是在 Middleware 中处理的？
- 哪些安全点是在 `http.Server` 中处理的？
- 哪些安全点需要部署层配合？

能分清这些边界，说明你已经不是把安全当成一个单独功能，而是在按后端工程方式处理它。
