# 0. 安全基础与跨域总览

本阶段目标：建立 Go HTTP 服务的基础安全意识。安全不是项目最后才补的功能，而是从读取请求、设置 Header、处理错误时就要考虑。

本阶段不会把你变成安全专家，但会帮你形成 Go Web 服务最基本的防线。很多线上事故并不是复杂攻击造成的，而是因为：

- 请求体没有限制。
- 服务没有超时。
- Cookie 缺少安全属性。
- CORS 随便放开。
- 错误信息把内部细节暴露给客户端。
- 日志里打印了敏感 Token。

---

## 一、本阶段学习顺序

1. 请求体大小限制。
2. Server 超时与慢请求防护。
3. Cookie 安全属性。
4. Token 鉴权边界。
5. CORS 白名单。
6. 敏感错误信息保护。
7. 反向代理后的真实 IP。
8. 安全检查清单与阶段练习。

---

## 二、本阶段关注的不是“绝对安全”

初学阶段不要试图一次性覆盖所有安全问题。本阶段重点是让你每写一个 HTTP 服务时，都能自然地问：

```text
请求会不会太大？
请求会不会太慢？
响应会不会泄露内部信息？
Cookie 是否能被脚本读取？
CORS 是否只允许可信前端？
日志里有没有敏感数据？
```

这些问题会让你的服务比“只写业务功能”稳很多。

---

## 三、本阶段达标标准

完成本阶段后，你应该能：

- 给 JSON API 和上传接口设置请求体限制。
- 配置基础 Server 超时。
- 正确设置 `HttpOnly`、`Secure`、`SameSite`。
- 区分 `401` 和 `403`。
- 写出 CORS 白名单中间件。
- 避免把内部错误和完整 Token 输出给客户端或日志。
- 理解代理后的 `RemoteAddr` 不一定是真实用户 IP。
