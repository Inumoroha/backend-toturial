# 0. 请求与响应处理总览

本阶段目标：从“能返回文本”升级到“能写 JSON API”。真实后端服务大多数都在处理输入、校验参数、调用业务逻辑、返回结构化响应。

第 1 阶段你已经知道 Handler 的基本形态：

```go
func handler(w http.ResponseWriter, r *http.Request) {
}
```

本阶段要深入这两个参数：

- 从 `r` 中读取 Query、Path、Header、Cookie、Body。
- 通过 `w` 写 Header、状态码、JSON、错误信息。

---

## 一、本阶段要解决什么问题

真实 API 不会只返回 `hello`。它通常需要：

- 接收 JSON 请求体。
- 校验字段是否为空。
- 返回 JSON 响应。
- 返回正确状态码。
- 给错误响应统一格式。
- 限制请求体大小。
- 处理读取 Body 和 JSON 解析错误。

这些就是本阶段重点。

## 三、本阶段核心原则

写 API 时要记住：

```text
输入要校验。
错误要明确。
状态码要正确。
响应格式要统一。
```

如果一个接口只有成功路径，没有错误路径，它还不能算写完。

---

## 四、本阶段达标标准

完成本阶段后，你应该能独立写出：

- 读取 Query 参数的接口。
- 读取 Path 参数的接口。
- 读取 JSON Body 的接口。
- 返回 JSON 的接口。
- 统一错误响应函数。
- 一个内存版 Todo API。
