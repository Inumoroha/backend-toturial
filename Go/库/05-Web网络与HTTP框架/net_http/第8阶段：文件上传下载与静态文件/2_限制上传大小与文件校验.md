# 2. 限制上传大小与文件校验

本节目标：给文件上传增加基本安全限制，包括大小、文件名、后缀和 Content-Type。

---

## 一、限制请求体大小

```go
const maxUploadSize = 5 << 20 // 5MB

r.Body = http.MaxBytesReader(w, r.Body, maxUploadSize)
```

要放在解析表单前：

```go
func upload(w http.ResponseWriter, r *http.Request) {
	r.Body = http.MaxBytesReader(w, r.Body, maxUploadSize)

	if err := r.ParseMultipartForm(maxUploadSize); err != nil {
		http.Error(w, "file too large or invalid form", http.StatusBadRequest)
		return
	}
}
```

---

## 二、不要直接使用原始文件名

不安全：

```go
path := "./uploads/" + header.Filename
```

如果文件名包含：

```text
../../something
```

就可能造成路径穿越。

至少要使用：

```go
filename := filepath.Base(header.Filename)
```

更推荐生成自己的文件名：

```go
filename := fmt.Sprintf("%d%s", time.Now().UnixNano(), ext)
```

---

## 三、校验后缀

```go
ext := strings.ToLower(filepath.Ext(header.Filename))
allowed := map[string]bool{
	".jpg":  true,
	".jpeg": true,
	".png":  true,
	".gif":  true,
}

if !allowed[ext] {
	http.Error(w, "unsupported file type", http.StatusBadRequest)
	return
}
```

后缀校验不能代表绝对安全，但它是基础防线。

---

## 四、校验 Content-Type

可以读取文件前 512 字节：

```go
buf := make([]byte, 512)
n, err := file.Read(buf)
if err != nil && err != io.EOF {
	http.Error(w, "read file failed", http.StatusBadRequest)
	return
}

contentType := http.DetectContentType(buf[:n])
```

读完后要把文件指针移回开头：

```go
if _, err := file.Seek(0, io.SeekStart); err != nil {
	http.Error(w, "reset file failed", http.StatusInternalServerError)
	return
}
```

注意：`multipart.File` 不一定总是支持 Seek，具体取决于实现。学习阶段可以先掌握思路，生产项目要更严谨处理。
