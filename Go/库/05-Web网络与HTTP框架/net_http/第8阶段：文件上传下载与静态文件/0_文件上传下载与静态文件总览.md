# 0. 文件上传下载与静态文件总览

本阶段目标：掌握 Web 后端常见文件处理能力，包括上传、下载、静态文件服务和基础安全校验。

文件处理看起来只是“把文件收进来、再把文件发出去”，但真实后端里它很容易引出安全和资源问题：

- 用户上传的文件可能非常大。
- 文件名可能带有路径穿越内容。
- 文件后缀和真实内容可能不一致。
- 私有文件不能直接暴露成静态目录。
- 下载响应要考虑浏览器预览和附件下载的差异。

所以本阶段不只是学习 API，而是建立文件服务的工程边界。

---

## 一、本阶段学习顺序

1. Multipart 文件上传。
2. 限制上传大小。
3. 校验文件名、后缀和 Content-Type。
4. 使用 `http.ServeFile` 下载文件。
5. 使用 `http.FileServer` 提供静态目录。
6. 文件服务阶段练习。
7. 文件服务安全检查清单。

---

## 二、本阶段核心 API

```text
r.ParseMultipartForm
r.FormFile
http.MaxBytesReader
http.ServeFile
http.FileServer
http.StripPrefix
http.DetectContentType
```

学习时要把这些 API 放回两个流程中理解：

```text
上传流程：限制大小 -> 解析 multipart -> 校验文件 -> 保存文件
下载流程：解析文件名 -> 校验路径和权限 -> 设置 Header -> 返回文件
```

---

## 三、本阶段达标标准

完成本阶段后，你应该能：

- 使用 curl 上传文件。
- 限制上传大小。
- 不直接信任用户上传的文件名。
- 校验文件后缀和内容类型。
- 使用 `ServeFile` 返回单个文件。
- 使用 `FileServer` 暴露公开静态目录。
- 说清楚公开文件和私有文件的处理差异。
