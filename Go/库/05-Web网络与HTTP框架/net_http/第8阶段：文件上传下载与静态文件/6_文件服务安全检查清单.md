# 6. 文件服务安全检查清单

本节目标：把前面上传、下载、静态文件的知识整理成一份可执行检查清单。文件服务真正上线前，不能只验证“能上传、能下载”，还要验证边界。

---

## 一、上传接口检查

上传接口至少检查：

- 是否使用 `http.MaxBytesReader` 限制请求体大小。
- 是否限制 `ParseMultipartForm` 的内存使用。
- 是否处理没有文件字段的请求。
- 是否处理空文件。
- 是否限制允许的文件后缀。
- 是否检查文件内容类型。
- 是否生成服务端文件名，而不是直接使用用户文件名。
- 是否把上传目录和代码目录分开。

示例验证：

```bash
curl -i -X POST http://localhost:8080/upload
curl -i -X POST http://localhost:8080/upload -F "file=@./avatar.png"
curl -i -X POST http://localhost:8080/upload -F "file=@./large.zip"
```

---

## 二、下载接口检查

下载接口至少检查：

- 是否使用 `filepath.Base` 或更严格方式处理文件名。
- 是否避免 `../../` 路径穿越。
- 文件不存在时是否返回清晰错误。
- 私有文件是否做权限校验。
- 是否需要设置 `Content-Disposition`。

验证路径穿越：

```bash
curl -i http://localhost:8080/files/../../secret.txt
```

服务不应该返回上传目录外的文件。

---

## 三、静态文件检查

静态文件目录适合放公开资源，不适合放：

- 配置文件。
- 私钥。
- 用户隐私文件。
- 未校验的上传文件。
- 数据库备份。

如果使用：

```go
http.FileServer(http.Dir("./public"))
```

就要确保 `public` 目录里的内容都是允许公开访问的。

---

## 四、阶段验收标准

完成本阶段后，你的文件服务应该能通过：

```text
正常图片可以上传。
超大文件会被拒绝。
不支持的后缀会被拒绝。
下载不存在文件返回 404。
路径穿越请求不会读到目录外文件。
公开静态目录只包含可公开资源。
```

这份检查清单以后可以直接放进项目 README 或发布前检查文档。
