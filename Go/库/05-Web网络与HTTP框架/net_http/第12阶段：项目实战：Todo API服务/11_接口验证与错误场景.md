# 11. 接口验证与错误场景

本节目标：用 curl 完整验证 Todo API 的成功路径和错误路径。项目没经过错误场景验证，就不算完成。

---

## 一、启动服务

```bash
go run ./cmd/server
```

健康检查：

```bash
curl -i http://localhost:8080/health
```

预期：

```text
200 OK
```

---

## 二、创建 Todo

```bash
curl -i -X POST http://localhost:8080/todos `
  -H "Content-Type: application/json" `
  -H "Authorization: Bearer dev-token" `
  -d "{\"title\":\"learn net/http\"}"
```

预期：

```text
201 Created
```

响应体包含：

```json
{"id":1,"title":"learn net/http","done":false}
```

---

## 三、查询列表和详情

```bash
curl -i http://localhost:8080/todos
curl -i http://localhost:8080/todos/1
```

预期都是 `200 OK`。

---

## 四、更新 Todo

```bash
curl -i -X PATCH http://localhost:8080/todos/1 `
  -H "Content-Type: application/json" `
  -H "Authorization: Bearer dev-token" `
  -d "{\"done\":true}"
```

再查详情，确认 `done` 变成 `true`。

---

## 五、删除 Todo

```bash
curl -i -X DELETE http://localhost:8080/todos/1 `
  -H "Authorization: Bearer dev-token"
```

预期：

```text
204 No Content
```

再查询：

```bash
curl -i http://localhost:8080/todos/1
```

预期：

```text
404 Not Found
```

---

## 六、错误场景验证

非法 JSON：

```bash
curl -i -X POST http://localhost:8080/todos `
  -H "Content-Type: application/json" `
  -H "Authorization: Bearer dev-token" `
  -d "{"
```

空 title：

```bash
curl -i -X POST http://localhost:8080/todos `
  -H "Content-Type: application/json" `
  -H "Authorization: Bearer dev-token" `
  -d "{\"title\":\"\"}"
```

非法 ID：

```bash
curl -i http://localhost:8080/todos/abc
```

未授权写操作：

```bash
curl -i -X POST http://localhost:8080/todos `
  -H "Content-Type: application/json" `
  -d "{\"title\":\"learn\"}"
```

---

## 七、观察日志

每次请求后看服务端日志，确认至少有：

```text
request_id
method
path
status
duration_ms
```

如果状态码和预期不一致，优先检查：

- 路由是否注册正确。
- 是否带了 Authorization。
- JSON 是否合法。
- Handler 是否正确 return。

---

## 八、本节检查点

你应该能完成一个完整闭环：

```text
创建 -> 查询列表 -> 查询详情 -> 更新 -> 删除 -> 再查询 404
```

并且能验证：

```text
非法 JSON -> 400
空 title -> 400
非法 ID -> 400
不存在资源 -> 404
未授权写操作 -> 401
```
