# 6. 初始化 Go 模块与目录

本节目标：从零创建 Todo API 项目的目录结构，让后续代码有清晰位置。

---

## 一、创建项目目录

在练习位置执行：

```bash
mkdir todo-api
cd todo-api
go mod init example.com/todo-api
```

生成：

```text
go.mod
```

这个项目暂时只使用标准库，所以不会立刻出现第三方依赖。

---

## 二、创建目录结构

```bash
mkdir cmd
mkdir cmd\server
mkdir internal
mkdir internal\httpapi
mkdir internal\middleware
mkdir internal\todo
```

最终结构：

```text
todo-api/
  cmd/
    server/
  internal/
    httpapi/
    middleware/
    todo/
  go.mod
```

---

## 三、先放一个最小 main.go

文件：`cmd/server/main.go`

```go
package main

import "fmt"

func main() {
	fmt.Println("todo api")
}
```

运行：

```bash
go run ./cmd/server
```

看到：

```text
todo api
```

说明目录和 module 基本可用。

---

## 四、为什么先做这个小闭环

不要一上来就写很多文件。先确认：

- module 初始化成功。
- 入口文件路径正确。
- `go run ./cmd/server` 能跑。
- 包路径后续可以正常导入。

这个小闭环能避免后面写了很多代码，才发现目录或 module 名称有问题。

---

## 五、本节检查点

请确认：

- `todo-api/go.mod` 已生成。
- `cmd/server/main.go` 可以运行。
- `internal/httpapi`、`internal/middleware`、`internal/todo` 已创建。
- 你知道每个目录后面要放什么。
