# 1. 需求分析与接口设计

本节目标：先把 Todo API 的边界定义清楚，再写代码。项目实战最怕一边写一边改方向。

---

## 一、项目目标

实现一个标准 JSON API 服务，支持：

```text
创建 Todo
查看 Todo 列表
查看 Todo 详情
更新 Todo
删除 Todo
```

本项目重点不是 Todo 本身，而是训练：

- RESTful 路由设计。
- JSON 请求响应。
- 错误处理。
- Middleware。
- `http.Server` 启动。
- 优雅关闭。
- 测试。

---

## 二、接口设计

```text
GET    /health
GET    /todos
POST   /todos
GET    /todos/{id}
PATCH  /todos/{id}
DELETE /todos/{id}
```

---

## 三、请求响应示例

创建：

```http
POST /todos
Content-Type: application/json

{"title":"learn net/http"}
```

响应：

```http
HTTP/1.1 201 Created
Content-Type: application/json

{"id":1,"title":"learn net/http","done":false}
```

列表：

```json
{
  "items": [
    {"id":1,"title":"learn net/http","done":false}
  ]
}
```

错误：

```json
{
  "error": {
    "code": "invalid_request",
    "message": "title is required"
  }
}
```

---

## 四、状态码规则

```text
创建成功 -> 201
查询成功 -> 200
更新成功 -> 200
删除成功 -> 204
参数错误 -> 400
未授权 -> 401
不存在 -> 404
内部错误 -> 500
```
