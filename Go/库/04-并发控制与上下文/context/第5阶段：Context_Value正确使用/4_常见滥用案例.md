# 4. 常见滥用案例

本节目标：识别 `context.Value` 的坏味道，避免把代码写成隐式依赖。

`context.Value` 滥用后，代码通常会变得：

- 函数签名看不出依赖。
- 类型断言到处都是。
- 测试很难写。
- 重构容易漏。
- 运行时才发现缺值。

---

## 一、把业务参数塞进 context

不推荐：

```go
ctx = context.WithValue(ctx, "product_id", productID)
orderService.CreateOrder(ctx)
```

推荐：

```go
orderService.CreateOrder(ctx, productID)
```

如果没有 `productID` 就不能创建订单，那它就是业务必需参数。

业务必需参数应该显式出现在函数签名中。

---

## 二、把数据库连接塞进 context

不推荐：

```go
ctx = context.WithValue(ctx, "db", db)
repo.Save(ctx)
```

推荐：

```go
type Repo struct {
	db *sql.DB
}

func (r *Repo) Save(ctx context.Context, input Input) error {
	_, err := r.db.ExecContext(ctx, "...", input.Name)
	return err
}
```

依赖对象应该通过结构体字段注入。

---

## 三、把可变 map 塞进 context

不推荐：

```go
ctx = context.WithValue(ctx, "data", map[string]any{})
```

问题：

- 多层都能修改。
- 并发访问可能有数据竞争。
- 不知道谁写了什么。
- 类型不清楚。

context value 应该尽量是小的、稳定的、请求级元数据。

---

## 四、到处直接类型断言

不推荐：

```go
requestID := ctx.Value("request_id").(string)
```

如果不存在或类型不对，会 panic。

推荐：

```go
requestID, ok := RequestIDFromContext(ctx)
if !ok {
	requestID = "unknown"
}
```

---

## 五、在底层偷偷要求 value

坏味道：

```go
func (r *Repo) Find(ctx context.Context) (*User, error) {
	userID := ctx.Value(userIDKey).(int64)
	// query by userID
}
```

调用方看函数签名完全不知道必须提前放 userID。

推荐：

```go
func (r *Repo) Find(ctx context.Context, userID int64) (*User, error)
```

---

## 六、本节达标标准

学完本节后，你应该能够做到：

- 识别业务参数塞 context 的问题。
- 识别依赖对象塞 context 的问题。
- 避免把可变 map 放进 context。
- 避免直接强制类型断言。
- 让函数依赖通过签名表达出来。

---

## 七、坏味道检查清单

看到 `context.WithValue` 时，逐项检查：

```text
这个 value 是不是请求级元数据？
这个 value 是否应该出现在函数参数里？
这个 value 是否是应用依赖？
这个 value 是否可能很大？
这个 value 是否会被修改？
key 是否是自定义私有类型？
读取时是否有 ok 判断？
```

只要有一个答案不对，就要重新考虑设计。

---

## 八、重构案例：业务参数移出 context

重构前：

```go
func CreateOrder(ctx context.Context) error {
	userID := ctx.Value(userIDKey).(int64)
	productID := ctx.Value(productIDKey).(int64)
	return saveOrder(ctx, userID, productID)
}
```

重构后：

```go
func CreateOrder(ctx context.Context, userID int64, productID int64) error {
	return saveOrder(ctx, userID, productID)
}
```

签名更清晰，测试也更简单。

---

## 九、重构案例：依赖对象移出 context

重构前：

```go
func Save(ctx context.Context, input Input) error {
	db := ctx.Value(dbKey).(*sql.DB)
	_, err := db.ExecContext(ctx, "...", input.Name)
	return err
}
```

重构后：

```go
type Repo struct {
	db *sql.DB
}

func (r *Repo) Save(ctx context.Context, input Input) error {
	_, err := r.db.ExecContext(ctx, "...", input.Name)
	return err
}
```

依赖关系从隐式变成显式。

---

## 十、什么时候可以接受 Value

可以接受：

```text
RequestIDFromContext
TraceIDFromContext
AuthUserFromContext
TenantIDFromContext
```

这些都是请求级横切数据。

但即便如此，也应该封装读写函数。

---

## 十一、本节练习

请重构下面伪代码：

```go
ctx = context.WithValue(ctx, "db", db)
ctx = context.WithValue(ctx, "user_id", userID)
ctx = context.WithValue(ctx, "page", page)

service.ListOrders(ctx)
```

目标：

- db 放进 repository 结构体。
- userID 和 page 作为函数参数。
- context 只保留 request id。
