# 5. 请求链路综合练习

本节目标：完成一个从 HTTP handler 到 service 再到 repository 的 context 传递练习。

这个练习不要求连接真实数据库，可以先用 `time.Sleep` 模拟 repository 查询。

---

## 一、项目结构

建议创建：

```text
cmd/04-request-chain/main.go
```

也可以拆成多个文件：

```text
internal/user/handler.go
internal/user/service.go
internal/user/repository.go
```

初学阶段放在一个 `main.go` 里也可以。

---

## 二、完整示例

```go
package main

import (
	"context"
	"encoding/json"
	"fmt"
	"net/http"
	"time"
)

type User struct {
	ID   int64  `json:"id"`
	Name string `json:"name"`
}

type UserRepo struct{}

func (r *UserRepo) FindByID(ctx context.Context, id int64) (*User, error) {
	ctx, cancel := context.WithTimeout(ctx, 500*time.Millisecond)
	defer cancel()

	select {
	case <-time.After(300 * time.Millisecond):
		return &User{ID: id, Name: "Tom"}, nil
	case <-ctx.Done():
		return nil, fmt.Errorf("find user stopped: %w", ctx.Err())
	}
}

type UserService struct {
	repo *UserRepo
}

func (s *UserService) GetUser(ctx context.Context, id int64) (*User, error) {
	if id <= 0 {
		return nil, fmt.Errorf("invalid user id")
	}
	return s.repo.FindByID(ctx, id)
}

type UserHandler struct {
	service *UserService
}

func (h *UserHandler) GetUser(w http.ResponseWriter, r *http.Request) {
	ctx := r.Context()

	user, err := h.service.GetUser(ctx, 1001)
	if err != nil {
		http.Error(w, err.Error(), http.StatusInternalServerError)
		return
	}

	_ = json.NewEncoder(w).Encode(user)
}

func main() {
	repo := &UserRepo{}
	service := &UserService{repo: repo}
	handler := &UserHandler{service: service}

	mux := http.NewServeMux()
	mux.HandleFunc("/users/1001", handler.GetUser)

	fmt.Println("listen on :8080")
	_ = http.ListenAndServe(":8080", mux)
}
```

## 三、改造练习

请尝试：

1. 把 repository 模拟耗时从 300ms 改成 800ms。
2. 观察接口返回错误。
3. 把 repository 超时从 500ms 改成 1s。
4. 给 handler 增加 700ms 的请求级 timeout。
5. 思考 handler timeout 和 repository timeout 哪个先触发。

---

## 四、本阶段总结

你现在应该掌握：

- HTTP 入口用 `r.Context()`。
- Handler 把 ctx 传给 service。
- Service 把 ctx 传给 repository。
- Repository 把 ctx 传给数据库或模拟耗时任务。
- 中间件可以通过 `r.WithContext` 添加请求级元数据。
- 不要把 context 存进结构体。

下一阶段专门学习 `context.Value` 的边界。

## 六、观察实验

### 实验一：正常返回

repository 耗时：

```go
300 * time.Millisecond
```

repository timeout：

```go
500 * time.Millisecond
```

应该返回成功。

### 实验二：repository 超时

repository 耗时改成：

```go
800 * time.Millisecond
```

应该返回错误。

### 实验三：handler 总超时更短

handler 中增加：

```go
ctx, cancel := context.WithTimeout(r.Context(), 200*time.Millisecond)
defer cancel()
```

即使 repository timeout 是 500ms，也会先被 handler 的 200ms 取消。

## 七、建议扩展

给综合练习加入 request id：

1. 写 `WithRequestID`。
2. 写 `RequestIDFromContext`。
3. 写中间件。
4. 在 repository 超时时打印 request id。

这样你能把第 4 阶段和第 5 阶段连接起来。

---

## 八、常见错误检查

请检查你的代码里有没有：

- service 中使用 `context.Background()`。
- repository 方法没有 ctx 参数。
- `WithTimeout` 后没有 `defer cancel()`。
- handler 直接访问 repository。
- 超时后没有保留 `ctx.Err()`。
