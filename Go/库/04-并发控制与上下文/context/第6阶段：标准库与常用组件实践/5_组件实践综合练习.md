# 5. 组件实践综合练习

本节目标：把 HTTP、数据库模拟、signal、errgroup 串成一个练习。

---

## 一、练习目标

实现一个 HTTP 服务：

```text
GET /profile
```

内部并发调用三个模拟函数：

- `fetchUser`
- `fetchOrders`
- `fetchWallet`

要求：

- handler 使用 `r.Context()`。
- 总超时 1 秒。
- 使用 `errgroup.WithContext` 并发执行。
- 任一任务失败时取消其他任务。
- 程序支持 Ctrl+C 优雅关闭。

---

## 二、模拟下游函数

```go
func call(ctx context.Context, name string, cost time.Duration, fail bool) error {
	select {
	case <-time.After(cost):
		if fail {
			return fmt.Errorf("%s failed", name)
		}
		fmt.Println(name, "done")
		return nil
	case <-ctx.Done():
		return fmt.Errorf("%s stopped: %w", name, ctx.Err())
	}
}
```

---

## 三、Handler 结构

```go
func profileHandler(w http.ResponseWriter, r *http.Request) {
	ctx, cancel := context.WithTimeout(r.Context(), time.Second)
	defer cancel()

	g, ctx := errgroup.WithContext(ctx)

	g.Go(func() error {
		return call(ctx, "fetchUser", 100*time.Millisecond, false)
	})
	g.Go(func() error {
		return call(ctx, "fetchOrders", 500*time.Millisecond, false)
	})
	g.Go(func() error {
		return call(ctx, "fetchWallet", 300*time.Millisecond, false)
	})

	if err := g.Wait(); err != nil {
		http.Error(w, err.Error(), http.StatusBadGateway)
		return
	}

	fmt.Fprintln(w, "profile ok")
}
```

ctx, stop := signal.NotifyContext(context.Background(), os.Interrupt, syscall.SIGTERM)

## 六、完整 main.go 结构建议

综合练习可以组织成下面几个部分：

```text
main.go
  - call 模拟下游
  - profileHandler 聚合接口
  - newServer 创建 HTTP server
  - main 启动服务和优雅关闭
```

这样即使暂时不拆包，代码也不会太乱。

## 八、建议补充的日志

在 `call` 函数中打印：

```go
start := time.Now()
defer func() {
	fmt.Println(name, "cost:", time.Since(start))
}()
```

这样你能看到每个下游实际耗时。

在 handler 中打印：

```go
fmt.Println("profile request done")
```

如果某个下游超时，可以观察其他下游是否提前停止。

## 十、常见问题

### 1. 端口被占用

换端口：

```go
Addr: ":8081"
```

### 2. errgroup 包找不到

执行：

```powershell
go get golang.org/x/sync/errgroup
```

### 3. Ctrl+C 后没有退出

检查：

- 是否使用 `signal.NotifyContext`。
- 是否调用 `server.Shutdown`。
- worker 是否监听 ctx。
- 是否有 goroutine 卡住。

## 十一、完成标准

完成这个综合练习后，你应该能独立解释：

- HTTP 请求 ctx 从哪里来。
- errgroup ctx 如何取消其他任务。
- signal ctx 如何触发服务关闭。
- Shutdown 为什么要单独创建 timeout context。
- 下游任务为什么必须监听 `ctx.Done()`。
