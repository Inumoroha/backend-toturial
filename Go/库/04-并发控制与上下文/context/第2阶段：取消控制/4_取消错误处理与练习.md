# 4. 取消错误处理与练习

本节目标：学习取消发生后应该如何返回错误、判断错误，并完成本阶段综合练习。

取消不是异常情况里的“小事”。在后端服务中，它经常代表：

- 客户端断开。
- 上游超时。
- 服务关闭。
- 业务主动停止子任务。

正确处理取消错误，有助于日志、监控和排障。

---

## 一、取消时返回 ctx.Err()

推荐：

```go
case <-ctx.Done():
	return ctx.Err()
```

这样调用方可以判断：

```go
if errors.Is(err, context.Canceled) {
	// 主动取消
}
```

不推荐：

```go
case <-ctx.Done():
	return errors.New("failed")
```

这样会丢失真实原因。

---

## 二、可以包装错误

如果你想添加业务语义，可以用 `%w` 包装：

```go
return fmt.Errorf("fetch user stopped: %w", ctx.Err())
```

调用方仍然可以：

```go
errors.Is(err, context.Canceled)
```

完整示例：

```go
err := fetchUser(ctx)
if errors.Is(err, context.Canceled) {
	fmt.Println("fetch user canceled")
}
```

---

## 三、不要把取消都打成 error 日志

如果用户主动取消请求，或者客户端断开连接，这在很多系统里是正常现象。

日志级别可以区分：

```text
context.Canceled：通常 debug 或 info。
context.DeadlineExceeded：可能是 warn，需要观察。
其他错误：error。
```

当然具体级别要结合业务。

---

## 四、综合练习：可取消下载任务

实现一个下载任务：

```go
func download(ctx context.Context, url string) error
```

要求：

1. 每 300ms 打印一次下载进度。
2. 总共模拟 10 个分片。
3. 如果 context 被取消，返回 `ctx.Err()`。
4. main 中 1 秒后主动取消。
5. main 中使用 `errors.Is` 判断是否被取消。

参考代码：

```go
package main

import (
	"context"
	"errors"
	"fmt"
	"time"
)

func download(ctx context.Context, url string) error {
	for i := 1; i <= 10; i++ {
		select {
		case <-ctx.Done():
			return fmt.Errorf("download %s stopped: %w", url, ctx.Err())
		case <-time.After(300 * time.Millisecond):
			fmt.Printf("download %s chunk %d/10\n", url, i)
		}
	}

	return nil
}

func main() {
	ctx, cancel := context.WithCancel(context.Background())

	go func() {
		time.Sleep(time.Second)
		cancel()
	}()

	err := download(ctx, "https://example.com/file.zip")
	if errors.Is(err, context.Canceled) {
		fmt.Println("download canceled")
		return
	}
	if err != nil {
		fmt.Println("download error:", err)
		return
	}

	fmt.Println("download success")
}
```

## 五、本阶段总结

你现在应该掌握：

- `WithCancel` 创建可取消 context。
- `cancel()` 会关闭 `Done()` channel。
- 父 context 取消会传播给子 context。
- 子 context 取消不会反向影响父 context。
- goroutine 要主动监听 `ctx.Done()`。
- 取消时应返回 `ctx.Err()` 或包装它。

下一阶段学习超时和截止时间，它们是后端外部调用稳定性的关键。

## 七、把主动取消改成用户输入取消

可以用简单输入模拟用户取消：

```go
go func() {
	fmt.Println("press Enter to cancel")
	fmt.Scanln()
	cancel()
}()
```

这样你可以手动控制取消时机。

## 八、日志级别建议

取消不一定是 error。

例如用户关闭页面、主动停止下载，属于正常行为。

可以这样处理：

```text
context.Canceled：info。
context.DeadlineExceeded：warn。
其他错误：error。
```

当然，具体还要看业务。

---

## 九、常见错误

### 1. 取消后继续处理

收到 `ctx.Done()` 后应该尽快 return。

### 2. 取消时返回 nil

调用方会误以为任务完成。

### 3. 包装错误不用 %w

导致 `errors.Is` 判断失败。
