这一阶段还不急着大量使用 `context`，先把“等待、超时、取消”的问题感受出来。

## 练习一：等待一个异步任务

创建：

```text
cmd/00-practice-01/main.go
```

代码：

```go
package main

import (
	"fmt"
	"time"
)

func main() {
	resultCh := make(chan string)

	go func() {
		time.Sleep(time.Second)
		resultCh <- "task result"
	}()

	result := <-resultCh
	fmt.Println(result)
}
```

运行：

```powershell
go run ./cmd/00-practice-01
```

你应该看到：

```text
task result
```

这个练习说明：channel 可以让主 goroutine 等待子 goroutine 的结果。

## 练习二：给异步任务加超时

创建：

```text
cmd/00-practice-02/main.go
```

代码：

```go
package main

import (
	"fmt"
	"time"
)

func main() {
	resultCh := make(chan string)

	go func() {
		time.Sleep(2 * time.Second)
		resultCh <- "task result"
	}()

	select {
	case result := <-resultCh:
		fmt.Println("result:", result)
	case <-time.After(time.Second):
		fmt.Println("timeout")
	}
}
```

运行后会输出：

```text
timeout
```

这个练习说明：调用方可以选择不再等待。

但它还没有通知子 goroutine 停止。

## 练习三：手写一个 done 信号

创建：

```text
cmd/00-practice-03/main.go
```

代码：

```go
package main

import (
	"fmt"
	"time"
)

func worker(done <-chan struct{}) {
	for {
		select {
		case <-done:
			fmt.Println("worker stopped")
			return
		default:
			fmt.Println("working...")
			time.Sleep(300 * time.Millisecond)
		}
	}
}

func main() {
	done := make(chan struct{})

	go worker(done)

	time.Sleep(time.Second)
	close(done)
	time.Sleep(500 * time.Millisecond)
}
```

这里 `done` 就是一个取消信号。

当主 goroutine 调用：

```go
close(done)
```

worker 会收到信号并退出。

后面学习 `context.WithCancel` 时，你会发现它帮我们封装了类似能力。

## 本阶段总结

你现在应该已经具备学习 `context` 的基础：

- goroutine 可以并发执行任务。
- channel 可以传结果或信号。
- select 可以等待多个事件。
- 超时和取消是后端服务的常见需求。
- goroutine 不会因为调用方不等了就自动退出。

## 五、综合练习加强版：模拟请求生命周期

写一个程序模拟一次请求：

```text
main 表示请求入口。
worker 表示下游任务。
done 表示请求取消。
```

要求：

1. worker 每 300ms 打印一次工作中。
2. main 在 1 秒后关闭 done。
3. worker 收到 done 后退出。
4. main 等待 worker 退出后再结束。

参考结构：

```go
func worker(done <-chan struct{}, stopped chan<- struct{}) {
	defer close(stopped)

	for {
		select {
		case <-done:
			fmt.Println("worker stopped")
			return
		case <-time.After(300 * time.Millisecond):
			fmt.Println("working")
		}
	}
}
```

这个练习和 `context.WithCancel` 的思想非常接近。

## 六、观察点

运行时观察：

```text
worker 是否继续打印？
main 是否等到 worker 退出？
如果 worker 不监听 done 会怎样？
```

请故意删掉 worker 中的 done 分支，再运行一次。

你会看到 main 即使“不想等了”，worker 也不知道应该停。
