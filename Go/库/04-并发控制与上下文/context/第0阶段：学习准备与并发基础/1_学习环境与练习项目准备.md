# 1. 学习环境与练习项目准备

本节目标：准备一个可以持续写 demo 的 Go 练习项目，并形成统一的运行方式。

学习 `context` 时会写很多小程序。如果每次都随便新建文件，后面复习会很乱。建议从一开始就建一个清晰的练习项目。

---

## 一、检查 Go 环境

在 PowerShell 中执行：

```powershell
go version
```

如果能看到类似输出：

```text
go version go1.22.0 windows/amd64
```

说明 Go 环境可用。

如果命令不存在，需要先安装 Go，并确认 `go` 命令已经加入系统 PATH。

---

## 二、创建练习项目

建议建立一个单独目录：

```powershell
mkdir go-context-lab
cd go-context-lab
go mod init go-context-lab
```

项目结构可以先这样：

```text
go-context-lab/
  go.mod
  cmd/
    01-basic/
      main.go
    02-cancel/
      main.go
    03-timeout/
      main.go
    04-http/
      main.go
    05-project/
      main.go
```

每个主题一个小目录，运行时指定对应目录：

```powershell
go run ./cmd/01-basic
```

这样做的好处是：

- 每个 demo 互不干扰。
- 后面可以逐个回看。
- 不需要反复改同一个 `main.go`。

---

## 三、推荐编辑器设置

如果使用 VS Code，建议安装：

- Go 官方扩展。
- gopls。

保存时自动格式化：

```json
{
  "editor.formatOnSave": true,
  "go.useLanguageServer": true
}
```

Go 代码一定要保持 `gofmt` 格式。

手动格式化：

```powershell
gofmt -w .
```

---

## 四、写第一个 main.go

创建：

```text
cmd/01-basic/main.go
```

内容：

```go
package main

import (
	"context"
	"fmt"
)

func main() {
	ctx := context.Background()
	fmt.Println("ctx:", ctx)
}
```

运行：

```powershell
go run ./cmd/01-basic
```

你可能会看到：

```text
ctx: context.Background
```

现在练习项目就准备好了。

---

## 五、学习时建议固定一个模板

后续每个 demo 都可以从这个模板开始：

```go
package main

import (
	"context"
	"fmt"
	"time"
)

func main() {
	ctx := context.Background()
	_ = ctx

	fmt.Println("start")
	time.Sleep(time.Second)
	fmt.Println("done")
}
```

先保证能跑，再逐步加入：

- `context.WithCancel`
- `context.WithTimeout`
- `select`
- `ctx.Done()`
- goroutine

不要一开始就写复杂项目。

## 六、常见环境问题

### 1. go run 找不到包

如果看到类似：

```text
go: go.mod file not found
```

说明当前目录不在 Go module 里。

解决方式：

```powershell
go mod init go-context-lab
```

或者切换到包含 `go.mod` 的目录。

### 2. import 了包但没使用

Go 不允许未使用的 import。

例如：

```go
import "time"
```

但代码中没有使用 `time`，会编译失败。

解决方式是删除不用的 import，或者真正使用它。

### 3. 代码格式很乱

执行：

```powershell
gofmt -w .
```

Go 项目里不需要争论代码缩进风格，统一交给 `gofmt`。

## 七、本节达标标准

学完本节后，你应该能够做到：

- 用 `go version` 检查环境。
- 创建一个 Go module。
- 使用 `go run ./cmd/xxx` 运行 demo。
- 知道后续代码应该放在哪里。
- 遇到 import 未使用、找不到 go.mod 时能自己处理。

## 八、推荐目录进一步细化

如果你想把所有练习长期保存，可以这样组织：

```text
go-context-lab/
  go.mod
  cmd/
    00-goroutine-basic/
      main.go
    01-channel-result/
      main.go
    02-select-timeout/
      main.go
    03-context-basic/
      main.go
    04-with-cancel/
      main.go
    05-with-timeout/
      main.go
    06-http-request-context/
      main.go
    07-errgroup/
      main.go
```

这种结构和后端项目的 `cmd` 目录习惯接近。

每个目录都是一个独立小程序，互相不影响。

---

## 九、如何运行指定 demo

例如运行取消控制 demo：

```powershell
go run ./cmd/04-with-cancel
```

运行超时 demo：

```powershell
go run ./cmd/05-with-timeout
```

如果你想检查所有包是否能编译：

```powershell
go test ./...
```

即使暂时没有测试文件，这个命令也能帮助发现编译错误。

---

## 十、推荐每个 demo 的文件头注释

可以在每个 `main.go` 顶部写：

```go
// 本 demo 演示 context.WithTimeout 的基本行为。
// 运行：go run ./cmd/05-with-timeout
```

以后复习时会很方便。

---

## 十一、依赖管理

大多数阶段只用标准库。

需要额外安装的是 errgroup：

```powershell
go get golang.org/x/sync/errgroup
```

安装后 `go.mod` 会多出依赖。

不要手动编辑 `go.sum`，交给 Go 工具维护。

---

## 十二、常见问题补充

### 1. Windows PowerShell 路径问题

如果目录里有中文，一般 Go 可以正常处理。

但如果遇到奇怪问题，可以把练习项目放到英文路径，例如：

```text
C:\code\go-context-lab
```

### 2. go run 当前目录失败

如果你在项目根目录执行：

```powershell
go run .
```

但根目录没有 `main.go`，会失败。

应该执行：

```powershell
go run ./cmd/xxx
```

### 3. 多个 main 包没有关系

每个 `cmd/xxx` 目录都可以有自己的 `package main`。

这在 Go 项目中很常见。
