# 1. 原子计数器

本节目标：用 `atomic.Int64` 实现并发安全计数器，理解原子加法和读取。

---

## 一、基本实现

```go
type Counter struct {
    value atomic.Int64
}

func (c *Counter) Add(n int64) {
    c.value.Add(n)
}

func (c *Counter) Value() int64 {
    return c.value.Load()
}

func (c *Counter) Reset() {
    c.value.Store(0)
}
```

`Add`、`Load`、`Store` 都是并发安全的。

---

## 二、并发测试

```go
func TestAtomicCounter(t *testing.T) {
    var c Counter
    var wg sync.WaitGroup

    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            c.Add(1)
        }()
    }

    wg.Wait()

    if got := c.Value(); got != 1000 {
        t.Fatalf("got %d, want 1000", got)
    }
}
```

## 三、不要普通读写同一个变量

错误示例：

```go
var count int64

atomic.AddInt64(&count, 1)
fmt.Println(count)
```

如果一个变量使用 atomic 访问，就应该所有并发访问都使用 atomic。

使用 typed atomic 可以减少这种错误：

```go
var count atomic.Int64
```

外部不能直接普通读写内部值。

---

## 四、计数器适合 atomic

计数器适合 atomic，因为操作很简单：

```text
加一；
读取；
清零。
```

如果需求变成：

```text
value 不能小于 0；
更新 value 时同时更新 lastUpdated；
超过阈值后记录一次告警；
```

就要考虑 `Mutex`，因为已经涉及多个状态之间的一致性。
