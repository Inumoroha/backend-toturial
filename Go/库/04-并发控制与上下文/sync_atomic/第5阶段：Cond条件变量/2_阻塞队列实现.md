# 2. 阻塞队列实现

本节目标：使用 `sync.Cond` 实现一个简单阻塞队列，理解生产者消费者模型。

---

## 一、需求

队列需要支持：

```text
Push(item)：放入一个元素
Pop()：如果队列为空则阻塞等待
Close()：关闭队列并唤醒全部等待者
```

---

## 二、数据结构

```go
type BlockingQueue struct {
    cond   *sync.Cond
    items  []string
    closed bool
}

func NewBlockingQueue() *BlockingQueue {
    return &BlockingQueue{
        cond: sync.NewCond(&sync.Mutex{}),
    }
}
```

`items` 和 `closed` 都是共享状态，必须由 `cond.L` 保护。

---

## 三、Push

```go
func (q *BlockingQueue) Push(item string) bool {
    q.cond.L.Lock()
    defer q.cond.L.Unlock()

    if q.closed {
        return false
    }

    q.items = append(q.items, item)
    q.cond.Signal()
    return true
}
```

新增一个元素后，唤醒一个等待的消费者即可。

---

## 四、Pop

```go
func (q *BlockingQueue) Pop() (string, bool) {
    q.cond.L.Lock()
    defer q.cond.L.Unlock()

    for len(q.items) == 0 && !q.closed {
        q.cond.Wait()
    }

    if len(q.items) == 0 && q.closed {
        return "", false
    }

    item := q.items[0]
    q.items = q.items[1:]
    return item, true
}
```

这里等待条件是：

```text
队列为空，并且还没有关闭。
```

被唤醒后要重新检查。

---

## 五、Close

```go
func (q *BlockingQueue) Close() {
    q.cond.L.Lock()
    defer q.cond.L.Unlock()

    if q.closed {
        return
    }

    q.closed = true
    q.cond.Broadcast()
}
```

关闭时要唤醒全部等待者，否则正在 `Pop` 的 goroutine 可能永远睡眠。
