# 2. 并发任务收集结果

本节目标：掌握使用 `WaitGroup` 等待多个任务，同时安全收集结果和错误。

`WaitGroup` 本身不保存结果。要收集结果，需要配合其他结构。

---

## 一、使用 channel 收集结果

```go
type Result struct {
    Index int
    Value string
    Err   error
}

func Run(tasks []func() (string, error)) []Result {
    var wg sync.WaitGroup
    results := make(chan Result, len(tasks))

    for i, task := range tasks {
        i, task := i, task

        wg.Add(1)
        go func() {
            defer wg.Done()
            value, err := task()
            results <- Result{Index: i, Value: value, Err: err}
        }()
    }

    wg.Wait()
    close(results)

    out := make([]Result, 0, len(tasks))
    for r := range results {
        out = append(out, r)
    }
    return out
}
```

这里 channel 有缓冲，容量等于任务数，goroutine 写入结果时不会因为主 goroutine 尚未读取而阻塞。

---

## 二、使用 Mutex 收集错误

```go
var (
    wg   sync.WaitGroup
    mu   sync.Mutex
    errs []error
)

for _, task := range tasks {
    task := task
    wg.Add(1)
    go func() {
        defer wg.Done()

        if err := task(); err != nil {
            mu.Lock()
            errs = append(errs, err)
            mu.Unlock()
        }
    }()
}

wg.Wait()
```

`errs` 是共享 slice，多个 goroutine append 时必须加锁。

---

## 三、保持结果顺序

如果结果需要和任务顺序一致，可以提前创建 slice：

```go
results := make([]Result, len(tasks))

for i, task := range tasks {
    i, task := i, task
    wg.Add(1)
    go func() {
        defer wg.Done()
        value, err := task()
        results[i] = Result{Index: i, Value: value, Err: err}
    }()
}

wg.Wait()
```

为什么这里可以不加锁？

因为每个 goroutine 只写自己独占的下标，`Wait` 后主 goroutine 再读取全部结果。不要让多个 goroutine 写同一个元素，也不要在任务未完成时并发读取。

---

## 四、限制最大并发数

`WaitGroup` 不限制并发。限制并发可以用 channel 信号量：

```go
sem := make(chan struct{}, 10)

for _, task := range tasks {
    task := task
    wg.Add(1)
    go func() {
        defer wg.Done()

        sem <- struct{}{}
        defer func() { <-sem }()

        task()
    }()
}
```

这样最多只有 10 个任务同时执行。
