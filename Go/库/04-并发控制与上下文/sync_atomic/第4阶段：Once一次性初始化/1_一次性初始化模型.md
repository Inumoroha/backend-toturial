# 1. 一次性初始化模型

本节目标：理解 `sync.Once` 的核心语义，掌握并发安全初始化的标准写法。

---

## 一、为什么需要 Once

假设你有一个全局配置：

```go
var cfg *Config
```

你希望它第一次被使用时才加载：

```go
func GetConfig() *Config {
    if cfg == nil {
        cfg = LoadConfig()
    }
    return cfg
}
```

这段代码在并发下不安全。多个 goroutine 可能同时看到 `cfg == nil`，然后重复初始化。

---

## 二、使用 Once 修复

```go
var (
    cfg  *Config
    once sync.Once
)

func GetConfig() *Config {
    once.Do(func() {
        cfg = LoadConfig()
    })
    return cfg
}
```

`Once` 保证：

- 初始化函数只执行一次。
- 其他 goroutine 会等待初始化完成。
- 初始化中写入的数据对后续调用可见。

---

## 三、Once 和双重检查

有些人会手写双重检查锁：

```go
if cfg == nil {
    mu.Lock()
    if cfg == nil {
        cfg = LoadConfig()
    }
    mu.Unlock()
}
```

这种写法更复杂，也更容易写错。Go 中优先使用 `sync.Once` 表达一次性初始化。

---

## 四、Once 不只是互斥

`Once` 不只是保证同一时刻只有一个 goroutine 执行初始化，它还保证初始化结果对其他 goroutine 可见。

也就是说：

```text
once.Do 返回后，初始化函数中完成的写入对调用方可见。
```

这就是它比普通 `if cfg == nil` 更可靠的原因。

---

## 五、本节练习

1. 写一个不安全懒加载配置示例。
2. 用 `sync.Once` 修复。
3. 启动 100 个 goroutine 同时调用 `GetConfig`，统计初始化函数执行次数。
4. 尝试用 `Mutex` 手写一次性初始化，然后对比 `Once` 的表达力。

---

## 六、本节达标标准

学完本节后，你应该能做到：

- 正确使用 `once.Do`。
- 解释为什么普通 nil 判断不是并发安全初始化。
- 知道 `Once` 同时提供互斥和可见性保证。
