# 2. 懒加载配置与客户端

本节目标：通过配置加载和客户端初始化两个后端场景，掌握 `sync.Once` 的工程用法。

---

## 一、懒加载配置

```go
type Config struct {
    Addr string
}

var (
    configOnce sync.Once
    config     *Config
)

func GetConfig() *Config {
    configOnce.Do(func() {
        config = &Config{
            Addr: "localhost:8080",
        }
    })
    return config
}
```

适合场景：

- 配置加载成本较高。
- 配置在进程生命周期内不变。
- 多处代码都可能第一次触发加载。

---

## 二、初始化 HTTP 客户端

```go
var (
    clientOnce sync.Once
    client     *http.Client
)

func HTTPClient() *http.Client {
    clientOnce.Do(func() {
        client = &http.Client{
            Timeout: 3 * time.Second,
        }
    })
    return client
}
```

这样可以避免多个包重复创建客户端。

不过要注意：很多客户端本身就是并发安全的，初始化一次后可以复用。

---

## 三、初始化数据库连接是否适合 Once

有些场景可以：

```text
进程启动后只初始化一次连接池；
初始化失败直接让程序启动失败；
后续不做动态重连。
```

但如果需要：

- 初始化失败后重试。
- 配置热更新。
- 连接池重建。
- 多租户不同配置。

就不适合只用简单 `Once`。

---

## 四、测试 Once 初始化次数

```go
func TestOnceInit(t *testing.T) {
    var once sync.Once
    var count atomic.Int64
    var wg sync.WaitGroup

    for i := 0; i < 100; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            once.Do(func() {
                count.Add(1)
            })
        }()
    }

    wg.Wait()

    if got := count.Load(); got != 1 {
        t.Fatalf("got %d, want 1", got)
    }
}
```
