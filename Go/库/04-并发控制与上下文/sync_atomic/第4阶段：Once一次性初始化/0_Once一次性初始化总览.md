# 0. Once 一次性初始化总览

本阶段目标：掌握 `sync.Once` 的使用方式，理解并发安全的一次性初始化，以及它在失败重试场景中的限制。

`sync.Once` 解决的问题是：

```text
某段初始化逻辑，在多个 goroutine 同时调用时，也只执行一次。
```

常见用途：

- 懒加载配置。
- 初始化全局客户端。
- 初始化只读缓存。
- 注册只执行一次的资源。

---

## 一、基本写法

```go
var once sync.Once

once.Do(func() {
    // init
})
```

无论多少 goroutine 同时调用，传给 `Do` 的函数都只会执行一次。

## 三、Once 的关键限制

需要特别注意：

- `Do` 中的函数只执行一次。
- 如果函数 panic，本次也会被认为执行过。
- `Once` 不适合需要失败后自动重试的复杂初始化。
- 不要复制已经使用过的 `Once`。
