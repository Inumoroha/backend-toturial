# 0. Mutex 互斥锁总览

本阶段目标：掌握 `sync.Mutex` 的正确使用方式，理解临界区、锁粒度、死锁和工程中的锁设计。

`Mutex` 是最基础、最常用的同步原语。它解决的问题很直接：

```text
同一时刻，只允许一个 goroutine 进入某段访问共享状态的代码。
```

注意：锁保护的不是变量名，而是访问共享状态的临界区。

## 三、Mutex 的基本写法

```go
var mu sync.Mutex

mu.Lock()
defer mu.Unlock()

// 访问共享状态
```

这段代码表达的含义是：

```text
从 Lock 成功开始，到 Unlock 执行结束，这中间的代码同一时刻只能有一个 goroutine 执行。
```

---

## 四、后端中常见使用场景

`Mutex` 常用于：

- 本地缓存。
- 限流器。
- 后台任务状态。
- 连接池内部状态。
- 统计指标聚合。
- 内存队列。
- WebSocket 连接写入保护。

只要有共享状态，就可能需要 `Mutex`。
