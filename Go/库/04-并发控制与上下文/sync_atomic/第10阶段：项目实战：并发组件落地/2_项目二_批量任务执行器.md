# 2. 项目二：批量任务执行器

本项目目标：实现一个支持最大并发数、错误收集、context 取消的批量任务执行器。

会使用：

- `sync.WaitGroup`
- channel 信号量
- `sync.Mutex`
- `context.Context`
- `atomic.Int64`

---

## 一、需求分析

任务执行器需要支持：

```text
提交多个任务；
限制最大并发数；
等待全部任务完成；
收集错误；
支持 context 取消；
统计成功和失败数量。
```

---

## 二、任务定义

```go
type Task func(ctx context.Context) error

type Result struct {
    Success int64
    Failed  int64
    Errors  []error
}
```

---

## 三、执行器结构

```go
type Runner struct {
    concurrency int
}

func NewRunner(concurrency int) *Runner {
    if concurrency <= 0 {
        concurrency = 1
    }
    return &Runner{concurrency: concurrency}
}
```

---

## 四、核心执行逻辑

```go
func (r *Runner) Run(ctx context.Context, tasks []Task) Result {
    sem := make(chan struct{}, r.concurrency)
    var wg sync.WaitGroup
    var mu sync.Mutex
    var result Result
    var success atomic.Int64
    var failed atomic.Int64

submitLoop:
    for _, task := range tasks {
        task := task

        select {
        case <-ctx.Done():
            break submitLoop
        default:
        }

        wg.Add(1)
        go func() {
            defer wg.Done()

            select {
            case sem <- struct{}{}:
                defer func() { <-sem }()
            case <-ctx.Done():
                failed.Add(1)
                return
            }

            if err := task(ctx); err != nil {
                failed.Add(1)
                mu.Lock()
                result.Errors = append(result.Errors, err)
                mu.Unlock()
                return
            }

            success.Add(1)
        }()
    }

    wg.Wait()
    result.Success = success.Load()
    result.Failed = failed.Load()
    return result
}
```

注意：这里使用了 `submitLoop` 标签。普通 `break` 只会跳出 `select`，不会跳出外层 `for`，这是 Go 里很容易写错的地方。

---

## 五、错误收集为什么要加锁

`result.Errors` 是共享 slice。

多个 goroutine 同时 append 会产生数据竞争，所以必须使用 `Mutex`。

而成功失败计数是简单数字，可以使用 `atomic.Int64`。

---

## 六、并发限制原理

```go
sem := make(chan struct{}, concurrency)
```

每个任务执行前放入一个 token：

```go
sem <- struct{}{}
```

执行结束归还：

```go
<-sem
```

channel 满时，新的任务会阻塞，从而限制最大并发数。

## 七、测试清单

必须测试：

- 所有任务都执行成功。
- 部分任务返回错误。
- 最大并发数不超过限制。
- context 取消后任务能停止。
- 错误收集无 race。
