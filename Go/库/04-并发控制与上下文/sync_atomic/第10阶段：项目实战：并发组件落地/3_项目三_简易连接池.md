# 3. 项目三：简易连接池

本项目目标：实现一个简化版连接池，理解资源获取、归还、等待、关闭的同步设计。

会使用：

- `sync.Mutex`
- `sync.Cond`
- `sync.Once`
- `context.Context`

---

## 一、需求分析

连接池需要支持：

```text
初始化固定数量连接；
Get(ctx) 获取连接；
Put(conn) 归还连接；
Close() 关闭连接池；
Get 在无可用连接时等待；
context 超时时 Get 返回错误；
Close 唤醒所有等待者。
```

---

## 二、连接模型

```go
type Conn struct {
    ID int
}

func (c *Conn) Close() error {
    return nil
}
```

真实项目中可以替换为数据库连接、TCP 连接或客户端对象。

---

## 三、连接池结构

```go
type Pool struct {
    mu     sync.Mutex
    cond   *sync.Cond
    conns  []*Conn
    closed bool
    once   sync.Once
}

func NewPool(size int) *Pool {
    p := &Pool{}
    p.cond = sync.NewCond(&p.mu)
    for i := 0; i < size; i++ {
        p.conns = append(p.conns, &Conn{ID: i + 1})
    }
    return p
}
```

`conns` 和 `closed` 都由 `mu` 保护。

---

## 四、Get 的基本逻辑

`Cond.Wait` 本身不直接支持 context，所以需要谨慎设计。

第一版可以先实现无 context：

```go
func (p *Pool) Get() (*Conn, error) {
    p.mu.Lock()
    defer p.mu.Unlock()

    for len(p.conns) == 0 && !p.closed {
        p.cond.Wait()
    }

    if p.closed {
        return nil, ErrClosed
    }

    n := len(p.conns)
    conn := p.conns[n-1]
    p.conns = p.conns[:n-1]
    return conn, nil
}
```

等待条件是：

```text
没有可用连接，并且池未关闭。
```

---

## 五、Put

```go
func (p *Pool) Put(conn *Conn) {
    if conn == nil {
        return
    }

    p.mu.Lock()
    defer p.mu.Unlock()

    if p.closed {
        _ = conn.Close()
        return
    }

    p.conns = append(p.conns, conn)
    p.cond.Signal()
}
```

归还连接后唤醒一个等待者。

---

## 六、Close

```go
func (p *Pool) Close() {
    p.once.Do(func() {
        p.mu.Lock()
        defer p.mu.Unlock()

        p.closed = true
        for _, conn := range p.conns {
            _ = conn.Close()
        }
        p.conns = nil
        p.cond.Broadcast()
    })
}
```

关闭时必须 `Broadcast`，否则等待中的 `Get` 可能永远睡眠。

---

## 七、context 支持思路

因为 `Cond.Wait` 不能直接 select context，可以用两种方式：

1. 使用 channel 重写连接池，更容易支持 context。
2. 启动辅助 goroutine 在 ctx.Done 时 Broadcast，但要非常小心避免 goroutine 泄漏。

学习阶段建议：

```text
先用 Cond 实现无 context 版本；
再用 channel 实现支持 context 的版本；
对比两者。
```

## 八、测试清单

必须测试：

- 能获取固定数量连接。
- 没有连接时 `Get` 阻塞。
- `Put` 后等待者被唤醒。
- `Close` 后 `Get` 返回错误。
- `Close` 可重复调用。
- `Close` 会唤醒等待者。
