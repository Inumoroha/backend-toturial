# Go sync 后续进阶学习方向

学完 `sync` 教程后，你已经掌握了 Go 并发同步的主体工具。下一步不要继续孤立地背 API，而是把 `sync` 放进完整的 Go 后端并发体系里学习。

后续进阶可以按下面顺序推进。

---

## 一、context：请求生命周期与取消传播

`sync` 负责同步，`context` 负责取消、超时和请求范围内的值传递。

必须掌握：

- `context.WithCancel`
- `context.WithTimeout`
- `context.WithDeadline`
- `ctx.Done()`
- `ctx.Err()`
- HTTP 请求中如何传递 context
- 数据库、Redis、RPC 调用如何使用 context

重点练习：

- 实现一个可取消 worker。
- 给批量任务执行器加入超时取消。
- HTTP 请求取消后，让后台 goroutine 停止工作。
- 服务关闭时，用 context 通知全部后台任务退出。

你需要形成的判断：

```text
WaitGroup 负责等任务结束；
context 负责通知任务应该结束；
二者经常一起使用。
```

---

## 二、channel 并发模式

`sync` 适合保护共享状态，channel 更适合表达数据流和事件通知。

必须掌握：

- 无缓冲 channel
- 有缓冲 channel
- channel close 语义
- `for range ch`
- `select`
- fan-in
- fan-out
- pipeline
- semaphore 模式
- done channel

重点练习：

- 实现生产者消费者模型。
- 实现 worker pool。
- 实现 fan-out/fan-in 数据处理。
- 使用 channel 限制最大并发数。
- 使用 close channel 广播退出。

需要注意：

```text
不要为了不用锁而滥用 channel；
也不要为了不用 channel 而把所有状态都塞进锁里。
```

选择标准是：共享状态优先考虑锁，数据流优先考虑 channel。

---

## 三、errgroup：带错误传播的并发编排

`sync.WaitGroup` 不负责错误传播，也不负责取消其他任务。真实后端请求里，经常需要：

```text
并发调用多个下游；
任意一个失败就返回错误；
同时取消其他还没完成的任务。
```

这时可以学习：

```text
golang.org/x/sync/errgroup
```

必须掌握：

- `errgroup.Group`
- `errgroup.WithContext`
- `g.Go(func() error)`
- `g.Wait()`
- 第一个错误返回机制
- context 取消传播

重点练习：

- 并发请求多个 HTTP API。
- 任意一个 API 失败后取消其他请求。
- 给每个请求加超时。
- 对比 `WaitGroup + Mutex 收集错误` 和 `errgroup`。

---

## 四、singleflight：防止缓存击穿

`sync.Map.LoadOrStore` 可以避免重复存储，但不能避免昂贵初始化函数重复执行。

后端缓存场景中，经常遇到：

```text
热点 key 过期；
大量请求同时打进来；
每个请求都去查数据库；
数据库瞬间压力暴涨。
```

这类问题可以学习：

```text
golang.org/x/sync/singleflight
```

必须掌握：

- `singleflight.Group`
- `Do(key, fn)`
- 多个请求共享同一次加载结果
- 缓存击穿与请求合并

重点练习：

- 在 TTL 缓存中加入 singleflight。
- 模拟 1000 个 goroutine 同时读取同一个过期 key。
- 对比加入 singleflight 前后的数据库查询次数。

---

## 五、goroutine 生命周期管理

很多 Go 后端事故不是因为不会加锁，而是因为 goroutine 泄漏。

必须掌握：

- goroutine 谁创建，谁负责停止。
- 后台任务如何退出。
- channel 是否会被永远阻塞。
- context 是否传递到了最深层。
- `WaitGroup` 是否等待了所有后台任务。
- Close 是否可重复调用。

重点练习：

- 实现一个可优雅关闭的 worker pool。
- 实现一个定时刷新配置的后台任务。
- 实现一个服务 shutdown 流程。
- 使用 pprof 查看 goroutine 数量变化。

验收标准：

```text
启动服务后 goroutine 数稳定；
压测后 goroutine 数能回落；
关闭服务时所有后台 goroutine 都能退出。
```

---

## 六、pprof、trace 与线上并发排障

会写并发代码只是第一步。线上真正重要的是能排查问题。

必须掌握：

- CPU profile
- memory profile
- goroutine profile
- block profile
- mutex profile
- `go tool pprof`
- `go tool trace`
- HTTP pprof endpoint

重点练习：

- 制造一个锁竞争严重的程序，用 mutex profile 分析。
- 制造一个 channel 阻塞程序，用 goroutine profile 分析。
- 制造一个 goroutine 泄漏程序，观察 goroutine profile。
- 用 trace 观察 goroutine 阻塞、唤醒和调度。

线上排障时常见问题：

- 锁竞争导致请求延迟升高。
- goroutine 泄漏导致内存上涨。
- channel 无人接收导致请求卡住。
- 持锁做 IO 导致吞吐下降。
- worker 无法退出导致服务关闭卡住。

---

## 七、Go runtime 调度器与内存模型

这一阶段偏底层，不需要一开始就学很深，但想成为更强的 Go 后端工程师，迟早要理解。

建议学习：

- GMP 调度模型。
- goroutine 栈增长。
- syscall 对调度的影响。
- network poller。
- 抢占式调度。
- Go 内存模型。
- happens-before 规则。
- atomic 的内存语义。

重点不是背 runtime 源码，而是理解这些问题：

- goroutine 为什么轻量。
- 阻塞 syscall 会怎样影响调度。
- 为什么同步原语能保证可见性。
- 为什么数据竞争会让程序行为不可预测。

---

## 八、高并发后端组件设计

学完基础后，建议继续做这些项目：

### 1. Worker Pool

功能：

- 固定 worker 数。
- 支持提交任务。
- 支持 context 取消。
- 支持关闭。
- 支持错误收集。
- 支持队列长度限制。

重点：

- channel 和 WaitGroup 配合。
- 关闭流程。
- 队列满时的策略。

### 2. 限流器

功能：

- 固定窗口限流。
- 滑动窗口限流。
- token bucket。
- leaky bucket。

重点：

- `Mutex` 保护计数状态。
- atomic 简单计数。
- 时间窗口更新。
- 高并发下的正确性。

### 3. 本地缓存增强版

功能：

- TTL。
- LRU。
- 最大容量。
- singleflight 防击穿。
- 后台清理。
- 指标统计。

重点：

- `Mutex/RWMutex`。
- `container/list`。
- 后台 goroutine 生命周期。
- benchmark 和 pprof。

### 4. 连接池增强版

功能：

- 最大连接数。
- 空闲连接数。
- 获取超时。
- 健康检查。
- 空闲回收。
- 优雅关闭。

重点：

- `Mutex + Cond` 或 channel。
- context。
- Close 幂等。
- 等待者唤醒。

---

## 九、推荐进阶顺序

建议按下面顺序继续：

```text
context
-> channel 并发模式
-> errgroup
-> singleflight
-> goroutine 生命周期管理
-> pprof/trace 排障
-> runtime 调度器与内存模型
-> 高并发组件项目
```

这个顺序比较适合 Go 后端工程师，因为它从业务最常用的能力开始，逐步走向底层。

---

## 十、最终进阶目标

继续深入后，你应该能做到：

- 设计完整的 goroutine 生命周期。
- 用 context 控制取消和超时。
- 用 channel 表达数据流。
- 用 sync 保护共享状态。
- 用 errgroup 编排有错误传播的并发任务。
- 用 singleflight 解决缓存击穿。
- 用 pprof 和 trace 定位线上并发问题。
- 设计缓存、连接池、限流器、worker pool 等后端基础组件。

到这一步，你掌握的就不只是 `sync` 包，而是 Go 后端并发编程的完整工程体系。
