# 2. goroutine 与共享状态

本节目标：理解 goroutine 并发执行时，为什么共享变量会带来不确定性，以及为什么 `sync` 包的核心任务是保护共享状态。

goroutine 很轻量，但它不是魔法。只要多个 goroutine 同时访问同一份数据，就必须考虑并发安全。

---

## 一、什么是共享状态

共享状态就是被多个执行单元共同访问的数据。

在 Go 后端中，常见共享状态包括：

- 全局配置缓存。
- 本地内存缓存。
- 连接状态。
- 限流计数器。
- 用户 session。
- 后台任务状态。
- 复用的 buffer 或对象。

例如：

```go
var count int

func add() {
    count++
}
```

如果只有一个 goroutine 调用 `add`，通常没问题。

如果有 1000 个 goroutine 同时调用 `add`，这个代码就不安全。

---

## 二、为什么 count++ 不安全

`count++` 看起来是一句代码，但它不是一个不可分割的动作。它大致包含三个步骤：

```text
读取 count 当前值
计算 count + 1
把新值写回 count
```

两个 goroutine 可能这样交错执行：

```text
goroutine A 读取 count = 10
goroutine B 读取 count = 10
goroutine A 写回 11
goroutine B 写回 11
```

执行了两次加一，结果却只增加了一次。

这就是典型的丢失更新。

---

## 三、并发不等于并行

并发关注的是“多个任务在同一时间段内推进”。

并行关注的是“多个任务在同一时刻真正同时执行”。

Go 程序即使在单核 CPU 上，也可能发生并发交错。因为 goroutine 可能在不同时间点被调度、暂停和恢复。

所以不要认为“我的机器只有一个核，所以不会有并发问题”。只要多个 goroutine 访问共享状态，问题就存在。

---

## 四、局部变量是不是安全的

如果变量只被一个 goroutine 拥有，通常是安全的。

例如：

```go
go func() {
    total := 0
    for i := 0; i < 100; i++ {
        total += i
    }
}()
```

这里的 `total` 只在当前 goroutine 中使用，没有被其他 goroutine 同时访问。

但是，如果把变量地址传出去，情况就变了：

```go
total := 0

go func() {
    total++
}()

go func() {
    total++
}()
```

此时 `total` 被多个 goroutine 共享，就需要同步保护。

---

## 五、后端工程里的共享状态例子

### 1. 本地缓存

```go
var cache = map[string]string{}
```

多个 HTTP 请求 goroutine 同时读写 `cache`，需要加锁或使用其他并发安全设计。

### 2. 限流计数器

```go
var requests int64
```

多个请求同时增加计数，需要 `Mutex` 或 `atomic`。

### 3. 后台任务状态

```go
type Job struct {
    Status string
}
```

如果一个 goroutine 修改任务状态，另一个 goroutine 查询任务状态，需要同步保护。

---

## 六、sync 解决的不是“多线程语法”，而是共享规则

学习 `sync` 时不要只记：

```text
加锁
解锁
等待
初始化一次
```

要进一步问：

```text
这段代码里哪些数据被共享？
哪些操作必须互斥？
什么时候读者可以并发？
什么时候写者必须独占？
谁负责关闭或释放资源？
```

这就是从“会写语法”走向“能做后端工程”的关键。

## 七、本节练习

练习一：写一个不安全计数器。

要求：

- 创建变量 `count := 0`。
- 启动 1000 个 goroutine。
- 每个 goroutine 执行 `count++`。
- 等待全部 goroutine 结束。
- 打印最终结果。

观察：最终结果是否每次都是 1000？

练习二：把 goroutine 数量改成 10、100、10000，观察错误出现概率。

练习三：思考下面问题：

- 为什么 goroutine 数量少时可能看不出问题？
- 为什么不能用“我运行了几次都正常”来证明并发安全？

---

## 八、本节达标标准

学完本节后，你应该能做到：

- 解释什么是共享状态。
- 解释 `count++` 为什么不是并发安全的。
- 区分局部变量和被多个 goroutine 捕获的共享变量。
- 能在后端场景中识别哪些数据可能需要同步保护。
