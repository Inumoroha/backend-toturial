# 5. 练习规范与学习方法

本节目标：建立一套适合长期学习 `sync` 的练习规范，让每个阶段都能沉淀可运行代码、测试、benchmark 和总结。

学习并发最怕两件事：

```text
只看文章，不写代码；
只写能跑的 main，不写能验证的测试。
```

本教程建议你每学一个同步原语，就完成一组固定动作。

---

## 一、每个主题的推荐产物

每个主题目录建议包含：

```text
README.md
example_test.go
xxx.go
xxx_test.go
xxx_bench_test.go
```

其中：

- `README.md` 记录概念、适用场景和踩坑。
- `example_test.go` 写最小示例。
- `xxx.go` 写可复用实现。
- `xxx_test.go` 写功能和并发测试。
- `xxx_bench_test.go` 写性能对比。

不要把所有代码都塞到一个 `main.go`。后端工程更需要可测试、可复用的包。

---

## 二、每个知识点的学习闭环

建议用下面的闭环学习：

```text
先写一个错误版本
-> 用测试或 race detector 观察问题
-> 使用 sync 工具修复
-> 补充并发测试
-> 写 benchmark 验证性能
-> 在 README 中总结适用边界
```

例如学习 `Mutex`：

1. 先写不加锁的计数器。
2. 用 `go test -race` 看到数据竞争。
3. 用 `sync.Mutex` 修复。
4. 写 1000 个 goroutine 并发调用的测试。
5. 跟 atomic 版本做 benchmark。
6. 总结：复杂状态用锁更清晰，简单计数可用 atomic。

## 三、并发测试怎么写

并发测试要尽量制造压力：

```go
func TestConcurrentAccess(t *testing.T) {
    var wg sync.WaitGroup
    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go func(i int) {
            defer wg.Done()
            // call target method
        }(i)
    }
    wg.Wait()
}
```

注意点：

- `wg.Add(1)` 放在启动 goroutine 前。
- goroutine 中用 `defer wg.Done()`。
- 不要在测试里靠 `time.Sleep` 等待任务完成。
- 最终要断言结果，而不是只打印。

## 四、benchmark 怎么写

最小 benchmark：

```go
func BenchmarkCounterAdd(b *testing.B) {
    var c Counter
    b.ReportAllocs()

    for i := 0; i < b.N; i++ {
        c.Add(1)
    }
}
```

并发 benchmark 可以用：

```go
func BenchmarkCounterAddParallel(b *testing.B) {
    var c Counter
    b.ReportAllocs()

    b.RunParallel(func(pb *testing.PB) {
        for pb.Next() {
            c.Add(1)
        }
    })
}
```

学习时关注：

- `ns/op`：每次操作耗时。
- `B/op`：每次操作分配多少字节。
- `allocs/op`：每次操作分配次数。

不要为了追求一个更低的数字牺牲代码正确性和可维护性。

## 五、阶段总结模板

每学完一个主题，建议在对应目录写一段总结：

```markdown
## 我学会了什么

## 适合使用的场景

## 不适合使用的场景

## 常见错误

## 我写过的测试

## 我还有哪些疑问
```

这会让你从“看过”变成“掌握过”。

## 六、本阶段完成后的下一步

完成准备阶段后，就可以进入 `Mutex`。

`Mutex` 是学习 `sync` 的第一块基石。很多后端并发问题，本质都是：

```text
有一份共享状态；
多个请求或后台任务会访问它；
必须让某些操作互斥执行。
```

掌握 `Mutex` 后，再看 `RWMutex`、`WaitGroup`、`Once` 会自然很多。

## 七、本节达标标准

学完本节后，你应该能做到：

- 为每个阶段建立独立练习目录。
- 为并发组件写普通测试、race 测试和 benchmark。
- 知道如何从错误版本逐步演进到正确版本。
- 能用 README 记录每个同步原语的适用边界。
