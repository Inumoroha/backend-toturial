# 夯实基础

- [ ] **基本语法：** 变量声明（`:=`）、数据类型、控制流（`for` 是唯一的循环语句）。
- [ ] **核心数据结构：** 数组、**切片 (Slice)**（重点掌握扩容机制）、**字典 (Map)**。
- [ ] **函数与指针：** 多返回值、闭包、延迟执行 (`defer`，非常重要)、指针的基础用法（Go 的指针比 C/C++ 安全且简单，主要用于避免值拷贝）。

- **面向“对象”：** 结构体 (`Struct`) 和方法 (`Method`)。
- **接口 (Interface)：** Go 语言设计的灵魂。理解“鸭子类型”（隐式实现），体会组合优于继承的理念。
- **错误处理：** 习惯 `if err != nil`。这是 Go 与 Java 最大的不同之一（没有 `try-catch`），强调显式处理错误。

### 第二阶段：进阶核心（Go 的灵魂：并发）

Go 能够在后端大放异彩，很大程度上归功于其轻量级的并发模型。

- **Goroutine：** 理解它是如何比系统线程更轻量（几 KB 内存即可启动）。
- **Channel（通道）：** 学习使用 Channel 在不同的 Goroutine 之间传递数据。“不要通过共享内存来通信，而应通过通信来共享内存”。
- **并发控制 (sync 包)：** `WaitGroup`（等待一组协程完成）、`Mutex/RWMutex`（读写锁）。
- **Context（上下文）：** 后端开发必考点。用于在 API 请求中传递截止日期、取消信号和其他请求范围的值。

### 第三阶段：Web与后端开发（实战技能）

Go 的标准库非常强大，很多时候不需要依赖庞大的框架。

- **标准库 `net/http`：** 学习如何只用标准库搭建一个 HTTP 服务，理解 Request 和 ResponseWriter。
- **Web 框架：** 学习 **Gin**（目前最流行的轻量级框架）或 Fiber。掌握路由、中间件 (Middleware)、参数绑定和 JSON 序列化。
- **RESTful API 设计：** 规范地设计和开发接口。

### 第四阶段：数据库与中间件

作为一个后端工程师，和数据打交道是日常。

- **关系型数据库 (MySQL/PostgreSQL)：** * 使用标准库 `database/sql` 进行原生的 CRUD 操作。
  - 学习 ORM 框架，首推 **GORM**（最流行）或 Ent。
- **缓存与消息队列：** * **Redis：** 掌握 Go 操作 Redis 的库（如 `go-redis`），实现缓存、分布式锁。
  - 了解基础的消息队列（RabbitMQ 或 Kafka），理解异步解耦的概念。

### 第五阶段：微服务与云原生生态（高阶技能）

当你掌握了单体应用开发后，这是让你在求职中脱颖而出的加分项。

- **RPC 与 Protobuf：** 学习 **gRPC** 框架和 Protocol Buffers 序列化协议，这是 Go 微服务间的标准通信方式。
- **工程化建设：** * 包依赖管理 (`Go Modules`)。
  - 单元测试 (`testing` 包，Go 自带强大的测试工具)。
  - 项目目录结构规范（参考社区标准的 `golang-standards/project-layout`）。
- **Docker 基础：** 学习如何编写 `Dockerfile` 把你的 Go 编译成极小的镜像（通常只有十几兆）并运行。

### 给你的实战项目建议

不要只看书和视频，Go 是一门非常注重实践的语言。你可以按顺序尝试写这几个项目：

1. **命令行工具 (CLI)：** 写一个类似 `ls` 或 `grep` 的小工具，或者一个查天气的命令行程序（练习基础和 HTTP 客户端）。
2. **并发爬虫：** 利用 Goroutine 和 Channel，写一个爬取特定网站新闻并保存到本地的爬虫（体会并发的威力）。
3. **博客/待办事项 API 接口：** 使用 Gin + GORM + MySQL 写一套完整的增删改查 RESTful API，加上 JWT 身份认证。
4. **实时聊天室：** 使用 WebSocket 构建一个可以在浏览器里多人聊天的后端服务。

### 推荐学习资源

- **入门必看：** `A Tour of Go` (Go 官方指南，可以在线运行代码)。
- **查漏补缺：** `Go by Example` (通过简洁的代码例子学习所有特性)。
- **经典书籍：** 《The Go Programming Language》（Go 语言圣经，作者之一是 C 语言的设计者）。
